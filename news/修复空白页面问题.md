# 修复空白页面问题

## 问题原因

系统显示空白页面的原因是：`app_frontend` Docker卷为空或未正确填充前端文件。

在 `docker-compose.yml` 中，前端文件通过命名卷 `app_frontend` 挂载：
- 应用容器：`app_frontend:/app/client/dist`
- Nginx容器：`app_frontend:/usr/share/nginx/html`

当卷为空时，Nginx会返回一个很小的HTML文件（约487字节），导致页面空白。

## 快速修复方案

### 方案1：直接执行命令（最简单，推荐）

在服务器上直接执行以下命令（不需要脚本文件）：

```bash
cd /opt/newsapp/news

# 1. 查找volume路径
VOLUME_PATH=$(docker volume inspect news_app_frontend | grep -i "Mountpoint" | awk '{print $2}' | tr -d '",')

# 2. 创建临时目录并复制文件
TEMP_DIR=$(mktemp -d)
docker cp newsapp:/app/client/dist/. "$TEMP_DIR/"

# 3. 复制到volume路径
sudo rm -rf "$VOLUME_PATH"/*
sudo cp -r "$TEMP_DIR"/* "$VOLUME_PATH/"

# 4. 清理临时目录
rm -rf "$TEMP_DIR"

# 5. 重启nginx
docker compose restart nginx
```

### 方案2：使用修复脚本

如果脚本文件已正确上传（LF换行符），可以执行：

```bash
cd /opt/newsapp/news

# 给脚本添加执行权限（首次使用）
chmod +x deploy/fix-blank-page-direct.sh

# 执行修复脚本
sudo bash deploy/fix-blank-page-direct.sh
```

### 方案2：手动执行命令

在服务器上执行以下命令：

```bash
cd /opt/newsapp/news

# 1. 创建临时容器并复制文件
docker rm -f temp-frontend-fix 2>/dev/null || true
TEMP_CONTAINER=$(docker run -d --name temp-frontend-fix \
    -v news_app_frontend:/target \
    alpine sleep 3600)

# 2. 从应用容器复制前端文件到卷
docker cp newsapp:/app/client/dist/. "${TEMP_CONTAINER}:/target/"

# 3. 清理临时容器
docker rm -f "${TEMP_CONTAINER}"

# 4. 重启nginx
docker compose restart nginx

# 5. 验证文件
docker compose exec nginx ls -la /usr/share/nginx/html/ | head -10
```

### 方案3：重新构建并删除旧卷（最彻底）

如果上述方案无效，可以重新构建镜像并删除旧卷：

```bash
cd /opt/newsapp/news

# 1. 停止容器
docker compose down

# 2. 删除前端卷
docker volume rm news_app_frontend 2>/dev/null || true

# 3. 重新构建镜像
docker compose build app

# 4. 启动容器
docker compose up -d

# 5. 等待容器启动后，复制文件到卷
sleep 10
TEMP_CONTAINER=$(docker run -d --name temp-frontend-fix \
    -v news_app_frontend:/target \
    alpine sleep 3600)
docker cp newsapp:/app/client/dist/. "${TEMP_CONTAINER}:/target/"
docker rm -f "${TEMP_CONTAINER}"

# 6. 重启nginx
docker compose restart nginx
```

## 验证修复

1. **检查卷中的文件**：
   ```bash
   docker compose exec nginx ls -la /usr/share/nginx/html/ | head -20
   ```

2. **检查index.html大小**（应该大于1KB）：
   ```bash
   docker compose exec nginx stat -c%s /usr/share/nginx/html/index.html
   ```

3. **查看nginx日志**：
   ```bash
   docker compose logs nginx | tail -20
   ```

4. **清除浏览器缓存并刷新**：
   - 按 `Ctrl + Shift + Delete` 清除缓存
   - 或按 `Ctrl + F5` 硬刷新页面

## 预防措施

为了避免将来再次出现此问题，可以考虑以下改进：

### 选项1：修改docker-compose.yml，在启动时自动复制文件

创建一个启动脚本，在应用容器启动时自动将文件复制到卷中。

### 选项2：使用本地目录挂载（开发环境）

修改 `docker-compose.yml`：

```yaml
volumes:
  - ./client/dist:/app/client/dist  # 改为本地目录
```

**注意**：这需要服务器上有Node.js环境来构建前端。

### 选项3：修改Dockerfile，确保文件在正确位置

确保构建的前端文件在镜像中的位置与卷挂载点一致。

## 常见问题

### Q: 为什么会出现空白页面？

A: Docker卷 `app_frontend` 是空的，Nginx无法找到正确的前端文件。

### Q: 修复后仍然空白怎么办？

A: 
1. 检查浏览器控制台是否有错误
2. 检查网络请求，确认JS和CSS文件是否加载成功
3. 清除浏览器缓存
4. 检查nginx配置是否正确

### Q: 如何避免将来再次出现？

A: 每次更新前端代码后，确保：
1. 重新构建镜像，或
2. 使用 `update-frontend-only.sh` 脚本更新前端文件
