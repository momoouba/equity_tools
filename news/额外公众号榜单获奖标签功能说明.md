# 额外公众号榜单获奖标签功能说明

## 📋 功能概述

对于来自额外公众号（`additional_wechat_accounts`表）的新闻，系统会：
1. 在AI分析时，特别关注榜单、获奖、荣誉等相关信息，并添加"榜单"或"获奖"标签
2. 如果新闻包含"榜单"或"获奖"标签且企业名称为空，会推送到邮件
3. 在邮件中，企业名称为空时，显示为"——榜单或获奖信息"

## 🔧 实现逻辑

### 1. AI分析增强（`news/server/utils/newsAnalysis.js`）

#### 1.1 标签扩展
在类型标签中新增：
- **榜单**：各类榜单、排名、TOP榜单、排行榜等
- **获奖**：获奖、荣誉、认证、资质、称号等

#### 1.2 额外公众号特殊处理
当新闻来自额外公众号时，AI分析会：
- 在提示词中特别强调关注榜单、获奖、荣誉相关信息
- 如果内容涉及榜单或获奖，会添加相应标签

**代码位置：** `analyzeNewsSentimentAndType` 函数

**实现逻辑：**
```javascript
// 检查是否是额外公众号的新闻
const isAdditionalAccount = await db.query(
  `SELECT id FROM additional_wechat_accounts 
   WHERE wechat_account_id = ? 
   AND status = 'active' 
   AND delete_mark = 0`,
  [newsItem.wechat_account]
);

// 传递额外公众号标志给AI分析
const analysis = await this.analyzeNewsSentimentAndType(
  newsItem.title,
  newsItem.content,
  newsItem.source_url,
  isAdditionalAccount.length > 0
);
```

**AI提示词增强：**
```
**额外公众号新闻特殊处理（重要）：**
- 如果新闻内容涉及榜单、排名、获奖、荣誉、认证等信息，请务必添加"榜单"或"获奖"标签
- 榜单相关：各类榜单、排名、TOP榜单、排行榜等
- 获奖相关：获奖、荣誉、认证、资质、称号等
- 请仔细分析内容，确保不遗漏榜单或获奖相关信息
```

### 2. 邮件推送逻辑（`news/server/utils/scheduledEmailTasks.js`）

#### 2.1 查询条件扩展
在邮件查询中，增加对额外公众号新闻的查询：

**查询条件：**
- 新闻来自额外公众号（`wechat_account` 在 `additional_wechat_accounts` 表中）
- 企业名称为空（`enterprise_full_name IS NULL`）
- 包含"榜单"或"获奖"标签（`keywords` 字段中包含相应标签）

**SQL查询：**
```sql
-- 额外公众号新闻：包含"榜单"或"获奖"标签且企业名称为空
(nd.enterprise_full_name IS NULL 
 AND nd.wechat_account IN (
   SELECT wechat_account_id 
   FROM additional_wechat_accounts 
   WHERE status = 'active' 
   AND delete_mark = 0
 )
 AND (
   nd.keywords LIKE '%"榜单"%'
   OR nd.keywords LIKE '%"获奖"%'
   OR nd.keywords LIKE '%榜单%'
   OR nd.keywords LIKE '%获奖%'
 ))
```

#### 2.2 分组逻辑
在按企业分组时，对于企业名称为空且包含"榜单"或"获奖"标签的新闻，使用 `null` 作为分组键：

```javascript
// 如果企业名称为空且包含"榜单"或"获奖"标签，使用null作为分组键
let enterpriseName = news.enterprise_full_name;
if ((!enterpriseName || enterpriseName === '' || enterpriseName === 'null')) {
  // 检查是否包含"榜单"或"获奖"标签
  let keywords = [];
  if (news.keywords) {
    try {
      if (typeof news.keywords === 'string') {
        keywords = JSON.parse(news.keywords);
      } else if (Array.isArray(news.keywords)) {
        keywords = news.keywords;
      }
    } catch (e) {
      // 解析失败，忽略
    }
  }
  
  const hasAwardTag = keywords.some(k => k === '榜单' || k === '获奖');
  if (hasAwardTag) {
    enterpriseName = null; // 使用null作为分组键
  }
}

const groupKey = enterpriseName || null;
if (!newsByEnterprise[groupKey]) {
  newsByEnterprise[groupKey] = [];
}
newsByEnterprise[groupKey].push(news);
```

## 📊 功能流程

### 完整流程示例

1. **新闻同步**
   - 从额外公众号同步新闻
   - `enterprise_full_name` 为空（因为不是被投企业公众号）

2. **AI分析**
   - 检测到是额外公众号的新闻
   - AI分析时特别关注榜单、获奖信息
   - 如果内容涉及榜单或获奖，添加相应标签

3. **邮件查询**
   - 查询条件包括：
     - 被投企业公众号的新闻
     - 被投企业相关的新闻
     - **额外公众号的新闻（包含"榜单"或"获奖"标签且企业名称为空）**

4. **邮件生成**
   - 按企业分组
   - 企业名称为空且包含"榜单"或"获奖"标签的新闻，分组到 `null` 键
   - 在邮件中显示为"——榜单或获奖信息"

## ✅ 验证方法

### 1. 验证AI标签生成

```sql
-- 查看额外公众号的新闻，检查是否包含"榜单"或"获奖"标签
SELECT 
  nd.id,
  nd.title,
  nd.wechat_account,
  nd.keywords,
  nd.enterprise_full_name
FROM news_detail nd
WHERE nd.wechat_account IN (
  SELECT wechat_account_id 
  FROM additional_wechat_accounts 
  WHERE status = 'active' 
  AND delete_mark = 0
)
AND (
  nd.keywords LIKE '%"榜单"%'
  OR nd.keywords LIKE '%"获奖"%'
)
AND nd.delete_mark = 0
ORDER BY nd.created_at DESC
LIMIT 10;
```

### 2. 验证邮件查询

```sql
-- 查看会被推送到邮件的额外公众号新闻
SELECT 
  nd.id,
  nd.title,
  nd.wechat_account,
  nd.keywords,
  nd.enterprise_full_name
FROM news_detail nd
WHERE nd.enterprise_full_name IS NULL 
AND nd.wechat_account IN (
  SELECT wechat_account_id 
  FROM additional_wechat_accounts 
  WHERE status = 'active' 
  AND delete_mark = 0
)
AND (
  nd.keywords LIKE '%"榜单"%'
  OR nd.keywords LIKE '%"获奖"%'
)
AND nd.public_time >= DATE_SUB(CURDATE(), INTERVAL 1 DAY)
AND nd.public_time < CURDATE()
AND nd.delete_mark = 0;
```

### 3. 验证邮件显示

- 发送测试邮件
- 检查邮件中是否包含"——榜单或获奖信息"分组
- 确认该分组下的新闻都包含"榜单"或"获奖"标签

## 📝 注意事项

1. **标签识别**：AI分析需要准确识别榜单和获奖相关内容，确保不遗漏
2. **邮件推送**：只有包含"榜单"或"获奖"标签且企业名称为空的额外公众号新闻才会推送
3. **显示格式**：企业名称为空时，统一显示为"——榜单或获奖信息"
4. **兼容性**：使用 `LIKE` 查询而非 `JSON_CONTAINS`，确保MySQL版本兼容性
