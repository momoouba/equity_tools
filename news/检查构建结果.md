# 检查构建结果

## 问题
本地构建的 `index.html` 只有 487 字节，这看起来异常小。

## 检查步骤

### 1. 查看构建后的 index.html 内容

在本地 PowerShell 中执行：

```powershell
cd E:\USER\SUREAL\Desktop\equity_news\news\client
Get-Content dist\index.html
```

**正常情况下**，构建后的 `index.html` 应该包含类似这样的内容：

```html
<!doctype html>
<html lang="zh-CN">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>股权投资小工具锦集</title>
    <link rel="stylesheet" href="/assets/index-CtRj0PG0.css">
  </head>
  <body>
    <div id="root"></div>
    <script type="module" src="/assets/index-XjvSr8Nz.js"></script>
  </body>
</html>
```

### 2. 检查 assets 目录

```powershell
ls dist\assets
```

应该看到：
- `index-CtRj0PG0.css`
- `index-XjvSr8Nz.js`
- 以及其他 vendor 文件

### 3. 如果 index.html 内容不正确

如果构建后的 `index.html` 仍然包含 `/src/main.jsx` 而不是 `/assets/index-XjvSr8Nz.js`，说明 Vite 构建有问题。

**可能的原因：**
1. Vite 配置问题
2. 构建过程被中断
3. 需要清理后重新构建

**解决方案：**

```powershell
# 清理旧的构建
Remove-Item -Recurse -Force dist

# 重新构建
npm run build

# 再次检查
Get-Content dist\index.html
```

### 4. 如果构建正确，上传到服务器

如果构建后的 `index.html` 包含正确的资源引用（指向 `/assets/` 目录中的文件），那么：

1. 使用 WinSCP 将整个 `dist` 目录上传到服务器的 `/tmp/dist`
2. 在服务器上执行：

```bash
cd /opt/newsapp/news
VOLUME_PATH=$(docker volume inspect news_app_frontend | grep -i "Mountpoint" | awk '{print $2}' | tr -d '",')
sudo rm -rf "$VOLUME_PATH"/*
sudo cp -r /tmp/dist/* "$VOLUME_PATH/"
docker compose restart nginx
```
