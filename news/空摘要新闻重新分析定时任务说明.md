# 空摘要新闻重新分析定时任务说明

## 功能概述

新增了一个定时任务，用于自动检查并重新分析新闻摘要为空的新闻数据，确保所有有效新闻都有摘要内容。

## 实现文件

- **定时任务模块**: `server/utils/scheduledNewsReanalysisTasks.js`
- **初始化位置**: `server/index.js` 第220-224行

## 功能特性

### 1. 自动定时执行

- **执行时间**: 每天凌晨2点（Asia/Shanghai时区）
- **Cron表达式**: `0 2 * * *`
- **执行频率**: 每天执行一次

### 2. 查询条件

定时任务会查询满足以下条件的新闻：

- `news_abstract IS NULL OR news_abstract = ''` - 摘要为空
- `content IS NOT NULL AND content != ''` - 有正文内容
- `LENGTH(content) > 20` - 内容长度大于20字符
- `delete_mark = 0` - 未被删除

### 3. 处理逻辑

1. **查询空摘要新闻**: 每次处理最多50条记录
2. **过滤乱码内容**: 自动跳过内容被污染（乱码）的新闻
3. **智能分析**:
   - 如果有企业关联，使用 `processNewsWithEnterprise` 方法
   - 如果没有企业关联，使用 `processNewsWithoutEnterprise` 方法
4. **结果验证**: 分析后验证是否成功生成摘要
5. **错误处理**: 记录成功和失败的统计信息

### 4. 性能优化

- **分批处理**: 每次最多处理50条，避免一次性处理过多数据
- **延迟控制**: 每条新闻处理间隔1秒，避免API调用过于频繁
- **错误隔离**: 单条新闻处理失败不影响其他新闻

## 使用方法

### 自动执行

定时任务会在服务器启动时自动初始化，每天凌晨2点自动执行，无需手动干预。

### 手动执行（用于测试）

如果需要手动执行定时任务，可以在代码中调用：

```javascript
const { executeEmptyAbstractReanalysis } = require('./server/utils/scheduledNewsReanalysisTasks');

// 手动执行，处理50条记录
executeEmptyAbstractReanalysis(50).then(result => {
  console.log('执行结果:', result);
});
```

或者通过Node.js REPL测试：

```bash
cd /opt/newsapp/news
node
> const task = require('./server/utils/scheduledNewsReanalysisTasks');
> task.executeEmptyAbstractReanalysis(10).then(r => console.log(r));
```

## 配置说明

### 修改执行时间

如果需要修改定时任务的执行时间，可以编辑 `server/utils/scheduledNewsReanalysisTasks.js`：

```javascript
// 在 initializeScheduledTask 函数中修改 cronExpression 参数
// 例如：每天凌晨3点执行
initializeScheduledTask('0 3 * * *');
```

### 修改批处理数量

在 `server/index.js` 中修改每次处理的记录数：

```javascript
await executeEmptyAbstractReanalysis(100); // 改为每次处理100条
```

## 统计信息

执行结果包含以下统计信息：

- `total`: 查询到的新闻总数
- `processed`: 实际处理的新闻数
- `successCount`: 成功分析的新闻数
- `errorCount`: 失败的新闻数
- `duration`: 执行耗时（秒）
- `processedNews`: 处理详情（前20条）

## 注意事项

1. **API限制**: 由于调用AI分析接口，需要确保API配额充足
2. **执行时间**: 建议在业务低峰期执行（默认凌晨2点）
3. **数据量**: 如果空摘要新闻较多，可能需要多次执行才能处理完
4. **内容质量**: 只有有效内容（非乱码、长度足够）的新闻才会被分析

## 故障排查

### 定时任务未执行

1. 检查服务器日志，查看是否有初始化错误
2. 确认服务器时区设置为 `Asia/Shanghai`
3. 检查 cron 表达式是否正确

### 分析失败较多

1. 检查AI配置是否正确
2. 查看日志中的具体错误信息
3. 确认API配额是否充足
4. 检查新闻内容是否有效

### 性能问题

1. 如果处理速度较慢，可以减少每批处理的记录数
2. 增加处理间隔时间（当前为1秒）
3. 检查数据库查询性能
