# 管理员舆情信息功能说明

## 📋 功能概述

已成功实现管理员舆情信息页面，管理员可以查看系统中的所有舆情信息，而普通用户只能查看与自己创建的被投企业相关的舆情信息。

## 🎯 主要功能

### 1. 权限区分显示
- **管理员**：可以查看所有舆情信息
- **普通用户**：只能查看自己创建的被投企业相关的舆情信息

### 2. 页面标识
- 管理员页面显示橙色标识："（管理员 - 全部数据）"
- 普通用户页面显示蓝色标识："（我的企业相关）"

### 3. 管理员专属功能

#### 统计面板
- 显示总舆情数量
- 显示当前页数据量
- 显示总页数

#### 增强表格
- 额外显示"创建时间"列
- 额外显示"操作"列
- 在标题下方显示文章摘要（如果有）

#### 详情查看功能
- **详情按钮**：查看舆情的完整信息
  - 公众号名称
  - 微信账号
  - 发布时间
  - 创建时间
  - 文章标题
  - 文章摘要
  - 原文链接
  - 关键词列表

- **正文按钮**：查看文章完整正文
  - 文章标题
  - 发布信息
  - 完整正文内容（支持HTML格式）

#### 批量选择和AI重新分析功能
- **复选框选择**：在所有Tab页签中支持批量选择新闻
  - 表头全选/取消全选功能
  - 每行独立复选框
  - 实时显示已选择数量
  
- **AI重新分析**：对选中的新闻进行批量AI重新分析
  - 重新进行情绪分析
  - 重新生成类型标签和摘要
  - 重新评估企业关联性
  - 实时显示分析进度
  - 支持异步批量处理
  
- **适用范围**：所有时间维度的Tab页签
  - 昨日舆情
  - 本周舆情
  - 上周舆情
  - 本月舆情
  - 全部舆情

```javascript
// 获取用户信息并判断是否为管理员
const userData = localStorage.getItem('user')
const userInfo = JSON.parse(userData)
setIsAdmin(userInfo.role === 'admin')
```

2. **API调用区分**
```javascript
// 根据用户角色选择不同的API端点
const endpoint = isAdmin ? '/api/news/' : '/api/news/user-news'
```

3. **条件渲染**
```javascript
// 管理员专属列和功能
{isAdmin && <th>创建时间</th>}
{isAdmin && <th>操作</th>}
```

```javascript
const userRole = req.headers['x-user-role'] || 'user'
```

## 📊 数据流程

### 管理员访问流程
1. 用户登录 → 检查角色为 `admin`
2. 访问舆情页面 → 显示管理员标识和统计面板
3. 调用 `/api/news/` → 返回所有舆情数据
4. 显示增强表格 → 包含额外列和操作按钮

### 普通用户访问流程
1. 用户登录 → 检查角色为 `user`
2. 访问舆情页面 → 显示用户标识
3. 调用 `/api/news/user-news` → 返回用户相关舆情数据
4. 显示基础表格 → 标准列显示

## 🔒 安全机制

1. **前端权限控制**
   - 基于用户角色显示不同界面

2. **后端权限验证**
   - 检查请求头中的用户角色

3. **数据隔离**

## 🚀 使用方法

### 管理员登录
1. 使用管理员账号登录（默认：`admin` / `wenchao`）
2. 进入"舆情信息"页面
3. 可以看到所有舆情数据和管理功能

### 普通用户登录
1. 使用普通用户账号登录
2. 进入"舆情信息"页面
3. 只能看到与自己创建的被投企业相关的舆情

### 批量AI重新分析操作
1. **选择新闻**：
   - 在任意Tab页签中，勾选需要重新分析的新闻
   - 可点击表头复选框全选当前页所有新闻
   - 或单独勾选需要的新闻

2. **执行分析**：
   - 选中新闻后，页面右上角显示"AI分析(数量)"按钮
   - 点击按钮，确认后开始批量分析
   - 系统会显示确认对话框，提示将要分析的新闻数量

3. **查看进度**：
   - 分析开始后，页面顶部显示进度条
   - 实时显示：已处理数量/总数量、百分比
   - 显示当前正在处理的新闻标题

4. **完成处理**：
   - 分析完成后自动刷新新闻列表
   - 显示分析结果统计信息
   - 可查看更新后的情绪标签、关键词和摘要

## 📝 注意事项

1. **权限继承**：管理员在后端API中会自动降级为用户权限，如果前端调用用户接口

2. **数据同步**：舆情数据需要通过系统配置中的新闻接口同步获取

3. **性能优化**：大量数据时建议使用分页和搜索功能

4. **安全提醒**：生产环境中请修改默认管理员密码
