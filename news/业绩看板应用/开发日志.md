## 🎯 项目概述

基于业绩看板需求文档，开发完整的业绩看板管理系统，包含数据版本管理、指标展示、数据导出、分享功能等核心模块。系统采用前后端分离架构，集成到现有新闻舆情系统中。

### 核心原则

1. **独立文件管理**: 所有新增的业绩看板功能文件独立创建专属文件夹

3. **后端路径**: `server/routes/业绩看板应用/`
4. **公用修改标注**: 涉及修改原有公用文件处标注「业绩看板应用扩展」备注

#### 2. 版本管理路由 - `version.js` ✅
**开发时间**: 2026-03-05  
**核心功能**:
- ✅ 获取日期列表（`GET /dates`）- 查询所有历史月份
- ✅ 获取版本列表（`GET /?date=YYYY-MM-DD`）- 按日期查询版本号
- ✅ 获取版本历史（`GET /history`）- 包含已删除的版本
- ✅ 创建版本（`POST /`）- 支持多月份选择（最多 6 个）
- ✅ 锁定/解锁版本（`PATCH /:version/lock`）- Admin权限
- ✅ 删除版本（`DELETE /:version`）- 软删除 17 张关联表

**关键技术实现**:
```sql
-- 版本号生成（带数据库行锁防并发）
SELECT version FROM b_version 
WHERE DATE(b_date) = ? 
ORDER BY CAST(SUBSTRING_INDEX(version, 'V', -1) AS UNSIGNED) DESC 
LIMIT 1 FOR UPDATE;

-- 版本号格式：YYYYMMDDVxx（如 20260228V01）
const version = `${monthDate.replace(/-/g, '')}V${String(newVersionNum).padStart(2, '0')}`;
```

**软删除涉及的 17 张表**:
`b_version`, `b_investment_indicator`, `b_investment_sum`, `b_investor_list`, 
`b_manage_indicator`, `b_project_all`, `b_transaction_indicator`, `b_all_indicator`, 
`b_investment`, `b_ipo`, `b_manage`, `b_project`, `b_transaction`, 
`b_project_a`, `b_region_a`, `b_region`, `b_ipo_a`

## 🔑 关键技术实现

### 1. 版本号生成与并发控制

**问题**: 多用户同时创建同一月份的版本时，可能产生重复版本号

**解决方案**:
```javascript
await connection.beginTransaction();

// 使用 FOR UPDATE 行锁
const [maxVersionRow] = await connection.query(
  `SELECT version FROM b_version 
   WHERE DATE(b_date) = ? 
   ORDER BY CAST(SUBSTRING_INDEX(version, 'V', -1) AS UNSIGNED) DESC 
   LIMIT 1 FOR UPDATE`,
  [monthDate]
);

// 计算新版本号
let newVersionNum = 1;
if (maxVersionRow.length > 0) {
  const maxVersion = maxVersionRow[0].version;
  const match = maxVersion.match(/V(\d+)$/);
  if (match) {
    newVersionNum = parseInt(match[1]) + 1;
  }
}

const version = `${monthDate.replace(/-/g, '')}V${String(newVersionNum).padStart(2, '0')}`;
```

**特点**:
- 数据库事务保证原子性
- 行级锁防止并发冲突
- 版本号自动递增（V01, V02, ... V11, V12...）

### 2. 软删除机制

**场景**: 删除某个版本时，需要同步删除 17 张关联表的数据

**实现**:
```javascript
const tables = [
  'b_version', 'b_investment_indicator', 'b_investment_sum',
  'b_investor_list', 'b_manage_indicator', 'b_project_all',
  // ... 共 17 张表
];

for (const table of tables) {
  await db.execute(
    `UPDATE ${table} 
     SET F_DeleteMark = 1, F_DeleteUserId = ?, F_DeleteTime = NOW()
     WHERE version = ? AND F_DeleteMark = 0`,
    [userId, version]
  );
}
```

**优点**:
- 数据可恢复（物理删除不可逆）
- 审计追踪（记录删除人和删除时间）
- 查询简单（WHERE F_DeleteMark = 0）

### 3. SQL安全检查

**风险**: 用户配置的 SQL 可能包含危险操作

**防护机制**:
```javascript
// 1. 只允许SELECT语句
if (!upperSql.startsWith('SELECT')) {
  return res.status(400).json({ message: '只允许执行 SELECT 查询语句' });
}

// 2. 禁止危险关键字
const forbiddenKeywords = ['INSERT', 'UPDATE', 'DELETE', 'DROP', 'TRUNCATE', 'ALTER'];
for (const keyword of forbiddenKeywords) {
  if (upperSql.includes(keyword)) {
    return res.status(400).json({ message: `SQL 中包含禁止的关键字：${keyword}` });
  }
}

// 3. 参数化查询执行（防止注入）
const result = await db.query(config.sql_content, [test_date]);
```

### 4. 数据格式化统一规则

**需求**: 不同字段类型有不同的显示规则

**实现**:
```javascript
// 金额转亿（decimal类型字段）
const formatAmount = (val) => {
  if (val === null || val === undefined) return '/';
  if (val === 0) return '-';
  return (val / 100000000).toFixed(2);
};

// 整数格式化（int 类型字段）
const formatNumber = (val) => {
  if (val === null || val === undefined) return '/';
  if (val === 0) return '-';
  return Math.round(val).toLocaleString();
};

// 比例（TVPI/DPI/RVPI/MOC）
const formatRatio = (val) => {
  if (val === null || val === undefined) return '/';
  if (val === 0) return '-';
  return val.toFixed(2) + 'x';
};

// 百分比（NIRR/GIRR）
const formatPercent = (val) => {
  if (val === null || val === undefined) return '/';
  if (val === 0) return '-';
  return (val * 100).toFixed(2) + '%';
};
```

**应用场景**:
- 后端返回原始金额（元）
- 前端看板部分统一除以 100000000 转为亿
- 弹窗中的数据根据截图参考为准

### 三、Gross IRR 和 Net IRR 计算功能

**新增文件**: `server/routes/业绩看板应用/transactionIrr.js`

**核心功能**:

- 支持复杂现金流模式的稳健求解

**Net IRR 计算逻辑**（LP 视角）:
- 现金流出：`lp IS NOT NULL AND transaction_type = '实缴'`（负数）
- 现金流入：`lp IS NOT NULL AND transaction_type = '分配'`（正数）
- 终值：`lp IS NULL AND sub_fund IS NULL AND company IS NULL AND transaction_type = '未实现价值'`（正数）

**Gross IRR 计算逻辑**（Fund 视角）:
- 现金流出：`lp IS NULL AND transaction_type IN ('实缴','出资')`（负数）

- 终值：`lp IS NULL AND (sub_fund IS NOT NULL OR company IS NOT NULL) AND transaction_type = '未实现价值'`（正数）

**触发时机**: 在 `version.js` 的创建版本流程中，`b_transaction_indicator` 数据写入完成后自动调用

**文件**: `server/routes/业绩看板应用/version.js`

### 七、时间标准化（上海时间 UTC+8）

**文件**:
- `server/routes/业绩看板应用/version.js`
- `server/routes/业绩看板应用/scheduled.js`

**实现**:
```javascript
// 统一使用中国上海时间（UTC+8）
function getShanghaiNow() {
  const now = new Date();
  const shanghaiOffsetMinutes = -8 * 60; // UTC+8
  const localOffsetMinutes = now.getTimezoneOffset();
  const diffMs = (localOffsetMinutes - shanghaiOffsetMinutes) * 60 * 1000;
  return new Date(now.getTime() + diffMs);
}
```

**应用场景**:
- 版本创建时间 `F_CreatorTime`
- 定时任务计算「当前月份最后一天」日期
- 数据清理逻辑中的日期判断（是否为 1 日/4 日）
