> **部署方式（方案A）**：业绩看板以路由模块形式挂载到现有 Express 服务，在 `server/index.js` 中添加：
> ```javascript
> const performanceRoutes = require('./routes/performance');
> app.use('/api/performance', performanceRoutes);
> ```
> 无需新增容器或服务进程，共享同一 MySQL 连接池、用户体系和 `externalDb.js` 连接池。

## 4. 性能优化策略

### 4.1 数据库优化

1. **索引优化**
   - 所有表建立 `(version, F_DeleteMark)` 组合索引
   - 高频查询字段单独建立索引
   - 定期分析查询执行计划

2. **查询优化**
   - 使用覆盖索引减少回表
   - 大数据量查询强制使用索引
   - 避免 SELECT *，只查询需要的字段

3. **连接池配置**

   复用新闻舆情系统的 `db.js` 连接池（`connectionLimit: 10`），无需额外配置：
   ```javascript
   // server/db.js（现有系统，直接复用）
   pool = mysql.createPool({
     host: DB_HOST,
     port: DB_PORT,
     user: DB_USER,
     password: DB_PASSWORD,
     database: DB_NAME,
     waitForConnections: true,
     connectionLimit: 10,    // 最大连接数
     charset: 'utf8mb4'
   });
   ```
   > 如业绩看板作为独立服务部署，可适当调大 `connectionLimit`，建议不超过 20。

### 4.2 缓存策略

1. **内存缓存（替代Redis）**

   使用 Node.js 进程内 `Map` 实现简单 TTL 缓存（与现有系统一致，不引入 Redis 依赖）：

   ```javascript
   const cache = new Map();
   
   function setCache(key, value, ttlSeconds) {
     cache.set(key, { value, expireAt: Date.now() + ttlSeconds * 1000 });
   }
   
   function getCache(key) {
     const item = cache.get(key);
     if (!item) return null;
     if (Date.now() > item.expireAt) { cache.delete(key); return null; }
     return item.value;
   }
   ```

   **缓存Key规划**：
   ```
   performance:version:{version}:dashboard   TTL: 300s
   performance:config:indicators             TTL: 600s
   performance:dates:list                    TTL: 300s
   ```

2. **缓存更新策略**
   - 数据版本更新后清除相关 Key
   - 配置修改后清除配置缓存
   - 进程重启时缓存自动清空（无需额外处理）

### 4.3 前端优化

1. **虚拟滚动**
   - 基金产品横向滚动（基金数量 > 20）
   - 弹窗表格纵向滚动（数据量 > 50）

2. **懒加载**
   - 弹窗数据按需加载
   - 图表组件懒加载

3. **数据预加载**
   - 首页数据优先加载
   - 弹窗数据预加载（悬停时）

## 5. 安全设计

### 5.1 认证授权

业绩看板复用新闻舆情系统的认证机制，**不使用 JWT**，保持技术栈一致性。

**内部用户认证（登录态）**：
```http
X-User-Id: <userId>
```
- 前端登录后从接口获取 `user.id`，后续请求统一在 header 中携带
- 后端从 `req.headers['x-user-id']` 读取用户ID（参考 `middleware/auth.js`）

**外部接口鉴权（API Token）**：
```http
Authorization: Bearer <api_token>
# 或
X-Api-Token: <api_token>
```
- api_token 存储在 `users.api_token` 字段，注册时自动生成，永久有效
- 适用于分享页面对外接口（参考 `middleware/apiTokenAuth.js`）

**权限控制**：
- 通过 `users.account_status` 判断账号是否激活（`active`/`inactive`）
- Admin 用户通过 `users.role = 'admin'` 判断（数据库已有 role 字段，默认值为 `'user'`，初始 admin 账户的 account 字段值为 `'admin'`）
- 业绩看板权限判断示例：
  ```javascript
  // 检查是否为管理员
  const users = await db.query('SELECT role FROM users WHERE id = ?', [userId]);
  const isAdmin = users.length > 0 && users[0].role === 'admin';
  ```

### 5.2 SQL注入防护

1. **参数化查询**
   ```typescript
   // 正确
   const result = await repository.query(
     'SELECT * FROM b_manage WHERE version = ?',
     [version]
   );
   
   // 错误 - 存在SQL注入风险
   const result = await repository.query(
     `SELECT * FROM b_manage WHERE version = '${version}'`
   );
   ```

2. **SQL配置审核**
   - 禁止包含 DROP/DELETE/UPDATE/INSERT/TRUNCATE/ALTER
   - 只允许 SELECT 语句
   - 语法解析验证

### 5.3 数据脱敏

```typescript
// 投资人名称脱敏
function maskInvestorName(name: string): string {
  if (!name || name.length <= 4) return '***';
  return name.substring(0, 2) + '***' + name.substring(name.length - 2);
}

// 示例：上海国方私募基金管理有限公司 -> 上海***公司
```
