# 业绩看板数据库设计文档

## 1. 设计概述

### 1.1 数据库选型

- **数据库**: MySQL 8.0+
- **存储引擎**: InnoDB
- **字符集**: utf8mb4
- **排序规则**: utf8mb4_0900_ai_ci

### 1.2 设计原则

1. **软删除**: 所有业务表使用 `F_DeleteMark` 字段实现软删除
2. **乐观锁**: 使用 `F_Lock` 字段实现数据锁定
3. **审计字段**: 所有表包含创建人、创建时间、修改人、修改时间
4. **版本控制**: 业务数据按版本号隔离，支持多版本并存
5. **主键规范**: `F_Id` 使用 `varchar(50)` 类型，与新闻舆情系统的 ID 生成规则一致

   **ID 生成规则**（复用 `utils/idGenerator.js`）：
   ```
   格式：YYYYMMDDHHmmss + 5位自增序列
   示例：20260305143001200001（19位）
   ```
   > 注意：业绩看板的 `F_Id` 虽沿用原有业务表字段名，但内部实际使用上述 ID 生成规则，与新闻舆情系统保持完全一致。

> **多数据源说明**：`external_db_config_id` 字段复用 `external_db_config` 表（新闻舆情系统已有），通过 `utils/externalDb.js` 的 `createExternalPool` 建立连接。值为 NULL 时使用主库 `db.js` 连接池。

**索引设计**:
```sql
CREATE INDEX idx_exec_order ON b_sql(exec_order);
CREATE INDEX idx_delete_mark ON b_sql(F_DeleteMark);
```

**SQL执行器逻辑概要**（`utils/performanceSqlExecutor.js`）：
```javascript
async function executeSql(sqlConfig, params) {
  // 根据 external_db_config_id 选择连接池
  let pool;
  if (sqlConfig.external_db_config_id) {
    // 复用 externalDb.js
    const { getExternalPool, createExternalPool } = require('./externalDb');
    pool = getExternalPool(sqlConfig.external_db_config_id) 
      || await createExternalPool(dbConfig);
  } else {
    pool = db; // 主库
  }
  return pool.query(sqlConfig.sql_content, params);
}
```

```sql

CREATE INDEX idx_version ON {table_name}(version);

CREATE INDEX idx_delete_mark ON {table_name}(F_DeleteMark);

-- 组合索引（最常用查询条件）
CREATE INDEX idx_version_delete ON {table_name}(version, F_DeleteMark);
```

## 5. 数据清理策略

### 5.1 软删除数据清理

```sql
-- 清理1年前的软删除数据（物理删除）
DELETE FROM b_version 
WHERE F_DeleteMark = 1 
  AND F_DeleteTime < DATE_SUB(NOW(), INTERVAL 1 YEAR);

-- 其他表同理
```

### 5.2 定时任务清理逻辑

```sql
-- 删除最新版本的 F_CreatorUserId 为 null 且创建时间不是1日或4日的数据
DELETE FROM b_version 
WHERE version = (
  SELECT version FROM (
    SELECT version FROM b_version 
    WHERE F_DeleteMark = 0 
    ORDER BY b_date DESC, version DESC 
    LIMIT 1
  ) t
) 
AND F_CreatorUserId IS NULL 
AND DAY(F_CreatorTime) NOT IN (1, 4);
```
