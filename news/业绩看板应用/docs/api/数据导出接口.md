## 通用说明

### 导出模式

| 模式 | 数据量 | 处理方式 | 响应方式 |
|------|--------|----------|----------|
| 同步导出 | < 1000 条 | 即时生成 | 直接返回文件流 |
| 异步导出 | >= 1000 条 | 后台生成 | 返回任务ID，轮询进度 |

### 文件命名规范

```
{版本号}-{模块名称}-{下载日期}.xlsx
```

示例：`20260228V05-在管产品清单-20260305.xlsx`

### Excel 格式规范

- 文件格式：.xlsx (Office Open XML)
- 字符编码：UTF-8
- 日期格式：YYYY-MM-DD
- 金额格式：千分位分隔，保留2位小数
- 百分比格式：保留2位小数，带%符号
- 倍数格式：保留2位小数，带x后缀

## 1. 导出在管产品清单

### 基本信息

- **接口地址**: `POST /api/performance/exports/manager-funds`
- **接口说明**: 导出在管产品清单Excel，包含两个Sheet

### 请求参数

```typescript
interface ExportManagerFundsRequest {
  version: string;           // 版本号
}
```

### 响应数据

```typescript
interface ExportResponse {
  taskId: string;            // 任务ID
  status: 'processing' | 'completed' | 'failed';
  downloadUrl?: string;      // 下载链接（同步导出时直接返回）
  estimatedTime?: number;    // 预计完成时间（秒）
}
```

### Excel 结构

**Sheet1: 在管产品清单**

| 序号 | 列名 | 对应字段 | 说明 |
|------|------|----------|------|
| 1 | 时间条件 | b_date | 日期格式 |
| 2 | 版本号 | version | |
| 3 | 基金类型 | fund_type | |
| 4 | 基金名称 | fund | |
| 5 | 成立时间 | set_up_date | 日期格式 |
| 6 | 认缴规模 | sub_amount | 金额格式 |
| 7 | 本年新增认缴 | sub_add | 金额格式 |
| 8 | 实缴规模 | paid_in_amount | 金额格式 |
| 9 | 本年新增实缴 | paid_in_add | 金额格式 |
| 10 | 累计分配金额 | dis_amount | 金额格式 |
| 11 | 本年新增分配 | dis_add | 金额格式 |

**Sheet2: 数据明细表**

| 序号 | 列名 | 对应字段 |
|------|------|----------|
| 1 | 时间条件 | b_date |
| 2 | 版本号 | version |
| 3 | 基金名称 | fund |
| 4 | 投资人名称 | lp |
| 5 | spv名称 | spv |
| 6 | 子基金名称 | sub_fund |
| 7 | 被投企业名称 | company |
| 8 | 被投企业全称 | company_name |
| 9 | 子基金全称 | sub_fund_name |
| 10 | 交易类型 | transaction_type |
| 11 | 交易时间 | transaction_date |
| 12 | 交易金额 | transaction_amount |
| 13 | 分配成本 | capital |
| 14 | 分配收益 | profit |
| 15 | 其中:分红 | dividend |

### SQL参考

```sql
-- Sheet1
SELECT 
  b_date, version, fund_type, fund, set_up_date,
  sub_amount, sub_add, paid_in_amount, paid_in_add,
  dis_amount, dis_add
FROM b_manage
WHERE version = ? AND F_DeleteMark = 0
ORDER BY fund_type, set_up_date;

-- Sheet2
SELECT 
  b_date, version, fund, lp, spv, sub_fund, company,
  company_name, sub_fund_name, transaction_type,
  transaction_date, transaction_amount, capital, profit, dividend
FROM b_transaction
WHERE version = ? AND F_DeleteMark = 0
  AND transaction_type IN ('实缴','分配','认缴')
  AND lp IS NOT NULL
ORDER BY fund, transaction_date ASC
```

## 8. 获取导出进度

### 基本信息

- **接口地址**: `GET /api/performance/exports/:taskId/progress`
- **接口说明**: 查询异步导出任务的执行进度

### 响应数据

```typescript
interface ExportProgressResponse {
  taskId: string;
  status: 'pending' | 'processing' | 'completed' | 'failed';
  progress: number;          // 进度百分比 0-100
  message: string;           // 状态说明
  downloadUrl?: string;      // 下载链接（完成后返回）
  fileSize?: number;         // 文件大小（字节）
  createdAt: string;         // 创建时间
  completedAt?: string;      // 完成时间
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "taskId": "export-123456",
    "status": "processing",
    "progress": 65,
    "message": "正在写入Excel数据...",
    "createdAt": "2026-03-05 14:30:00"
  },
  "timestamp": 1704067200000
}
```

## 9. 下载导出文件

### 基本信息

- **接口地址**: `GET /api/performance/exports/:taskId/download`
- **接口说明**: 下载导出的Excel文件


### 响应头

```http

Content-Disposition: attachment; filename="20260228V05-在管产品清单-20260305.xlsx"
Content-Length: 10240
```

## 导出任务状态流转

```
pending -> processing -> completed
                    -> failed
```

| 状态 | 说明 |
|------|------|
| pending | 任务已创建，等待执行 |
| processing | 正在生成文件 |
| completed | 文件生成完成，可下载 |
| failed | 生成失败，可查看错误原因 |

## 性能优化建议

1. **大数据量导出**（> 10000 条）
   - 使用异步任务
   - 采用流式写入，分批处理
   - 每批 1000 条，避免内存溢出

2. **文件存储**
   - 临时文件存储于本地或 OSS
   - 设置 24 小时自动清理
   - 文件名使用 UUID 避免冲突

3. **并发控制**
   - 同一用户同时只能有一个导出任务
   - 系统总并发导出任务限制为 5 个
