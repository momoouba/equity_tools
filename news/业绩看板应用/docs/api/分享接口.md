# 分享接口

## 复用说明

> **与新闻舆情系统保持一致**：分享功能复用现有 `news_share_links` 表，通过新增 `link_type` 字段区分业绩看板分享（`'performance'`）与新闻舆情分享（`'news'`，默认值）。Token 生成使用 `crypto.randomBytes(32).toString('hex')`，存入数据库，不使用 JWT。

### news_share_links 表扩展

在现有表基础上，通过数据库迁移脚本新增以下字段：

```sql
ALTER TABLE news_share_links 
ADD COLUMN link_type VARCHAR(20) DEFAULT 'news' COMMENT '链接类型：news-新闻舆情分享，performance-业绩看板分享',
ADD COLUMN performance_version VARCHAR(300) NULL COMMENT '业绩看板版本号（link_type=performance时使用）',
ADD COLUMN can_export TINYINT(1) DEFAULT 0 COMMENT '是否允许导出：1-是，0-否';
```

## 接口列表

| 序号 | 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|------|
| 1 | 创建/更新分享链接 | POST | /performance/share/create | 创建或更新业绩看板分享链接 |
| 2 | 验证分享Token | GET | /performance/share/verify | 验证分享链接有效性 |

## 1. 创建/更新分享链接

### 基本信息

- **接口地址**: `POST /api/performance/share/create`
- **接口说明**: 为当前用户创建或更新业绩看板分享链接（每个用户只能有一条 `status=active` 的分享链接，与新闻舆情系统逻辑相同）
- **认证方式**: `X-User-Id` header（需要登录）

### 请求参数

```json
{
  "version": "20260228V05",
  "hasExpiry": true,
  "expiryTime": "2026-03-12 23:59:59",
  "hasPassword": false,
  "password": "",
  "canExport": true
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| version | string | 是 | 当前版本号，存入 performance_version 字段 |
| hasExpiry | boolean | 否 | 是否设置有效期，默认 false |
| expiryTime | string | 否 | 有效期至，格式 YYYY-MM-DD HH:mm:ss |
| hasPassword | boolean | 否 | 是否设置密码，默认 false |
| password | string | 否 | 访问密码（明文，后端 bcrypt 加密存储）|
| canExport | boolean | 否 | 是否允许被分享者导出数据，默认 false |

### 响应数据

```json
{
  "success": true,
  "message": "分享链接已创建",
  "data": {
    "shareId": "20260305143001200001",
    "shareToken": "a1b2c3d4e5f6...",
    "shareUrl": "https://example.com/performance/share/a1b2c3d4e5f6...",
    "version": "20260228V05",
    "hasExpiry": true,
    "expiryTime": "2026-03-12 23:59:59",
    "hasPassword": false,
    "canExport": true,
    "createdAt": "2026-03-05 14:30:00"
  }
}
```

### 实现说明

参考 `routes/newsShare.js` 的 `/create` 接口逻辑：

```javascript
// 检查是否已有活跃的分享链接
const existingLinks = await db.query(
  `SELECT id, share_token FROM news_share_links
   WHERE user_id = ? AND status = 'active' AND link_type = 'performance'
   ORDER BY created_at DESC LIMIT 1`,
  [userId]
);

// 有则更新，无则创建（使用 generateId('news_share_links')）
// Token 生成: crypto.randomBytes(32).toString('hex')
// 密码加密: bcrypt.hash(password, 10)
```

## 2. 验证分享Token

### 基本信息

- **接口地址**: `GET /api/performance/share/verify`
- **接口说明**: 验证分享链接有效性，用于分享页面加载时校验
- **认证方式**: 无需登录，通过 token 参数鉴权

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| token | string | 是 | 分享 Token，URL query 参数 |
| password | string | 否 | 访问密码（如果链接设了密码则必填）|

### 响应数据

**成功（无密码或密码正确）**：
```json
{
  "success": true,
  "data": {
    "valid": true,
    "shareId": "20260305143001200001",
    "version": "20260228V05",
    "hasExpiry": true,
    "expiryTime": "2026-03-12 23:59:59",
    "isExpired": false,
    "canExport": true
  }
}
```

**失败（密码错误 / 已过期 / Token无效）**：
```json
{
  "success": false,
  "message": "分享链接已过期"
}
```

### 实现说明

```javascript
// 查询分享链接
const links = await db.query(
  `SELECT * FROM news_share_links 
   WHERE share_token = ? AND link_type = 'performance' AND status = 'active'`,
  [token]
);
if (!links.length) return res.status(404).json({ success: false, message: 'Token 无效' });

const link = links[0];

// 检查有效期
if (link.has_expiry && link.expiry_time && new Date() > new Date(link.expiry_time)) {
  return res.status(200).json({ success: false, message: '分享链接已过期' });
}

// 检查密码
if (link.has_password && password) {
  const isValid = await bcrypt.compare(password, link.password_hash);
  if (!isValid) return res.status(200).json({ success: false, message: '密码错误' });
}
```

## 3. 获取分享页面数据

### 基本信息

- **接口地址**: `GET /api/performance/share/data`
- **接口说明**: 通过 token 获取业绩看板完整数据，无需登录
- **认证方式**: 无需登录，通过 token 参数鉴权（需先调用 verify 接口验证密码）

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| token | string | 是 | 分享 Token |

### 响应数据

与 `/api/performance/dashboard/manager` 等数据查询接口格式相同，但投资人姓名等敏感字段做脱敏处理。

### 数据脱敏规则

| 字段 | 脱敏规则 | 示例 |
|------|----------|------|
| 投资人名称（lp） | 保留首2字+末2字，中间替换为 `***` | 上海***公司 |
| lp_type | 保持原样，不脱敏 | - |

## 4. 关闭分享链接

### 基本信息

- **接口地址**: `POST /api/performance/share/close`
- **接口说明**: 将分享链接状态置为 `inactive`，访问链接将失效
- **认证方式**: `X-User-Id` header（需要登录，只能关闭自己创建的链接）

### 请求参数

```json
{
  "shareToken": "a1b2c3d4e5f6..."
}
```

### 响应数据

```json
{
  "success": true,
  "message": "分享链接已关闭"
}
```

## 安全考虑

1. **Token 强随机性**：使用 `crypto.randomBytes(32).toString('hex')` 生成 64 位十六进制 Token，与新闻舆情系统一致
2. **密码 bcrypt 加密**：使用 `bcrypt.hash(password, 10)` 存储，验证使用 `bcrypt.compare()`
3. **有效期控制**：可选，过期后自动拦截
4. **数据脱敏**：分享页面中投资人等敏感信息脱敏显示
5. **访问类型隔离**：`link_type = 'performance'` 确保不会与新闻舆情分享混用
