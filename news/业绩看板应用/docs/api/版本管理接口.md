# 版本管理接口

## 接口列表

| 序号 | 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|------|
| 1 | 获取日期列表 | GET | /versions/dates | 获取有数据的日期列表 |
| 2 | 获取版本列表 | GET | /versions | 获取指定日期下的版本列表 |
| 3 | 获取版本历史 | GET | /versions/history | 获取指定日期的版本历史 |
| 4 | 创建版本 | POST | /versions | 创建新版本并生成数据 |
| 5 | 锁定版本 | PATCH | /versions/:version/lock | 锁定指定版本 |
| 6 | 删除版本 | DELETE | /versions/:version | 软删除指定版本 |

## 1. 获取日期列表

### 基本信息

- **接口地址**: `GET /api/performance/versions/dates`
- **接口说明**: 获取数据库中所有有数据的日期列表，用于日期选择下拉框

### 请求参数

无

### 响应数据

```typescript
interface DateListResponse {
  dates: string[];  // 日期列表，格式 YYYY-MM-DD，按降序排列
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "dates": [
      "2026-02-28",
      "2026-01-31",
      "2025-12-31",
      "2025-11-30"
    ]
  },
  "timestamp": 1704067200000
}
```

### 实现说明

```sql
SELECT DISTINCT DATE(b_date) as date
FROM b_version
WHERE F_DeleteMark = 0
ORDER BY b_date DESC
```

## 2. 获取版本列表

### 基本信息

- **接口地址**: `GET /api/performance/versions`
- **接口说明**: 获取指定日期下的所有版本列表

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| date | string | 是 | 日期，格式 YYYY-MM-DD |

### 响应数据

```typescript
interface Version {
  version: string;           // 版本号，如 20260228V01
  bDate: string;             // 日期，格式 YYYY-MM-DD
  creatorId: string | null;  // 创建人ID，系统生成为 null
  creatorName: string;       // 创建人名称
  createTime: string;        // 创建时间，格式 YYYY-MM-DD HH:mm:ss
  isLocked: boolean;         // 是否锁定
}

interface VersionListResponse {
  versions: Version[];       // 版本列表，按版本号降序排列
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "versions": [
      {
        "version": "20260228V05",
        "bDate": "2026-02-28",
        "creatorId": "user123",
        "creatorName": "张三",
        "createTime": "2026-03-01 10:30:00",
        "isLocked": true
      },
      {
        "version": "20260228V04",
        "bDate": "2026-02-28",
        "creatorId": null,
        "creatorName": "系统",
        "createTime": "2026-02-28 00:00:00",
        "isLocked": false
      }
    ]
  },
  "timestamp": 1704067200000
}
```

### 实现说明

```sql
SELECT 
  version,
  b_date,
  F_CreatorUserId,
  F_CreatorTime,
  F_Lock
FROM b_version
WHERE F_DeleteMark = 0 
  AND DATE(b_date) = ?
ORDER BY CAST(SUBSTRING_INDEX(version, 'V', -1) AS UNSIGNED) DESC
```

## 4. 创建版本

### 基本信息

- **接口地址**: `POST /api/performance/versions`
- **接口说明**: 创建新版本并触发数据生成任务

### 请求参数

```typescript
interface CreateVersionRequest {
  date: string;              // 目标日期，格式 YYYY-MM-DD
  months: string[];          // 需要生成数据的月份列表，最多6个
}
```

### 请求示例

```json
{
  "date": "2026-02-28",
  "months": ["2026-02-28", "2026-01-31"]
}
```

### 响应数据

```typescript
interface CreateVersionResponse {
  taskId: string;            // 任务ID，用于查询生成进度
  versions: string[];        // 生成的版本号列表
  status: 'pending' | 'processing' | 'completed' | 'failed';
  estimatedTime: number;     // 预计完成时间（秒）
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "taskId": "task-123456",
    "versions": ["20260228V06", "20260131V03"],
    "status": "processing",
    "estimatedTime": 120
  },
  "timestamp": 1704067200000
}
```

### 错误码

| 错误码 | 说明 |
|--------|------|
| 400 | 参数错误：月份超过6个 |
| 409 | 版本生成冲突，已有任务在执行 |
| 1003 | 数据生成中，请稍后重试 |

### 实现说明

1. 使用数据库行锁防止并发生成同一月份的版本号
2. 按 `b_sql` 表的 `exec_order` 顺序执行 SQL
3. 每个 SQL 执行结果插入对应数据表
4. 所有操作在事务中执行，失败回滚

```sql
-- 获取下一个版本号（带锁）
START TRANSACTION;
SELECT version FROM b_version 
WHERE DATE(b_date) = ? 
ORDER BY CAST(SUBSTRING_INDEX(version, 'V', -1) AS UNSIGNED) DESC 
LIMIT 1 FOR UPDATE;

-- 生成新版本号并插入
INSERT INTO b_version (F_Id, version, b_date, ...) VALUES (...);

-- 按顺序执行 SQL 配置并插入数据
-- ...

COMMIT;
```

## 5. 锁定版本

### 基本信息

- **接口地址**: `PATCH /api/performance/versions/:version/lock`
- **接口说明**: 锁定或解锁指定版本

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| version | string | 是 | 版本号，路径参数 |
| locked | boolean | 是 | true=锁定，false=解锁 |

### 请求示例

```json
{
  "locked": true
}
```

### 响应数据

```typescript
interface LockVersionResponse {
  version: string;
  isLocked: boolean;
  operator: string;
  operateTime: string;
}
```

### 权限控制

- 普通用户（`role = 'user'`）：只能锁定，不能解锁
- Admin 用户（`role = 'admin'`）：可以锁定和解锁

Admin 判断方式（与新闻舆情系统一致）：
```javascript
const users = await db.query('SELECT role FROM users WHERE id = ?', [userId]);
const isAdmin = users.length > 0 && users[0].role === 'admin';
```

### 错误码

| 错误码 | 说明 |
|--------|------|
| 403 | 权限不足，无法解锁 |
| 404 | 版本不存在 |

## 6. 删除版本

### 基本信息

- **接口地址**: `DELETE /api/performance/versions/:version`
- **接口说明**: 软删除指定版本（将 F_DeleteMark 置为 1）

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| version | string | 是 | 版本号，路径参数 |

### 响应数据

```typescript
interface DeleteVersionResponse {
  version: string;
  deletedAt: string;
  operator: string;
}
```

### 实现说明

删除版本时，需要同时更新所有相关表的 F_DeleteMark 字段：

```sql
START TRANSACTION;

UPDATE b_version SET F_DeleteMark = 1, F_DeleteTime = NOW() WHERE version = ?;
UPDATE b_all_indicator SET F_DeleteMark = 1 WHERE version = ?;
UPDATE b_investment SET F_DeleteMark = 1 WHERE version = ?;
UPDATE b_investment_indicator SET F_DeleteMark = 1 WHERE version = ?;
UPDATE b_investment_sum SET F_DeleteMark = 1 WHERE version = ?;
UPDATE b_investor_list SET F_DeleteMark = 1 WHERE version = ?;
UPDATE b_ipo SET F_DeleteMark = 1 WHERE version = ?;
UPDATE b_ipo_a SET F_DeleteMark = 1 WHERE version = ?;
UPDATE b_manage SET F_DeleteMark = 1 WHERE version = ?;
UPDATE b_manage_indicator SET F_DeleteMark = 1 WHERE version = ?;
UPDATE b_project SET F_DeleteMark = 1 WHERE version = ?;
UPDATE b_project_a SET F_DeleteMark = 1 WHERE version = ?;
UPDATE b_project_all SET F_DeleteMark = 1 WHERE version = ?;
UPDATE b_region SET F_DeleteMark = 1 WHERE version = ?;
UPDATE b_region_a SET F_DeleteMark = 1 WHERE version = ?;
UPDATE b_transaction SET F_DeleteMark = 1 WHERE version = ?;
UPDATE b_transaction_indicator SET F_DeleteMark = 1 WHERE version = ?;

COMMIT;
```

### 错误码

| 错误码 | 说明 |
|--------|------|
| 1002 | 版本已被锁定，无法删除 |
| 404 | 版本不存在 |

## 数据模型

### Version 对象

```typescript
interface Version {
  F_Id: string;              // 主键
  version: string;           // 版本号
  b_date: Date;              // 日期
  F_CreatorUserId: string;   // 创建人ID
  F_CreatorTime: Date;       // 创建时间
  F_LastModifyUserId: string;// 修改人ID
  F_LastModifyTime: Date;    // 修改时间
  F_DeleteUserId: string;    // 删除人ID
  F_DeleteMark: number;      // 删除标记 0=未删除 1=已删除
  F_DeleteTime: Date;        // 删除时间
  F_Lock: number;            // 锁定状态 0=未锁定 1=已锁定
}
```
