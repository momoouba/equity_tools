# 通用规范

## 1. 请求规范

### 1.1 HTTP 方法

| 方法 | 用途 | 示例 |
|------|------|------|
| GET | 获取资源 | 查询数据列表 |
| POST | 创建资源 | 创建新版本 |
| PUT | 更新资源 | 更新配置 |
| DELETE | 删除资源 | 删除版本 |
| PATCH | 部分更新 | 锁定/解锁版本 |

### 1.2 请求头

**内部用户（登录态）**：
```http
Content-Type: application/json
X-User-Id: <userId>          // 登录用户ID，从登录接口获取
```

**外部接口（分享页面等无登录态场景）**：
```http
Content-Type: application/json
Authorization: Bearer <api_token>
# 或
X-Api-Token: <api_token>
```

> 与新闻舆情系统保持一致，不使用 JWT Token。api_token 存储在 `users.api_token` 字段中，注册时自动生成。

### 1.3 URL 规范

- 使用小写字母
- 单词间使用连字符 `-` 分隔
- 不使用下划线 `_`
- 资源使用名词复数形式

```
正确: /api/performance/versions
错误: /api/performance/getVersions
错误: /api/performance/get_versions
```

## 2. 响应规范

### 2.1 统一响应格式

> 与新闻舆情系统保持一致，使用 `success` 字段而非 `code` 字段作为主要成功标识。

```typescript
// 成功响应
interface SuccessResponse<T> {
  success: true;
  message: string;
  data: T;          // 业务数据，部分接口直接在顶层展开
}

// 失败响应
interface ErrorResponse {
  success: false;
  message: string;  // 错误描述
}
```

### 2.2 成功响应示例

```json
{
  "success": true,
  "message": "获取成功",
  "data": {
    "id": "20260305143001200001",
    "version": "20260228V01"
  }
}
```

### 2.3 错误响应示例

```json
{
  "success": false,
  "message": "版本号不能为空"
}
```

### 2.4 HTTP 状态码使用规范

| 场景 | HTTP状态码 | success字段 |
|------|-----------|------------|
| 操作成功 | 200 | true |
| 参数错误 | 400 | false |
| 未登录 | 401 | false |
| 权限不足 | 403 | false |
| 资源不存在 | 404 | false |
| 服务器错误 | 500 | false |
| 服务启动中 | 503 | false |

## 3. 错误信息规范

> 与新闻舆情系统一致，错误信息直接通过 `message` 字段描述，不使用独立的业务错误码数字体系。

### 3.1 常见业务错误信息参考

| 场景 | message 示例 |
|------|-------------|
| 版本号已存在 | 该日期今日版本已存在 |
| 版本已锁定无法删除 | 版本已被锁定，无法删除 |
| 数据生成中 | 数据生成中，请稍后重试 |
| SQL执行失败 | SQL执行失败：{错误原因} |
| 导出文件失败 | 导出文件生成失败 |
| 分享链接过期 | 分享链接已过期 |
| 未登录 | 未登录 |
| Token无效 | Token 无效或已失效 |

## 4. 分页规范

### 4.1 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | number | 否 | 页码，默认 1 |
| pageSize | number | 否 | 每页条数，默认 20，最大 100 |
| sort | string | 否 | 排序字段，如 `-createTime` 表示倒序 |

### 4.2 响应格式

```typescript
interface PaginatedResponse<T> {
  list: T[];              // 数据列表
  page: number;           // 当前页码
  pageSize: number;       // 每页条数
  total: number;          // 总条数
  totalPages: number;     // 总页数
  hasNext: boolean;       // 是否有下一页
  hasPrev: boolean;       // 是否有上一页
}
```

### 4.3 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [],
    "page": 1,
    "pageSize": 20,
    "total": 100,
    "totalPages": 5,
    "hasNext": true,
    "hasPrev": false
  },
  "timestamp": 1704067200000
}
```

## 5. 日期时间规范

### 5.1 格式标准

- **日期**: `YYYY-MM-DD` (如: 2026-03-05)
- **日期时间**: `YYYY-MM-DD HH:mm:ss` (如: 2026-03-05 14:30:00)
- **时间戳**: 毫秒级 Unix 时间戳 (如: 1704067200000)

### 5.2 时区处理

- 服务器统一使用 UTC 时间存储
- API 返回的时间戳统一为 UTC+8 (Asia/Shanghai)
- 前端展示时根据用户时区转换

## 6. 金额处理规范

### 6.1 存储单位

- 数据库存储：元（原始金额）
- API 返回：元（原始金额）
- 前端展示：根据需求转换为亿/万

### 6.2 精度处理

- 金额字段：保留 2 位小数
- 百分比字段：保留 2 位小数，如 5.87%
- 倍数字段：保留 2 位小数，如 1.49x

## 7. 空值处理规范

| 数据类型 | 空值表示 | 前端展示 |
|----------|----------|----------|
| 数字 | null | `/` |
| 金额 | null 或 0 | `-` |
| 字符串 | null 或 "" | `-` |
| 日期 | null | `-` |

## 8. 版本号格式

### 8.1 格式定义

```
YYYYMMDD + "V" + 两位数字序号
```

### 8.2 示例

- `20260228V01` - 2026年2月28日第1个版本
- `20260228V02` - 2026年2月28日第2个版本
- `20260228V11` - 2026年2月28日第11个版本

### 8.3 排序规则

按版本号数字部分降序排列，V 后的数字按数值比较（非字符串比较）。

```sql
ORDER BY CAST(SUBSTRING_INDEX(version, 'V', -1) AS UNSIGNED) DESC
```
