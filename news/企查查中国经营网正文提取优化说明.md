# 企查查中国经营网正文提取优化说明

## 📋 更新内容

针对企查查接入的中国经营网新闻，优化了正文提取逻辑，支持从多页标签中提取并合并正文内容。

## 🎯 功能说明

### 支持的标签格式

中国经营网的新闻正文分布在多个页面标签中，格式如下：
- `<div class="content_page p_page">` - 第一页（或唯一页）
- `<div class="content_page p_page_1">` - 第一页
- `<div class="content_page p_page_2">` - 第二页
- `<div class="content_page p_page_3">` - 第三页
- ... 以此类推

### 提取逻辑

1. **URL检测**：自动检测包含 `cbnet.com` 或 `cb.com.cn` 的URL
2. **多页提取**：查找所有包含 `content_page` 和 `p_page` 类的div元素
3. **内容合并**：按页面在HTML中的位置顺序合并所有页面的内容
4. **内容清理**：自动清理script、style等无关标签

## 📝 修改的文件

- `server/utils/newsAnalysis.js`
  - 在 `extractArticleContent` 方法中添加了中国经营网的特殊处理逻辑
  - 位置：在东方财富网处理逻辑之前

## 🔍 代码逻辑

```javascript
// 检测中国经营网URL
if (url && (url.includes('cbnet.com') || url.includes('cb.com.cn'))) {
  // 查找所有包含 content_page p_page 的div元素
  // 支持：content_page p_page、content_page p_page_1、content_page p_page_2 等
  // 按位置排序并合并所有页面的内容
}
```

## ⚠️ 注意事项

1. **URL匹配**：目前支持 `cbnet.com` 和 `cb.com.cn` 域名，如果中国经营网使用其他域名，需要添加相应的URL检测

2. **标签格式**：代码会匹配所有同时包含 `content_page` 和 `p_page`（可能带数字后缀）的div标签

3. **内容长度**：如果提取的内容少于100字符，会回退到通用提取逻辑

4. **多页合并**：所有页面的内容会按在HTML中的位置顺序合并，使用换行符 `\n` 分隔

## 🔧 故障排查

### 问题1：未检测到中国经营网URL

**检查**：
- 确认URL中包含 `cbnet.com` 或 `cb.com.cn`
- 查看日志：`docker compose logs app | grep "检测到中国经营网"`

**解决**：
- 如果使用其他域名，修改URL检测条件

### 问题2：未找到内容页面

**检查**：
- 查看日志：`docker compose logs app | grep "未找到中国经营网"`
- 检查HTML中是否确实存在 `content_page p_page` 类的div

**解决**：
- 检查实际的HTML结构
- 可能需要调整正则表达式匹配规则

### 问题3：提取的内容不完整

**检查**：
- 查看日志中显示的页面数量：`找到 X 个内容页面`
- 检查是否所有页面都被正确提取

**解决**：
- 确认HTML结构是否与预期一致
- 检查是否有嵌套的div导致提取不完整
