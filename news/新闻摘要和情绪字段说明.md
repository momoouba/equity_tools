# 新闻摘要和情绪字段功能说明

## 功能概述

在`news_detail`表中新增了两个字段，用于存储新闻的关键信息提取和情绪分析结果：

1. **新闻摘要（news_abstract）**：存储对文章内容的关键信息提取
2. **新闻情绪（news_sentiment）**：存储中性、正面、负面的情绪标签

## 数据库结构变更

### 新增字段

```sql
-- 新闻摘要字段
ALTER TABLE news_detail 
ADD COLUMN news_abstract TEXT COMMENT '新闻摘要（AI提取的关键信息）' AFTER content;

-- 新闻情绪字段
ALTER TABLE news_detail 
ADD COLUMN news_sentiment ENUM('positive', 'neutral', 'negative') DEFAULT 'neutral' 
COMMENT '新闻情绪：positive-正面，neutral-中性，negative-负面' AFTER news_abstract;

-- 为新字段创建索引
CREATE INDEX IF NOT EXISTS idx_news_sentiment ON news_detail(news_sentiment);
```

### 字段详细说明

| 字段名 | 数据类型 | 默认值 | 说明 |
|--------|----------|--------|------|
| `news_abstract` | TEXT | NULL | 新闻摘要，存储AI提取的关键信息 |
| `news_sentiment` | ENUM | 'neutral' | 新闻情绪：positive/neutral/negative |

## 技术实现

### 1. 数据库自动迁移

系统启动时会自动检查并添加新字段：

```javascript
// 检查并添加 news_abstract 字段
const [abstractColumns] = await dbPool.query(`
  SELECT COLUMN_NAME 
  FROM INFORMATION_SCHEMA.COLUMNS 
  WHERE TABLE_SCHEMA = DATABASE() 
  AND TABLE_NAME = 'news_detail' 
  AND COLUMN_NAME = 'news_abstract'
`);

if (abstractColumns.length === 0) {
  await dbPool.query(`
    ALTER TABLE news_detail 
    ADD COLUMN news_abstract TEXT COMMENT '新闻摘要（AI提取的关键信息）' AFTER content
  `);
}

// 检查并添加 news_sentiment 字段
const [sentimentColumns] = await dbPool.query(`
  SELECT COLUMN_NAME 
  FROM INFORMATION_SCHEMA.COLUMNS 
  WHERE TABLE_SCHEMA = DATABASE() 
  AND TABLE_NAME = 'news_detail' 
  AND COLUMN_NAME = 'news_sentiment'
`);

if (sentimentColumns.length === 0) {
  await dbPool.query(`
    ALTER TABLE news_detail 
    ADD COLUMN news_sentiment ENUM('positive', 'neutral', 'negative') DEFAULT 'neutral' 
    COMMENT '新闻情绪：positive-正面，neutral-中性，negative-负面' AFTER news_abstract
  `);
}
```

## 使用场景

### 1. 新闻摘要（news_abstract）

- **用途**：存储AI提取的文章关键信息摘要
- **数据来源**：
  - 可通过AI模型（如GPT、文心一言等）分析文章内容生成
  - 可通过关键词提取算法生成简要摘要
  - 可手动编辑和完善
- **应用场景**：
  - 快速浏览新闻要点
  - 新闻列表展示简要信息
  - 搜索和筛选相关内容

### 2. 新闻情绪（news_sentiment）

- **用途**：标识新闻的情绪倾向
- **取值范围**：
  - `positive`：正面新闻（如获奖、融资、业务增长等）
  - `neutral`：中性新闻（如一般性报道、事实陈述等）
  - `negative`：负面新闻（如争议、问题、风险等）
- **应用场景**：
  - 舆情监控和分析
  - 风险预警系统
  - 情绪趋势统计
  - 分类展示和筛选

## 数据处理策略

### 1. 历史数据处理

- **策略**：历史数据不做批量更新，保持原有字段为默认值
- **原因**：避免对大量历史数据进行处理，影响系统性能
- **后续处理**：可根据需要对特定历史数据进行单独分析和更新

### 2. 新数据处理

- **默认值**：
  - `news_abstract`：NULL（待后续AI分析填充）
  - `news_sentiment`：'neutral'（默认中性，待情感分析更新）
- **处理流程**：
  1. 数据同步时插入默认值
  2. 后台异步进行AI分析
  3. 更新摘要和情绪字段

### 3. 扩展功能

未来可以基于这两个字段实现：

1. **智能摘要生成**：
   - 集成AI模型自动生成新闻摘要
   - 支持多种摘要长度和风格

2. **情感分析引擎**：
   - 自动分析文章情绪倾向
   - 支持更细粒度的情绪分类

3. **舆情分析面板**：
   - 基于情绪字段统计正负面新闻比例
   - 生成情绪趋势图表

4. **智能推荐**：
   - 基于摘要内容进行相关新闻推荐
   - 根据情绪标签进行内容筛选

## 性能优化

1. **索引优化**：为`news_sentiment`字段创建了索引，提高按情绪筛选的查询性能
2. **存储优化**：`news_abstract`使用TEXT类型，适合存储中等长度的摘要内容
3. **查询优化**：ENUM类型的`news_sentiment`字段占用空间小，查询效率高

## 注意事项

1. **数据一致性**：新字段的添加不影响现有功能和数据
2. **向下兼容**：所有现有API和功能保持不变
3. **扩展性**：字段设计考虑了未来AI分析功能的集成需求
4. **性能影响**：新字段的添加对现有查询性能影响极小
