# 定时任务管理功能说明

## 📋 功能概述

定时任务管理功能是系统管理员用于管理和监控所有定时任务的统一入口。系统支持两种类型的定时任务：

1. **邮件发送舆情**：定时向指定收件人发送昨日舆情信息的邮件（包含Excel附件）
2. **新闻接口同步**：定时从新闻接口同步舆情数据

管理员可以查看、编辑、启用/禁用、手动执行和删除这些定时任务，并查看任务的执行日志。

## 🎯 主要功能

### 1. Tab页签切换

系统提供两个Tab页签，分别管理不同类型的定时任务：

- **邮件发送舆情**：管理邮件发送定时任务
- **新闻接口同步**：管理新闻数据同步定时任务

### 2. 邮件发送舆情任务管理

#### 功能特点
- 查看所有用户的邮件发送定时任务
- 显示任务详细信息（用户账号、收件人邮箱、邮件主题、发送频率、发送时间等）
- 计算并显示下次执行时间
- 支持编辑任务配置（发送频率、发送时间、启用状态）
- 支持手动立即执行任务
- 支持查看任务执行日志
- 支持删除任务（软删除）

#### 任务配置项
- **发送频率**：
  - `daily`：每天
  - `weekly`：每周（每周一）
  - `monthly`：每月（每月1号）
- **发送时间**：格式为 `HH:mm:ss`，例如 `09:00:00`
- **启用状态**：启用/禁用

#### 任务执行逻辑
- 自动获取用户可见的昨日舆情信息（根据用户权限过滤）
- 按企业分组新闻数据
- 生成HTML格式的邮件内容
- 生成Excel附件（包含完整的舆情信息）
- 发送邮件到配置的收件人邮箱
- 记录发送日志（成功/失败）

### 3. 新闻接口同步任务管理

#### 功能特点
- 查看所有新闻接口同步配置
- 显示配置详细信息（应用名称、接口类型、请求地址、同步频率等）
- 支持新增、编辑、复制、删除任务
- 支持手动立即执行同步任务
- 显示最后同步时间
- 计算并显示下次执行时间

#### 任务配置项
- **应用名称**：选择关联的应用
- **新闻接口类型**：
  - `新榜`：新榜接口
  - `企查查`：企查查舆情接口
- **请求地址**：新闻接口的API地址
- **同步频率**：
  - `daily`：每天
  - `weekly`：每周（需指定星期几）
  - `monthly`：每月（需指定日期）
- **星期**（仅weekly时）：`monday`（星期一）到 `sunday`（星期日）
- **日期**（仅monthly时）：
  - `first`：第一天
  - `last`：最后一天
  - `15`：15日（或其他数字日期）
- **同步时间**：格式为 `HH:mm:ss`，例如 `00:00:00`
- **启用状态**：启用/禁用

#### 任务执行逻辑
- 根据接口类型调用相应的同步函数
- 新榜接口：调用 `syncNewsData`
- 企查查接口：调用 `syncQichachaNewsData`
- 同步完成后更新 `last_sync_time` 字段

### 4. 任务操作功能

#### 编辑任务
- 修改任务的发送/同步频率、时间、启用状态
- 新闻接口同步任务还可以修改应用、接口类型、请求地址
- 保存后自动更新定时任务调度

#### 立即执行
- 管理员可以手动触发任务立即执行
- 邮件发送任务：立即发送昨日舆情邮件
- 新闻同步任务：立即执行数据同步

#### 查看日志
- 邮件发送任务可以查看发送日志
- 显示操作类型、发件人、收件人、主题、状态、错误信息、时间
- 支持分页查看历史日志

#### 复制任务（仅新闻接口同步）
- 支持复制现有任务配置
- 复制时会保留原始配置的所有字段（包括API密钥等）
- 方便快速创建相似配置

#### 删除任务
- 软删除，数据保留在数据库中
- 删除后任务不再执行
- 邮件发送任务删除后不再显示在列表中

```sql
CREATE TABLE recipient_management (
  id VARCHAR(19) PRIMARY KEY,
  user_id VARCHAR(19) NOT NULL,
  recipient_email TEXT NOT NULL,
  email_subject VARCHAR(255),
  send_frequency ENUM('daily', 'weekly', 'monthly') DEFAULT 'daily',
  send_time TIME DEFAULT '09:00:00',
  is_active TINYINT DEFAULT 1,
  is_deleted TINYINT DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  deleted_at DATETIME NULL,
  deleted_by VARCHAR(19) NULL

```

#### news_interface_config表（新闻接口同步任务）
```sql
CREATE TABLE news_interface_config (
  id VARCHAR(19) PRIMARY KEY,

  interface_type VARCHAR(50) DEFAULT '新榜',
  request_url VARCHAR(500) NOT NULL,

  send_time TIME DEFAULT '00:00:00',
  weekday VARCHAR(20) NULL,
  month_day VARCHAR(20) NULL,
  is_active TINYINT DEFAULT 1,
  is_deleted TINYINT DEFAULT 0,

  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 定时任务调度

#### Cron表达式生成

系统使用 `node-cron` 库进行定时任务调度：

```javascript
// 每天执行
daily: `${minutes} ${hours} * * *`

// 每周执行（每周一）
weekly: `${minutes} ${hours} * * 1`

// 每月执行（每月1号）
monthly: `${minutes} ${hours} 1 * *`
```

#### 下次执行时间计算

系统会根据发送频率、发送时间和当前时间计算下次执行时间：

- **每天**：如果今天的时间已过，则明天执行；否则今天执行
- **每周**：根据配置的星期几计算，如果已过则计算下周
- **每月**：根据配置的日期（第一天/最后一天/具体日期）计算，如果已过则计算下个月

## 🚀 使用方法

### 1. 查看定时任务

1. 管理员登录系统
2. 进入"定时任务管理"页面
3. 选择对应的Tab页签查看任务列表
4. 使用分页控件浏览所有任务

### 2. 编辑邮件发送任务

1. 在"邮件发送舆情"Tab中找到要编辑的任务
2. 点击"编辑"按钮
3. 修改发送频率、发送时间或启用状态
4. 点击"保存"完成修改

### 3. 新增新闻接口同步任务

1. 切换到"新闻接口同步"Tab
2. 点击"新增"按钮
3. 填写配置信息：
   - 选择应用名称
   - 选择接口类型（新榜/企查查）
   - 输入请求地址
   - 设置同步频率和时间
   - 如果是weekly，选择星期
   - 如果是monthly，选择日期
   - 设置启用状态
4. 点击"保存"创建任务

### 4. 复制新闻接口同步任务

1. 在任务列表中点击"复制"按钮
2. 系统会打开编辑弹窗，显示原始配置
3. 可以修改配置项
4. 点击"保存"创建新任务

### 5. 立即执行任务

1. 在任务列表中点击"立即执行"按钮
2. 确认执行操作
3. 系统会立即执行任务（发送邮件或同步数据）
4. 执行完成后显示提示信息

### 6. 查看任务日志

1. 在邮件发送任务列表中点击"日志"按钮
2. 查看该任务的发送历史记录
3. 可以查看成功/失败的详细信息
4. 使用分页浏览历史日志

### 7. 删除任务

1. 在任务列表中点击"删除"按钮
2. 确认删除操作
3. 任务将被软删除，不再执行

## 📝 注意事项

### 1. 任务执行时间
- 系统使用 `Asia/Shanghai` 时区
- 下次执行时间根据当前服务器时间计算
- 确保服务器时间准确

### 2. 邮件发送任务
- 只发送用户可见的昨日舆情信息
- 管理员可以看到所有新闻
- 普通用户只能看到自己创建的被投企业相关新闻
- 如果没有可见的新闻，任务会跳过发送

### 3. 新闻同步任务
- 同步频率为weekly时，必须配置weekday
- 同步频率为monthly时，必须配置month_day
- 同步时间默认为 `00:00:00`
- 最后同步时间会在同步完成后自动更新

### 4. 任务状态
- 只有启用状态的任务才会自动执行
- 禁用的任务不会自动执行，但可以手动执行
- 已删除的任务不会显示在列表中（邮件发送任务）

### 5. 定时任务更新
- 编辑任务后，系统会自动更新定时任务调度
- 删除任务后，系统会自动停止该任务的调度
- 服务器重启后，系统会自动初始化所有启用的定时任务

### 6. 性能考虑
- 大量任务时建议使用分页查看
- 立即执行任务可能需要较长时间，请耐心等待
- 邮件发送任务会生成Excel附件，注意服务器存储空间

### 7. 错误处理
- 任务执行失败会记录在日志中
- 可以查看日志了解失败原因
- 邮件发送失败不会影响其他任务的执行

## 🔍 常见问题

### Q: 为什么邮件发送任务没有执行？
**A:** 请检查：
1. 任务是否处于启用状态
2. 用户是否有可见的昨日舆情信息
3. 邮件配置是否正确
4. 查看任务日志了解详细错误信息

### Q: 如何修改任务的执行时间？
**A:** 点击"编辑"按钮，修改"发送时间"或"同步时间"字段，保存后系统会自动更新定时任务。

### Q: 新闻同步任务的星期和日期字段什么时候需要填写？
**A:** 
- 当同步频率为 `weekly` 时，需要选择星期（星期一至星期日）
- 当同步频率为 `monthly` 时，需要选择日期（第一天/最后一天/具体日期）
- 当同步频率为 `daily` 时，这两个字段不需要填写

### Q: 可以同时有多个相同时间的任务吗？
**A:** 可以。系统支持多个任务在同一时间执行，但需要注意服务器性能和资源限制。

### Q: 任务删除后可以恢复吗？
**A:** 邮件发送任务删除后不会显示在列表中，但数据仍在数据库中。新闻接口同步任务删除后可以通过数据库操作恢复。
