# 前端页面 503 错误排查

## 🔍 问题

访问 `/login` 页面返回 503，说明 Nginx 无法提供前端静态文件。

## 🚀 立即执行以下诊断命令

### 步骤1：检查前端文件是否存在

```bash
cd /opt/newsapp/news

# 检查前端构建文件是否存在
ls -la client/dist/

# 检查 index.html 是否存在
ls -la client/dist/index.html

# 检查 assets 目录
ls -la client/dist/assets/ | head -10
```

### 步骤2：检查 Nginx volume 中的文件

```bash
# 检查 Nginx volume 中的文件
sudo docker compose exec nginx ls -la /usr/share/nginx/html/

# 检查 index.html
sudo docker compose exec nginx cat /usr/share/nginx/html/index.html | head -20
```

### 步骤3：测试 Nginx 能否提供静态文件

```bash
# 测试根路径
curl -v http://localhost/

# 测试 index.html
curl -v http://localhost/index.html
```

### 步骤4：检查前端 volume 挂载

```bash
# 检查 docker-compose.yml 中的 volume 配置
grep -A 5 "app_frontend" docker-compose.yml

# 检查 volume 是否存在
sudo docker volume ls | grep frontend
```

### 步骤5：重新构建前端（如果文件不存在或过旧）

```bash
cd /opt/newsapp/news

# 进入前端目录
cd client

# 安装依赖（如果需要）
npm install

# 构建前端
npm run build

# 返回项目根目录
cd ..

# 检查构建结果
ls -la client/dist/
```

### 步骤6：更新前端文件到 volume

```bash
# 方法1：重新构建 Docker 镜像（包含最新前端文件）
sudo docker compose build app
sudo docker compose up -d app

# 方法2：手动复制文件到 volume（如果 volume 存在）
# 先查找 volume 路径
VOLUME_NAME="news_app_frontend"
VOLUME_PATH=$(sudo docker volume inspect $VOLUME_NAME 2>/dev/null | grep -i "Mountpoint" | awk '{print $2}' | tr -d '",')
if [ -n "$VOLUME_PATH" ]; then
  echo "Volume 路径: $VOLUME_PATH"
  sudo cp -r client/dist/* "$VOLUME_PATH/"
  sudo docker compose restart nginx
fi
```

## 🔧 快速修复方案

### 方案1：重新构建前端并更新镜像

```bash
cd /opt/newsapp/news

# 1. 构建前端
cd client
npm run build
cd ..

# 2. 重新构建 Docker 镜像
sudo docker compose build app

# 3. 删除旧的 volume（如果存在）
sudo docker compose down
sudo docker volume rm news_app_frontend 2>/dev/null || true

# 4. 重新启动
sudo docker compose up -d

# 5. 等待 30 秒
sleep 30

# 6. 测试
curl http://localhost/
```

### 方案2：使用快速更新脚本（如果存在）

```bash
cd /opt/newsapp/news

# 如果存在更新脚本
if [ -f "deploy/update-frontend-only.sh" ]; then
  chmod +x deploy/update-frontend-only.sh
  ./deploy/update-frontend-only.sh
else
  echo "脚本不存在，使用方案1"
fi
```
