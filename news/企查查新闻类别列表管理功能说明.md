# 企查查新闻类别列表管理功能说明

## 📋 功能概述

企查查新闻类别列表管理功能允许管理员在系统中维护企查查新闻的类别编码和描述映射关系。类别数据存储在数据库中，支持新增、编辑、删除和批量导入等操作，系统在调用接口时会使用这些数据进行类别匹配和映射。

## 🎯 功能特点

### 1. 类别数据存储
- 类别数据存储在 `qichacha_news_categories` 数据库表中
- 每个类别包含：类别编码（唯一）、类别描述、创建时间、更新时间
- 系统初始化时会自动导入175个默认类别

### 2. 类别管理功能
- **列表展示**：分页显示所有类别，支持搜索
- **新增类别**：手动添加新的类别编码和描述
- **编辑类别**：修改已有类别的描述（编码不可修改）
- **删除类别**：删除不再使用的类别
- **Excel批量导入**：支持Excel文件批量导入类别数据
  - 提供标准Excel模板下载
  - 自动检测重复的类别编码（重复的不导入）
  - 导入完成后显示详细结果（成功、重复、失败）

### 3. 数据同步机制
- 类别映射工具（`qichachaCategoryMapper.js`）优先从数据库加载类别数据
- 如果数据库中没有数据，则使用默认映射作为后备
- 类别数据更新后会自动清除缓存，确保使用最新数据
- 缓存有效期为5分钟，减少数据库查询

## 🔧 技术实现

### 数据库表结构

```sql
CREATE TABLE qichacha_news_categories (
  id VARCHAR(19) PRIMARY KEY COMMENT '数据ID：年月日时分秒+5位自增序列',
  category_code VARCHAR(50) NOT NULL UNIQUE COMMENT '类别编码',
  category_name VARCHAR(255) NOT NULL COMMENT '类别描述',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  INDEX idx_category_code (category_code)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 后端API接口

#### 1. 获取类别列表
```javascript
GET /api/system/qichacha-news-categories?page=1&pageSize=50&search=关键词
```

**请求参数：**
- `page`：页码（默认1）
- `pageSize`：每页数量（默认50）
- `search`：搜索关键词（可选，搜索类别编码或描述）

**返回格式：**
```json
{
  "success": true,
  "data": [
    {
      "id": "20250101120000001",
      "category_code": "10000",
      "category_name": "信用预警",
      "created_at": "2025-01-01T12:00:00.000Z",
      "updated_at": "2025-01-01T12:00:00.000Z"
    }
  ],
  "total": 175,
  "page": 1,
  "pageSize": 50
}
```

#### 2. 获取单个类别
```javascript
GET /api/system/qichacha-news-category/:id
```

#### 3. 创建类别
```javascript
POST /api/system/qichacha-news-category
{
  "category_code": "10000",
  "category_name": "信用预警"
}
```

**验证规则：**
- `category_code`：必填，不能为空
- `category_name`：必填，不能为空
- `category_code`：必须唯一，如果已存在会返回错误

#### 4. 更新类别
```javascript
PUT /api/system/qichacha-news-category/:id
{
  "category_name": "信用预警（更新）"
}
```

**说明：**
- 可以更新 `category_name`（类别描述）
- `category_code`（类别编码）不可修改
- 如果更新 `category_code` 且与其他记录重复，会返回错误

#### 5. 删除类别
```javascript
DELETE /api/system/qichacha-news-category/:id
```

#### 6. 下载导入模板
```javascript
GET /api/system/qichacha-news-categories/template
```

**返回：** Excel文件（`.xlsx`格式）

**模板格式：**
- 第一行：表头（类别编号、类别描述）
- 第二行开始：示例数据
- 列宽：类别编号（15字符）、类别描述（30字符）

#### 7. 批量导入类别（Excel文件）
```javascript
POST /api/system/qichacha-news-categories/import
Content-Type: multipart/form-data

FormData:
  file: <Excel文件>
```

**请求说明：**
- 使用 `multipart/form-data` 格式上传Excel文件
- 支持 `.xlsx` 和 `.xls` 格式
- 文件大小限制：5MB

**Excel文件格式要求：**
- 第一行必须是表头：`类别编号`、`类别描述`
- 从第二行开始为数据行
- 类别编号和类别描述都是必填字段

**返回格式：**
```json
{
  "success": true,
  "message": "导入完成：成功 2 条，重复 1 条，失败 0 条",
  "data": {
    "success": 2,
    "duplicate": [
      {
        "row": 3,
        "category_code": "10000",
        "category_name": "信用预警"
      }
    ],
    "failed": 0,
    "errors": []
  }
}
```

**返回字段说明：**
- `success`: 成功导入的数量
- `duplicate`: 重复的类别列表（包含行号、类别编号、类别描述），这些类别不会导入
- `failed`: 失败的数量
- `errors`: 错误详情列表（包含行号、类别编号、错误信息）

**导入逻辑：**
1. **验证Excel格式**：检查表头是否为"类别编号"、"类别描述"
2. **批量查询已存在类别**：一次性查询所有已存在的类别编码（提高性能）
3. **逐行处理数据**：
   - 验证类别编号和类别描述是否为空
   - 检查类别编号是否已存在（与数据库中已有数据对比）
   - **如果类别编号已存在**：标记为重复，不导入（不会更新已有数据）
   - **如果类别编号不存在**：创建新类别
4. **返回结果**：包含成功、重复、失败的详细统计
5. **清除缓存**：如果成功导入数据，自动清除类别映射缓存

### 类别映射工具

**文件位置：** `news/server/utils/qichachaCategoryMapper.js`

**主要函数：**
- `getCategoryMap()`：异步获取类别映射（优先从数据库加载）
- `getCategoryMapSync()`：同步获取类别映射（使用缓存或默认映射）
- `convertCategoryCodeToChinese(categoryCode)`：将类别编码转换为中文（同步）
- `convertCategoryCodeToChineseAsync(categoryCode)`：将类别编码转换为中文（异步）
- `convertCategoryCodesToChinese(categoryCodes)`：将多个类别编码转换为中文数组（同步）
- `convertCategoryCodesToChineseAsync(categoryCodes)`：将多个类别编码转换为中文数组（异步）
- `clearCategoryMapCache()`：清除类别映射缓存

**缓存机制：**
- 类别映射数据会缓存5分钟
- 在类别创建、更新、删除、导入后会自动清除缓存
- 确保使用最新的类别数据

## 🎨 用户界面

### 功能入口
- **路径**：系统配置 → 企查查接口配置 → 企查查新闻类别列表（第二个Tab）

### 界面功能

#### 1. 列表展示
- 表格显示所有类别
- 列包括：类别编码、类别描述、创建时间、更新时间、操作
- 支持分页显示（每页50条）
- 支持搜索（按类别编码或描述搜索）

#### 2. 操作按钮
- **刷新**：刷新类别列表
- **下载模板**：下载Excel导入模板
- **导入**：打开批量导入弹窗
- **新增类别**：打开新增类别表单

#### 3. 行操作
- **编辑**：编辑类别描述
- **删除**：删除类别（需确认）

#### 4. 新增/编辑表单
- **类别编码**：必填，必须唯一（编辑时不可修改）
- **类别描述**：必填
- **操作**：保存、取消

#### 5. 批量导入弹窗
- **下载模板**：点击按钮下载Excel模板

  - 失败的记录及错误信息
- **操作**：导入、取消、确定（查看结果后）

## 🚀 使用方法

### 1. 查看类别列表

1. 进入"系统配置"页面
2. 点击"企查查接口配置"Tab
3. 点击"企查查新闻类别列表"Tab（第二个Tab）
4. 查看所有类别列表
5. 可以使用搜索框搜索特定类别

### 2. 新增类别

1. 在类别列表页面，点击"新增类别"按钮
2. 填写类别编码和类别描述
3. 点击"创建"按钮保存
4. 如果类别编码已存在，会提示错误

### 3. 编辑类别

1. 在类别列表中找到要编辑的类别
2. 点击"编辑"按钮
3. 修改类别描述（类别编码不可修改）
4. 点击"更新"按钮保存

### 4. 删除类别

1. 在类别列表中找到要删除的类别
2. 点击"删除"按钮
3. 确认删除操作
4. 删除后，该类别的映射关系将不再可用

### 5. 批量导入类别（Excel）

1. 在类别列表页面，点击"下载模板"按钮
2. 下载Excel模板文件
3. 按照模板格式填写类别数据：
   - 第一行是表头（类别编号、类别描述），不要修改
   - 从第二行开始填写数据
   - 类别编号和类别描述都是必填字段
4. 点击"导入"按钮
5. 选择填写好的Excel文件
6. 点击"导入"提交
7. 查看导入结果：
   - **成功导入**：显示成功导入的数量
   - **重复的类别**：显示重复的类别列表（包含行号、类别编号、类别描述），这些类别不会导入
   - **导入失败**：显示失败的记录及错误原因

**Excel模板示例：**

| 类别编号 | 类别描述 |
|---------|---------|
| 10000   | 信用预警 |
| 10001   | 承诺失信 |

**注意事项：**
- 必须使用系统提供的模板格式
- 表头必须完全匹配（类别编号、类别描述）
- 类别编号不能为空
- 类别描述不能为空
- 如果类别编号已存在，该行数据不会导入（不会更新已有数据）
- 支持 `.xlsx` 和 `.xls` 格式
- 文件大小不能超过5MB

## 📊 数据流程

### 类别数据使用流程

```
企查查接口返回新闻数据
    ↓
获取新闻的Category字段（类别编码）
    ↓
调用convertCategoryCodeToChinese()转换
    ↓
优先从数据库qichacha_news_categories表查询
    ↓
如果数据库中有数据：使用数据库中的映射
如果数据库中没有数据：使用默认映射（qichachaCategoryMapper.js）
    ↓
返回中文类别名称
    ↓
存储到news_detail表的news_category字段
```

### 类别数据更新流程

```
管理员在界面中更新类别数据
    ↓
调用后端API（创建/更新/删除/导入）
    ↓
更新数据库qichacha_news_categories表
    ↓
调用clearCategoryMapCache()清除缓存
    ↓
下次查询时使用最新数据
```

## 🔍 应用场景

### 1. 企查查新增类别
- 当企查查接口新增了新的类别编码时
- 管理员可以在系统中添加新的类别映射
- 确保新类别的新闻能够正确显示中文名称

### 2. 修改类别描述
- 当需要修改某个类别的描述时
- 可以直接在系统中编辑
- 修改后立即生效，无需重启服务

### 3. 批量导入类别（Excel）
- 当需要一次性导入大量类别数据时
- 可以使用Excel批量导入功能
- 下载模板 → 填写数据 → 上传导入
- 系统会自动检测重复，避免重复导入
- 导入完成后显示详细结果，方便核对

### 4. 删除无效类别
- 当某个类别不再使用时
- 可以删除该类别
- 删除后，该类别的映射关系将不再可用

### 5. 数据迁移
- 从其他系统迁移类别数据时
- 可以导出为Excel格式
- 按照模板格式整理后导入
- 系统会自动处理重复数据

## 📝 注意事项

### 1. 类别编码唯一性
- 类别编码必须唯一
- 如果尝试创建重复的类别编码，会返回错误
- 编辑时不能修改类别编码

### 2. 数据一致性
- 类别数据更新后会自动清除缓存
- 确保系统使用最新的类别数据
- 如果数据库中没有数据，系统会使用默认映射

### 3. Excel导入格式
- 必须使用系统提供的Excel模板
- 表头格式：第一行必须是"类别编号"、"类别描述"
- 数据格式：从第二行开始填写数据
- 必填字段：类别编号和类别描述都不能为空
- 文件格式：支持 `.xlsx` 和 `.xls` 格式
- 文件大小：不能超过5MB

### 4. 导入重复处理
- 系统会根据类别编码判断是否重复
- **重复的类别不会导入**（不会更新已有数据）
- 导入结果中会详细列出所有重复的类别（包含行号、类别编号、类别描述）
- 如果需要更新已有类别的描述，请使用"编辑"功能

### 5. 删除影响
- 删除类别后，该类别的映射关系将不再可用
- 已存储的新闻数据中的类别名称不会改变
- 新同步的新闻如果使用已删除的类别编码，将无法正确映射

### 6. 性能考虑
- 类别映射数据会缓存5分钟
- 减少数据库查询，提高性能
- 类别数据更新后会自动清除缓存
- Excel导入时会批量查询已存在的类别编码，提高导入性能
