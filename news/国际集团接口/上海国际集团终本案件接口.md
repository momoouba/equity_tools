终本案件查询接口
接口基础信息
接口说明：上海国际企业数据总线系统的终本案件查询接口实现了通过标准接口查询上海国际集团外部融合平台终本案件信息数据的功能。
接口地址（测试环境 - 内网）：http://114.141.181.181:8000/dofp/v2/ipaas/query/finalCase
请求方式：POST
正文格式：application/json; charset=UTF-8
Header 头信息：与 1.12 接口一致
请求报文格式
json
{
    "query_type": "queryByCodeAndDate",
    "exec_idtfn_cd":"91620100MA74AADC6G",
    "query_date" : "2025-12-15"
}

属性	名称	数据类型	是否必填	说明
query_type	查询方式	String	是	queryByCode:根据编码查询 / queryByCodeAndDate:根据编码和日期查询
exec_idtfn_cd	被执行人唯一识别码	String	是	
query_date	查询日期	String	是	格式为：yyyy-MM-dd

接口调用与日期补拉规则
1. 接口由定时任务或任务调度系统按天触发，**正常情况**：当天 \(T\) 日调用时，`query_date` 取前一日 \(T-1\) 的日期。
2. 系统需要记录**上一次成功调用时实际使用的 `query_date`**（记为 `last_query_date`），而不是仅记录调用时间。
3. 当再次触发接口时，应从 `last_query_date` 的下一天开始，直到本次调用日的前一日，**逐日补拉**，每个自然日调用一次接口：
   - 例如：上一次请求时间为 2026-02-27，使用的 `query_date` 为 2026-02-26，今天为 2026-03-02，
   - 则本次触发时需要调用 3 次接口，`query_date` 分别为：2026-02-27、2026-02-28、2026-03-01。
4. **节假日场景**：节假日期间不会触发调用，节后第一次触发时，需要按照上述规则，
   - 从节前最后一次查询的 `query_date` 的下一天开始，
   - 一直到节后首次触发日的前一日为止，
   - 对中间所有自然日执行补拉，确保节假日前一天以及节假日期间的数据不会遗漏。
