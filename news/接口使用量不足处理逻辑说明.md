# 接口使用量不足处理逻辑说明

## 📋 功能概述

当接口使用量不足时（如配额用完、调用次数超限等），系统会：
1. **先保存已获取的数据**：将已经成功获取并处理的数据保存到数据库
2. **停止后续调用**：不再继续调用接口，避免浪费
3. **返回已处理结果**：返回已处理的数据统计和错误信息

## 🔧 实现逻辑

### 1. 新榜接口处理（`news/server/routes/news.js`）

#### 1.1 接口返回错误检查

**位置：** `executeNewsSyncForConfig` 函数

**检查时机：**
- 接口返回 `code !== 0` 时
- 检查错误消息中是否包含使用量不足的关键词

**错误识别关键词：**
- 中文：`使用量`、`配额`、`不足`、`次数`
- 英文：`quota`、`limit`

**处理逻辑：**
```javascript
if (response.data.code !== 0) {
  const errorMsg = response.data.msg || response.data.message || '接口返回错误';
  
  // 检查是否是使用量不足的错误
  const isQuotaExceeded = errorMsg.includes('使用量') || 
                         errorMsg.includes('配额') || 
                         errorMsg.includes('不足') ||
                         errorMsg.includes('quota') ||
                         errorMsg.includes('limit') ||
                         errorMsg.includes('次数');
  
  if (isQuotaExceeded && totalSynced > 0) {
    // 先返回已处理的数据
    return {
      success: true,
      message: `接口使用量不足，已处理 ${totalSynced} 条数据`,
      data: {
        synced: totalSynced,
        total: uniqueAccounts.length,
        errors: errors,
        quotaExceeded: true
      }
    };
  }
}
```

#### 1.2 异常捕获处理

**位置：** `executeNewsSyncForConfig` 函数的 catch 块

**检查时机：**
- 接口调用抛出异常时
- 检查错误类型和HTTP状态码

**错误识别：**
- HTTP状态码：`403`（权限不足）、`429`（请求过多）
- 错误消息中包含使用量不足的关键词

**处理逻辑：**
```javascript
catch (error) {
  // 检查是否是使用量不足的错误
  const isQuotaExceeded = errorMessage.includes('使用量') || 
                         errorMessage.includes('配额') || 
                         errorMessage.includes('不足') ||
                         errorType === '403-权限不足' ||
                         (error.response && (error.response.status === 403 || error.response.status === 429));
  
  if (isQuotaExceeded && totalSynced > 0) {
    // 先返回已处理的数据
    return {
      success: true,
      message: `接口使用量不足，已处理 ${totalSynced} 条数据`,
      data: {
        synced: totalSynced,
        total: uniqueAccounts.length,
        errors: errors,
        quotaExceeded: true
      }
    };
  }
}
```

## 📊 功能特点

### 1. 数据保护
- **优先保存数据**：已获取的数据会先保存到数据库
- **避免数据丢失**：不会因为使用量不足而丢失已获取的数据

### 2. 智能识别
- **多语言支持**：识别中文和英文的使用量不足错误
- **多状态码支持**：识别HTTP 403、429等状态码
- **关键词匹配**：通过错误消息中的关键词识别

### 3. 优雅降级
- **部分成功**：即使部分失败，也返回已处理的数据
- **明确提示**：返回结果中包含 `quotaExceeded: true` 标志
- **错误记录**：记录详细的错误信息，便于排查

## 🎯 使用场景

### 场景1：新榜接口使用量不足

**情况：**
- 正在同步100个公众号
- 已成功同步50个公众号的数据
- 第51个公众号调用时返回"使用量不足"错误

**处理：**
1. 保存已同步的50个公众号的数据
2. 停止后续调用
3. 返回结果：`已处理 X 条数据，接口使用量不足`

### 场景2：企查查接口使用量不足

**情况：**
- 正在同步100个企业
- 已成功同步30个企业的数据
- 第31个企业调用时返回"配额不足"错误

**处理：**
1. 保存已同步的30个企业的数据
2. 停止后续调用
3. 返回结果：`已处理 X 条数据，接口使用量不足`

## 📝 返回结果格式

### 成功返回（使用量不足但已处理数据）

```json
{
  "success": true,
  "message": "接口使用量不足，已处理 50 条数据",
  "data": {
    "synced": 50,
    "total": 100,
    "errors": [
      {
        "account": "xxx",
        "message": "接口使用量不足"
      }
    ],
    "quotaExceeded": true
  }
}
```

### 字段说明

- `success`: `true` - 表示操作成功（即使使用量不足）
- `message`: 提示信息，说明已处理的数据量
- `data.synced`: 已成功同步的数据条数
- `data.total`: 总数据量
- `data.errors`: 错误列表
- `data.quotaExceeded`: `true` - 标识为使用量不足

## ⚠️ 注意事项

1. **数据完整性**：已获取的数据会完整保存，不会丢失
2. **错误识别**：依赖错误消息和状态码，如果接口返回的错误格式不同，可能需要调整识别逻辑
3. **后续处理**：使用量不足后，需要等待配额恢复或增加配额后再次同步
4. **日志记录**：所有使用量不足的情况都会记录在日志中
