# 浏览器 503 错误解决方案

## ✅ 已确认

- ✅ 应用服务器正常运行（`localhost:3001` 返回 200 OK）
- ✅ Nginx 代理正常（`localhost/api/system/basic-config` 返回 200 OK）
- ❌ 浏览器中仍然显示 503 错误

## 🔍 问题原因

这是**浏览器缓存**导致的。浏览器缓存了之前的 503 错误响应。

## 🚀 解决方案

### 方法1：清除浏览器缓存（推荐）

**Chrome/Edge：**
1. 按 `F12` 打开开发者工具
2. 右键点击浏览器刷新按钮
3. 选择"清空缓存并硬性重新加载"（Empty Cache and Hard Reload）

**或者：**
1. 按 `Ctrl + Shift + Delete` 打开清除浏览数据
2. 选择"缓存的图片和文件"
3. 点击"清除数据"
4. 刷新页面（`Ctrl + F5`）

### 方法2：使用无痕模式测试

```bash
# 打开无痕窗口测试，确认是否是缓存问题
# Chrome: Ctrl + Shift + N
# Edge: Ctrl + Shift + P
```

### 方法3：强制刷新

- Windows/Linux: `Ctrl + F5` 或 `Ctrl + Shift + R`
- Mac: `Cmd + Shift + R`

### 方法4：清除特定网站的缓存

1. 按 `F12` 打开开发者工具
2. 切换到 "Application" 标签
3. 左侧选择 "Storage" > "Clear site data"
4. 点击 "Clear site data"
5. 刷新页面

## 🎯 验证修复

清除缓存后：
1. 刷新页面
2. 打开开发者工具（F12）> Network 标签
3. 查看 API 请求，应该都是 200 OK，不再是 503

## 📝 如果清除缓存后仍然有 503

如果清除缓存后仍然有 503 错误，请检查：

1. **检查浏览器控制台的具体错误**
   - 打开开发者工具（F12）> Console 标签
   - 查看具体的错误信息

2. **检查 Network 标签**
   - 打开开发者工具（F12）> Network 标签
   - 刷新页面
   - 查看哪些请求返回 503
   - 点击失败的请求，查看 Response 内容

3. **检查请求头**
   - 在 Network 标签中，点击失败的请求
   - 查看 Request Headers，确认请求是否正确

4. **测试特定 API**
   ```bash
   # 在服务器上测试浏览器访问的完整 URL
   curl -v -H "Host: news.gf-dsai.com" http://localhost/api/system/basic-config
   ```

## ⚠️ 注意事项

- `/vite.svg` 的 503 错误可能是正常的（如果文件不存在）
- 主要关注 `/api/*` 路径的请求
- 如果只有部分 API 返回 503，可能是那些接口有特殊问题
