# 大模型新闻分析功能说明

## 功能概述

本系统集成了大模型AI功能，用于对新闻进行智能分析，包括情绪分析、内容分类、摘要生成和企业关联性分析。系统支持多种AI模型提供商，并提供完整的配置管理界面。

## 核心功能

### 1. AI模型配置管理

**位置**: 系统配置 → AI模型配置

**功能特点**:
- 支持多种AI提供商（阿里云千问、OpenAI、百度文心一言、腾讯混元）
- 完整的模型参数配置（温度、Token数、Top P等）
- 配置测试功能，确保模型连接正常
- 支持多个配置并存，可灵活切换

**配置项说明**:
- **配置名称**: 自定义配置名称，便于识别
- **提供商**: AI服务提供商选择
- **模型名称**: 具体的模型版本
- **API类型**: Chat或Completion类型
- **API密钥**: 模型访问密钥
- **API端点**: 模型服务地址
- **温度**: 控制输出随机性（0.0-2.0）
- **最大Token数**: 单次请求最大Token限制
- **Top P**: 核采样参数（0.0-1.0）
- **应用类型**: 新闻分析或通用

### 2. 新闻智能分析

#### 2.1 有被投企业的新闻分析

对于已关联被投企业的新闻，系统会进行：

**情绪分析**:
- `positive`: 正面新闻（获奖、融资、业务增长等）
- `neutral`: 中性新闻（一般性报道、事实陈述等）
- `negative`: 负面新闻（争议、问题、风险等）

**类型分类**:
- 企业发展重点节点
- 企业荣誉
- 产品发布
- 融资消息
- 合作伙伴
- 技术创新
- 市场拓展
- 人事变动
- 财务报告
- 行业分析
- 广告推广
- 其他

**摘要生成**:
- 提取文章关键信息
- 生成100字左右的精炼摘要
- 保留核心要点和关键数据

#### 2.2 无被投企业的新闻分析

对于来自第三方公众号的新闻，系统会：

**企业关联性分析**:
- 分析新闻内容与数据库中被投企业的关联度
- 计算相关性得分（百分比）
- 筛选相关度30%以上的企业

**数据处理策略**:
- 如果没有相关企业：保持`enterprise_full_name`为空
- 如果有1家相关企业：更新该字段为企业全称
- 如果有多家相关企业：复制多条记录，每条对应一家企业

**后续分析**:
- 对所有记录进行情绪分析
- 进行内容类型分类
- 生成新闻摘要

### 3. 自动化工作流

#### 3.1 数据同步后自动分析

- 每次新闻数据同步完成后，自动触发AI分析
- 异步处理，不影响同步响应速度
- 只分析新同步的数据，提高效率

#### 3.2 手动分析功能

**批量分析**:
- 管理员可手动触发批量分析
- 可设置分析条数限制
- 提供详细的分析统计报告

**单条分析**:
- 支持对特定新闻进行重新分析
- 适用于修正分析结果或测试功能

### 4. 分析统计面板

**统计指标**:
- 总新闻数量
- 已分析新闻数量
- 待分析新闻数量
- 情绪分布统计
- 最近7天分析趋势

**待分析列表**:
- 显示所有待分析的新闻
- 支持分页浏览
- 可直接对单条新闻进行分析

## 技术架构

### 1. 数据库设计

**ai_model_config表**:
```sql
CREATE TABLE ai_model_config (
  id VARCHAR(19) PRIMARY KEY,
  config_name VARCHAR(255) NOT NULL,
  provider VARCHAR(100) NOT NULL,
  model_name VARCHAR(255) NOT NULL,
  api_type VARCHAR(50) NOT NULL,
  api_key TEXT NOT NULL,
  api_endpoint VARCHAR(500) NOT NULL,
  temperature DECIMAL(3,2) DEFAULT 0.7,
  max_tokens INT DEFAULT 2000,
  top_p DECIMAL(3,2) DEFAULT 1.0,
  is_active TINYINT DEFAULT 1,
  application_type ENUM('news_analysis', 'general') DEFAULT 'news_analysis',
  -- 其他字段...
);
```

**news_detail表新增字段**:
- `news_abstract`: TEXT - 新闻摘要
- `news_sentiment`: ENUM('positive', 'neutral', 'negative') - 新闻情绪
- `keywords`: JSON - 新闻类型标签（替换原关键词）

### 2. API接口

**AI配置管理**:
- `GET /api/ai-config` - 获取配置列表
- `POST /api/ai-config` - 新增配置
- `PUT /api/ai-config/:id` - 更新配置
- `DELETE /api/ai-config/:id` - 删除配置
- `POST /api/ai-config/:id/test` - 测试配置
- `GET /api/ai-config/models/available` - 获取可用模型

**新闻分析**:
- `POST /api/news-analysis/analyze` - 批量分析
- `POST /api/news-analysis/analyze/:id` - 单条分析
- `GET /api/news-analysis/stats` - 分析统计
- `GET /api/news-analysis/pending` - 待分析列表

### 3. 核心算法

**情绪分析算法**:
- 基于文章标题和内容进行综合分析
- 考虑关键词、语境和行业特点
- 输出情绪标签和判断理由

**企业关联性算法**:
- 文本相似度计算
- 实体识别和匹配
- 行业领域关联分析
- 相关性得分计算

**摘要生成算法**:
- 关键信息提取
- 重要性排序
- 内容压缩和重组
- 可读性优化

## 使用指南

### 1. 初始配置

1. **配置AI模型**:
   - 进入"系统配置" → "AI模型配置"
   - 点击"新增配置"
   - 填写阿里云千问的配置信息
   - 测试连接确保配置正确
   - 启用配置

2. **验证功能**:
   - 进入新闻分析统计页面
   - 查看待分析新闻数量
   - 手动触发小批量分析测试

### 2. 日常使用

1. **自动分析**:
   - 系统会在每次新闻同步后自动分析
   - 无需人工干预

2. **手动分析**:
   - 在分析统计页面可手动触发
   - 适用于补充分析或重新分析

3. **结果查看**:
   - 在新闻信息页面查看分析结果
   - 情绪标签、类型标签、摘要等

### 3. 监控和维护

1. **性能监控**:
   - 关注分析成功率和错误率
   - 监控API调用频率和成本
   - 观察分析质量和准确性

2. **配置优化**:
   - 根据分析效果调整模型参数
   - 切换不同的模型进行对比
   - 优化提示词和分析逻辑

## 注意事项

### 1. API成本控制

- AI模型调用会产生费用，建议设置合理的Token限制
- 监控每日API调用量，避免超出预算
- 可通过温度参数控制输出质量和成本平衡

### 2. 分析质量

- 定期检查分析结果的准确性
- 对于重要新闻可进行人工复核
- 根据反馈持续优化分析逻辑

### 3. 系统性能

- AI分析是异步处理，不影响正常业务
- 大批量分析时注意API频率限制
- 建议在业务低峰期进行大规模分析

### 4. 数据安全

- API密钥需要妥善保管
- 定期更换访问密钥
- 确保网络连接安全
