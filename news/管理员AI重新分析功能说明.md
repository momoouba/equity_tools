# 管理员AI重新分析功能说明

## 功能概述

在管理员的舆情信息管理页面新增了AI重新分析功能，专门用于重新分析额外公众号中已被关联企业的新闻数据，使用优化后的AI算法提高企业关联的准确性。

## 功能位置

**访问路径**：舆情信息 → 管理员统计面板 → "AI重新分析"按钮

## 主要功能

### 1. **时间范围选择**
- **开始时间**：选择要重新分析的新闻创建时间起点
- **结束时间**：选择要重新分析的新闻创建时间终点
- **默认范围**：最近7天（可自定义调整）

### 2. **处理数量控制**
- **数量限制**：1-1000条，控制单次处理的新闻数量
- **默认值**：100条
- **用途**：避免一次处理过多数据导致系统负载过高

### 3. **预览模式**
- **预览功能**：勾选"预览模式"可查看将要处理的数据
- **安全机制**：建议先预览，确认无误后再执行
- **详细信息**：显示将要处理的新闻标题、当前关联企业等

### 4. **智能重新分析**
- **清除错误关联**：自动清除不合理的企业关联
- **重新评估**：使用优化后的AI算法重新分析企业关联性
- **准确性提升**：应用30%关联度阈值和二次验证机制

## 使用流程

### 步骤1：打开功能
1. 以管理员身份登录系统
2. 进入"舆情信息"页面
3. 在管理员统计面板中点击"AI重新分析"按钮

### 步骤2：设置参数
1. **选择时间范围**：
   - 开始时间：如 2025-11-15
   - 结束时间：如 2025-11-21
   
2. **设置处理数量**：
   - 建议首次使用时设置较小数量（如50条）
   
3. **启用预览模式**：
   - 勾选"预览模式"进行安全预览

### 步骤3：预览分析
1. 点击"预览"按钮
2. 查看分析结果：
   - 总计数量
   - 将要处理的新闻列表
   - 当前企业关联情况

### 步骤4：执行分析
1. 确认预览结果无误
2. 取消勾选"预览模式"
3. 点击"执行重新分析"
4. 等待处理完成

## 处理对象

### **目标数据**
- ✅ 来自额外公众号的新闻（additional_wechat_accounts表中的公众号）
- ✅ 已被关联企业的新闻（enterprise_full_name不为空）
- ✅ 指定时间范围内的新闻
- ✅ 未被删除的有效数据

### **排除数据**
- ❌ 来自被投企业公众号的新闻
- ❌ 未关联企业的新闻
- ❌ 已删除的新闻数据

## 分析逻辑

### 1. **清空现有分析**
```sql
UPDATE news_detail 
SET enterprise_full_name = NULL, 
    news_abstract = NULL, 
    news_sentiment = 'neutral', 
    keywords = NULL 
WHERE id = ?
```

### 2. **重新企业关联分析**
- 使用优化后的AI提示词
- 应用严格的30%关联度阈值
- 进行二次验证（检查企业名称是否在内容中出现）

### 3. **生成新的分析结果**
- 情绪分析（正面/中性/负面）
- 内容类型分类
- 新闻摘要生成
- 企业关联评估

## API接口

### **后端接口**
```
POST /api/news-analysis/reanalyze-additional
```

### **请求参数**
```json
{
  "startDate": "2025-11-15 00:00:00",
  "endDate": "2025-11-21 23:59:59",
  "limit": 100,
  "dryRun": true
}
```

### **响应格式**
```json
{
  "success": true,
  "message": "重新分析完成：成功处理 45 条，总计 50 条",
  "data": {
    "total": 50,
    "processed": 45,
    "failed": 5,
    "results": [...]
  }
}
```

## 注意事项

### 1. **性能考虑**
- 单次处理建议不超过200条
- 处理过程中会有1秒延迟，避免API频率限制
- 大批量数据建议分批处理

### 2. **安全机制**
- 必须以管理员身份操作
- 建议先使用预览模式
- 处理过程不可逆，请谨慎操作

### 3. **最佳实践**
- 定期检查和清理错误关联
- 优先处理最近的数据
- 关注处理结果，及时调整参数

### 4. **故障排除**
- 如果处理失败，检查AI模型配置
- 网络超时可能导致部分失败
- 查看服务器日志获取详细错误信息
