# 微信公众号图片识别优化说明

## 问题描述

在分析日志中发现，微信公众号文章的图片没有被正确抓取和执行AI分析。具体表现为：

1. **图片提取问题**：前端页面中的图片标签（`<img>`）没有被正确提取
2. **图片识别未执行**：即使提取到图片，也没有执行AI图片识别分析

### 问题示例

前端HTML中的图片标签格式：
```html
<section style="text-align: center; margin-bottom: 0px;">
  <img class="rich_pages wxw-img" 
       data-imgfileid="100003619" 
       data-ratio="10.058333333333334" 
       data-s="300,640" 
       data-src="https://mmbiz.qpic.cn/mmbiz_jpg/.../640?wx_fmt=jpeg&from=appmsg#imgIndex=0" 
       data-type="jpeg" 
       data-w="1080" 
       src="https://mmbiz.qpic.cn/mmbiz_jpg/.../640?wx_fmt=jpeg&from=appmsg&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=0" 
       ...>
</section>
```

图片URL格式：
- `data-src`: `https://mmbiz.qpic.cn/mmbiz_jpg/.../640?wx_fmt=jpeg&from=appmsg#imgIndex=0`
- `src`: `https://mmbiz.qpic.cn/mmbiz_jpg/.../640?wx_fmt=jpeg&from=appmsg&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=0`

## 解决方案

### 1. 优化图片提取逻辑

**文件：** `news/server/utils/wechatArticleExtractor.py`

**修改位置：** `_extract_images` 方法

**改进点：**

1. **优化URL提取顺序**：
   - 优先使用 `data-src`（微信公众号懒加载图片的原始高清图）
   - 其次使用 `src`（已加载的图片）
   - 最后使用 `data-original`（备用）

2. **增强URL处理**：
   - 正确处理HTML实体编码（如 `&amp;` 转换为 `&`）
   - 移除URL锚点（`#imgIndex=0` 等）
   - 清理微信公众号图片URL中的无效参数（`imgIndex`, `wxfrom`, `wx_lazy`, `tp`）
   - 保留重要参数（如 `wx_fmt` 格式参数）

3. **增强日志输出**：
   - 记录找到的img标签数量
   - 记录每个图片使用的URL来源（data-src/src/data-original）
   - 记录URL清理过程
   - 记录最终提取的图片数量和去重后的数量

### 2. 优化图片识别流程

**文件：** `news/server/utils/wechatArticleExtractor.py`

**修改位置：** `extract_article_content` 方法中的图片识别部分

**改进点：**

1. **增强日志输出**：
   - 记录图片识别模型配置状态
   - 记录每张图片的识别过程
   - 记录识别成功/失败的原因
   - 记录API调用的详细过程

2. **错误处理优化**：
   - 区分网络请求错误和其他错误
   - 输出详细的错误堆栈信息
   - 记录API响应的详细信息

### 3. 优化图片识别API调用

**文件：** `news/server/utils/wechatArticleExtractor.py`

**修改位置：** `_recognize_image_text` 方法

**改进点：**

1. **增强日志输出**：
   - 记录图片下载过程（URL、大小、类型）
   - 记录base64转换过程
   - 记录API请求和响应的详细信息
   - 记录响应解析过程

2. **错误处理优化**：
   - 区分网络错误和API错误
   - 输出详细的错误信息
   - 记录API响应的原始内容（用于调试）

## 关键代码修改

### 1. 图片URL提取优化

```python
# 优先使用data-src（微信公众号懒加载图片），然后是src（已加载的图片），最后是data-original
data_src = img.get('data-src')
src = img.get('src')
data_original = img.get('data-original')

img_url = data_src or src or data_original

# 记录使用的URL来源
url_source = 'data-src' if data_src else ('src' if src else 'data-original')
print(f"图片 {idx + 1} 使用 {url_source} 属性，原始URL: {img_url[:100]}...", file=sys.stderr)

# 移除HTML实体编码
import html
img_url = html.unescape(img_url)

# 移除URL锚点
if '#' in img_url:
    img_url = img_url.split('#')[0]
```

### 2. 微信公众号图片URL清理

```python
# 对于微信公众号图片，清理可能影响下载的参数
if 'mmbiz.qpic.cn' in img_url or 'mmbizurl.cn' in img_url:
    from urllib.parse import urlparse as parse_url, urlencode, parse_qs
    parsed_url = parse_url(img_url)
    query_params = parse_qs(parsed_url.query)
    # 移除可能影响下载的参数，但保留格式参数（wx_fmt）
    params_to_remove = ['imgIndex', 'wxfrom', 'wx_lazy', 'tp']
    for param in params_to_remove:
        if param in query_params:
            del query_params[param]
    # 重新构建URL
    new_query = urlencode(query_params, doseq=True)
    if new_query:
        img_url = f"{parsed_url.scheme}://{parsed_url.netloc}{parsed_url.path}?{new_query}"
    else:
        img_url = f"{parsed_url.scheme}://{parsed_url.netloc}{parsed_url.path}"
```

## 使用说明

### 1. 配置图片识别模型

要使用图片识别功能，必须在AI模型配置中添加图片识别模型：

1. **进入系统管理** → **AI配置**
2. **添加新配置** 或 **编辑现有配置**
3. **设置配置信息**：
   - **用途类型**：选择 `image_recognition`（图片识别）
   - **模型名称**：推荐使用 `Qwen2.5-VL-32B-Instruct` 或其他支持图片识别的模型
   - **API Endpoint**：配置正确的API地址
   - **API Key**：配置API密钥
   - **API类型**：通常选择 `chat` 或 `chat_completion`
4. **保存配置**

### 2. 测试图片识别功能

1. **找到包含图片的微信公众号文章**
2. **在舆情信息页面，点击"重新分析"**
3. **查看日志**，确认：
   - 图片是否被正确提取
   - 图片识别是否执行
   - 识别结果是否正确

### 3. 查看日志

```bash
# 查看应用日志
sudo docker compose logs app | grep "图片"

# 查看图片提取日志
sudo docker compose logs app | grep "\[图片提取\]"

# 查看图片识别日志
sudo docker compose logs app | grep "\[图片识别\]"

# 查看图片识别API日志
sudo docker compose logs app | grep "\[图片识别API\]"
```

## 故障排查

### 问题1：图片没有被提取

**可能原因：**
- HTML结构变化，img标签不在预期位置
- URL格式不符合预期

**排查方法：**
```bash
# 查看图片提取日志
sudo docker compose logs app | grep "\[图片提取\]"
```

**解决方案：**
- 检查日志中的"找到 X 个img标签"信息
- 如果为0，说明HTML结构可能有问题
- 如果找到但未提取，检查URL提取逻辑

### 问题2：图片识别未执行

**可能原因：**
- 未配置图片识别模型
- 图片识别模型配置错误

**排查方法：**
```bash
# 查看图片识别配置日志
sudo docker compose logs app | grep "图片识别模型配置"
```

**解决方案：**
- 检查AI模型配置中是否有 `usage_type='image_recognition'` 的配置
- 检查配置是否激活（`is_active=1`）
- 检查配置是否正确（API endpoint、API key等）

### 问题3：图片识别失败

**可能原因：**
- 图片下载失败
- API调用失败
- API响应格式不符合预期

**排查方法：**
```bash
# 查看图片识别详细日志
sudo docker compose logs app | grep "\[图片识别API\]"
```

**解决方案：**
- 检查图片下载是否成功（查看"图片下载成功"日志）
- 检查API调用是否成功（查看"API响应状态码"日志）
- 检查API响应格式（查看"API响应解析"日志）
