# 用户个人信息管理功能说明

## 功能概述

用户个人信息管理功能允许登录用户在系统中管理自己的个人信息，包括查看和编辑手机号、修改密码等操作。该功能通过点击登录后页面右上角的用户名来访问。

## 功能特性

### 1. 个人信息编辑
- **用户名显示**：显示当前登录用户的用户名（不可编辑）
- **手机号编辑**：用户可以修改自己的手机号
- **实时验证**：手机号格式验证（11位，1开头）
- **重复检查**：确保手机号未被其他用户使用

### 2. 修改密码
- **旧密码验证**：必须输入正确的旧密码才能修改
- **新密码要求**：新密码至少6位
- **密码确认**：需要重复输入新密码进行确认
- **安全检查**：新密码不能与旧密码相同

### 3. 忘记密码
- **提示信息**：显示联系管理员重置密码的提示

## 使用说明

### 访问个人信息
1. 登录系统后，在页面右上角找到"欢迎，{用户名}"的提示
2. 点击用户名（蓝色可点击链接）
3. 弹出个人信息编辑弹窗

### 编辑手机号
1. 在个人信息弹窗中，找到"手机号"输入框
2. 修改手机号（必须符合11位手机号格式）
3. 点击"保存"按钮
4. 系统会验证手机号格式和唯一性
5. 保存成功后，弹窗会在2秒后自动关闭

### 修改密码
1. 在个人信息弹窗中，点击"修改密码"按钮
2. 在弹出的修改密码弹窗中：
   - 输入旧密码
   - 输入新密码（至少6位）
   - 再次输入新密码进行确认
3. 点击"确认修改"按钮
4. 系统会验证：
   - 旧密码是否正确
   - 新密码长度是否符合要求
   - 两次输入的新密码是否一致
   - 新密码是否与旧密码相同
5. 验证通过后，密码修改成功，弹窗会在2秒后自动关闭

### 忘记密码
1. 在个人信息弹窗中，点击"忘记密码"按钮
2. 查看提示信息："如需重置密码，请联系管理员处理。"
3. 点击"知道了"按钮关闭弹窗

### 后端API接口

#### 1. 获取当前用户信息
**接口**：`GET /api/auth/profile`

**请求头**：
```
x-user-id: {用户ID}
```

**响应示例**：
```json
{
  "success": true,
  "data": {
    "id": "20240101120000001",
    "account": "testuser",
    "phone": "13800138000",
    "email": "test@example.com",
    "company_name": "测试公司"
  }
}
```

**错误响应**：
- `401`：未登录
- `404`：用户不存在
- `500`：服务器错误

#### 2. 更新用户信息
**接口**：`PUT /api/auth/profile`

**请求头**：
```
x-user-id: {用户ID}
Content-Type: application/json
```

**请求体**：
```json
{
  "phone": "13800138000"
}
```

**响应示例**：
```json
{
  "success": true,
  "message": "个人信息更新成功"
}
```

**错误响应**：
- `400`：手机号格式不正确或已被使用
- `401`：未登录
- `500`：服务器错误

**验证规则**：
- 手机号格式：`/^1[3-9]\d{9}$/`
- 手机号唯一性检查

#### 3. 修改密码
**接口**：`PUT /api/auth/change-password`

**请求头**：
```
x-user-id: {用户ID}
Content-Type: application/json
```

**请求体**：
```json
{
  "oldPassword": "oldpass123",
  "newPassword": "newpass123"
}
```

**响应示例**：
```json
{
  "success": true,
  "message": "密码修改成功"
}
```

**错误响应**：
- `400`：旧密码错误、新密码长度不足、新密码与旧密码相同
- `401`：未登录
- `404`：用户不存在
- `500`：服务器错误

**验证规则**：
- 旧密码不能为空
- 新密码至少6位
- 旧密码必须正确
- 新密码不能与旧密码相同

### 数据库操作

#### 用户表结构
表名：`users`

相关字段：
- `id`：用户ID（主键）
- `account`：用户名（唯一）
- `phone`：手机号（唯一）
- `email`：邮箱
- `password`：密码（bcrypt加密）

#### 更新操作
- 更新手机号：`UPDATE users SET phone = ? WHERE id = ?`
- 更新密码：`UPDATE users SET password = ? WHERE id = ?`

## 安全特性

1. **身份验证**：所有API接口都需要通过 `x-user-id` 请求头验证用户身份
2. **密码加密**：使用 bcrypt 进行密码加密存储（10轮加密）
3. **密码验证**：修改密码时必须验证旧密码正确性
4. **数据验证**：前后端双重验证，确保数据格式正确
5. **唯一性检查**：手机号更新时检查是否已被其他用户使用

## 管理员重置密码功能

### 功能说明
管理员可以在用户管理页面中重置任意用户的密码，重置后的密码统一为 `123456`。

### 使用说明
1. 登录管理员账号
2. 进入"用户管理"页面
3. 在用户列表的"操作"列中，找到"重置密码"按钮
4. 点击"重置密码"按钮
5. 确认对话框提示后，系统将用户密码重置为 `123456`
6. 重置成功后，用户可以使用新密码登录

### 技术实现

#### 前端实现
位置：`news/client/src/pages/UserManagement.jsx`

主要功能：
- 在操作列中添加"重置密码"按钮
- 点击时弹出确认对话框
- 调用后端API重置密码
- 显示操作结果提示

关键代码：
```jsx
const handleResetPassword = async (user) => {
  if (!window.confirm(`确定要将用户 "${user.account}" 的密码重置为 "123456" 吗？`)) {
    return
  }

  try {
    await axios.put(`/api/auth/users/${user.id}/reset-password`)
    alert(`用户 "${user.account}" 的密码已重置为 "123456"`)
    fetchUsers()
  } catch (error) {
    console.error('重置密码失败:', error)
    alert('重置密码失败：' + (error.response?.data?.message || '未知错误'))
  }
}
```

#### 后端API接口

**接口**：`PUT /api/auth/users/:id/reset-password`

**权限要求**：仅管理员（admin）可用

**请求头**：
```
x-user-role: admin
```

**路径参数**：
- `id`：用户ID

**响应示例**：
```json
{
  "success": true,
  "message": "密码重置成功"
}
```

**错误响应**：
- `403`：无权访问（非管理员）
- `404`：用户不存在
- `500`：服务器错误

**安全特性**：
- 仅管理员可以重置密码
- 重置后的密码统一为 `123456`（使用 bcrypt 加密存储）
- 操作前需要确认对话框

**注意事项**：
1. 只有管理员角色才能使用此功能
2. 重置后的密码为 `123456`，建议用户首次登录后立即修改
3. 重置密码操作不可逆，请谨慎操作
