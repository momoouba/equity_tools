# 登录 503 错误详细排查

## 🔍 问题现象

- 登录请求 `/api/auth/login` 返回 503
- Response Headers 显示 `content-length: 0`（无响应体）
- `connection: close`（连接被关闭）

## 🚀 立即执行以下诊断命令

### 步骤1：查看服务器日志（关键）

```bash
cd /opt/newsapp/news

# 查看应用日志，特别关注登录相关的错误
sudo docker compose logs app --tail 100 | grep -i "login\|auth\|error\|503\|fail"

# 查看完整的最近日志
sudo docker compose logs app --tail 200
```

### 步骤2：测试登录接口（从服务器）

```bash
# 测试登录接口
curl -v -X POST http://localhost:3001/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"account":"admin","password":"your_password"}'

# 或者使用域名 Host 头
  curl -v -X POST -H "Host: news.gf-dsai.com" http://localhost/api/auth/login \
    -H "Content-Type: application/json" \
    -d '{"account":"admin","password":"your_password"}'
```

### 步骤3：检查服务器就绪状态

```bash
# 确认服务器是否就绪
sudo docker compose logs app | grep -i "服务器核心功能已就绪"

# 检查 serverReady 标志
sudo docker compose logs app | grep -i "serverReady\|503\|starting"
```

### 步骤4：检查 Nginx 日志

```bash
# 查看 Nginx 错误日志
sudo docker compose logs nginx --tail 50 | grep -i "error\|503\|upstream\|connect"

# 查看 Nginx 访问日志（查看登录请求）
sudo docker compose logs nginx --tail 50 | grep -i "login\|auth"
```

### 步骤5：检查应用是否崩溃

```bash
# 检查容器状态
sudo docker compose ps

# 检查容器是否频繁重启
sudo docker compose ps -a | grep app
```

## 🔧 可能的原因和解决方案

### 原因1：服务器在处理登录时崩溃

**症状：** 日志中有错误，容器可能重启

**解决方案：**
```bash
# 查看详细错误日志
sudo docker compose logs app --tail 300 | grep -A 20 "error\|exception\|crash"
```

### 原因2：serverReady 标志被重置

**症状：** 服务器启动后，某个操作导致 serverReady 变为 false

**检查：**
```bash
# 查看服务器启动日志
sudo docker compose logs app | grep -i "serverReady\|服务器核心功能已就绪"
```

### 原因3：Nginx 代理问题

**症状：** Nginx 无法正确代理 POST 请求

**解决方案：**
```bash
# 重启 Nginx
sudo docker compose restart nginx

# 检查 Nginx 配置
sudo docker compose exec nginx nginx -t
```

### 原因4：数据库连接问题

**症状：** 登录时需要查询数据库，但连接失败

**检查：**
```bash
# 测试数据库连接
sudo docker compose exec mysql mysql -u newsapp -pNewsApp@2024 investment_tools -e "SELECT 1;"
```

## 🎯 快速修复

```bash
cd /opt/newsapp/news

# 1. 完全重启所有服务
sudo docker compose restart

# 2. 等待 60 秒让服务器完全启动
sleep 60

# 3. 检查服务器状态
sudo docker compose logs app --tail 50 | grep -i "服务器核心功能已就绪"

# 4. 测试登录接口
curl -X POST http://localhost:3001/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"account":"admin","password":"your_password"}'
```
