## 二、过滤逻辑

### 1. **邮件推送过滤规则**

在 `news/server/utils/scheduledEmailTasks.js` 的 `filterNewsByCategory` 函数中：

- **企查查数据源**：根据收件配置的类别编码进行过滤
  - **自定义类别**：如果收件配置中设置了 `qichacha_category_codes` 字段（非空），则只保留这些类别编码对应的新闻
  - **默认类别**：如果收件配置中 `qichacha_category_codes` 字段为空（null），则使用默认类别（80000系列、40000系列、14004）
- **非企查查数据源**（如新榜）：全部保留，不受类别限制
- **类别为空**：企查查数据源中类别为空的新闻会被过滤掉

### 2. **自定义类别选择功能**

#### 功能说明
- 在收件管理编辑页面，用户可以点击"选择消息类型"按钮
- 弹窗中显示所有企查查类别编码和对应的中文名称
- 支持全选、清空、使用默认类别等操作
- 支持多选，可以选择任意数量的类别
- 选择的类别会保存到 `recipient_management` 表的 `qichacha_category_codes` 字段（JSON格式）

#### 使用方式
1. 进入"收件管理"页面，点击"编辑"按钮
2. 在编辑表单中，找到"企查查接口消息类型"字段
3. 点击"选择消息类型"按钮
4. 在弹窗中选择需要的类别（可全选、清空或使用默认类别）
5. 点击"确定"保存选择
6. 保存收件配置后，邮件发送时会根据选择的类别进行过滤

#### 默认类别
如果未选择任何类别（或清空选择），系统将使用以下默认类别：
- **80000系列**（9个类别）：其他预警类
- **40000系列**（31个类别）：经营预警类
- **14004**（1个类别）：荣誉奖项

### 3. **收件人14004的特殊规则**

在 `news/server/utils/scheduledEmailTasks.js` 的 `executeEmailTask` 函数中：

- 当收件人ID为 `14004` 时，会额外查询并添加：
  - 类别为"荣誉奖项"（编码14004）的企查查新闻
  - 关键词/标题/摘要中包含"荣誉"、"奖项"、"获奖"、"榜单"的企查查新闻

## 三、类别编码映射

### 3.1 类别数据存储

所有企查查类别编码与中文名称的映射关系存储在数据库 `qichacha_news_categories` 表中：

**数据库表结构：**
```sql
CREATE TABLE qichacha_news_categories (
  id VARCHAR(19) PRIMARY KEY COMMENT '数据ID',
  category_code VARCHAR(50) NOT NULL UNIQUE COMMENT '类别编码',
  category_name VARCHAR(255) NOT NULL COMMENT '类别描述',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  INDEX idx_category_code (category_code)
);
```

**类别数据管理：**
- 类别数据可以在"系统配置 → 企查查接口配置 → 企查查新闻类别列表"Tab中维护
- 支持新增、编辑、删除、批量导入类别
- 系统初始化时会自动导入175个默认类别
- 类别映射工具（`qichachaCategoryMapper.js`）优先从数据库加载，如果数据库中没有数据则使用默认映射

### 3.2 类别管理功能

#### 功能入口
- 路径：系统配置 → 企查查接口配置 → 企查查新闻类别列表（第二个Tab）

#### 功能特点
1. **列表展示**
   - 显示所有企查查新闻类别（类别编码、类别描述、创建时间、更新时间）
   - 支持分页显示（每页50条）
   - 支持搜索（按类别编码或描述搜索）

2. **新增类别**
   - 点击"新增类别"按钮
   - 填写类别编码和类别描述
   - 类别编码必须唯一

3. **编辑类别**
   - 点击列表中的"编辑"按钮
   - 可以修改类别描述
   - 类别编码不可修改

4. **删除类别**
   - 点击列表中的"删除"按钮
   - 确认后删除该类别

5. **批量导入**
   - 点击"导入"按钮
   - 支持多种格式：
     - **JSON格式**：`[{"category_code":"10000","category_name":"信用预警"},...]`
     - **CSV格式**：第一行为标题，后续行为数据
     - **简单格式**：每行一个，格式为 `编码|描述` 或 `编码,描述`
   - 如果类别编码已存在，则更新描述；如果不存在，则创建新类别

#### 使用场景
- 企查查新增了新的类别编码，需要添加到系统中
- 需要修改某个类别的描述
- 需要批量导入类别数据
- 需要删除不再使用的类别

### 3.3 获取类别映射API

系统提供了两个API接口用于获取企查查类别映射：

**1. 获取类别列表（推荐）**
```javascript
GET /api/system/qichacha-news-categories?page=1&pageSize=1000&search=关键词
```

返回格式：
```json
{
  "success": true,
  "data": [
    {
      "id": "20250101120000001",
      "category_code": "10000",
      "category_name": "信用预警",
      "created_at": "2025-01-01T12:00:00.000Z",
      "updated_at": "2025-01-01T12:00:00.000Z"
    },
    ...
  ],
  "total": 175,
  "page": 1,
  "pageSize": 1000
}
```

**2. 获取类别映射（兼容旧接口）**
```javascript
GET /api/news/qichacha-categories
```

返回格式：
```json
{
  "success": true,
  "data": {
    "00000": "其他",
    "1000": "高管信息",
    "10000": "信用预警",
    ...
  }
}
```

**说明：**
- 第一个API返回完整的类别列表，包含创建时间等信息
- 第二个API返回简化的映射对象，用于类别选择弹窗显示
- 两个API都优先从数据库获取数据，如果数据库中没有数据则使用默认映射
- 前端类别选择弹窗使用第二个API获取类别映射

## 四、数据同步说明

### 企查查接口同步时：

1. **获取所有类别**：企查查接口同步时，不指定 `category` 参数，会获取**所有类别**的新闻
2. **存储到数据库**：所有类别的新闻都会存储到 `news_detail` 表中
3. **邮件推送时过滤**：只有在邮件推送时，才会根据收件配置的类别规则过滤

### 数据流程：

```
企查查接口同步
    ↓
获取所有类别的新闻（不限制类别）
    ↓
存储到 news_detail 表
    ↓
邮件推送时
    ↓
读取收件配置的 qichacha_category_codes
    ↓
如果为空（null）：使用默认类别（80000系列、40000系列、14004）
如果非空：使用自定义类别
    ↓
过滤：只保留配置的类别
    ↓
发送邮件
```

### 数据库字段说明

**recipient_management 表新增字段：**

```sql
qichacha_category_codes JSON COMMENT '企查查新闻类别编码列表（JSON数组），为空时使用默认类别'
```

- **字段类型**：JSON
- **存储格式**：JSON数组，例如：`["80000", "80001", "14004"]`
- **默认值**：NULL（表示使用默认类别）
- **说明**：
  - 如果字段为 NULL 或空数组，系统使用默认类别（80000系列、40000系列、14004）
  - 如果字段包含类别编码数组，系统只推送这些类别对应的新闻

## 五、总结

### 默认邮件推送的企查查新闻类型：

1. **80000系列**（9个类别）：其他预警类
2. **40000系列**（31个类别）：经营预警类
3. **14004**（1个类别）：荣誉奖项

**总计：41个类别编码**

### 自定义类别功能：

- 用户可以在收件管理编辑页面自定义选择需要的企查查类别
- 支持选择任意数量的类别（最多可选择所有类别）
- 如果未选择任何类别，系统自动使用默认类别
- 每个收件配置可以独立设置自己的类别过滤规则

### 特殊规则：

- 收件人ID为14004时，额外推送包含荣誉奖项关键词的新闻（即使类别不在上述列表中）
- 非企查查数据源（如新榜）的新闻不受类别限制，全部推送
