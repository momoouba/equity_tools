# AI 分析 503 错误处理指南

## 🔍 问题分析

根据错误信息，AI 分析进度查询接口 `/api/news-analysis/analysis-progress/:taskId` 返回了 **503 Service Unavailable** 错误。

### 原因

在 `server/index.js` 中，有一个服务器就绪检查中间件：

```javascript
// 如果服务器未完全就绪，返回503
if (!serverReady) {
  return res.status(503).json({
    success: false,
    message: '服务器正在启动中，请稍后重试',
    status: 'starting'
  });
}
```

**503 错误的可能原因：**
1. ✅ **服务器正在启动中**：容器重启后，服务器还在初始化（数据库连接、定时任务等）
2. ✅ **服务器启动失败**：初始化过程中遇到错误，`serverReady` 一直为 `false`
3. ✅ **容器崩溃重启**：应用容器异常退出后自动重启，但还未完全就绪

## ✅ 解决方案

### 方法一：使用 fix-503-error.sh 脚本（推荐）

这个脚本专门用于诊断和修复 503 错误：

```bash
# 1. 进入项目目录
cd /opt/newsapp/news

# 2. 给脚本添加执行权限（首次使用）
chmod +x deploy/fix-503-error.sh

# 3. 执行脚本
sudo ./deploy/fix-503-error.sh
```

**脚本会自动：**
1. ✅ 检查 Docker 容器状态
2. ✅ 检查服务器是否就绪
3. ✅ 查看应用日志，查找错误
4. ✅ 检查 Nginx 配置
5. ✅ 重启容器（如果需要）
6. ✅ 验证修复是否成功

### 方法二：手动检查和修复

如果脚本无法解决问题，可以手动执行：

```bash
# 1. 检查容器状态
sudo docker compose ps

# 2. 查看应用日志（重点关注服务器启动相关）
sudo docker compose logs app --tail 100 | grep -i "ready\|error\|fail\|503"

# 3. 检查服务器是否就绪
sudo docker compose logs app | grep -i "服务器核心功能已就绪\|server.*ready"

# 4. 如果服务器未就绪，查看详细错误
sudo docker compose logs app --tail 200

# 5. 重启应用容器
sudo docker compose restart app

# 6. 等待服务器完全启动（通常需要 30-60 秒）
sleep 30

# 7. 再次检查日志
sudo docker compose logs app --tail 50
```

### 方法三：等待服务器完全启动

如果服务器正在启动中，只需等待：

```bash
# 监控服务器启动进度
sudo docker compose logs app -f | grep -i "ready\|error"
```

**等待看到以下日志：**
```
✓ 数据库连接已就绪
✓ 服务器运行在 http://localhost:3001
✓ 服务器核心功能已就绪，可以接收请求
```

## 🔍 诊断步骤

```bash
# 查看服务器是否已就绪

# 如果返回 200，说明服务器已就绪
```

```bash
# 查找错误信息

sudo docker compose logs app | grep -i "服务器核心功能已就绪"
```

### 3. 检查数据库连接

```bash
# 检查数据库容器状态
sudo docker compose ps mysql

# 测试数据库连接（从应用容器内）
sudo docker compose exec app node -e "require('./db').query('SELECT 1').then(() => console.log('DB OK')).catch(e => console.error('DB Error:', e.message))"
```

### 4. 检查容器资源

```bash
# 检查容器资源使用情况
sudo docker stats --no-stream newsapp

# 如果内存或 CPU 使用率过高，可能需要优化或增加资源
```

## 🛠️ 常见问题和解决方案

### 问题1：服务器一直显示"正在启动中"

**可能原因：**
- 数据库连接失败
- 初始化任务卡住
- 代码语法错误

**解决方案：**
```bash
# 1. 查看详细错误日志
sudo docker compose logs app --tail 200

# 2. 检查数据库连接
sudo docker compose exec mysql mysql -u newsapp -pNewsApp@2024 investment_tools -e "SELECT 1;"

# 3. 如果数据库连接失败，检查环境变量
sudo docker compose exec app env | grep DB_

# 4. 重启所有服务
sudo docker compose restart
```

### 问题2：容器频繁重启

**可能原因：**
- 应用崩溃
- 内存不足
- 代码错误

**解决方案：**
```bash
# 1. 查看容器退出原因
sudo docker compose ps -a

# 2. 查看崩溃前的日志
sudo docker compose logs app --tail 200

# 3. 检查系统资源
free -h
df -h

# 4. 如果内存不足，增加 Docker 内存限制或优化应用
```

### 问题3：Nginx 返回 503，但应用直接访问正常

**可能原因：**
- Nginx upstream 配置问题
- 容器网络问题

**解决方案：**
```bash
# 1. 检查 Nginx 配置
sudo docker compose exec nginx nginx -t

# 2. 检查容器网络
sudo docker network inspect newsapp_newsapp-network

# 3. 测试从 Nginx 容器访问应用
sudo docker compose exec nginx wget -q -O - http://app:3001/api/health

# 4. 重启 Nginx
sudo docker compose restart nginx
```

## ⚠️ 预防措施

1. **监控服务器启动时间**：如果启动时间过长（>2分钟），需要优化初始化流程
2. **设置健康检查超时**：确保健康检查有足够的等待时间
3. **添加启动超时保护**：如果启动超过一定时间，记录警告日志
4. **优化初始化顺序**：先启动核心功能，后台任务异步执行
