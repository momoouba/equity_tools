# 企查查正文提取增强说明

## 更新时间
2025-12-18

## 更新内容

### 问题描述
在抓取企查查的数据中，如果是链接的，需要增加查询前端类似 `article class="main-news article-with-html"` 这种后面的正文内容的提取。

### 修复位置
`news/server/utils/newsAnalysis.js:99-118` - `extractArticleContent` 方法

### 修复内容
在 `extractArticleContent` 方法中，增加了对企查查等网站常用的 `article` 标签的匹配规则，特别是包含以下 class 的：
- `main-news`
- `article-with-html`
- `main-news article-with-html`（两个class同时存在）

### 匹配优先级
新的匹配规则被设置为**最高优先级**，确保在提取正文时优先匹配这些标记：

```javascript
// 最高优先级：企查查等网站的 article 标签，包含 main-news、article-with-html 等 class
/<article[^>]*class="[^"]*main-news[^"]*"[^>]*>([\s\S]*?)<\/article>/i,
/<article[^>]*class="[^"]*article-with-html[^"]*"[^>]*>([\s\S]*?)<\/article>/i,
/<article[^>]*class="[^"]*main-news[^"]*\s+article-with-html[^"]*"[^>]*>([\s\S]*?)<\/article>/i,
/<article[^>]*class="[^"]*article-with-html[^"]*\s+main-news[^"]*"[^>]*>([\s\S]*?)<\/article>/i,
```

### 匹配规则说明

1. **`/<article[^>]*class="[^"]*main-news[^"]*"[^>]*>([\s\S]*?)<\/article>/i`**
   - 匹配包含 `main-news` class 的 `article` 标签

2. **`/<article[^>]*class="[^"]*article-with-html[^"]*"[^>]*>([\s\S]*?)<\/article>/i`**
   - 匹配包含 `article-with-html` class 的 `article` 标签

3. **`/<article[^>]*class="[^"]*main-news[^"]*\s+article-with-html[^"]*"[^>]*>([\s\S]*?)<\/article>/i`**
   - 匹配同时包含 `main-news` 和 `article-with-html` class 的 `article` 标签（`main-news` 在前）

4. **`/<article[^>]*class="[^"]*article-with-html[^"]*\s+main-news[^"]*"[^>]*>([\s\S]*?)<\/article>/i`**
   - 匹配同时包含 `main-news` 和 `article-with-html` class 的 `article` 标签（`article-with-html` 在前）

### 工作流程

当企查查接口返回的新闻 `Content` 字段为空时：

1. **第一步**：使用 `fetchContentFromUrl` 方法抓取网页HTML内容
2. **第二步**：调用 `extractArticleContent` 方法提取正文
   - 优先匹配 `article class="main-news article-with-html"` 等标记
   - 如果匹配成功且内容长度大于200字符，使用该内容
   - 如果匹配失败，继续尝试其他匹配规则
3. **第三步**：如果 `extractArticleContent` 提取失败或内容太短，使用 AI 提取作为备用方案

### 示例HTML结构

```html
<article class="main-news article-with-html">
  <div class="article-header">
    <h1>新闻标题</h1>
  </div>
  <div class="article-body">
    <p>正文内容...</p>
    <p>更多正文内容...</p>
  </div>
</article>
```

系统会提取 `<article class="main-news article-with-html">` 标签内的所有内容作为正文。

## 验证方法

### 1. 测试企查查新闻正文提取

```sql
-- 查询最近同步的企查查新闻，检查是否有正文内容
SELECT id, title, 
       LENGTH(content) as content_length,
       source_url,
       APItype,
       created_at
FROM news_detail
WHERE APItype = '企查查'
  AND created_at >= DATE('2025-12-18')
  AND content IS NOT NULL 
  AND content != ''
  AND LENGTH(content) > 50
  AND delete_mark = 0
ORDER BY created_at DESC
LIMIT 10;
```

### 2. 查看提取日志

在应用日志中查找以下信息：
- `[extractArticleContent] 找到正文容器，长度: XXX字符`
- `[企查查同步] ✓ 使用extractArticleContent方法成功提取正文，长度: XXX 字符`

### 3. 手动测试提取

可以创建一个测试脚本，手动测试特定URL的正文提取：

```javascript
const newsAnalysis = require('./server/utils/newsAnalysis');
const url = 'https://example.com/news/123'; // 企查查新闻链接

newsAnalysis.fetchContentFromUrl(url).then(content => {
  console.log('提取的正文长度:', content.length);
  console.log('正文预览:', content.substring(0, 200));
});
```

## 注意事项

1. **匹配优先级**：新的匹配规则被设置为最高优先级，确保优先匹配企查查网站的 `article` 标签
2. **内容长度检查**：提取的内容必须大于200字符才会被使用，避免提取到导航栏等非正文内容
3. **备用方案**：如果 `extractArticleContent` 提取失败，系统会自动使用 AI 提取作为备用方案
4. **兼容性**：新的匹配规则不会影响其他网站的正文提取，因为它们是按优先级顺序匹配的
