# 外部访问数据库指南

## 📋 概述

本文档介绍如何从外部（如 Navicat、本地应用等）访问部署在 Docker 中的 MySQL 数据库。

---

## 🔑 关键信息

### 数据库连接信息

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **数据库类型** | MySQL 8.0 | 数据库版本 |
| **主机地址** | 服务器 IP 地址 | 例如：192.168.1.100 或公网 IP |
| **端口** | **3307** | ⚠️ 重要！不是 3306 |
| **数据库名** | investment_tools | 应用数据库名 |
| **用户名** | newsapp | 应用专用用户 |
| **密码** | NewsApp@2024 | 用户密码 |
| **字符集** | utf8mb4 | 支持中文 |

### 端口说明

- **容器内端口**: `3306`（MySQL 默认端口）
- **宿主机映射端口**: `3307`（对外暴露的端口）
- **端口映射配置**: `3307:3306`

⚠️ **重要提示**：外部访问时必须使用 **3307** 端口，不是 3306！

## 🔗 Navicat 连接配置

### 步骤1：创建新连接

1. 打开 Navicat
2. 点击 **连接** → **MySQL**
3. 填写连接信息

### 步骤2：填写连接参数

```
连接名: Equity News Server（或自定义名称）
连接类型: MySQL

基本设置:
- 主机: 服务器 IP 地址（如 192.168.1.100）
- 端口: 3307  ← ⚠️ 必须是 3307，不是 3306
- 用户名: newsapp
- 密码: NewsApp@2024
- 数据库: investment_tools

高级设置:
- 使用 SSL: 否
- 连接超时: 30
- 读取超时: 30
- 写入超时: 30
```

### 步骤3：测试连接

1. 点击 **测试连接**
2. 如果显示 **连接成功**，点击 **确定**
3. 双击连接名即可访问数据库

### 步骤4：验证表数量

连接成功后，在查询窗口执行：

```sql
-- 查看当前数据库
SELECT DATABASE();

-- 列出所有表
SHOW TABLES;

-- 统计表数量（应该接近 48 张）
SELECT COUNT(*) as table_count
FROM information_schema.tables
WHERE table_schema = 'investment_tools';
```

## 🌐 网络配置

### 查看服务器 IP 地址

在服务器上执行：

```bash
# 查看所有网络接口
ip addr show

# 或查看主 IP
hostname -I | awk '{print $1}'
```

### 检查端口映射状态

在服务器上执行：

```bash
cd /opt/newsapp/news

# 查看 MySQL 容器状态和端口映射
docker compose ps mysql

# 应该看到类似输出：
# NAME                  STATUS         PORTS
# newsapp-mysql         Up 2 hours     0.0.0.0:3307->3306/tcp

# 检查端口监听
netstat -tlnp | grep 3307
# 应该看到：0.0.0.0:3307
```

### 防火墙配置

如果从外部网络（非同一局域网）连接，需要开放防火墙端口：

```bash
# Ubuntu/Debian
sudo ufw allow 3307/tcp
sudo ufw status

# CentOS/RHEL
sudo firewall-cmd --permanent --add-port=3307/tcp
sudo firewall-cmd --reload
sudo firewall-cmd --list-ports

# 如果使用云服务器，需要在云平台控制台添加安全组规则：
# 入站规则: TCP 3307 允许访问（可以限制特定 IP）
```

## ❓ 常见问题和解决方案

### 问题1：Navicat 显示表数量不正确（如只有 20 张表）

**原因分析：**
- 连接到了本地 MySQL 而不是服务器
- 端口配置错误（使用了 3306 而不是 3307）
- 数据库名称错误

**解决方案：**

1. **检查 Navicat 配置**
   - Host: 必须是 **服务器 IP**，不是 `localhost` 或 `127.0.0.1`
   - Port: 必须是 **3307**，不是 3306
   - Database: `investment_tools`

2. **验证连接信息**
   在 Navicat 查询窗口执行：
   ```sql
   SELECT
       DATABASE() as current_db,
       USER() as current_user,
       @@hostname as mysql_hostname,
       @@port as mysql_port;
   ```
   - `current_db` 应该是 `investment_tools`
   - `current_user` 应该是 `newsapp@%`
   - `mysql_port` 应该是 `3306`（容器内端口）

3. **关闭本地 MySQL**
   如果本地运行了 MySQL，关闭它避免混淆：
   ```powershell
   # Windows
   net stop mysql80
   # 或在服务管理器中停止 MySQL 服务
   ```

### 问题2：连接超时

**原因分析：**
- 防火墙阻止连接
- 服务器 IP 错误
- MySQL 容器未运行

**解决方案：**

1. **检查 MySQL 容器是否运行**
   ```bash
   docker compose ps mysql
   ```

2. **检查端口是否监听**
   ```bash
   netstat -tlnp | grep 3307
   ```

3. **检查防火墙**
   ```bash
   sudo ufw status
   sudo ufw allow 3307/tcp
   ```

4. **测试端口连通性**
   从本地执行：
   ```powershell
   # Windows PowerShell
   Test-NetConnection -ComputerName 服务器IP -Port 3307

   # 或使用 telnet（如果已安装）
   telnet 服务器IP 3307
   ```

### 问题3：Access denied for user 'newsapp'@'xxx'

**原因分析：**
- 用户名或密码错误
- 用户权限配置问题

**解决方案：**

1. **确认用户名和密码**
   - 用户名：`newsapp`（小写）
   - 密码：`NewsApp@2024`

2. **检查用户权限**
   ```bash
   docker compose exec mysql mysql -u root -pRootPassword123! -e "
   SELECT user, host FROM mysql.user WHERE user='newsapp';
   "
   ```

3. **重新授权（如果需要）**
   ```bash
   docker compose exec mysql mysql -u root -pRootPassword123! investment_tools << EOF
   DROP USER IF EXISTS 'newsapp'@'%';
   DROP USER IF EXISTS 'newsapp'@'localhost';
   CREATE USER 'newsapp'@'%' IDENTIFIED BY 'NewsApp@2024';
   CREATE USER 'newsapp'@'localhost' IDENTIFIED BY 'NewsApp@2024';
   GRANT ALL PRIVILEGES ON investment_tools.* TO 'newsapp'@'%';
   GRANT ALL PRIVILEGES ON investment_tools.* TO 'newsapp'@'localhost';
   FLUSH PRIVILEGES;
   EOF
   ```

### 问题4：只能看到部分表（如系统表）

**原因分析：**
- 连接到了错误的数据库
- 用户权限不足

**解决方案：**

1. **切换到正确的数据库**
   ```sql
   USE investment_tools;
   SHOW TABLES;
   ```

2. **在 Navicat 连接配置中指定数据库**
   - 数据库字段填写：`investment_tools`

### 问题5：字符编码问题（中文乱码）

**解决方案：**

1. **Navicat 连接设置**
   - 右键连接 → 编辑连接
   - 高级选项 → 字符集: `utf8mb4`

2. **应用程序连接配置**
   - 在连接字符串中添加：`charset=utf8mb4`

## 🔒 安全建议

### 1. 使用受限用户权限

当前 `newsapp` 用户拥有 `investment_tools` 数据库的所有权限。如果只需要只读访问：

```bash
docker compose exec mysql mysql -u root -pRootPassword123! << EOF
-- 创建只读用户
CREATE USER 'newsapp_readonly'@'%' IDENTIFIED BY 'Readonly@2024';
GRANT SELECT ON investment_tools.* TO 'newsapp_readonly'@'%';
FLUSH PRIVILEGES;
EOF
```

### 2. 限制访问 IP

在生产环境中，建议只允许特定 IP 访问：

```bash
# 只允许特定 IP 访问
docker compose exec mysql mysql -u root -pRootPassword123! << EOF
CREATE USER 'newsapp'@'特定IP' IDENTIFIED BY 'NewsApp@2024';
GRANT ALL PRIVILEGES ON investment_tools.* TO 'newsapp'@'特定IP';
DROP USER 'newsapp'@'%';
FLUSH PRIVILEGES;
EOF
```

### 3. 使用 SSL 连接

在生产环境中，建议启用 SSL 加密连接：

```bash
# 生成 SSL 证书（在 MySQL 容器内）
docker compose exec mysql mysql -u root -pRootPassword123! << EOF
CREATE USER 'newsapp_ssl'@'%' IDENTIFIED BY 'NewsApp@2024' REQUIRE SSL;
GRANT ALL PRIVILEGES ON investment_tools.* TO 'newsapp_ssl'@'%';
FLUSH PRIVILEGES;
EOF
```

### 4. 定期修改密码

```bash
docker compose exec mysql mysql -u root -pRootPassword123! << EOF
ALTER USER 'newsapp'@'%' IDENTIFIED BY '新密码';
ALTER USER 'newsapp'@'localhost' IDENTIFIED BY '新密码';
FLUSH PRIVILEGES;
EOF
```

### 5. 防火墙限制

```bash
# 只允许特定 IP 访问 3307 端口
sudo ufw allow from 特定IP to any port 3307
```

## 📊 数据库操作常用命令

### 在服务器上执行

```bash
cd /opt/newsapp/news

# 查看所有数据库
docker compose exec mysql mysql -u root -pRootPassword123! -e "SHOW DATABASES;"

# 查看所有表
docker compose exec mysql mysql -u newsapp -pNewsApp@2024 investment_tools -e "SHOW TABLES;"

# 查看表结构
docker compose exec mysql mysql -u newsapp -pNewsApp@2024 investment_tools -e "DESC users;"

# 查看表数据
docker compose exec mysql mysql -u newsapp -pNewsApp@2024 investment_tools -e "SELECT * FROM users LIMIT 10;"

# 执行 SQL 文件
docker compose exec -T mysql mysql -u newsapp -pNewsApp@2024 investment_tools < /path/to/script.sql

# 导出数据库（备份）
docker compose exec mysql mysqldump -u root -pRootPassword123! investment_tools > backup.sql

# 导入数据库（恢复）
docker compose exec -T mysql mysql -u root -pRootPassword123! investment_tools < backup.sql
```

### 在 Navicat 中执行常用 SQL

```sql
-- 查看数据库大小
SELECT
    table_schema AS 'Database',
    ROUND(SUM(data_length + index_length) / 1024 / 1024, 2) AS 'Size (MB)'
FROM information_schema.tables
WHERE table_schema = 'investment_tools'
GROUP BY table_schema;

-- 查看各表行数
SELECT
    table_name AS 'Table',
    table_rows AS 'Rows',
    ROUND((data_length + index_length) / 1024 / 1024, 2) AS 'Size (MB)'
FROM information_schema.tables
WHERE table_schema = 'investment_tools'
ORDER BY table_rows DESC;

-- 查看表结构
SHOW CREATE TABLE users;

-- 查看索引
SHOW INDEX FROM users;

-- 查看外键约束
SELECT
    CONSTRAINT_NAME,
    TABLE_NAME,
    COLUMN_NAME,
    REFERENCED_TABLE_NAME,
    REFERENCED_COLUMN_NAME
FROM information_schema.KEY_COLUMN_USAGE
WHERE TABLE_SCHEMA = 'investment_tools'
  AND REFERENCED_TABLE_NAME IS NOT NULL;
```

## 📝 配置文件参考

### docker-compose.yml 中的端口配置

```yaml
services:
  mysql:
    image: mysql:8.0
    container_name: newsapp-mysql
    ports:
      - "${MYSQL_PORT:-3307}:3306"  # 宿主机:3307 -> 容器:3306
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD:-RootPassword123!}
      MYSQL_DATABASE: ${DB_NAME:-investment_tools}
      MYSQL_USER: ${DB_USER:-newsapp}
      MYSQL_PASSWORD: ${DB_PASSWORD:-NewsApp@2024}
    volumes:
      - mysql_data:/var/lib/mysql
      - ./deploy/mysql-init:/docker-entrypoint-initdb.d
```
