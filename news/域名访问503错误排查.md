# 域名访问 503 错误排查

## 🔍 问题

浏览器通过域名 `news.gf-dsai.com` 访问返回 503，但通过 `localhost` 访问正常。

## 🚀 立即执行以下诊断命令

### 步骤1：测试域名访问（在服务器上）

```bash
cd /opt/newsapp/news

# 使用域名测试（模拟浏览器请求）
curl -v -H "Host: news.gf-dsai.com" http://localhost/api/system/basic-config

# 或者直接使用域名（如果 DNS 已配置）
curl -v http://news.gf-dsai.com/api/system/basic-config
```

### 步骤2：检查 Nginx 日志

```bash
# 查看 Nginx 访问日志
sudo docker compose logs nginx --tail 50

# 查看 Nginx 错误日志
sudo docker compose logs nginx --tail 50 | grep -i "error\|503\|upstream"
```

### 步骤3：检查 Nginx 配置中的 server_name

```bash
# 查看容器内的实际配置
sudo docker compose exec nginx cat /etc/nginx/conf.d/default.conf | grep -A 5 "server_name"
```

### 步骤4：检查服务器日志

```bash
# 查看应用日志，确认请求是否到达应用
sudo docker compose logs app --tail 50 | grep -i "basic-config\|system"
```

## 🔧 可能的原因和解决方案

### 原因1：Nginx server_name 配置问题

如果 Nginx 配置中的 `server_name` 不匹配，可能返回默认错误。

**检查：**
```bash
sudo docker compose exec nginx cat /etc/nginx/conf.d/default.conf | grep "server_name"
```

**应该看到：**
```nginx
server_name news.gf-dsai.com _;
```

### 原因2：DNS 解析问题

域名可能没有正确解析到服务器 IP。

**检查：**
```bash
# 在服务器上检查域名解析
nslookup news.gf-dsai.com

# 或者
dig news.gf-dsai.com
```

### 原因3：Nginx 配置未正确加载

**解决方案：**
```bash
# 重启 Nginx 并检查配置
sudo docker compose exec nginx nginx -t
sudo docker compose restart nginx
sleep 5
```

### 原因4：前端请求的域名不正确

检查前端构建后的代码中是否硬编码了域名。

## 🎯 快速修复

```bash
cd /opt/newsapp/news

# 1. 检查 Nginx 配置
sudo docker compose exec nginx nginx -t

# 2. 重启 Nginx
sudo docker compose restart nginx

# 3. 等待 5 秒
sleep 5

# 4. 测试域名访问
curl -v -H "Host: news.gf-dsai.com" http://localhost/api/system/basic-config
```
