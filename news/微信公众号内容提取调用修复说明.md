# 微信公众号内容提取调用修复说明

## 问题描述

在分析日志中发现，对于微信公众号文章，系统没有调用Python脚本进行内容提取和图片识别，而是使用了JavaScript的常规提取方法，导致：

1. **验证页面URL无法处理**：无法从验证页面URL中提取真实文章URL
2. **图片识别未执行**：即使文章是纯图片，也没有执行图片识别
3. **内容提取失败**：提取到的仍然是JavaScript/CSS代码（乱码）

### 问题日志示例

```
[ensureNewsContent] 新闻ID 2025121909493600003 content包含乱码，尝试从source_url抓取内容
[fetchContentFromUrl] 开始抓取网页内容: https://mp.weixin.qq.com/s?__biz=...
[extractArticleContent] ⚠️ 最终提取的文本太短（43字符），可能提取失败
[ensureNewsContent] 清理后的内容仍然包含脏信息，但继续使用清理后的版本
```

**问题根源**：`ensureNewsContent` 方法在检测到内容乱码时，直接调用了 `fetchContentFromUrl`（JavaScript方法），而没有检查是否是微信公众号URL，应该优先使用 `extractWeChatArticleContent`（Python脚本）。

## 解决方案

### 修改文件

**文件：** `news/server/utils/newsAnalysis.js`

**修改位置：** `ensureNewsContent` 方法

### 修改内容

在 `ensureNewsContent` 方法中，添加对微信公众号URL的特殊处理：

1. **检测微信公众号URL**：检查 `source_url` 是否包含 `mp.weixin.qq.com`
2. **优先使用Python脚本**：如果是微信公众号文章，优先调用 `extractWeChatArticleContent` 方法
3. **备用方案**：如果Python脚本提取失败，再使用常规的 `fetchContentFromUrl` 方法

### 代码逻辑

```javascript
// 对于微信公众号文章，优先使用Python脚本提取（支持验证页面URL和图片识别）
let fetchedContent = null;
if (newsItem.source_url && newsItem.source_url.includes('mp.weixin.qq.com')) {
  try {
    console.log(`[ensureNewsContent] 检测到微信公众号URL，使用Python脚本提取内容: ${newsItem.source_url}`);
    const extractResult = await this.extractWeChatArticleContent(newsItem.source_url);
    
    if (extractResult && extractResult.success && extractResult.content && extractResult.content.trim().length > 0) {
      fetchedContent = extractResult.content;
      console.log(`[ensureNewsContent] ✓ Python脚本提取成功，内容长度: ${fetchedContent.length}字符`);
      console.log(`[ensureNewsContent] 图片统计: 找到 ${extractResult.image_count || 0} 张图片，识别 ${extractResult.recognized_image_count || 0} 张`);
    } else {
      console.warn(`[ensureNewsContent] Python脚本提取失败或内容为空，错误: ${extractResult?.error || '未知错误'}`);
      // Python脚本提取失败，尝试使用常规方法作为备用
      console.log(`[ensureNewsContent] 尝试使用常规方法作为备用方案`);
      fetchedContent = await this.fetchContentFromUrl(newsItem.source_url);
    }
  } catch (wechatError) {
    console.error(`[ensureNewsContent] Python脚本提取时出错: ${wechatError.message}`);
    // Python脚本提取失败，尝试使用常规方法作为备用
    console.log(`[ensureNewsContent] 尝试使用常规方法作为备用方案`);
    fetchedContent = await this.fetchContentFromUrl(newsItem.source_url);
  }
} else {
  // 非微信公众号文章，使用常规提取
  fetchedContent = await this.fetchContentFromUrl(newsItem.source_url);
}
```

## 修复效果

### 修复前

1. 检测到内容乱码
2. 调用 `fetchContentFromUrl`（JavaScript方法）
3. 提取失败，仍然是乱码
4. 没有调用Python脚本
5. 没有执行图片识别

### 修复后

1. 检测到内容乱码
2. **检测到微信公众号URL**
3. **调用 `extractWeChatArticleContent`（Python脚本）**
4. **处理验证页面URL，提取真实文章URL**
5. **提取正文内容**
6. **识别图片文字（如果配置了图片识别模型）**
7. **返回整合后的内容**

## 配置要求

### 图片识别功能

要使用图片识别功能，必须在AI模型配置中添加图片识别模型：

1. **用途类型**：`image_recognition`
2. **模型名称**：推荐使用 `qwen3-vl-plus` 或其他支持图片识别的模型
3. **API配置**：配置正确的API endpoint和API key

**配置位置：** 系统管理 → AI配置 → 添加/编辑配置 → 选择用途类型为"图片识别"

根据您提供的配置信息，图片识别模型已正确配置：
- **配置名称**：图片处理
- **提供商**：alibaba
- **模型名称**：qwen3-vl-plus
- **用途类型**：image_recognition
- **状态**：已激活（is_active=1）

## 故障排查

### 问题1：仍然没有调用Python脚本

**排查方法：**
```bash
# 查看是否有"检测到微信公众号URL"的日志
sudo docker compose logs app | grep "检测到微信公众号URL"
```

**可能原因：**
- URL格式不符合预期（不包含 `mp.weixin.qq.com`）
- 代码未正确部署

**解决方案：**
- 检查URL格式
- 确认代码已正确部署（重启容器）

### 问题2：Python脚本提取失败

**排查方法：**
```bash
# 查看Python脚本提取日志
sudo docker compose logs app | grep "提取微信公众号文章"
```

**可能原因：**
- Python环境问题
- 图片识别模型配置错误
- 网络问题

**解决方案：**
- 检查Python环境
- 检查图片识别模型配置
- 查看详细错误日志

### 问题3：图片识别未执行

**排查方法：**
```bash
# 查看图片识别配置日志
sudo docker compose logs app | grep "图片识别模型配置"
```

**可能原因：**
- 未配置图片识别模型
- 图片识别模型配置错误

**解决方案：**
- 检查AI模型配置中是否有 `usage_type='image_recognition'` 的配置
- 检查配置是否激活（`is_active=1`）
