# 用户舆情统计功能说明

## 📊 功能概述

为用户端舆情信息页面添加了昨日新闻统计功能，在页面上方显示两个关键统计指标，帮助用户快速了解昨日的舆情动态。

## 🎯 统计指标

### 1. 昨日发布新闻企业个数
- **说明**：统计昨日有发布新闻的被投企业数量（去重统计）
- **计算逻辑**：基于用户创建的被投企业，统计昨日（00:00:00 - 23:59:59）期间有新闻发布的企业个数
- **显示位置**：页面顶部第一个统计卡片（橙色）

### 2. 昨日累计新闻条数
- **说明**：统计昨日新增的新闻文章总数
- **计算逻辑**：统计昨日时间范围内，用户相关企业发布的所有新闻条数
- **显示位置**：页面顶部第二个统计卡片（橙色）

### 3. 当前总关注被投企业个数
- **说明**：统计用户当前关注的被投企业总数
- **计算逻辑**：统计用户创建的所有被投企业中，退出状态不为"完全退出"和"已上市"且未删除的企业数量
- **显示位置**：页面顶部第三个统计卡片（蓝色）

## 🔧 技术实现

### 后端API (`/api/news/user-stats`)

```javascript
// 新增统计接口
router.get('/user-stats', async (req, res) => {
  // 1. 获取用户ID和权限验证
  // 2. 查询用户创建的被投企业微信公众号
  // 3. 计算昨日时间范围
  // 4. 统计昨日发布新闻的企业个数（去重）
  // 5. 统计昨日新闻总条数
  // 6. 返回统计结果
})
```

**关键查询逻辑**：
```sql
-- 当前总关注被投企业个数
SELECT COUNT(*) as count
FROM invested_enterprises 
WHERE creator_user_id = ? 
AND exit_status NOT IN ('完全退出', '已上市')
AND delete_mark = 0

-- 昨日发布新闻的企业个数（去重）
SELECT COUNT(DISTINCT wechat_account) as count 
FROM news_detail 
WHERE wechat_account IN (用户企业列表)
AND created_at >= 昨日开始时间 
AND created_at <= 昨日结束时间

-- 昨日新闻总条数
SELECT COUNT(*) as count 
FROM news_detail 
WHERE wechat_account IN (用户企业列表)
AND created_at >= 昨日开始时间 
AND created_at <= 昨日结束时间
```

### 前端实现

#### 1. 数据获取
```javascript

  }
```

## 📱 用户体验

### 1. 权限区分
- **普通用户**：显示昨日统计面板
- **管理员**：显示原有的管理员统计面板
- **未登录用户**：不显示统计信息

### 2. 数据更新
- 页面加载时自动获取统计数据
- 用户角色变更时重新获取对应数据
- 统计数据基于用户创建的被投企业范围

### 3. 响应式适配
- **桌面端**：水平排列的统计卡片
- **移动端**：垂直排列，全宽显示
- **交互反馈**：悬浮效果和阴影变化

## 🔍 数据说明

### 时间范围定义
- **昨日**：指当前日期的前一天（00:00:00 - 23:59:59）
- **时区**：使用服务器本地时区
- **更新频率**：每次页面加载时实时计算

### 统计范围
- **企业范围**：仅统计用户创建的被投企业
- **退出状态过滤**：排除退出状态为"完全退出"和"已上市"的企业
- **数据来源**：
  - 舆情数据：news_detail表中的created_at字段
  - 企业数据：invested_enterprises表
- **去重逻辑**：按wechat_account字段去重统计企业个数

## 📋 使用场景

1. **日常监控**：用户每日查看舆情页面时，快速了解昨日动态
2. **趋势分析**：通过每日统计数据，观察企业舆情活跃度变化
3. **工作汇报**：为管理层提供简洁的昨日舆情数据摘要
