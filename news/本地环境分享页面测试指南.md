# 本地环境分享页面测试指南

## 问题描述
在生产环境上，分享新闻页面可以正常打开，但在本地环境打开是空白的。

## 可能的原因

### 1. 后端服务未启动
分享页面需要调用后端 API (`/api/news-share/verify/:token`)，如果后端服务未启动，页面会显示空白或错误。

**解决方法：**
- 确保后端服务正在运行：`npm run dev` 或 `npm run server`
- 后端应该运行在 `http://localhost:3001`
- 检查后端服务是否正常：访问 `http://localhost:3001/api/health`

### 2. 路由配置问题
在本地开发环境中，需要确保 Vite 开发服务器正确支持客户端路由。

**解决方法：**
- 直接访问 Vite 开发服务器：`http://localhost:5173/share/:token`
- 或者通过后端代理访问：`http://localhost:3001/share/:token`（后端会自动代理到 Vite）

### 3. API 请求失败
如果 API 请求失败，页面会显示错误信息而不是空白。

**检查方法：**
1. 打开浏览器开发者工具 (F12)
2. 查看 Console 标签页，应该能看到调试信息：
   - `[ShareNewsPage] 组件已加载`
   - `[ShareNewsPage] 版本: 2.0.0-simplified`
   - `[ShareNewsPage] 开始验证token: ...`
3. 查看 Network 标签页，检查 API 请求是否成功

## 测试步骤

### 步骤 1: 启动服务
```bash
# 在项目根目录运行
npm run dev
```

这将同时启动：
- 后端服务器：http://localhost:3001
- 前端开发服务器：http://localhost:5173

### 步骤 2: 获取分享链接
1. 访问 http://localhost:5173
2. 登录系统
3. 进入"舆情信息"页面
4. 创建或查看分享链接，获取 token

### 步骤 3: 测试分享页面
**方法 1: 直接访问 Vite 开发服务器**
```
http://localhost:5173/share/YOUR_TOKEN_HERE
```

**方法 2: 通过后端代理访问**
```
http://localhost:3001/share/YOUR_TOKEN_HERE
```

### 步骤 4: 检查问题
如果页面仍然空白，请检查：

1. **浏览器控制台 (F12)**
   - 查看是否有 JavaScript 错误
   - 查看是否有调试信息输出
   - 查看 Network 标签页，检查 API 请求状态

2. **后端服务状态**
   ```bash
   # 检查后端是否运行
   curl http://localhost:3001/api/health
   ```

3. **API 请求测试**
   ```bash
   # 替换 YOUR_TOKEN_HERE 为实际的 token
   curl http://localhost:3001/api/news-share/verify/YOUR_TOKEN_HERE
   ```

## 调试信息

现在 ShareNewsPage 组件已经添加了详细的调试信息，在浏览器控制台中可以看到：

- `[ShareNewsPage] 组件已加载` - 确认组件已渲染
- `[ShareNewsPage] 版本: 2.0.0-simplified` - 确认使用的是最新版本
- `[ShareNewsPage] 开始验证token: ...` - 显示正在验证的 token
- `[ShareNewsPage] Token验证响应: ...` - 显示 API 响应
- `[ShareNewsPage] 当前状态: ...` - 显示组件当前状态

如果看到这些信息，说明组件已正常加载，问题可能在 API 请求或后端服务。

## 常见错误及解决方法

### 错误 1: "无法连接到服务器"
**原因：** 后端服务未启动或无法访问

**解决方法：**
1. 确保后端服务正在运行
2. 检查端口 3001 是否被占用
3. 检查防火墙设置

### 错误 2: "分享链接验证失败"
**原因：** Token 无效或已过期

**解决方法：**
1. 重新创建分享链接
2. 检查分享链接是否已过期
3. 检查数据库中是否存在该 token

### 错误 3: 页面完全空白（无任何内容）
**原因：** 可能是以下问题之一：
- JavaScript 错误导致组件无法渲染
- 路由配置问题
- 组件导入错误

**解决方法：**
1. 打开浏览器控制台，查看是否有 JavaScript 错误
2. 检查是否有调试信息输出
3. 确认使用的是最新版本的代码
4. 清除浏览器缓存并硬刷新 (Ctrl+Shift+R)

## 验证修复

修复后，你应该能看到：

1. **正常情况：**
   - 页面显示"验证分享链接中..."
   - 然后显示新闻列表或密码输入框

2. **错误情况：**
   - 页面显示错误信息
   - 包含调试信息，帮助定位问题

3. **浏览器控制台：**
   - 有详细的调试日志
   - 没有 JavaScript 错误

## 注意事项

1. **开发环境 vs 生产环境：**
   - 开发环境：直接访问 `http://localhost:5173/share/:token`
   - 生产环境：通过 Nginx 访问，路由由 Nginx 配置处理

2. **API 请求：**
   - 开发环境：通过 Vite 代理转发到 `http://localhost:3001`
   - 生产环境：直接请求后端 API

3. **浏览器缓存：**
   - 如果修改了代码，可能需要清除浏览器缓存
   - 使用硬刷新 (Ctrl+Shift+R) 或无痕模式测试
