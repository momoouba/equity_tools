# 被投企业管理功能说明

## 功能概述

被投企业管理模块用于管理投资机构所投资的企业信息，包括企业基本信息、公众号信息、退出状态等，并支持从外部数据库定时同步企业数据。

## 功能特性

### 1. 企业信息管理

- **企业基本信息**：
  - 项目编号（自动生成）
  - 项目简称
  - 被投企业全称（必填）
  - 统一信用代码
  - 企业公众号ID
  - 企业官网
  - 退出状态

- **退出状态选项**：
  - **未退出**：企业仍在投资组合中
  - **部分退出**：已部分退出投资
  - **完全退出**：已完全退出投资（不抓取新闻数据）
  - **继续观察**：继续观察状态
  - **已上市**：企业已上市（不抓取新闻数据）

### 2. 新闻数据抓取规则

系统在从企查查接口获取新闻信息时，会自动排除以下状态的企业：

- **完全退出**：已完全退出投资的企业
- **已上市**：已上市的企业

**说明**：
- 状态为"完全退出"或"已上市"的企业，系统不会从企查查接口抓取新闻数据
- 这些企业的数据仍然保留在系统中，可以正常查看和管理
- 新榜新闻同步也会排除这些状态的企业

### 3. 批量导入功能

- 支持Excel批量导入企业数据
- 自动去重（基于统一信用代码）
- 支持导入模板下载
- 导入时自动生成项目编号

**批量导入模板字段**：
- 项目简称
- 被投企业全称
- 统一信用代码
- 企业公众号id
- 企业官网
- 退出状态（未退出/部分退出/完全退出/继续观察/已上市）

### 4. 定时数据同步

在被投企业管理页面，点击"定时更新"按钮可以配置从外部数据库定时同步企业数据。

**功能特点**：
- 选择外部数据库连接
- 输入SQL查询语句（必须是SELECT语句）
- 设置定时执行时间
- 支持手动执行同步
- 自动映射数据字段

**数据同步逻辑**：
- 根据企业全称和统一信用代码判断数据是否已存在
- 如果存在则更新，不存在则插入
- 自动生成项目编号（如果SQL结果中没有）

详细说明请参考：`外部数据库连接及数据同步功能说明.md`

## 操作说明

### 新增企业

1. 点击"新增"按钮
2. 填写企业信息：
   - 项目简称（可选，可点击"查询"按钮从企查查获取）
   - 被投企业全称（必填）
   - 统一信用代码（可选）
   - 企业公众号ID（可选）
   - 企业官网（可选）
   - 退出状态（默认：未退出）
3. 点击"确定"保存

### 编辑企业

1. 在列表中点击"编辑"按钮
2. 修改企业信息
3. 点击"确定"保存

### 删除企业

1. 在列表中点击"删除"按钮
2. 确认删除操作
3. 企业将被软删除（delete_mark=1）

### 批量导入

1. 点击"批量导入"按钮
2. 下载导入模板（如需要）
3. 填写企业数据
4. 上传Excel文件
5. 系统自动处理导入

### 定时更新

1. 点击"定时更新"按钮
2. 选择外部数据库
   - 如果该数据库配置已有保存的任务，系统会自动加载已保存的SQL、时间和描述
3. 输入SQL查询语句（如果已有保存的任务，会自动填充）
4. 设置定时执行时间（如果已有保存的任务，会自动填充）
5. 点击"保存"保存任务，或点击"手动执行"立即同步
   - **保存任务**：将SQL查询语句、执行时间等信息保存到数据库
   - **手动执行**：
     - 如果SQL输入框有内容，使用输入框中的SQL执行
     - 如果SQL输入框为空，自动使用已保存的SQL执行
     - 如果既没有输入SQL也没有保存的任务，会提示错误

**功能特点**：
- SQL查询语句会保存到数据库中，方便后续使用
- 选择数据库后自动加载已保存的任务信息
- 手动执行时，如果SQL为空，自动使用已保存的SQL
- 每个数据库配置只能有一个定时任务

## 数据字段说明

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 项目编号 | 文本 | 自动生成 | 格式：P+年月日+5位序列号 |
| 项目简称 | 文本 | 否 | 企业的简称或代号 |
| 被投企业全称 | 文本 | 是 | 企业的完整名称 |
| 统一信用代码 | 文本 | 否 | 企业的统一社会信用代码 |
| 企业公众号ID | 文本 | 否 | 企业微信公众号ID |
| 企业官网 | 文本 | 否 | 企业官方网站地址 |
| 退出状态 | 下拉 | 是 | 未退出/部分退出/完全退出/继续观察/已上市 |

## 退出状态说明

### 未退出
- 企业仍在投资组合中
- 正常抓取新闻数据
- 参与所有统计和查询

### 部分退出
- 已部分退出投资
- 正常抓取新闻数据
- 参与所有统计和查询

### 完全退出
- 已完全退出投资
- **不抓取新闻数据**（企查查和新榜接口都会排除）
- 不参与新闻统计
- 数据仍保留在系统中

### 继续观察
- 继续观察状态
- 正常抓取新闻数据
- 参与所有统计和查询

### 已上市
- 企业已上市
- **不抓取新闻数据**（企查查和新榜接口都会排除）
- 不参与新闻统计
- 数据仍保留在系统中

## 新闻数据抓取规则

### 企查查新闻接口

企查查新闻接口在获取新闻数据时，会：

1. 查询所有退出状态不为"完全退出"和"已上市"的企业
2. 获取这些企业的统一信用代码
3. 根据统一信用代码调用企查查新闻接口
4. 将新闻数据保存到系统中

**排除逻辑**：
```sql
SELECT DISTINCT unified_credit_code 
FROM invested_enterprises 
WHERE exit_status NOT IN ('完全退出', '已上市')
AND unified_credit_code IS NOT NULL 
AND unified_credit_code != ''
AND delete_mark = 0
```

### 新榜新闻接口

新榜新闻接口在同步新闻数据时，会：

1. 查询所有退出状态不为"完全退出"和"已上市"的企业
2. 获取这些企业的微信公众号ID
3. 根据公众号ID调用新榜接口
4. 将新闻数据保存到系统中

**排除逻辑**：
```sql
SELECT DISTINCT wechat_official_account_id 
FROM invested_enterprises 
WHERE exit_status NOT IN ('完全退出', '已上市')
AND wechat_official_account_id IS NOT NULL 
AND wechat_official_account_id != ''
AND delete_mark = 0
```

## 统计和查询规则

以下功能会自动排除"完全退出"和"已上市"状态的企业：

1. **用户舆情统计**：统计昨日发布新闻企业个数、累计新闻条数等
2. **新闻列表查询**：用户和管理员查看新闻列表时
3. **AI分析**：进行新闻分析时，只分析有效企业的新闻
4. **定时邮件任务**：发送邮件时，只包含有效企业的新闻

## 注意事项

1. **状态变更影响**：
   - 将企业状态改为"完全退出"或"已上市"后，系统将不再抓取该企业的新闻数据
   - 已存在的新闻数据不会删除，仍可查看

2. **数据完整性**：
   - 退出状态为"完全退出"或"已上市"的企业数据仍保留在系统中
   - 可以正常编辑、查看、删除这些企业记录

3. **批量导入**：
   - 批量导入时，退出状态字段支持所有选项
   - 建议使用标准格式：未退出/部分退出/完全退出/继续观察/已上市

4. **定时同步**：
   - 定时同步功能基于统一社会信用代码进行匹配
   - 如果统一信用代码一致，更新项目简称、企业全称和企业官网
   - 如果统一信用代码不一致或不存在，新增数据并自动生成项目编号
   - 建议在SQL查询中排除"完全退出"和"已上市"状态的企业，避免同步这些企业的数据

## API接口

### 获取企业列表
```
GET /api/enterprises?page=1&pageSize=10&search=关键词
```

### 新增企业
```
POST /api/enterprises
Body: {
  project_abbreviation: string,
  enterprise_full_name: string,
  unified_credit_code: string,
  wechat_official_account_id: string,
  official_website: string,
  exit_status: string
}
```

### 更新企业
```
PUT /api/enterprises/:id
Body: {
  project_abbreviation?: string,
  enterprise_full_name?: string,
  unified_credit_code?: string,
  wechat_official_account_id?: string,
  official_website?: string,
  exit_status?: string
}
```

### 删除企业
```
DELETE /api/enterprises/:id
```

### 批量导入模板
```
GET /api/enterprises/batch-import/template
```

### 批量导入
```
POST /api/enterprises/batch-import
Content-Type: multipart/form-data
Body: { file: File }
```

### 创建/更新定时同步任务
```
POST /api/enterprises/sync-task
Body: {
  db_config_id: string,
  sql_query: string,
  cron_expression: string,
  description?: string
}
```

### 手动执行同步
```
POST /api/enterprises/sync-task/execute
Body: {
  db_config_id: string,
  sql_query: string
}
```
