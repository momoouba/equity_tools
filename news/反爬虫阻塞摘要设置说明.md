# 反爬虫阻塞摘要设置说明

## 📋 功能说明

当新闻抓取遇到反爬虫机制（如HTTP 521错误）时，自动将新闻摘要设置为："网页设置了反爬虫机制，无法获取正文"。

## 🔧 实现逻辑

### 1. 错误检测

在 `fetchContentFromUrl` 方法中：
- 检测到HTTP 521错误时，抛出特殊错误 `ANTI_CRAWLER_BLOCKED`
- 错误对象包含 `status: 521` 和 `url` 信息

### 2. 标记设置

在 `ensureNewsContent` 方法中：
- 捕获 `ANTI_CRAWLER_BLOCKED` 错误
- 在 `newsItem` 对象上设置 `_antiCrawlerBlocked = true` 标记
- 记录警告日志

### 3. 摘要设置

在 `processNewsWithEnterprise` 和 `processNewsWithoutEnterprise` 方法中：
- 检测 `newsItem._antiCrawlerBlocked` 标记
- 如果标记为 `true`，设置特殊摘要：
  - **摘要**：`网页设置了反爬虫机制，无法获取正文`
  - **关键词**：`['政策信息']`
  - **情绪**：`neutral`

## 📝 修改的文件

- `server/utils/newsAnalysis.js`
  - `fetchContentFromUrl` 方法：521错误时抛出特殊错误
  - `ensureNewsContent` 方法：捕获错误并设置标记
  - `processNewsWithEnterprise` 方法：检测标记并设置摘要
  - `processNewsWithoutEnterprise` 方法：检测标记并设置摘要

## ✅ 验证方法

1. **测试反爬虫阻塞场景**：
   - 找到一条来自中国经营网（cb.com.cn）的新闻
   - 触发AI分析或正文提取
   - 如果遇到521错误，应该看到特殊摘要

2. **查看日志**：
   ```bash
   docker compose logs app | grep "反爬虫阻塞"
   ```

3. **预期日志输出**：
   ```
   [fetchContentFromUrl] HTTP 521错误（Cloudflare反爬虫机制）
   [ensureNewsContent] 检测到反爬虫阻塞（521错误）
   [processNewsWithEnterprise] 检测到反爬虫阻塞，设置特殊摘要
   [processNewsWithEnterprise] 反爬虫阻塞 - 情绪: neutral, 摘要: 网页设置了反爬虫机制，无法获取正文
   ```

4. **检查数据库**：
   - 新闻的 `news_abstract` 字段应该为："网页设置了反爬虫机制，无法获取正文"
   - 新闻的 `keywords` 字段应该为：`["政策信息"]`
   - 新闻的 `news_sentiment` 字段应该为：`neutral`

## 📊 处理流程

```
抓取网页内容
    ↓
检测到HTTP 521错误
    ↓
抛出 ANTI_CRAWLER_BLOCKED 错误
    ↓
ensureNewsContent 捕获错误
    ↓
设置 newsItem._antiCrawlerBlocked = true
    ↓
processNewsWithEnterprise/processNewsWithoutEnterprise 检测标记
    ↓
设置摘要为："网页设置了反爬虫机制，无法获取正文"
    ↓
保存到数据库
```

## ⚠️ 注意事项

1. **仅针对521错误**：当前实现仅针对HTTP 521错误（Cloudflare反爬虫机制）
2. **其他错误**：其他HTTP错误（如403、404等）不会触发此逻辑
3. **关键词设置**：反爬虫阻塞时，关键词固定为 `['政策信息']`
4. **情绪设置**：反爬虫阻塞时，情绪固定为 `neutral`

## 🔍 故障排查

### 问题1：摘要未设置为特殊文本

**检查**：
- 查看日志中是否有 "检测到反爬虫阻塞" 的日志
- 确认是否真的是521错误

**解决**：
- 检查 `newsItem._antiCrawlerBlocked` 标记是否正确设置
- 确认错误处理逻辑是否正确执行

### 问题2：摘要被其他逻辑覆盖

**检查**：
- 查看日志中摘要设置的顺序
- 确认是否有其他逻辑在反爬虫检测之后修改了摘要

**解决**：
- 检查 `validateAnalysisResult` 方法是否覆盖了摘要
- 确认摘要设置的优先级
