# API 请求 503 错误排查指南

## 🔍 问题现象

- ✅ 可以登录系统
- ❌ 登录后多个 API 请求返回 503 错误：
  - `/api/system/basic-config` - 503
  - `/api/enterprises?page=1&pageSize=20` - 503
  - `/vite.svg` - 503

## 🚀 立即执行以下诊断命令

### 步骤1：检查服务器日志，确认 serverReady 状态

```bash
cd /opt/newsapp/news

# 查看服务器是否已就绪
sudo docker compose logs app | grep -i "服务器核心功能已就绪\|serverReady"

# 查看最近的错误日志
sudo docker compose logs app --tail 100 | grep -i "error\|503\|fail"
```

### 步骤2：测试 API 请求（从宿主机直接访问应用）

```bash
# 测试系统配置接口
curl -v http://localhost:3001/api/system/basic-config

# 测试企业列表接口
curl -v http://localhost:3001/api/enterprises?page=1&pageSize=20

# 如果这些直接访问都正常，问题在 Nginx 代理
```

### 步骤3：测试通过 Nginx 访问

```bash
# 通过 Nginx 访问（模拟浏览器请求）
curl -v http://localhost/api/system/basic-config

# 查看 Nginx 日志
sudo docker compose logs nginx --tail 50 | grep -i "error\|503\|upstream"
```

### 步骤4：检查 Nginx 能否访问应用容器

```bash
# 从 Nginx 容器内测试应用容器
sudo docker compose exec nginx wget -q -O - http://app:3001/api/system/basic-config

# 应该返回 JSON 数据，而不是 503
```

## 🔧 可能的原因和解决方案

### 原因1：Nginx 无法访问应用容器

**症状：** 直接访问 `localhost:3001` 正常，但通过 Nginx 访问返回 503

**解决方案：**
```bash
# 重启 Nginx 容器
sudo docker compose restart nginx

# 等待 5 秒
sleep 5

# 再次测试
curl http://localhost/api/system/basic-config
```

### 原因2：Nginx upstream 配置问题

**检查配置：**
```bash
# 查看容器内的实际配置
sudo docker compose exec nginx cat /etc/nginx/conf.d/default.conf | grep -A 5 "upstream"

# 应该看到：
# upstream app_backend {
#     server app:3001;
# }
```

### 原因3：容器网络问题

**检查网络：**
```bash
# 检查容器是否在同一网络
sudo docker network inspect newsapp_newsapp-network | grep -A 10 "Containers"

# 测试容器间连接
sudo docker compose exec nginx ping -c 2 app
```

### 原因4：Nginx 缓存了旧的错误响应

**解决方案：**
```bash
# 重启 Nginx 清除缓存
sudo docker compose restart nginx

# 或者完全重启所有服务
sudo docker compose restart
```

## 🎯 快速修复（推荐）

```bash
cd /opt/newsapp/news

# 1. 重启 Nginx（最常见的问题）
sudo docker compose restart nginx

# 2. 等待 5 秒
sleep 5

# 3. 测试
curl http://localhost/api/system/basic-config

# 4. 如果还是不行，重启所有服务
sudo docker compose restart

# 5. 等待 30 秒
sleep 30

# 6. 再次测试
curl http://localhost/api/system/basic-config
```
