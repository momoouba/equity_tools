# 舆情信息公共链接分享功能说明

## 📋 功能概述

已成功实现舆情信息公共链接分享功能，用户可以通过生成分享链接，让任何人（无需登录）访问当前用户视角下的舆情信息页面。分享链接支持有效期设置和密码保护，确保数据安全。

## 🎯 主要功能

### 1. 创建分享链接
- **发布按钮**：在舆情信息页面右上角新增"发布"按钮
- **分享设置**：支持设置有效期和密码保护
- **链接生成**：系统自动生成唯一的分享链接token
- **一键复制**：创建成功后自动复制链接和密码到剪贴板

### 2. 分享链接管理
- **状态控制**：可以启用/禁用分享链接
- **有效期设置**：可设置链接的有效期，过期后自动失效
- **密码保护**：可设置访问密码，增强安全性
- **链接记录**：系统记录所有分享链接的创建时间和设置

### 3. 公共访问页面
- **无需登录**：访问者无需登录即可查看舆情信息
- **密码验证**：如果设置了密码，访问时需要输入密码
- **数据一致性**：访问者看到的数据与创建链接用户视角完全一致
- **功能完整**：支持时间筛选、搜索、企业过滤等所有功能

## 🔧 技术实现

### 数据库设计

#### news_share_links 表结构
```sql
CREATE TABLE news_share_links (
  id VARCHAR(19) PRIMARY KEY COMMENT '数据ID：年月日时分秒+5位自增序列',
  user_id VARCHAR(19) NOT NULL COMMENT '创建用户ID',
  share_token VARCHAR(64) NOT NULL UNIQUE COMMENT '分享链接token',
  status ENUM('active', 'inactive') DEFAULT 'active' COMMENT '状态：active-启用，inactive-禁用',
  has_expiry TINYINT(1) DEFAULT 0 COMMENT '是否有有效期：1-是，0-否',
  expiry_time DATETIME NULL COMMENT '有效期至',
  has_password TINYINT(1) DEFAULT 0 COMMENT '是否有密码：1-是，0-否',
  password_hash VARCHAR(255) NULL COMMENT '密码哈希值',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  INDEX idx_user_id (user_id),
  INDEX idx_share_token (share_token),
  INDEX idx_status (status),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 后端API实现

#### 1. 创建分享链接
**接口**：`POST /api/news-share/create`

**请求参数**：
```json
{
  "hasExpiry": true,           // 是否设置有效期
  "expiryTime": "2026-01-12T18:13:10",  // 有效期时间（ISO格式）
  "hasPassword": true,         // 是否设置密码保护
  "password": "GN5E$F_TKL"     // 访问密码
}
```

**响应示例**：
```json
{
  "success": true,
  "data": {
    "id": "2026011110434500001",
    "shareToken": "abc123...",
    "shareUrl": "https://example.com/share/abc123...",
    "status": "active",
    "hasExpiry": true,
    "expiryTime": "2026-01-12T18:13:10",
    "hasPassword": true
  }
}
```

#### 2. 获取分享链接列表
**接口**：`GET /api/news-share/list`

**响应示例**：
```json
{
  "success": true,
  "data": [
    {
      "id": "2026011110434500001",
      "share_token": "abc123...",
      "shareUrl": "https://example.com/share/abc123...",
      "status": "active",
      "has_expiry": 1,
      "expiry_time": "2026-01-12 18:13:10",
      "has_password": 1,
      "created_at": "2026-01-11 10:43:45",
      "updated_at": "2026-01-11 10:43:45"
    }
  ]
}
```

#### 3. 更新分享链接设置
**接口**：`PUT /api/news-share/:id`

**请求参数**：
```json
{
  "status": "active",          // 状态：active/inactive
  "hasExpiry": true,           // 是否设置有效期
  "expiryTime": "2026-01-12T18:13:10",  // 有效期时间
  "hasPassword": true,         // 是否设置密码保护
  "password": "new_password"    // 新密码（如果修改密码）
}
```

#### 4. 删除分享链接
**接口**：`DELETE /api/news-share/:id`

#### 5. 验证分享链接
**接口**：`GET /api/news-share/verify/:token`

**响应示例**：
```json
{
  "success": true,
  "data": {
    "userId": "2026011110434500001",
    "hasPassword": true
  }
}
```

#### 6. 验证密码
**接口**：`POST /api/news-share/verify-password/:token`

**请求参数**：
```json
{
  "password": "GN5E$F_TKL"
}
```

#### 7. 获取分享的舆情信息（公共接口）
**接口**：`GET /api/news-share/news/:token`

**查询参数**：
- `page`: 页码（默认：1）
- `pageSize`: 每页数量（默认：10）
- `timeRange`: 时间范围（yesterday/thisWeek/lastWeek/thisMonth/all）
- `search`: 搜索关键词
- `enterpriseFilter`: 企业过滤（enterprise/all）

**响应示例**：
```json
{
  "success": true,
  "data": [
    {
      "id": "2026011110434500001",
      "title": "新闻标题",
      "enterprise_full_name": "企业全称",
      "keywords": ["关键词1", "关键词2"],
      "public_time": "2026-01-11 10:43:00",
      "news_abstract": "新闻摘要",
      "source_url": "https://example.com/article",
      "account_name": "公众号名称",
      "wechat_account": "微信号"
    }
  ],
  "total": 100,
  "page": 1,
  "pageSize": 10
```


## 📝 使用说明

### 创建分享链接

1. **进入舆情信息页面**
   - 登录系统后，进入"舆情信息"页面

2. **点击发布按钮**
   - 在页面右上角找到红色的"发布"按钮
   - 点击打开分享设置对话框

3. **配置分享设置**
   - **开启分享**：打开"公共链接分享"开关
   - **设置有效期**（可选）：
     - 勾选"有效期"复选框
     - 选择有效期截止时间
   - **设置密码保护**（可选）：
     - 勾选"密码保护"复选框
     - 输入访问密码（或点击"重新生成"自动生成密码）

4. **创建链接**
   - 点击"创建链接"按钮
   - 系统自动生成分享链接
   - 链接和密码（如有）自动复制到剪贴板

### 访问分享链接

1. **打开分享链接**
   - 通过生成的链接访问（格式：`https://域名/share/{token}`）

2. **密码验证**（如果设置了密码）
   - 在密码输入框中输入访问密码
   - 点击"确认"按钮

3. **查看舆情信息**
   - 验证通过后，可以查看与创建用户视角一致的舆情信息
   - 支持时间筛选、搜索、企业过滤等功能

### 管理分享链接

**更新链接设置**：
- 通过 `PUT /api/news-share/:id` 接口更新链接状态、有效期、密码等设置

**删除链接**：
- 通过 `DELETE /api/news-share/:id` 接口删除不再需要的分享链接

**查看链接列表**：
- 通过 `GET /api/news-share/list` 接口查看当前用户创建的所有分享链接

## 🔒 安全特性

### 1. Token安全
- 使用64位随机token，确保链接唯一性
- Token存储在数据库中，支持状态管理

### 2. 密码保护
- 密码使用bcrypt加密存储
- 支持密码验证，防止未授权访问

### 3. 有效期控制
- 支持设置链接有效期
- 过期后自动失效，无法访问

### 4. 状态管理
- 支持启用/禁用链接
- 禁用后链接立即失效

### 5. 权限隔离
- 访问者只能看到创建用户视角下的数据
- 管理员和普通用户的数据权限保持一致

## 📊 数据流程

```
用户点击发布按钮
  ↓
打开分享设置对话框
  ↓
配置有效期和密码（可选）
  ↓
调用创建API
  ↓
生成token和链接
  ↓
保存到数据库
  ↓
返回链接给用户
  ↓
```

```
访问者打开分享链接
  ↓

  ↓
如果设置了密码，显示密码输入框
  ↓
验证密码（如果设置了）
  ↓
获取创建用户的数据权限
  ↓
显示舆情信息页面
```

## 🔍 注意事项

1. **前端域名配置**
   - 如果前端和后端不在同一域名，需要在环境变量中配置 `FRONTEND_HOST`
   - 确保分享链接指向正确的前端地址

2. **密码安全**
   - 建议使用强密码（包含大小写字母、数字、特殊字符）
   - 密码长度建议至少8位

3. **有效期设置**
   - 建议设置合理的有效期，避免链接长期有效
   - 过期后需要重新创建链接

4. **数据权限**
   - 分享链接显示的数据与创建用户视角完全一致
   - 管理员创建的链接显示全部数据
   - 普通用户创建的链接只显示其企业相关的数据

5. **性能考虑**
   - 分享链接访问不需要登录，但需要验证token
   - 大量并发访问时注意数据库性能
