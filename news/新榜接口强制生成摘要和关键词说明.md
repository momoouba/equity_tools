# 新榜接口强制生成摘要和关键词说明

## 修复时间
2025-12-18

## 修复内容

### 1. 强制生成摘要和关键词逻辑

**要求**：
- 新榜接口的新闻必须强制生成摘要和关键词
- 除非内容为空或者乱码，否则必须要输出摘要和关键词
- 如果第一次分析完后有部分不满足这个条件，需要对已入库的当前数据，再次检查，是否存在关键词和摘要为空的情况
- 如果有，需要再执行一次AI分析

## 使用说明

### 1. 自动处理

新榜接口的新闻在同步后会自动触发AI分析，系统会自动：
1. 检查内容是否是乱码
2. 如果内容有效，强制生成摘要和关键词
3. 如果第一次分析后摘要或关键词仍为空，会再次分析

### 2. 手动重新分析

如果发现已入库的新榜新闻中有摘要或关键词为空的情况，可以调用API手动重新分析：

```bash
POST /api/news-analysis/reanalyze-xinbang-missing
Content-Type: application/json
x-user-id: <用户ID>
x-user-role: admin

{
  "startDate": "2025-12-18",
  "endDate": "2025-12-18",
  "limit": 100
}
```

### 3. 查询分析进度

```bash
GET /api/news-analysis/analysis-progress/xinbang-missing-1234567890
x-user-id: <用户ID>
```

## 验证方法

### 1. 检查新榜新闻是否有摘要和关键词

```sql
SELECT id, title, 
       LENGTH(content) as content_length,
       news_abstract,
       keywords,
       APItype,
       created_at
FROM news_detail
WHERE APItype = '新榜'
  AND created_at >= DATE('2025-12-18')
  AND content IS NOT NULL 
  AND content != ''
  AND LENGTH(content) > 20
  AND (news_abstract IS NULL OR news_abstract = '' OR keywords IS NULL OR keywords = '' OR keywords = '[]')
  AND delete_mark = 0
ORDER BY created_at DESC
LIMIT 10;
```

### 2. 检查乱码内容是否被跳过

```sql
SELECT id, title, 
       content,
       news_abstract,
       keywords
FROM news_detail
WHERE APItype = '新榜'
  AND created_at >= DATE('2025-12-18')
  AND content IS NOT NULL 
  AND content != ''
  AND delete_mark = 0
ORDER BY created_at DESC
LIMIT 10;
```

## 注意事项

1. **乱码检测**：系统会自动检测乱码内容（包含JavaScript代码、CSS样式等），乱码内容不会强制生成摘要和关键词
2. **图片内容**：如果内容只是图片（无文字），系统会使用默认的摘要和关键词（如"正文无文字，无法生成摘要"、"图片内容"）
3. **内容长度**：只有内容长度大于20字符的新闻才会强制生成摘要和关键词
4. **性能考虑**：重新分析会调用AI模型，建议分批处理，避免一次性处理过多数据
