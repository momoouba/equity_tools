# 新闻同步和邮件发送时间范围分析

## 场景描述

假设：
- **1月1日（周四）**：法定假日，工作日（调休），执行同步
- **1月2日（周五）**：假日，跳过执行
- **1月3日（周六）**：假日，跳过执行
- **1月4日（周日）**：工作日（调休），执行同步和邮件发送

## 问题1：1月4日执行时，新闻同步查询的数据日期范围是什么？

### 答案

**时间范围：2025-01-01 00:00:00 到 2025-01-04 00:00:00**

**说明：** 包含1月1日、1月2日、1月3日三天的数据

**执行时间：** 2025-01-01 00:00:00

**同步数据：** 2024-12-31 00:00:00 到 2025-01-01 00:00:00 的新闻

**执行后更新：**
- `last_sync_time` = 2025-01-01 00:00:00

#### 1月2日（周五）跳过

**执行时间：** 2025-01-02 00:00:00

**判断：** 节假日，跳过执行

**结果：**
- 不执行同步
- **不更新 `last_sync_time`**
- `last_sync_time` 仍然是：2025-01-01 00:00:00

#### 1月3日（周六）跳过

**执行时间：** 2025-01-03 00:00:00

**判断：** 节假日，跳过执行

**结果：**
- 不执行同步
- **不更新 `last_sync_time`**
- `last_sync_time` 仍然是：2025-01-01 00:00:00

#### 1月4日（周日）执行

**执行时间：** 2025-01-04 00:00:00

**时间范围计算：**

根据代码逻辑（`server/routes/news.js` 第652-670行）：

```javascript
if (isManual) {
  // 手动触发：前一天00:00:00到当天00:00:00
} else if (config.last_sync_time) {
  // 定时任务：有上次同步时间，使用上次同步时间到当天00:00:00
```

**计算过程：**
1. `baseRunDate` = 2025-01-04 00:00:00（执行日期）
2. `toDate` = 2025-01-04 00:00:00（当天00:00:00）
3. `config.last_sync_time` = 2025-01-01 00:00:00（上次同步时间）
4. `fromDate` = `new Date(config.last_sync_time)` = 2025-01-01 00:00:00

**最终时间范围：**
- `fromDate`：2025-01-01 00:00:00（上次同步时间）
- `toDate`：2025-01-04 00:00:00（执行当天00:00:00）

**同步数据：** 2025-01-01 00:00:00 到 2025-01-04 00:00:00 的新闻

```javascript
function getYesterdayTimeRange() {
  const now = new Date();
  // 使用Asia/Shanghai时区计算本地日期
  const localDateStr = now.toLocaleDateString('zh-CN', { timeZone: 'Asia/Shanghai' });
  const [localYear, localMonth, localDay] = localDateStr.split('/').map(Number);
  
  // 创建本地时区的今天00:00:00
  const today = new Date(localYear, localMonth - 1, localDay, 0, 0, 0);
  
  // 创建本地时区的昨天00:00:00
  const yesterday = new Date(today);
  yesterday.setDate(yesterday.getDate() - 1);
  
  return {
    from: formatDate(yesterday),  // 2025-01-03 00:00:00
    to: formatDate(today)          // 2025-01-04 00:00:00
  };
}
```

**计算过程：**
1. `now` = 2025-01-04 00:00:00（执行时间）
2. `today` = 2025-01-04 00:00:00（今天）
3. `yesterday` = 2025-01-03 00:00:00（昨天）

**最终时间范围：**
- `fromDate`：2025-01-03 00:00:00（前一天）
- `toDate`：2025-01-04 00:00:00（当天）

**邮件内容：** 2025-01-03 00:00:00 到 2025-01-04 00:00:00 的新闻

**结果：** ✅ 发送邮件，内容为1月3日的新闻

## 关键区别

### 新闻同步 vs 邮件发送

| 项目 | 新闻同步 | 邮件发送 |
|------|---------|---------|
| **时间范围计算** | 从上次同步时间到当前时间 | 从前一天00:00:00到当天00:00:00 |
| **1月4日的时间范围** | 2025-01-01 00:00:00 → 2025-01-04 00:00:00 | 2025-01-03 00:00:00 → 2025-01-04 00:00:00 |
| **包含的数据** | 1月1日、1月2日、1月3日 | 1月3日 |
| **逻辑** | 增量同步（补上跳过期间的数据） | 每日发送（只发送前一天的数据） |

### 为什么会有这个区别？

1. **新闻同步（增量同步）**
   - 目的：确保数据不遗漏
   - 逻辑：从上次同步时间开始，补上所有跳过期间的数据
   - 结果：1月4日执行时，会同步1月1日、1月2日、1月3日三天的数据

2. **邮件发送（每日发送）**
   - 目的：每日发送前一天的新闻摘要
   - 逻辑：始终发送前一天的数据，不累积
   - 结果：1月4日执行时，只发送1月3日的数据

## 时间线总结

| 日期 | 类型 | 新闻同步 | 邮件发送 | 说明 |
|------|------|---------|---------|------|
| 1月1日 | 工作日 | ✅ 执行<br>范围：2024-12-31 → 2025-01-01 | ✅ 发送<br>范围：2024-12-31 → 2025-01-01 | 同步和发送都执行 |
| 1月2日 | 假日 | ❌ 跳过 | ❌ 跳过 | 都不执行 |
| 1月3日 | 假日 | ❌ 跳过 | ❌ 跳过 | 都不执行 |
| 1月4日 | 工作日 | ✅ 执行<br>范围：**2025-01-01 → 2025-01-04** | ✅ 发送<br>范围：**2025-01-03 → 2025-01-04** | 同步补上3天数据，邮件只发送1天数据 |

## 注意事项

1. **新闻同步会补上跳过期间的数据**
   - 如果连续跳过多天，下次执行时会同步多天的数据
   - 数据量可能较大

2. **邮件发送只发送前一天的数据**
   - 即使跳过多天，下次发送时也只发送前一天的数据
   - 不会累积发送

3. **两者逻辑不同**
   - 新闻同步：增量同步，确保数据完整性
   - 邮件发送：每日摘要，不累积

4. **建议**
   - 如果希望邮件也包含跳过期间的数据，需要修改邮件发送逻辑
   - 或者手动触发邮件发送，补发跳过期间的邮件

5. **上海国际来源的额外发送限制（邮件发送）**
   - 仅对 `news_detail.APItype = '上海国际'` 生效
   - 除 `created_at` 需落在邮件发送时间范围内外，还要求：
     - `public_time` 非空
     - `DATEDIFF(DATE(created_at), DATE(public_time)) <= 30`
   - 若 `public_time` 为空，或日期差大于 30 天，不发送该新闻
