# 上海国际集团新闻接口逻辑说明

## 一、触发方式

上海国际集团新闻接口可以通过以下方式触发：

1. **手动触发**：通过前端界面点击「同步」按钮
2. **定时任务**：根据配置的 Cron 表达式自动执行

## 二、完整执行流程

### 步骤1：获取配置信息

1. **获取新闻接口配置**（`news_interface_config` 表）
   - 查找 `interface_type = '上海国际集团'` 且 `is_active = 1` 的配置
   - 如果提供了 `config_id`，则使用指定的配置
   - 否则使用最新的上海国际集团配置

2. **获取集团接口凭证**（`shanghai_international_group_config` 表）
   - 查找 `is_active = 1` 的配置
   - 获取以下信息：
     - `x_app_id`：X-App-Id，Ipass 平台授权的消费方标识
     - `api_key`：APIkey，消费方认证
     - `daily_limit`：每日查询限制次数（默认 100 次）

### 步骤2：计算时间范围

时间范围计算逻辑与企查查类似：

- **有 last_sync_date**：从上次同步日期开始，到执行日期前一天
- **有 last_sync_time**：从上次同步时间的日期开始，到执行日期前一天
- **首次执行**：使用前一天 00:00:00 到执行日期前一天的日期范围

### 步骤3：获取企业列表

从 `invested_enterprises` 表查询需要同步的企业：

**查询条件：**
- `exit_status NOT IN ('完全退出', '已上市', '不再观察')`
- `unified_credit_code IS NOT NULL` 且不为空字符串
- `delete_mark = 0`

**企业类型过滤：** 若配置了 `entity_type`，按企业类型过滤。

### 步骤4：处理每日查询限制

- 每个企业查询 1 次接口
- 受 `daily_limit` 限制，每次同步最多处理 `dailyLimit` 个企业
- 剩余企业将在后续同步中处理

### 步骤5：遍历企业并调用接口

对每个企业执行以下操作：

#### 5.1 构建请求 Headers

```javascript
{
  'Content-Type': 'application/json; charset=UTF-8',
  'X-App-Id': xAppId,           // 从集团接口配置获取
  'X-Sequence-No': uuid,        // 每次请求生成新的 UUID
  'X-Timestamp': timestamp,     // 当前时间戳（毫秒）
  'APIkey': apiKey             // 从集团接口配置获取
}
```

#### 5.2 构建请求体

```javascript
{
  instn_idtfn_cd: creditCode,   // 统一信用代码
  start_time: startDate,        // 开始日期（yyyy-MM-dd）
  end_time: endDate             // 结束日期（yyyy-MM-dd）
}
```

#### 5.3 发送 HTTP 请求

- 方法：POST
- URL：从新闻接口配置的 `request_url` 获取（默认测试环境：`http://114.141.181.181:8000/dofp/v2/ipaas/query/newsAndPubnote`）

#### 5.4 处理返回数据

**成功响应（Code = '200'）：**

1. 合并 `Data.instn_news`（舆情）和 `Data.instn_pubnote`（公告）为新闻列表
2. 对每条新闻进行去重检查（`source_url` 或 `title + public_time`）
3. 数据转换并插入 `news_detail` 表：

| 接口字段 | 数据库字段 |
|----------|------------|
| news_title | title |
| news_url | source_url |
| disp_time | public_time |
| sentiment_typ（正面→positive，负面→negative，中性→neutral） | news_sentiment |
| pbls_src 或「上海国际集团」 | account_name, wechat_account |
| instn_nm | enterprise_full_name |
| event_nm_lv12 或 risk_nm_lv12 | summary |

- `APItype = '上海国际集团'`
- 企业全称、entity_type、fund、sub_fund 从 `invested_enterprises` 表按统一信用代码匹配

## 三、关键特性

### 1. 配置分层
- **集团接口配置**：存放 X-App-Id、APIkey、每日查询限制
- **新闻接口配置**：存放请求地址、定时规则、企业类型过滤

### 2. 去重机制
- 根据 `source_url` 或 `title + public_time` 去重
- 已删除的新闻（`delete_mark = 1`）不会重新插入

### 3. 数据完整性
- 同时获取舆情（instn_news）和公告（instn_pubnote）数据
- 自动匹配企业全称及 fund、sub_fund

## 四、配置入口

1. **系统配置 → 上海国际集团接口配置**：配置 X-App-Id、APIkey、每日查询限制
2. **系统配置 → 新闻接口配置**：新增配置，接口类型选择「上海国际集团」，填写请求地址和定时规则

## 五、注意事项

1. 需先配置「上海国际集团接口配置」，再创建「新闻接口配置」
2. 时间范围基于北京时区计算
3. 单个企业失败不影响整体同步
4. 接口文档参见：`国际集团新闻舆情接口.MD`
