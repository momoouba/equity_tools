# 企查查正文提取问题排查指南

## 问题描述
在抓取企查查的数据中，如果是链接的，需要增加查询前端类似 `article class="main-news article-with-html"` 这种后面的正文内容的提取。但发现有些新闻没有正确提取内容。

## 已实现的匹配规则

### 匹配规则（按优先级排序）

1. **精确匹配两个class同时存在**：
   - `class="main-news article-with-html"`
   - `class="article-with-html main-news"`

2. **匹配包含 main-news 的 article 标签**

3. **匹配包含 article-with-html 的 article 标签**

### 内容长度要求

- 对于企查查的 `main-news` 匹配规则（前4个规则）：**最低100字符**
- 对于其他匹配规则：**最低200字符**

## 调试日志

系统已添加详细的调试日志，包括：

1. **HTML检查日志**：
   - 是否包含 `article` 标签
   - 是否包含 `main-news` class
   - 是否包含 `article-with-html` class

2. **匹配过程日志**：
   - 每个匹配规则的尝试结果
   - 提取的内容长度
   - 是否满足最小长度要求

3. **匹配成功日志**：
   - 匹配到的规则编号
   - 提取的内容长度
   - 匹配到的article标签的class属性

## 排查步骤

### 1. 查看应用日志

在Linux服务器上查看应用日志：

```bash
# 查看应用容器日志
sudo docker compose logs app --tail 200 | grep -i "extractArticleContent\|fetchContentFromUrl\|企查查同步"

# 或者实时查看日志
sudo docker compose logs app -f | grep -i "extractArticleContent\|fetchContentFromUrl"
```

### 2. 查找特定新闻的提取日志

如果知道新闻ID或URL，可以搜索相关日志：

```bash
# 搜索特定URL的提取日志
sudo docker compose logs app | grep "开始抓取网页内容: https://example.com"

# 搜索包含main-news的日志
sudo docker compose logs app | grep "main-news"
```

### 3. 检查匹配规则是否生效

查看日志中是否有以下信息：

```
[extractArticleContent] 尝试匹配规则 1，提取内容长度: XXX字符，最小要求: 100字符
[extractArticleContent] ✓ 找到正文容器（规则1），长度: XXX字符
[extractArticleContent] 匹配到企查查等网站的article标签（main-news/article-with-html）
[extractArticleContent] 匹配到的article标签class: "main-news article-with-html"
```

### 4. 常见问题

#### 问题1：匹配规则未匹配到内容

**日志特征**：
```
[extractArticleContent] 规则 1（企查查main-news匹配）未匹配到内容
```

**可能原因**：
- HTML结构不符合预期
- class属性使用了单引号而不是双引号
- class属性中有额外的空格或特殊字符

**解决方法**：
1. 检查实际的HTML结构
2. 查看日志中的HTML检查结果
3. 如果HTML中包含 `main-news` 但未匹配，可能需要调整正则表达式

#### 问题2：匹配成功但内容太短

**日志特征**：
```
[extractArticleContent] ⚠️ 规则 1 匹配成功但内容太短（50字符 < 100字符），继续尝试下一个规则
```

**可能原因**：
- article标签内的内容确实很短
- 提取的内容被其他标签分割
- 需要进一步处理HTML结构

**解决方法**：
1. 查看提取的内容预览
2. 检查是否需要调整内容长度阈值
3. 检查是否需要处理嵌套的HTML结构

#### 问题3：HTML中包含main-news但提取失败

**日志特征**：
```
[fetchContentFromUrl] HTML检查: hasArticleTag=true, hasMainNews=true, hasArticleWithHtml=true
[fetchContentFromUrl] 提取完成，文本长度: 0字符
[fetchContentFromUrl] ⚠️ HTML中包含main-news或article-with-html，但提取失败，可能需要检查匹配规则
```

**可能原因**：
- 正则表达式未正确匹配
- HTML结构有特殊格式
- 需要调整匹配规则

**解决方法**：
1. 查看完整的article标签结构
2. 检查正则表达式是否需要调整
3. 可能需要添加新的匹配规则

## 手动测试

### 测试特定URL的提取

可以创建一个测试脚本来测试特定URL的提取：

```javascript
const newsAnalysis = require('./server/utils/newsAnalysis');
const url = 'https://example.com/news/123'; // 替换为实际的企查查新闻链接

newsAnalysis.fetchContentFromUrl(url).then(content => {
  console.log('提取的正文长度:', content.length);
  console.log('正文预览（前500字符）:', content.substring(0, 500));
}).catch(error => {
  console.error('提取失败:', error);
});
```

### 查看数据库中的提取结果

```sql
-- 查询最近同步的企查查新闻，检查正文提取情况
SELECT id, title, 
       LENGTH(content) as content_length,
       source_url,
       APItype,
       created_at,
       CASE 
         WHEN content IS NULL OR content = '' THEN '未提取'
         WHEN LENGTH(content) < 100 THEN '提取内容太短'
         ELSE '已提取'
       END as extract_status
FROM news_detail
WHERE APItype = '企查查'
  AND created_at >= DATE('2025-12-18')
  AND source_url IS NOT NULL
  AND source_url != ''
  AND delete_mark = 0
ORDER BY created_at DESC
LIMIT 20;
```

## 优化建议

如果发现某些网站的HTML结构特殊，可以：

1. **添加新的匹配规则**：在 `extractArticleContent` 方法的 `articleSelectors` 数组中添加新的匹配规则

2. **调整内容长度阈值**：根据实际情况调整 `minLength` 的值

3. **优化正则表达式**：根据实际的HTML结构优化正则表达式
