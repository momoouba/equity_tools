# 查看完整的邮件发送日志

## 问题说明

如果使用 `grep` 过滤日志，可能会漏掉一些重要的日志信息。建议先查看**完整日志**，然后再根据需要过滤。

## Docker 环境查看完整日志

### 方法1：查看最近100行日志（推荐）

```bash
# 进入项目目录
cd /opt/newsapp/news

# 查看最近100行日志（不过滤）
sudo docker compose logs --tail 100 app
```

### 方法2：实时查看所有日志（发送邮件时使用）

```bash
# 实时查看所有日志（不过滤）
sudo docker compose logs -f app
```

**使用步骤：**
1. 先运行上面的命令
2. 然后在另一个终端或浏览器中触发邮件发送
3. 观察日志输出，按 `Ctrl+C` 退出

### 方法3：查看最近10分钟的日志

```bash
# 查看最近10分钟的日志（不过滤）
sudo docker compose logs --since 10m app
```

### 方法4：查看最近5分钟的日志

```bash
# 查看最近5分钟的日志（不过滤）
sudo docker compose logs --since 5m app
```

## 关键日志信息

发送邮件时，应该能看到以下日志（按顺序）：

1. `[邮件发送] ========== 开始获取用户可见的舆情信息 ==========`
2. `[邮件发送] 用户ID: ...`
3. `[邮件发送] 时间范围: ...`
4. `[邮件发送] 用户权限检查: ...`
5. `[邮件发送] 用户角色: ...`
6. `[邮件发送] ========== 管理员查询逻辑 ==========`
7. `[邮件发送] 步骤1：查询满足条件的被投企业...`
8. `[邮件发送] 管理员：执行SQL查询`
9. `[邮件发送] 管理员：时间范围内总新闻数：...`
10. `[邮件发送] 管理员：公众号ID匹配的新闻总数（不限时间）：...`
11. `[邮件发送] 管理员：公众号ID匹配 + 时间范围的新闻数：...`
12. `[邮件发送] 管理员：查询到 X 条新闻`
13. `[邮件发送] ========== 开始过滤新闻 ==========`
14. `[邮件发送] 初始查询结果：X 条新闻`
15. `[邮件发送] 初始新闻详情（前5条）：...`
16. `[邮件发送] 企查查类别过滤后：X 条新闻`
17. `[邮件发送] 过滤掉 X 条摘要和正文都为空的数据，剩余 X 条`
18. `[邮件发送] ========== 过滤完成 ==========`
19. `[邮件发送] 最终返回：X 条新闻`

## 如果看不到详细日志

如果看不到上述详细日志，说明代码可能还没有更新到服务器。需要：

1. **确认代码已更新到服务器**
   ```bash
   # 检查文件修改时间
   ls -lh server/utils/scheduledEmailTasks.js
   ```

2. **重启应用容器**
   ```bash
   sudo docker compose restart app
   ```

3. **重新测试发送邮件**

4. **查看日志**
   ```bash
   # 实时查看日志
   sudo docker compose logs -f app
   ```

## 过滤日志（可选）

如果需要过滤日志，可以使用：

```bash
# 只查看邮件发送相关日志
sudo docker compose logs app | grep '邮件发送'

# 查看最近的邮件发送日志（最后50行）
sudo docker compose logs app | grep '邮件发送' | tail -50

# 实时查看邮件发送日志
sudo docker compose logs -f app | grep '邮件发送'
```

**注意：** 建议先查看完整日志，确认所有步骤都正常执行，然后再使用过滤。
