# 解决端口 3306 占用问题

## ❌ 错误信息

```
ERROR] Can't start server: Bind on TCP/IP port. Got error: 98: Address already in use
ERROR] Do you already have another server running on port: 3306 ?
```

## 🔍 问题原因

端口 3306 已经被其他进程占用，可能是：
1. 之前安装的 MySQL 进程还在运行
2. 其他数据库服务占用了端口
3. 残留的 MySQL/MariaDB 进程

## ✅ 解决方法

### 步骤1：查找占用端口的进程

```bash
# 方法1：使用 lsof
sudo lsof -i :3306

# 方法2：使用 netstat
sudo netstat -tulpn | grep 3306

# 方法3：使用 ss
sudo ss -tulpn | grep 3306

# 方法4：使用 fuser
sudo fuser 3306/tcp
```

### 步骤2：停止占用端口的进程

#### 方法A：如果是 MySQL/MariaDB 进程

```bash
# 查找所有 MySQL/MariaDB 进程
ps aux | grep mysql
ps aux | grep mariadb

# 停止所有 MySQL 相关进程
sudo pkill mysql
sudo pkill mysqld
sudo pkill mariadb
sudo pkill mariadbd

# 或使用 systemctl 停止（如果服务存在）
sudo systemctl stop mysql 2>/dev/null
sudo systemctl stop mysqld 2>/dev/null
```

#### 方法B：根据进程ID停止

```bash
# 从 lsof 或 netstat 输出中找到进程ID（PID）
# 然后停止该进程
sudo kill <PID>

# 如果普通 kill 不行，使用强制停止
sudo kill -9 <PID>
```

### 步骤3：确认端口已释放

```bash
# 再次检查端口
sudo netstat -tulpn | grep 3306

# 应该没有输出，说明端口已释放
```

### 步骤4：启动 MariaDB

```bash
# 启动 MariaDB
sudo systemctl start mariadb

# 检查状态
sudo systemctl status mariadb

# 测试连接
sudo mysql -u root -e "SELECT 1;"
```

## 🚀 一键修复命令

```bash
# 查找并停止占用 3306 端口的进程
sudo lsof -i :3306 | grep LISTEN | awk '{print $2}' | xargs -r sudo kill -9 && \
sudo pkill mysql 2>/dev/null; \
sudo pkill mysqld 2>/dev/null; \
sudo pkill mariadb 2>/dev/null; \
sudo pkill mariadbd 2>/dev/null; \
sleep 2 && \
sudo systemctl start mariadb && \
sudo systemctl status mariadb
```

## 🔧 详细操作步骤

### 步骤1：查找占用进程

```bash
# 查看占用 3306 端口的进程
sudo lsof -i :3306
```

输出示例：
```
COMMAND   PID  USER   FD   TYPE DEVICE SIZE/OFF NODE NAME
mysqld  12345 mysql   21u  IPv4  12345      0t0  TCP *:mysql (LISTEN)
```

### 步骤2：停止进程

```bash
# 如果看到进程ID（PID），停止它
sudo kill 12345

# 或强制停止
sudo kill -9 12345

# 或停止所有 MySQL 相关进程
sudo pkill -9 mysql
sudo pkill -9 mysqld
```

### 步骤3：清理残留进程

```bash
# 检查是否还有进程
ps aux | grep mysql
ps aux | grep mariadb

# 如果还有，强制停止
sudo pkill -9 mysql
sudo pkill -9 mysqld
sudo pkill -9 mariadb
sudo pkill -9 mariadbd
```

### 步骤4：确认端口已释放

```bash
# 检查端口
sudo netstat -tulpn | grep 3306

# 如果没有输出，说明端口已释放
```

### 步骤5：启动 MariaDB

```bash
# 启动服务
sudo systemctl start mariadb

# 检查状态
sudo systemctl status mariadb

# 如果成功，应该看到 "active (running)"
```

### 步骤6：创建数据库

```bash
# 创建应用数据库
sudo mysql -u root <<EOF
CREATE DATABASE IF NOT EXISTS investment_tools DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
SHOW DATABASES;
EOF
```

### 步骤7：重启应用

```bash
cd /opt/newsapp/news
pm2 restart newsapp
pm2 logs newsapp --lines 50
```

## 📋 完整修复流程

```bash
# 1. 查找占用端口的进程
echo "=== 查找占用 3306 端口的进程 ==="
sudo lsof -i :3306

# 2. 停止所有 MySQL/MariaDB 相关进程
echo -e "\n=== 停止所有 MySQL/MariaDB 进程 ==="
sudo pkill -9 mysql 2>/dev/null
sudo pkill -9 mysqld 2>/dev/null
sudo pkill -9 mariadb 2>/dev/null
sudo pkill -9 mariadbd 2>/dev/null
sudo systemctl stop mysql 2>/dev/null
sudo systemctl stop mysqld 2>/dev/null

# 3. 等待进程完全停止
sleep 3

# 4. 确认端口已释放
echo -e "\n=== 确认端口已释放 ==="
sudo netstat -tulpn | grep 3306 || echo "端口 3306 已释放"

# 5. 启动 MariaDB
echo -e "\n=== 启动 MariaDB ==="
sudo systemctl start mariadb

# 6. 检查状态
echo -e "\n=== 检查 MariaDB 状态 ==="
sudo systemctl status mariadb --no-pager -l | head -15

# 7. 测试连接
echo -e "\n=== 测试数据库连接 ==="
sudo mysql -u root -e "SELECT 1;" 2>&1
```

## ⚠️ 注意事项

1. **停止进程前确认**：确保停止的是正确的进程，不要误停止其他重要服务
2. **数据安全**：如果 MySQL 正在运行且有重要数据，先备份再停止
3. **服务冲突**：确保不会同时运行 MySQL 和 MariaDB
