# 手动上传前端文件部署指南

本文档说明如何在不使用Git的情况下，手动上传前端文件到Ubuntu Docker生产环境。

## 部署流程概览

1. **本地构建前端** → 2. **打包dist目录** → 3. **上传到服务器** → 4. **解压并更新** → 5. **重启服务**

---

## 步骤 1: 在本地构建前端

### 1.1 进入前端目录

```bash
# Windows PowerShell 或 CMD
cd E:\USER\SUREAL\Desktop\equity_news\news\client
```

### 1.2 安装依赖（如果需要）

```bash
npm install
```

### 1.3 构建前端

```bash
npm run build
```

### 1.4 验证构建结果

构建完成后，检查 `client/dist` 目录是否存在，并包含以下文件：
- `index.html`
- `assets/` 目录（包含JS和CSS文件）

---

## 步骤 2: 打包dist目录

### 方法A: 使用压缩工具（推荐）

1. 进入 `news/client` 目录
2. 右键点击 `dist` 文件夹
3. 选择"压缩"或"添加到压缩文件"
4. 创建 `dist.zip` 或 `dist.tar.gz`

### 方法B: 使用命令行（Windows PowerShell）

```powershell
# 进入client目录
cd E:\USER\SUREAL\Desktop\equity_news\news\client

# 压缩dist目录为zip
Compress-Archive -Path dist -DestinationPath dist.zip -Force

# 或使用tar（如果可用）
tar -czf dist.tar.gz dist
```

---

## 步骤 3: 上传文件到服务器

### 方法A: 使用SFTP工具（推荐）

**推荐工具：**
- WinSCP（Windows）
- FileZilla（跨平台）
- MobaXterm（Windows）

**连接信息：**
- 主机：你的服务器IP地址
- 端口：22（SSH默认端口）
- 用户名：你的服务器用户名
- 密码：你的服务器密码
- 协议：SFTP

**上传步骤：**
1. 连接到服务器
2. 导航到项目目录（例如：`/home/username/project/news` 或 `/opt/newsapp/news`）
3. 上传 `dist.zip` 或 `dist.tar.gz` 文件

### 方法B: 使用SCP命令（命令行）

```bash
# 在本地PowerShell或CMD中执行
scp E:\USER\SUREAL\Desktop\equity_news\news\client\dist.zip username@服务器IP:/path/to/project/news/

# 示例
scp E:\USER\SUREAL\Desktop\equity_news\news\client\dist.zip root@192.168.1.100:/opt/newsapp/news/
```

### 方法C: 使用FTP工具

如果服务器支持FTP，可以使用FTP客户端上传文件。

---

## 步骤 4: 在服务器上更新前端文件

### 4.1 连接到服务器

使用SSH连接到服务器：

```bash
ssh username@服务器IP
# 示例: ssh root@192.168.1.100
```

### 4.2 进入项目目录

```bash
cd /path/to/your/project/news
# 示例: cd /opt/newsapp/news
```

### 4.3 解压上传的文件

```bash
# 如果上传的是zip文件
unzip -o dist.zip

# 如果上传的是tar.gz文件
tar -xzf dist.tar.gz

# 验证解压结果
ls -la client/dist/
```

### 4.4 查找Docker Volume

```bash
# 查看所有volumes
sudo docker volume ls | grep frontend

# 查看volume详细信息（获取路径）
sudo docker volume inspect newsapp_app_frontend
```

**记录下Mountpoint路径**，例如：`/var/lib/docker/volumes/newsapp_app_frontend/_data`

### 4.5 更新前端文件到Volume

**方法1: 直接操作volume路径（推荐）**

```bash
# 替换VOLUME_PATH为实际的volume路径
VOLUME_PATH="/var/lib/docker/volumes/newsapp_app_frontend/_data"

# 清空旧文件
sudo rm -rf "$VOLUME_PATH"/*
sudo rm -rf "$VOLUME_PATH"/.[!.]* 2>/dev/null || true

# 复制新文件
sudo cp -r client/dist/* "$VOLUME_PATH/"

# 设置权限
sudo chown -R root:root "$VOLUME_PATH"
sudo chmod -R 755 "$VOLUME_PATH"
```

**方法2: 使用临时容器（如果无法直接访问volume路径）**

```bash
# 创建临时容器
TEMP_CONTAINER="temp-frontend-$(date +%s)"
sudo docker run -d --name "$TEMP_CONTAINER" -v newsapp_app_frontend:/target alpine sleep 3600

# 清空旧文件
sudo docker exec "$TEMP_CONTAINER" sh -c "rm -rf /target/* /target/.[!.]* 2>/dev/null || true"

# 复制新文件
sudo docker cp client/dist/. "$TEMP_CONTAINER:/target/"

# 清理临时容器
sudo docker rm -f "$TEMP_CONTAINER"
```

### 4.6 重启Nginx服务

```bash
# 方式1: 使用docker compose
sudo docker compose restart nginx

# 方式2: 直接重启容器
sudo docker restart newsapp-nginx
```

---

## 步骤 5: 验证更新

### 5.1 检查容器状态

```bash
sudo docker ps | grep newsapp
```

### 5.2 查看Nginx日志

```bash
sudo docker logs newsapp-nginx --tail 50
```

### 5.3 检查文件是否正确更新

```bash
# 进入nginx容器检查
sudo docker exec -it newsapp-nginx sh

# 在容器内查看文件
ls -la /usr/share/nginx/html/
cat /usr/share/nginx/html/index.html | head -20

# 退出容器
exit
```

### 5.4 浏览器验证

1. 清除浏览器缓存（`Ctrl+Shift+Delete`）
2. 强制刷新页面（`Ctrl+F5`）
3. 打开开发者工具（F12），检查：
   - 没有404错误
   - 资源加载正常
   - 控制台没有错误

## 一键部署脚本（服务器端）

为了简化操作，可以在服务器上创建一个脚本：

### 创建更新脚本

```bash
# 在服务器上创建脚本
nano /path/to/project/news/deploy/update-from-upload.sh
```

**脚本内容：**

```bash
#!/bin/bash
# 从上传的dist目录更新前端

set -e

PROJECT_ROOT="/path/to/project/news"  # 修改为实际项目路径
VOLUME_NAME="newsapp_app_frontend"    # 修改为实际的volume名称

cd "$PROJECT_ROOT"

# 检查dist目录是否存在
if [ ! -d "client/dist" ]; then
    echo "❌ 错误: client/dist 目录不存在"
    echo "请先解压上传的文件到 client/dist"
    exit 1
fi

echo "=========================================="
echo "更新前端文件"
echo "=========================================="

# 查找volume路径
VOLUME_PATH=$(sudo docker volume inspect "$VOLUME_NAME" 2>/dev/null | grep -i "Mountpoint" | awk '{print $2}' | tr -d '",' || echo "")

if [ -n "$VOLUME_PATH" ] && [ -d "$VOLUME_PATH" ]; then
    echo "找到volume路径: $VOLUME_PATH"
    echo "清空旧文件..."
    sudo rm -rf "$VOLUME_PATH"/*
    echo "复制新文件..."
    sudo cp -r client/dist/* "$VOLUME_PATH/"
    echo "✓ 文件已更新"
else
    echo "使用临时容器更新..."
    TEMP_CONTAINER="temp-frontend-$(date +%s)"
    sudo docker run -d --name "$TEMP_CONTAINER" -v "$VOLUME_NAME":/target alpine sleep 3600
    sudo docker exec "$TEMP_CONTAINER" sh -c "rm -rf /target/*"
    sudo docker cp client/dist/. "$TEMP_CONTAINER:/target/"
    sudo docker rm -f "$TEMP_CONTAINER"
    echo "✓ 文件已更新"
fi

# 重启nginx
echo "重启nginx..."
sudo docker compose restart nginx || sudo docker restart newsapp-nginx

echo ""
echo "✅ 更新完成！"
echo "请清除浏览器缓存并刷新页面"
```

**给脚本添加执行权限：**

```bash
chmod +x /path/to/project/news/deploy/update-from-upload.sh
```

**使用脚本：**

```bash
# 上传并解压文件后，执行脚本
./deploy/update-from-upload.sh
```

## 完整操作示例

### 本地操作（Windows）

```powershell
# 1. 进入前端目录
cd E:\USER\SUREAL\Desktop\equity_news\news\client

# 2. 构建前端
npm run build

# 3. 打包dist目录
Compress-Archive -Path dist -DestinationPath dist.zip -Force

# 4. 使用WinSCP或FileZilla上传 dist.zip 到服务器
# 上传到: /opt/newsapp/news/
```

### 服务器操作（Ubuntu）

```bash
# 1. SSH连接到服务器
ssh root@192.168.1.100

# 2. 进入项目目录
cd /opt/newsapp/news

# 3. 解压上传的文件
unzip -o dist.zip

# 4. 执行更新脚本（如果已创建）
./deploy/update-from-upload.sh

# 或手动执行：
VOLUME_PATH=$(sudo docker volume inspect newsapp_app_frontend | grep -i "Mountpoint" | awk '{print $2}' | tr -d '",')
sudo rm -rf "$VOLUME_PATH"/*
sudo cp -r client/dist/* "$VOLUME_PATH/"
sudo docker restart newsapp-nginx

# 5. 验证
sudo docker logs newsapp-nginx --tail 20
```

## 常见问题

### Q1: 找不到volume路径

**解决方案：**
```bash
# 查看所有volumes
sudo docker volume ls

# 查看docker-compose.yml中的volume名称
grep -A 2 "app_frontend" docker-compose.yml

# 尝试不同的volume名称
sudo docker volume inspect newsapp_app_frontend
sudo docker volume inspect news_app_frontend
```

### Q2: 权限被拒绝

**解决方案：**
```bash
# 使用sudo执行命令
sudo rm -rf "$VOLUME_PATH"/*
sudo cp -r client/dist/* "$VOLUME_PATH/"

# 或添加用户到docker组
sudo usermod -aG docker $USER
# 重新登录后生效
```

### Q3: 文件上传后找不到

**解决方案：**
```bash
# 查找上传的文件
find /home -name "dist.zip"
find /opt -name "dist.zip"

# 检查当前目录
pwd
ls -la
```

### Q4: 更新后页面没有变化

**解决方案：**
1. 确认文件已正确复制：`sudo docker exec newsapp-nginx ls -la /usr/share/nginx/html/`
2. 清除浏览器缓存（强制刷新 `Ctrl+F5`）
3. 检查nginx是否已重启：`sudo docker ps | grep nginx`
4. 查看nginx日志：`sudo docker logs newsapp-nginx`

---

## 注意事项

1. **备份**：虽然只更新前端，但建议在执行前备份volume
2. **文件完整性**：确保上传的dist目录完整，包含所有assets文件
3. **权限**：确保文件权限正确（通常755）
4. **服务不中断**：只重启nginx，不会影响应用服务
5. **浏览器缓存**：更新后务必清除浏览器缓存
