# 排查应用响应慢的问题

## ✅ 已确认

- 域名解析正常：`news.gf-dsai.com` → `119.3.127.211`
- curl 请求卡住或很慢

## 🔍 排查步骤

### 步骤1：使用 timeout 限制请求时间

```bash
# 使用 timeout 限制请求时间（10秒）
timeout 10 curl -v http://119.3.127.211:3001/api/system/basic-config

# 或者使用域名
timeout 10 curl -v http://news.gf-dsai.com:3001/api/system/basic-config
```

### 步骤2：检查应用是否正常响应

```bash
# 测试本地连接（应该很快）
curl http://localhost:3001/api/system/basic-config

# 如果本地正常，说明应用本身没问题
```

### 步骤3：检查防火墙和安全组

```bash
# 检查本地防火墙
sudo ufw status

# 检查 iptables
sudo iptables -L -n | grep 3001

# 检查应用日志，看是否有请求到达
pm2 logs newsapp --lines 20
```

### 步骤4：检查网络连接

```bash
# 测试端口是否开放
telnet 119.3.127.211 3001
# 或者
nc -zv 119.3.127.211 3001

# 从服务器内部测试外部访问
curl -v http://119.3.127.211:3001/api/system/basic-config
```

## 🚀 一键排查

```bash
# 完整排查
echo "=== 1. 测试本地连接 ===" && \
curl -s http://localhost:3001/api/system/basic-config | head -3 && \
echo -e "\n=== 2. 检查应用状态 ===" && \
pm2 status && \
echo -e "\n=== 3. 检查端口监听 ===" && \
sudo netstat -tulpn | grep 3001 && \
echo -e "\n=== 4. 测试外部访问（带超时）===" && \
timeout 5 curl -s http://119.3.127.211:3001/api/system/basic-config | head -3 || echo "请求超时或失败"
```

## 🔧 可能的问题和解决方案

### 问题1：应用只监听本地

如果应用只监听 `127.0.0.1:3001`，外部无法访问。

**检查**：
```bash
sudo netstat -tulpn | grep 3001
# 如果显示 127.0.0.1:3001，需要修改为 0.0.0.0:3001
```

### 问题2：防火墙阻止

**检查**：
```bash
# 检查防火墙
sudo ufw status
sudo ufw allow 3001/tcp

# 检查云服务器安全组
# 确保安全组开放了 3001 端口
```

### 问题3：应用响应慢

**检查应用日志**：
```bash
# 实时查看日志
pm2 logs newsapp

# 然后尝试访问，观察日志输出
```

## 📋 如果本地正常但外部访问失败

可能是：
1. 防火墙阻止
2. 云服务器安全组未开放端口
3. 应用只监听本地

**解决方案**：
1. 确保应用监听 `0.0.0.0:3001`（已确认）
2. 开放防火墙端口
3. 检查云服务器安全组
