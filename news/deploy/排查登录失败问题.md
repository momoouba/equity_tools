# 排查登录失败问题

## ❌ 错误现象

- 浏览器控制台显示：`POST /api/auth/login 500 (Internal Server Error)`
- 返回：`{"success": false, "message": "服务器错误"}`
- `basic-config` 请求也失败：`{"success": false, "message": "获取配置失败"}`

## ✅ 排查步骤

### 步骤1：检查应用日志（最重要）

```bash
# 查看最新的错误日志
pm2 logs newsapp --err --lines 50

# 或者查看日志文件
tail -50 /var/log/newsapp/error.log
tail -50 /var/log/newsapp/combined.log
```

### 步骤2：检查数据库配置

```bash
# 检查 .env 文件中的数据库密码
cd /opt/newsapp/news
cat .env | grep DB_PASSWORD

# 应该显示：
# DB_PASSWORD=Mqdqxygyqy!!!klklsys24678
```

### 步骤3：测试数据库连接

```bash
# 测试数据库是否能连接
mysql -u root -p'Mqdqxygyqy!!!klklsys24678' -e "SELECT 1;"
```

### 步骤4：检查应用状态

```bash
# 检查 PM2 状态
pm2 status

# 检查应用是否正常运行
pm2 describe newsapp
```

## 🔧 修复步骤

### 如果数据库密码未更新

```bash
# 1. 更新 .env 文件
cd /opt/newsapp/news
sed -i 's|^DB_PASSWORD=.*|DB_PASSWORD=Mqdqxygyqy!!!klklsys24678|' .env

# 2. 验证更新
cat .env | grep DB_PASSWORD

# 3. 重启应用
pm2 restart newsapp

# 4. 查看启动日志
pm2 logs newsapp --lines 20
```

### 如果应用未重启

```bash
# 重启应用
pm2 restart newsapp

# 等待几秒后查看状态
sleep 3
pm2 status
pm2 logs newsapp --lines 20
```

## 📋 常见错误和解决方法

### 错误1：数据库连接被拒绝

**错误信息**：`ER_ACCESS_DENIED_ERROR` 或 `Access denied`

**解决方法**：
```bash
# 更新密码并重启
cd /opt/newsapp/news
sed -i 's|^DB_PASSWORD=.*|DB_PASSWORD=Mqdqxygyqy!!!klklsys24678|' .env
pm2 restart newsapp
```

### 错误2：数据库服务未启动

**错误信息**：`ECONNREFUSED` 或 `Can't connect to MySQL server`

**解决方法**：
```bash
# 启动 MySQL
sudo systemctl start mysql
sudo systemctl status mysql
```

### 错误3：应用未运行

**解决方法**：
```bash
# 启动应用
cd /opt/newsapp/news
pm2 start ecosystem.config.js
pm2 save
```
