# 快速查看日志指南

## 🚀 最简单的方法（推荐）

### 直接使用 PM2 命令

```bash
# 实时查看所有日志（最简单）
pm2 logs newsapp

# 只查看错误日志
pm2 logs newsapp --err

# 只查看标准输出
pm2 logs newsapp --out

# 查看最后100行（不跟踪）
pm2 logs newsapp --lines 100 --nostream
```

**优点**：
- ✅ 不需要切换目录
- ✅ 不需要脚本
- ✅ 可以在任何目录使用
- ✅ 最简单直接

## 💡 创建快捷命令（可选）

### 方法1：创建别名

编辑 `~/.bashrc`：

```bash
nano ~/.bashrc
```

添加以下内容：

```bash
# 日志查看别名
alias pm2logs='pm2 logs newsapp'
alias viewlogs='cd /opt/newsapp/news && ./deploy/view-logs.sh'
```

重新加载配置：

```bash
source ~/.bashrc
```

然后可以使用：

```bash
pm2logs          # 直接查看PM2日志
viewlogs --pm2   # 使用脚本查看
```

### 方法2：创建全局命令

```bash
# 创建符号链接
sudo ln -s /opt/newsapp/news/deploy/view-logs.sh /usr/local/bin/view-logs

# 然后可以在任何目录使用
view-logs --pm2
```

## 🔧 故障排查

### 问题1：找不到脚本文件

**解决方法**：

```bash
# 检查文件是否存在
ls -la /opt/newsapp/news/deploy/view-logs.sh

# 如果不存在，检查应用目录
ls -la /opt/newsapp/news/deploy/

# 如果deploy目录不存在，可能需要同步文件
```

### 问题2：权限不足

**解决方法**：

```bash
# 添加执行权限
chmod +x /opt/newsapp/news/deploy/view-logs.sh

# 检查权限
ls -l /opt/newsapp/news/deploy/view-logs.sh
# 应该显示：-rwxr-xr-x
```

### 问题3：PM2未安装或应用未运行

**解决方法**：

```bash
# 检查PM2是否安装
pm2 --version

# 检查应用是否在PM2中运行
pm2 list

# 如果应用未运行，启动应用
cd /opt/newsapp/news
pm2 start deploy/ecosystem.config.js
```
