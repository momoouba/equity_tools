# 排查登录 500 错误

## ❌ 问题

- 浏览器显示：`POST /api/auth/login 500 (Internal Server Error)`
- 浏览器显示：`GET /api/system/basic-config 500 (Internal Server Error)`
- 应用已启动，但 API 请求失败

## ✅ 排查步骤

### 步骤1：查看服务器端错误日志

```bash
# 查看最新的错误日志
pm2 logs newsapp --err --lines 100

# 或者查看错误日志文件
tail -100 /var/log/newsapp/error.log

# 搜索包含 "login"、"auth"、"500" 的错误
grep -i "login\|auth\|error\|500\|exception" /var/log/newsapp/error.log | tail -30
```

### 步骤2：测试数据库连接

```bash
# 测试 newsapp 用户连接
mysql -u newsapp -p'NewsApp@123456' -e "USE investment_tools; SELECT 1;"

# 检查 users 表
mysql -u newsapp -p'NewsApp@123456' -e "USE investment_tools; SELECT account, role FROM users LIMIT 5;"
```

### 步骤3：检查 API 路由

```bash
# 查看应用是否正常监听
netstat -tulpn | grep 3001

# 测试本地 API
curl http://localhost:3001/api/system/basic-config
curl -X POST http://localhost:3001/api/auth/login -H "Content-Type: application/json" -d '{"account":"admin","password":"wenchao"}'
```

### 步骤4：查看实时日志

```bash
# 实时查看日志（在另一个终端）
pm2 logs newsapp

# 然后在浏览器中尝试登录，观察日志输出
```

## 🔍 常见问题排查

### 问题1：数据库连接失败

**检查**：
```bash
# 检查 .env 配置
cd /opt/newsapp/news
cat .env | grep DB_

# 测试数据库连接
mysql -u newsapp -p'NewsApp@123456' -e "USE investment_tools; SELECT 1;"
```

### 问题2：users 表不存在或为空

**检查**：
```bash
mysql -u newsapp -p'NewsApp@123456' -e "USE investment_tools; SHOW TABLES LIKE 'users';"
mysql -u newsapp -p'NewsApp@123456' -e "USE investment_tools; SELECT COUNT(*) FROM users;"
```

### 问题3：应用路由问题

**检查**：
```bash
# 查看应用代码中的路由
grep -n "app.use\|router\|/api/auth" /opt/newsapp/news/server/index.js | head -20
```
