# 文件更新后部署操作指南

## 📋 概述

本次更新涉及的文件：
- **前端文件**：`client/src/pages/NewsInfo.css`, `client/src/pages/NewsInfo.jsx`
- **后端文件**：`server/routes/news.js`, `server/utils/newsAnalysis.js`, `server/routes/enterprises.js`
- **配置文件**：`deploy/nginx-site.conf`

---

## 🐳 方式一：Docker 部署（推荐）

如果你的服务器使用 Docker Compose 部署，请按以下步骤操作：

### 步骤1：确认文件已复制到服务器

```bash
# 登录服务器，进入项目目录
cd /opt/newsapp/news  # 或你的实际项目路径

# 确认文件已更新
ls -la client/src/pages/NewsInfo.jsx
ls -la server/routes/news.js
ls -la server/utils/newsAnalysis.js
ls -la server/routes/enterprises.js
ls -la deploy/nginx-site.conf
```

### 步骤2：重新构建前端（必须）

```bash
# 进入项目根目录
cd /opt/newsapp/news

# 进入前端目录
cd client

# 安装依赖（如果package.json有变化）
npm install

# 重新构建前端
npm run build

# 返回项目根目录
cd ..
```

### 步骤3：更新前端文件到 Docker Volume

有两种方法：

#### 方法A：使用自动脚本（推荐）

```bash
# 使用清除缓存并更新脚本
chmod +x deploy/clear-cache-and-update.sh
./deploy/clear-cache-and-update.sh
```

#### 方法B：手动更新

```bash
# 查找前端volume路径
VOLUME_PATH=$(sudo docker volume inspect news_app_frontend | grep -i "Mountpoint" | awk '{print $2}' | tr -d '",')

# 复制前端文件到volume
sudo cp -r client/dist/* "$VOLUME_PATH/"

# 或者使用临时容器方式
sudo docker run --rm -v news_app_frontend:/target -v $(pwd)/client/dist:/source alpine sh -c "cp -r /source/* /target/"
```

### 步骤4：重启服务

```bash
# 重启应用容器（后端代码会自动同步，因为server目录是volume挂载）
sudo docker compose restart app

# 如果nginx配置有变化，需要重启nginx
sudo docker compose restart nginx

# 或者重启所有服务
sudo docker compose restart
```

### 步骤5：验证部署

```bash
# 检查容器状态
sudo docker compose ps

# 查看应用日志
sudo docker compose logs app --tail 50

# 查看nginx日志
sudo docker compose logs nginx --tail 50

# 测试API接口
curl http://localhost:3001/api/health
```

## ⚙️ 方式二：PM2 部署

如果你的服务器使用 PM2 直接运行 Node.js，请按以下步骤操作：

### 步骤1：确认文件已复制到服务器

```bash
# 切换到应用用户（如果使用非root用户）
su - guofang  # 或你的应用用户名

# 进入项目目录
cd /opt/newsapp/news

# 确认文件已更新
ls -la client/src/pages/NewsInfo.jsx
ls -la server/routes/news.js
ls -la server/utils/newsAnalysis.js
ls -la server/routes/enterprises.js
```

### 步骤2：重新构建前端（必须）

```bash
# 进入前端目录
cd client

# 安装依赖（如果package.json有变化）
npm install

# 重新构建前端
npm run build

# 返回项目根目录
cd ..
```

### 步骤3：检查后端依赖（如果有新依赖）

```bash
# 在项目根目录
npm install
```

### 步骤4：重启应用

```bash
# 重启PM2应用
pm2 restart newsapp

# 或重启所有PM2进程
pm2 restart all

# 查看应用状态
pm2 status

# 查看日志确认启动成功
pm2 logs newsapp --lines 50
```

### 步骤5：更新Nginx配置（如果nginx-site.conf有变化）

```bash
# 如果nginx配置有变化，需要更新并重启nginx
sudo cp deploy/nginx-site.conf /etc/nginx/sites-available/newsapp

# 测试nginx配置
sudo nginx -t

# 重启nginx
sudo systemctl restart nginx

# 查看nginx状态
sudo systemctl status nginx
```

### 步骤6：验证部署

```bash
# 检查PM2状态
pm2 status

# 检查端口是否监听
netstat -tulpn | grep 3001

# 查看应用日志
pm2 logs newsapp --lines 100

# 测试API接口
curl http://localhost:3001/api/health
```

## ⚠️ 重要注意事项

### 1. 前端缓存问题

更新前端后，用户浏览器可能缓存了旧文件，需要：

- **在浏览器中清除缓存**：按 `Ctrl+Shift+Delete` 清除缓存
- **强制刷新**：按 `Ctrl+F5` 强制刷新页面
- **或在开发者工具中**：右键刷新按钮，选择"清空缓存并硬性重新加载"

### 2. 检查文件修改时间

确认文件确实已更新：

```bash
# 查看文件修改时间
stat server/routes/news.js
stat client/src/pages/NewsInfo.jsx
```

### 3. 查看错误日志

如果部署后出现问题：

**Docker部署：**
```bash
sudo docker compose logs app --tail 100
sudo docker compose logs nginx --tail 100
```

**PM2部署：**
```bash
pm2 logs newsapp --err --lines 100
pm2 logs newsapp --out --lines 100
```

### 4. 回滚操作（如果需要）

**Docker部署：**
```bash
# 停止当前容器
sudo docker compose down

# 恢复之前的代码（如果有备份）
# 然后重新启动
sudo docker compose up -d
```

**PM2部署：**
```bash
# 恢复之前的代码（如果有备份）
# 然后重启
pm2 restart newsapp
```

## 🆘 故障排查

### 问题1：前端更新后看不到变化

**解决方法：**
1. 确认前端已重新构建：检查 `client/dist` 目录的修改时间
2. 清除浏览器缓存
3. 检查nginx是否重启（Docker部署）
4. 查看浏览器开发者工具的Network标签，确认加载的是新文件

### 问题2：应用无法启动

**解决方法：**
1. 查看错误日志（见上方"查看错误日志"部分）
2. 检查代码语法：`node -c server/routes/news.js`
3. 检查依赖是否安装：`npm install`
4. 检查环境变量配置：`cat .env`

### 问题3：端口被占用

**解决方法：**
```bash
# 查找占用端口的进程
sudo lsof -i :3001
# 或
sudo netstat -tulpn | grep 3001

# 停止占用端口的进程
sudo kill <进程ID>
```
