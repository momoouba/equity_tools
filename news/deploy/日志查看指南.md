## Linux服务器直接查看

### 方法1：使用提供的脚本（推荐）

项目提供了便捷的日志查看脚本 `deploy/view-logs.sh`：

```bash
# 1. 给脚本添加执行权限
chmod +x deploy/view-logs.sh

# 2. 实时查看合并日志（默认）
./deploy/view-logs.sh

# 3. 使用PM2查看日志
./deploy/view-logs.sh --pm2

# 4. 查看错误日志
./deploy/view-logs.sh --error

# 5. 查看最后200行日志（不跟踪）
./deploy/view-logs.sh --lines 200 --no-follow

# 6. 过滤包含ERROR的日志
./deploy/view-logs.sh --grep ERROR

# 7. 查看所有日志文件
./deploy/view-logs.sh --all
```

### 方法2：直接使用Linux命令

#### 使用PM2查看日志

```bash
# 实时查看所有日志
pm2 logs newsapp

# 只查看标准输出
pm2 logs newsapp --out

# 只查看错误日志
pm2 logs newsapp --err

# 查看最后100行（不跟踪）
pm2 logs newsapp --lines 100 --nostream

# 清空日志
pm2 flush newsapp
```

#### 使用tail命令查看日志文件

日志文件位置：`/var/log/newsapp/`

```bash
# 实时查看合并日志
tail -f /var/log/newsapp/combined.log

# 实时查看标准输出日志
tail -f /var/log/newsapp/out.log

# 实时查看错误日志
tail -f /var/log/newsapp/error.log

# 查看最后100行（不跟踪）
tail -n 100 /var/log/newsapp/combined.log

# 查看最后100行并实时跟踪
tail -n 100 -f /var/log/newsapp/combined.log

# 过滤包含ERROR的日志
tail -f /var/log/newsapp/combined.log | grep ERROR

# 过滤包含ERROR的日志（高亮显示）
tail -f /var/log/newsapp/combined.log | grep --color=always ERROR

# 同时查看多个日志文件
tail -f /var/log/newsapp/combined.log /var/log/newsapp/error.log
```

#### 使用less查看日志（支持搜索）

```bash
# 使用less查看日志（支持搜索和翻页）
less /var/log/newsapp/combined.log

# 在less中：
# - 按 / 然后输入关键词搜索
# - 按 n 查找下一个
# - 按 N 查找上一个
# - 按 F 实时跟踪（类似tail -f）
# - 按 q 退出
```

## Windows通过SSH查看

### 方法1：使用PowerShell脚本（推荐）

项目提供了PowerShell脚本 `deploy/view-logs.ps1`：

```powershell
# 1. 打开PowerShell，切换到项目目录
cd E:\USER\SUREAL\Desktop\news\deploy

# 2. 查看日志（需要提供服务器地址和用户名）
.\view-logs.ps1 -Server 192.168.1.100 -User root

# 3. 查看错误日志
.\view-logs.ps1 -Server 192.168.1.100 -User root -LogType error

# 4. 查看最后200行（不跟踪）
.\view-logs.ps1 -Server 192.168.1.100 -User root -Lines 200 -NoFollow

# 5. 过滤包含ERROR的日志
.\view-logs.ps1 -Server 192.168.1.100 -User root -Grep ERROR
```

### 方法2：使用SSH直接连接

#### 使用Windows内置OpenSSH

```powershell
# 1. 连接到服务器
ssh root@192.168.1.100

# 2. 连接后使用Linux命令查看日志
tail -f /var/log/newsapp/combined.log

# 或使用PM2
pm2 logs newsapp
```

#### 使用PuTTY（图形界面）

1. 下载并安装PuTTY：https://www.putty.org/
2. 打开PuTTY，输入服务器地址和端口
3. 连接后使用Linux命令查看日志

## 使用PM2查看日志

如果应用使用PM2管理，推荐使用PM2的日志功能：

### PM2日志位置

PM2日志默认存储在：`~/.pm2/logs/`

- `newsapp-out.log` - 标准输出日志
- `newsapp-error.log` - 错误日志

### PM2日志命令

```bash
# 实时查看所有日志
pm2 logs newsapp

# 只查看标准输出
pm2 logs newsapp --out

# 只查看错误日志
pm2 logs newsapp --err

# 查看最后100行（不跟踪）
pm2 logs newsapp --lines 100 --nostream

# 查看最后100行并实时跟踪
pm2 logs newsapp --lines 100

# 清空所有日志
pm2 flush newsapp

# 清空指定应用的日志
pm2 flush newsapp

# 查看日志文件大小
pm2 list
```

### PM2日志配置

日志配置在 `deploy/ecosystem.config.js` 中：

```javascript
log_file: '/var/log/newsapp/combined.log',
out_file: '/var/log/newsapp/out.log',
error_file: '/var/log/newsapp/error.log',
log_date_format: 'YYYY-MM-DD HH:mm:ss Z',
```

## 常用命令速查

### 快速查看日志

```bash
# 最简单的方式：实时查看合并日志
tail -f /var/log/newsapp/combined.log

# 或使用PM2
pm2 logs newsapp
```

### 查看最近的错误

```bash
# 查看最后50行错误日志
tail -n 50 /var/log/newsapp/error.log

# 实时查看错误日志
tail -f /var/log/newsapp/error.log

# 搜索包含ERROR的日志
grep ERROR /var/log/newsapp/combined.log | tail -n 50
```

### 查看特定时间段的日志

```bash
# 查看今天的日志
grep "$(date +%Y-%m-%d)" /var/log/newsapp/combined.log

# 查看最近1小时的日志
grep "$(date -d '1 hour ago' +%Y-%m-%d\ %H)" /var/log/newsapp/combined.log
```

### 日志文件管理

```bash
# 查看日志文件大小
du -h /var/log/newsapp/*.log

# 查看日志文件数量
ls -lh /var/log/newsapp/

# 压缩旧日志（保留最近7天）
find /var/log/newsapp/ -name "*.log" -mtime +7 -exec gzip {} \;

# 删除30天前的日志
find /var/log/newsapp/ -name "*.log" -mtime +30 -delete
```

## 没有 PM2 时如何查看日志

### 情况1：npm/Node.js 未安装

**问题**：运行脚本时提示 "PM2未安装"，安装 PM2 时提示 "npm 未找到"

**解决方法**：

```bash
# 方法1：使用一键安装脚本（推荐）
chmod +x deploy/install-node-pm2.sh
./deploy/install-node-pm2.sh

# 方法2：手动安装
# 1. 安装 Node.js 和 npm
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 2. 验证安装
node --version
npm --version

# 3. 安装 PM2
npm install -g pm2

# 4. 验证 PM2
pm2 --version
```

**详细指南**：查看 `deploy/安装Node和PM2指南.md`

### 情况2：应用直接运行（不使用 PM2）

如果应用是直接运行 `node server/index.js`，日志会输出到运行该命令的终端。

**查看方法**：

1. **找到运行应用的终端窗口**（如果有 SSH 会话）
2. **查看进程信息**：
```bash
# 查看 node 进程
ps aux | grep 'node.*index.js'

# 查看进程的详细信息
ps aux | grep node
```

3. **使用 screen 或 tmux**（如果应用在这些工具中运行）：
```bash
# 列出所有 screen 会话
screen -ls

# 连接到会话
screen -r <session_id>

# 列出所有 tmux 会话
tmux ls

# 连接到会话
tmux attach -t <session_id>
```

4. **使用 systemd 服务**（如果配置了）：
```bash
# 查看服务日志
sudo journalctl -u newsapp -f

# 查看最近100行
sudo journalctl -u newsapp -n 100
```

## 故障排查

### 问题1：日志文件不存在

**原因**：日志目录未创建或应用未启动

**解决方法**：
```bash
# 创建日志目录
sudo mkdir -p /var/log/newsapp
sudo chown $USER:$USER /var/log/newsapp

# 检查应用是否运行
pm2 list
# 或
ps aux | grep node
```

### 问题2：npm 或 Node.js 未安装

**症状**：运行 `npm install -g pm2` 时提示 "Command 'npm' not found"

**解决方法**：

```bash
# 1. 安装 Node.js 18.x（包含 npm）
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 2. 验证安装
node --version
npm --version

# 3. 安装 PM2
npm install -g pm2

# 4. 验证 PM2
pm2 --version
```

**或使用一键安装脚本**：
```bash
chmod +x deploy/install-node-pm2.sh
./deploy/install-node-pm2.sh
```

### 问题3：没有权限查看日志

**解决方法**：
```bash
# 使用sudo查看
sudo tail -f /var/log/newsapp/combined.log

# 或修改文件权限
sudo chmod 644 /var/log/newsapp/*.log
```

### 问题4：PM2日志为空

**原因**：应用可能未通过PM2启动

**解决方法**：
```bash
# 检查PM2进程
pm2 list

# 如果应用未在PM2中，使用PM2启动
pm2 start ecosystem.config.js
```

### 问题5：Windows SSH连接失败

**解决方法**：
1. 确保Windows已安装OpenSSH客户端
2. 检查服务器SSH服务是否运行
3. 检查防火墙设置
4. 使用PuTTY等第三方SSH工具

## 最佳实践

1. **定期清理日志**：避免日志文件过大占用磁盘空间
2. **使用日志轮转**：配置logrotate自动管理日志文件
3. **监控错误日志**：设置告警监控错误日志数量
4. **日志分级**：区分info、warn、error等级别
5. **集中管理**：考虑使用ELK、Graylog等日志管理系统
