# 修复登录错误和重启应用

## ❌ 问题

1. 登录页面显示"服务器错误"
2. 代码已更新，但日志中仍有外键约束警告
3. PM2 显示有多个 newsapp 进程在运行

## ✅ 解决步骤

### 步骤1：停止所有 PM2 进程

```bash
# 停止所有 newsapp 进程
pm2 stop newsapp

# 或者删除所有进程
pm2 delete newsapp
```

### 步骤2：确认代码已更新

```bash
# 检查代码是否已更新（确认迁移逻辑已注释）
cd /opt/newsapp/news
grep -n "已禁用.*迁移逻辑" server/db.js

# 应该看到注释说明
```

### 步骤3：重新启动应用

```bash
# 使用 ecosystem.config.js 启动（单实例）
cd /opt/newsapp/news
pm2 start ecosystem.config.js

# 或者直接启动（单实例）
pm2 start server/index.js --name newsapp -i 1

# 保存 PM2 配置
pm2 save
```

### 步骤4：检查应用状态

```bash
# 检查进程状态
pm2 status

# 应该只看到一个 newsapp 进程

# 查看启动日志
pm2 logs newsapp --lines 30
```

### 步骤5：检查登录相关错误

```bash
# 查看最新的错误日志
pm2 logs newsapp --err --lines 50

# 或者查看日志文件
tail -50 /var/log/newsapp/error.log | grep -i "error\|denied\|fail"
```

## 🚀 一键修复

```bash
# 停止所有进程
pm2 stop newsapp
pm2 delete newsapp

# 确认代码已更新
cd /opt/newsapp/news
echo "=== 检查代码更新 ==="
grep -c "已禁用.*迁移逻辑" server/db.js && echo "✓ 代码已更新" || echo "✗ 代码未更新，需要重新上传"

# 重新启动应用（单实例）
pm2 start server/index.js --name newsapp -i 1
pm2 save

# 等待启动
sleep 5

# 检查状态
echo -e "\n=== PM2 状态 ==="
pm2 status

echo -e "\n=== 最新日志 ==="
pm2 logs newsapp --lines 20 --nostream
```

## 🔍 检查登录错误的具体原因

```bash
# 查看包含 "login" 或 "auth" 的错误
grep -i "login\|auth\|denied\|500" /var/log/newsapp/error.log | tail -20

# 查看 API 错误
grep -i "api\|system\|basic-config" /var/log/newsapp/error.log | tail -20
```

## 📋 如果登录仍然失败

### 检查数据库连接

```bash
# 测试数据库连接
mysql -u newsapp -p'NewsApp@123456' -e "USE investment_tools; SELECT 1;"

# 如果失败，检查用户
sudo mysql -u root -p'Mqdqxygyqy!!!klklsys24678' -e "SELECT user, host FROM mysql.user WHERE user='newsapp';"
```

### 检查应用配置

```bash
# 检查 .env 配置
cd /opt/newsapp/news
cat .env | grep DB_
```

### 查看实时日志

```bash
# 实时查看日志（在另一个终端）
pm2 logs newsapp

# 然后在浏览器中尝试登录，观察日志输出
```

## ✅ 验证修复

修复后，检查：

1. **PM2 状态**：应该只有一个 newsapp 进程
2. **日志**：应该没有外键约束警告
3. **登录**：刷新浏览器，尝试登录
