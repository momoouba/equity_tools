# 舆情信息公共链接分享功能 - Ubuntu Docker 部署指南

## 📋 部署概述

本指南说明如何将舆情信息公共链接分享功能部署到Ubuntu系统的Docker环境中。

## 🎯 部署内容

本次部署包含以下更新：

### 后端更新
- ✅ `server/routes/newsShare.js` - 分享链接API路由
- ✅ `server/index.js` - SPA路由支持（支持 `/share/:token` 路径）
- ✅ `server/db.js` - 数据库表创建（news_share_links表）

### 前端更新
- ✅ `client/src/pages/ShareNewsPage.jsx` - 分享链接公共访问页面
- ✅ `client/src/pages/ShareNewsPage.css` - 分享页面样式
- ✅ `client/src/pages/NewsInfo.jsx` - 添加"发布"按钮和分享对话框
- ✅ `client/src/App.jsx` - 添加 `/share/:token` 路由
- ✅ `client/src/utils/axios.js` - 添加baseURL自动检测

## 🚀 部署步骤

### 方法一：使用自动化部署脚本（推荐）

#### 1. 上传代码到服务器

将更新后的代码文件上传到Ubuntu服务器：

```bash
# 在本地Windows PowerShell中执行
# 假设服务器IP为 your-server-ip，用户名为 root

# 上传后端文件
scp E:\USER\SUREAL\Desktop\equity_news\news\server\routes\newsShare.js root@your-server-ip:/opt/newsapp/news/server/routes/
scp E:\USER\SUREAL\Desktop\equity_news\news\server\index.js root@your-server-ip:/opt/newsapp/news/server/

# 上传前端文件
scp E:\USER\SUREAL\Desktop\equity_news\news\client\src\pages\ShareNewsPage.jsx root@your-server-ip:/opt/newsapp/news/client/src/pages/
scp E:\USER\SUREAL\Desktop\equity_news\news\client\src\pages\ShareNewsPage.css root@your-server-ip:/opt/newsapp/news/client/src/pages/
scp E:\USER\SUREAL\Desktop\equity_news\news\client\src\pages\NewsInfo.jsx root@your-server-ip:/opt/newsapp/news/client/src/pages/
scp E:\USER\SUREAL\Desktop\equity_news\news\client\src\App.jsx root@your-server-ip:/opt/newsapp/news/client/src/
scp E:\USER\SUREAL\Desktop\equity_news\news\client\src\utils\axios.js root@your-server-ip:/opt/newsapp/news/client/src/utils/

# 上传部署脚本
scp E:\USER\SUREAL\Desktop\equity_news\news\deploy\deploy-share-feature.sh root@your-server-ip:/opt/newsapp/news/deploy/
```

#### 2. 在服务器上执行部署脚本

```bash
# SSH登录到服务器
ssh root@your-server-ip

# 进入项目目录
cd /opt/newsapp/news

# 给脚本添加执行权限
chmod +x deploy/deploy-share-feature.sh

# 执行部署脚本
./deploy/deploy-share-feature.sh
```

脚本会自动完成以下操作：
1. ✅ 检查必要文件是否存在
2. ✅ 备份当前代码
3. ✅ 构建前端（npm run build）
4. ✅ 更新Docker volume中的前端文件
5. ✅ 重启应用容器（加载新的后端代码）
6. ✅ 重启Nginx容器
7. ✅ 验证部署结果

### 方法二：手动部署

如果自动化脚本无法使用，可以手动执行以下步骤：

#### 1. 更新代码文件

确保所有更新的文件都已上传到服务器。

#### 2. 构建前端

```bash
cd /opt/newsapp/news/client

# 安装依赖（如果需要）
npm install

# 构建前端
npm run build
```

#### 3. 更新Docker volume中的前端文件

```bash
# 查找volume名称
docker volume ls | grep frontend

# 使用临时容器更新文件
TEMP_CONTAINER="temp-frontend-update-$(date +%s)"
docker run -d --name "$TEMP_CONTAINER" -v newsapp_app_frontend:/target alpine sleep 3600

# 清空旧文件
docker exec "$TEMP_CONTAINER" sh -c "rm -rf /target/* /target/.[!.]* /target/..?* 2>/dev/null || true"

# 复制新文件
docker cp /opt/newsapp/news/client/dist/. "$TEMP_CONTAINER:/target/"

# 清理临时容器
docker rm -f "$TEMP_CONTAINER"
```

#### 4. 重启服务

```bash
cd /opt/newsapp/news

# 重启应用容器（加载新的后端代码）
docker compose restart app

# 重启Nginx容器
docker compose restart nginx
```

#### 5. 验证部署

```bash
# 检查容器状态
docker compose ps

# 检查应用健康状态
curl http://localhost:3001/api/health

# 查看应用日志
docker compose logs app --tail 100
```

## 🐛 故障排除

### 问题1：部署脚本执行失败

**解决方案：**
```bash
# 检查脚本权限
chmod +x deploy/deploy-share-feature.sh

# 检查文件是否存在
ls -la deploy/deploy-share-feature.sh

# 手动执行脚本中的步骤
```

### 问题2：前端文件未更新

**解决方案：**
```bash
# 检查volume中的文件
docker run --rm -v newsapp_app_frontend:/data alpine ls -la /data

# 重新执行前端更新步骤
```

### 问题3：后端代码未生效

**解决方案：**
```bash
# 检查容器中的文件
docker exec newsapp ls -la /app/server/routes/newsShare.js

# 强制重启容器
docker compose restart app

# 查看容器日志
docker compose logs app --tail 100
```

### 问题4：分享链接页面空白

**解决方案：**
1. 检查浏览器控制台是否有错误
2. 检查网络请求是否成功
3. 检查Nginx配置是否正确
4. 查看应用日志：`docker compose logs app --tail 100`

### 问题5：数据库表不存在

**解决方案：**
```bash
# 检查数据库初始化脚本
ls -la deploy/mysql-init/

# 手动创建表（如果需要）
docker exec -it newsapp-mysql mysql -uroot -p investment_tools < server/db.js
```
