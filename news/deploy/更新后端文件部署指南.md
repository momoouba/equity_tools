# 更新后端文件部署指南

## 📋 本次更新的文件

- `server/utils/newsAnalysis.js` - 新闻分析工具类（强制生成摘要和关键词逻辑）
- `server/routes/newsAnalysis.js` - 新闻分析路由（新增重新分析API端点）

## 🐳 Docker 部署操作步骤

### 方式一：快速重启（推荐）

由于 `server` 目录是通过 volume 挂载的（`./server:/app/server`），文件更新后只需重启应用容器即可：

```bash
# 1. 进入项目目录
cd /opt/newsapp/news

# 2. 确认文件已更新
ls -la server/utils/newsAnalysis.js
ls -la server/routes/newsAnalysis.js

# 3. 重启应用容器（后端代码会自动同步）
sudo docker compose restart app

# 4. 查看应用日志，确认启动成功
sudo docker compose logs app --tail 50 -f
```

### 方式二：使用重启脚本

```bash
# 进入项目目录
cd /opt/newsapp/news

# 使用重启脚本（会自动检查健康状态）
chmod +x deploy/restart-app.sh
./deploy/restart-app.sh
```

### 方式三：完整重启流程

```bash
# 1. 进入项目目录
cd /opt/newsapp/news

# 2. 停止应用容器
sudo docker compose stop app

# 3. 等待2秒
sleep 2

# 4. 启动应用容器
sudo docker compose up -d app

# 5. 等待应用启动（约30秒）
echo "等待应用启动..."
sleep 30

# 6. 检查容器状态
sudo docker compose ps app

# 7. 查看应用日志
sudo docker compose logs app --tail 50
```

## 🔍 故障排查

### 问题1：容器无法启动

**解决方法：**
```bash
# 查看详细错误日志
sudo docker compose logs app --tail 100

# 检查代码语法（在容器外）
node -c server/utils/newsAnalysis.js
node -c server/routes/newsAnalysis.js

# 如果语法错误，修复后重新部署
```

### 问题2：应用启动但返回503错误

**解决方法：**
```bash
# 等待更长时间（应用可能需要初始化数据库连接等）
sleep 60

# 再次检查健康状态
curl http://localhost:3001/api/health

# 查看应用日志
sudo docker compose logs app --tail 100
```

### 问题3：文件更新后没有生效

**解决方法：**
```bash
# 1. 确认文件确实已更新
stat server/utils/newsAnalysis.js
stat server/routes/newsAnalysis.js

# 2. 确认文件在容器中已更新
sudo docker exec newsapp ls -la /app/server/utils/newsAnalysis.js
sudo docker exec newsapp ls -la /app/server/routes/newsAnalysis.js

# 3. 如果容器中的文件没有更新，检查volume挂载
sudo docker inspect newsapp | grep -A 10 "Mounts"

# 4. 强制重启容器
sudo docker compose restart app
```

## ⚠️ 重要提示

1. **无需重新构建镜像**：由于 `server` 目录是 volume 挂载，文件更新后直接重启容器即可，不需要重新构建 Docker 镜像。

2. **无需更新前端**：本次更新只涉及后端文件，不需要重新构建前端。

3. **应用启动时间**：应用容器启动后可能需要30-60秒才能完全就绪（包括数据库连接、定时任务初始化等），请耐心等待。

4. **查看日志**：如果遇到问题，首先查看应用日志：`sudo docker compose logs app --tail 100`
