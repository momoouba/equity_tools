# 代码更新部署指南

## 📋 服务器端操作步骤

### 步骤1：同步代码到服务器

#### 方法1：使用 Git（推荐）

如果服务器使用 Git 管理代码：

```bash
# 1. 切换到应用目录
cd /opt/newsapp/news

# 2. 切换到应用用户（如果使用非root用户）
su - guofang
cd /opt/newsapp/news

# 3. 拉取最新代码
git pull origin main
# 或
git pull origin master
```

#### 方法2：手动上传文件

如果使用 FTP/SFTP 或其他方式：

```bash
# 1. 上传修改后的文件到服务器
# server/routes/news.js

# 2. 确认文件已上传到正确位置
ls -la /opt/newsapp/news/server/routes/news.js
```

### 步骤2：重启应用

#### 方法1：使用 PM2（推荐）

```bash
# 1. 切换到应用用户（如果使用非root用户）
su - guofang

# 2. 进入应用目录
cd /opt/newsapp/news

# 3. 重启应用
pm2 restart newsapp

# 或重启所有PM2进程
pm2 restart all

# 4. 查看应用状态
pm2 status

# 5. 查看日志确认启动成功
pm2 logs newsapp --lines 50
```

#### 方法2：直接运行 Node.js

如果应用是直接运行的：

```bash
# 1. 停止当前运行的应用（找到进程并停止）
ps aux | grep 'node.*index.js'
kill <进程ID>

# 2. 重新启动应用
cd /opt/newsapp/news
node server/index.js
```

#### 方法3：使用 systemd 服务

如果使用 systemd 管理：

```bash
# 重启服务
sudo systemctl restart newsapp

# 查看服务状态
sudo systemctl status newsapp

# 查看日志
sudo journalctl -u newsapp -f
```

### 步骤3：验证更新

#### 检查应用是否正常运行

```bash
# 1. 检查PM2状态
pm2 status

# 2. 检查端口是否监听
netstat -tulpn | grep 3001

# 3. 检查应用日志
pm2 logs newsapp --lines 100

# 4. 测试API接口
curl http://localhost:3001/api/health
```

#### 验证新功能

```bash
# 查看日志，确认拖底逻辑是否生效
pm2 logs newsapp | grep "拖底逻辑"

# 或查看完整日志
pm2 logs newsapp
```

## 📝 详细操作流程

### 完整流程示例

```bash
# 1. 登录服务器
ssh guofang@服务器IP

# 2. 切换到应用目录
cd /opt/newsapp/news

# 3. 备份当前代码（可选，但推荐）
cp -r server server.backup.$(date +%Y%m%d_%H%M%S)

# 4. 同步代码（根据你的部署方式选择）
# 方式A：Git
git pull origin main

# 方式B：手动上传后，确认文件存在
ls -la server/routes/news.js

# 5. 检查文件修改时间确认已更新
stat server/routes/news.js

# 6. 重启应用
pm2 restart newsapp

# 7. 等待几秒后检查状态
sleep 3
pm2 status

# 8. 查看日志确认启动成功
pm2 logs newsapp --lines 50

# 9. 测试API（可选）
curl http://localhost:3001/api/health
```

## ⚠️ 注意事项

### 1. 备份重要数据

在更新前建议备份：

```bash
# 备份代码
cp -r /opt/newsapp/news/server /opt/newsapp/news/server.backup.$(date +%Y%m%d_%H%M%S)

# 备份数据库（如果担心数据问题）
mysqldump -u newsapp -p investment_tools > /tmp/newsapp_backup_$(date +%Y%m%d_%H%M%S).sql
```

### 2. 检查依赖

如果代码添加了新的依赖：

```bash
cd /opt/newsapp/news
npm install
```

### 3. 检查环境变量

确保 `.env` 文件配置正确：

```bash
# 检查环境变量文件
cat /opt/newsapp/news/.env
```

### 4. 查看错误日志

如果重启失败，查看错误日志：

```bash
# PM2错误日志
pm2 logs newsapp --err

# 系统日志
sudo journalctl -u newsapp -n 100

# 应用日志文件
tail -f /var/log/newsapp/error.log
```

## 🐛 故障排查

### 问题1：PM2重启失败

**解决方法**：

```bash
# 查看详细错误
pm2 logs newsapp --err --lines 100

# 检查代码语法
cd /opt/newsapp/news
node -c server/routes/news.js

# 手动测试启动
node server/index.js
```

### 问题2：应用无法启动

**解决方法**：

```bash
# 检查端口占用
netstat -tulpn | grep 3001

# 检查数据库连接
mysql -u newsapp -p investment_tools

# 检查环境变量
cat .env

# 查看详细错误
pm2 logs newsapp --err
```

### 问题3：代码未生效

**解决方法**：

```bash
# 确认文件已更新
stat server/routes/news.js

# 确认PM2已重启
pm2 list

# 清除PM2缓存（如果需要）
pm2 flush newsapp
pm2 restart newsapp
```
