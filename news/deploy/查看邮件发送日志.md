# 查看邮件发送日志的方法

## Docker 环境查看日志

### 方法1：查看最近的邮件发送日志（推荐）

```bash
# 进入项目目录
cd /opt/newsapp/news

# 查看最近的邮件发送相关日志
sudo docker compose logs app | grep -E '邮件发送|管理员' | tail -50
```

### 方法2：实时查看日志（发送邮件时使用）

```bash
# 实时查看日志，发送邮件时可以看到详细输出
sudo docker compose logs -f app | grep '邮件发送'
```

### 方法3：查看最近10分钟的日志

```bash
sudo docker compose logs --since 10m app | grep -E '邮件发送|管理员'
```

### 方法4：查看所有应用日志（不筛选）

```bash
# 查看最近100行日志
sudo docker compose logs --tail 100 app

# 实时查看所有日志
sudo docker compose logs -f app
```

## 关键日志信息

发送邮件时，应该能看到以下日志：

1. `[邮件发送] 获取用户可见的舆情信息` - 开始查询
2. `[邮件发送] 管理员：执行SQL查询` - SQL查询开始
3. `[邮件发送] 管理员：查询到 X 条新闻` - 查询结果
4. `[邮件发送] 初始查询结果：X 条新闻` - 初始结果
5. `[邮件发送] 初始新闻详情` - 每条新闻的详细信息（APItype、news_category等）
6. `[邮件发送] 企查查类别过滤后：X 条新闻` - 类别过滤结果
7. `[邮件发送] ⚠️ 警告` - 如果有问题会显示警告
8. `[邮件发送] 最终返回：X 条新闻` - 最终结果

## 如果看不到详细日志

如果看不到上述详细日志，说明代码可能还没有更新到服务器。需要：

1. 确认代码已更新到服务器
2. 重启应用容器：
   ```bash
   sudo docker compose restart app
   ```
3. 重新测试发送邮件
