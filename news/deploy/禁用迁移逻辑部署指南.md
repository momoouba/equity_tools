# 禁用迁移逻辑部署指南

## ✅ 已完成的修改

已注释掉 `server/db.js` 中两处导致每次启动都执行外键约束迁移的代码：

1. **第 848-913 行**：移除 news_interface_config 表唯一约束的迁移逻辑
2. **第 1763-1857 行**：迁移 news_interface_config 唯一键约束的逻辑

## 📋 部署步骤

### 步骤1：将修改后的代码部署到服务器

```bash
# 在本地（Windows）提交代码
git add server/db.js
git commit -m "禁用每次启动的外键约束迁移逻辑"
git push

# 在服务器上拉取最新代码
cd /opt/newsapp/news
git pull
```

### 步骤2：重启应用

```bash
# 重启 PM2 应用
pm2 restart newsapp

# 查看启动日志
pm2 logs newsapp --lines 30
```

### 步骤3：验证修复

```bash
# 等待几秒后查看日志，应该没有外键约束警告
sleep 5
pm2 logs newsapp --lines 30 --nostream | grep -i "外键\|foreign\|Duplicate foreign key" || echo "✓ 没有外键相关警告"
```

## 🚀 一键部署

```bash
# 在服务器上执行
cd /opt/newsapp/news && \
git pull && \
pm2 restart newsapp && \
sleep 5 && \
pm2 logs newsapp --lines 30 --nostream | grep -i "外键\|foreign" || echo "✓ 没有外键相关警告"
```
