# 处理重启后数据库连接错误

## ❌ 错误现象

服务器重启后，应用启动时出现以下错误：

```
× 服务器启动失败: connect ECONNREFUSED 172.18.0.2:3306
错误堆栈: Error: connect ECONNREFUSED 172.18.0.2:3306
```

## 🔍 问题原因

1. **启动顺序问题**：应用容器在MySQL容器完全就绪前就尝试连接数据库
2. **MySQL容器启动时间**：MySQL容器需要时间初始化，健康检查可能还未通过
3. **网络延迟**：Docker网络可能需要时间建立连接
4. **健康检查配置**：虽然配置了 `depends_on: mysql: condition: service_healthy`，但可能健康检查还未完成

## ✅ 解决方法

### 方法1：使用恢复脚本（推荐）

使用 `restore-services.sh` 脚本，它会：
- 等待MySQL完全启动
- 检查MySQL健康状态
- 按正确顺序启动容器

```bash
# 1. 添加执行权限
chmod +x /opt/newsapp/news/deploy/restore-services.sh

# 2. 执行恢复脚本
/opt/newsapp/news/deploy/restore-services.sh
```

### 方法2：手动按顺序启动

```bash
# 1. 进入应用目录
cd /opt/newsapp/news

# 2. 先启动MySQL容器
sudo docker compose up -d mysql

# 3. 等待MySQL完全启动（至少30秒）
echo "等待MySQL启动..."
sleep 30

# 4. 检查MySQL健康状态
sudo docker inspect newsapp-mysql --format='{{.State.Health.Status}}'
# 应该显示：healthy

# 5. 测试MySQL连接
sudo docker exec newsapp-mysql mysqladmin ping -h localhost -u root -p$(grep MYSQL_ROOT_PASSWORD .env 2>/dev/null | cut -d'=' -f2) --silent

# 6. 启动应用容器
sudo docker compose up -d app

# 7. 等待应用启动
sleep 15

# 8. 启动Nginx容器
sudo docker compose up -d nginx

# 9. 检查所有容器状态
sudo docker compose ps
```

### 方法3：优化应用启动逻辑（代码层面）

如果应用代码中有数据库连接重试逻辑，可以增加重试次数和延迟：

```javascript
// 在 server/db.js 或 server/index.js 中
// 增加数据库连接重试逻辑
async function connectWithRetry(maxRetries = 10, delay = 3000) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      await createDatabaseIfNeeded();
      return true;
    } catch (error) {
      if (i === maxRetries - 1) throw error;
      console.log(`数据库连接失败，${delay/1000}秒后重试 (${i+1}/${maxRetries})...`);
      await new Promise(resolve => setTimeout(resolve, delay));
    }
  }
}
```

### 方法4：修改docker-compose.yml增加启动延迟

在 `docker-compose.yml` 中为应用容器添加启动命令延迟：

```yaml
app:
  # ... 其他配置
  command: >
    sh -c "
    echo '等待MySQL完全就绪...' &&
    sleep 30 &&
    node server/index.js
    "
```

或者使用 `wait-for-it` 脚本：

```yaml
app:
  # ... 其他配置
  command: >
    sh -c "
    ./wait-for-it.sh mysql:3306 --timeout=60 --strict -- node server/index.js
    "
```

## 🔧 完整修复流程

### 步骤1：停止所有容器

```bash
cd /opt/newsapp/news
sudo docker compose down
```

### 步骤2：检查MySQL配置

```bash
# 检查.env文件中的数据库配置
cat .env | grep DB_

# 如果使用Docker，DB_HOST应该是 "mysql"
# 如果使用本地MySQL，DB_HOST应该是 "localhost" 或 "127.0.0.1"
```

### 步骤3：按顺序启动服务

```bash
# 1. 启动MySQL
sudo docker compose up -d mysql

# 2. 等待并检查MySQL状态
echo "等待MySQL启动..."
for i in {1..60}; do
  STATUS=$(sudo docker inspect newsapp-mysql --format='{{.State.Health.Status}}' 2>/dev/null)
  if [ "$STATUS" = "healthy" ]; then
    echo "✓ MySQL已就绪"
    break
  fi
  if [ $i -eq 60 ]; then
    echo "⚠ MySQL启动超时，但继续执行"
  fi
  sleep 1
done

# 3. 额外等待5秒确保MySQL完全就绪
sleep 5

# 4. 启动应用
sudo docker compose up -d app

# 5. 等待应用启动
sleep 15

# 6. 启动Nginx
sudo docker compose up -d nginx

# 7. 检查状态
sudo docker compose ps
```

### 步骤4：验证连接

```bash
# 检查应用日志
sudo docker compose logs app --tail 50

# 应该看到：
# ✓ 数据库连接已就绪
# ✓ 服务器运行在 http://localhost:3001

# 测试健康检查
curl http://localhost:3001/api/health

# 应该返回：
# {"status":"ok","message":"服务器运行正常","database":"connected"}
```

## 🚨 故障排查

### 问题1：MySQL容器一直无法健康检查通过

```bash
# 查看MySQL容器日志
sudo docker compose logs mysql --tail 100

# 检查MySQL容器状态
sudo docker inspect newsapp-mysql

# 手动进入MySQL容器测试
sudo docker exec -it newsapp-mysql mysql -u root -p
```

### 问题2：应用容器启动后立即退出

```bash
# 查看应用日志
sudo docker compose logs app --tail 100

# 检查应用容器退出代码
sudo docker inspect newsapp --format='{{.State.ExitCode}}'

# 手动启动应用容器查看错误
sudo docker compose run --rm app node server/index.js
```

### 问题3：网络连接问题

```bash
# 检查Docker网络
sudo docker network ls
sudo docker network inspect newsapp-newsapp-network

# 测试容器间网络连接
sudo docker exec newsapp ping -c 3 mysql

# 检查DNS解析
sudo docker exec newsapp nslookup mysql
```

### 问题4：端口冲突

```bash
# 检查端口占用
sudo netstat -tulpn | grep -E ":(80|3001|3306|3307)"

# 检查Docker端口映射
sudo docker compose ps
sudo docker port newsapp-mysql
sudo docker port newsapp
```

## 📝 预防措施

### 1. 使用恢复脚本

将 `restore-services.sh` 添加到系统启动脚本：

```bash
# 编辑 /etc/rc.local（如果存在）
sudo nano /etc/rc.local

# 添加：
/opt/newsapp/news/deploy/restore-services.sh >> /var/log/restore-services.log 2>&1
```

### 2. 配置systemd服务

创建一个systemd服务来管理应用启动顺序：

```bash
# 创建服务文件
sudo nano /etc/systemd/system/newsapp-restore.service

# 内容：
[Unit]
Description=Restore NewsApp Services After Reboot
After=docker.service mysql.service
Requires=docker.service

[Service]
Type=oneshot
ExecStart=/opt/newsapp/news/deploy/restore-services.sh
RemainAfterExit=yes

[Install]
WantedBy=multi-user.target

# 启用服务
sudo systemctl enable newsapp-restore.service
```

### 3. 优化docker-compose.yml

确保 `depends_on` 配置正确：

```yaml
app:
  depends_on:
    mysql:
      condition: service_healthy
```

### 4. 增加应用启动延迟

在应用代码中增加数据库连接重试逻辑，或者使用启动脚本延迟启动。

## 🎯 快速修复命令

如果遇到连接错误，执行以下命令快速修复：

```bash
cd /opt/newsapp/news && \
sudo docker compose down && \
sudo docker compose up -d mysql && \
sleep 30 && \
sudo docker compose up -d app && \
sleep 15 && \
sudo docker compose up -d nginx && \
sudo docker compose ps && \
echo "等待10秒后检查应用..." && \
sleep 10 && \
curl -f http://localhost:3001/api/health && \
echo "✓ 应用启动成功"
```
