## 🚀 部署步骤

### 步骤1：确认文件已上传

```bash
# 进入项目目录
cd /opt/newsapp/news

# 检查前端文件
ls -la client/src/pages/NewsInfo.jsx
ls -la client/src/pages/ShareNewsPage.jsx
ls -la client/src/pages/RecipientManagement.jsx
ls -la client/src/pages/NewsConfig.jsx
ls -la client/src/pages/EnterpriseManagement.jsx
ls -la client/src/pages/ShareNewsPage.css

# 检查后端文件
ls -la server/routes/news.js
ls -la server/routes/system.js
ls -la server/utils/emailSender.js

# 检查文件修改时间（确认已更新）
stat client/src/pages/NewsInfo.jsx
stat server/routes/news.js
```

### 步骤2：重新构建前端（重要！）

由于前端代码有修改，**必须重新构建前端**：

```bash
# 进入前端目录
cd /opt/newsapp/news/client

# 安装依赖（如果package.json有变化，通常不需要）
# npm install

# 重新构建前端
npm run build

# 检查构建是否成功
ls -la dist/index.html
```

**注意**：如果构建失败，检查：
- Node.js版本是否正确（需要Node 18+）
- 内存是否充足（可能需要增加内存限制）
- 查看构建错误信息

### 步骤3：更新前端文件到Docker Volume

```bash
# 返回项目根目录
cd /opt/newsapp/news

# 使用自动脚本更新前端文件（推荐）
chmod +x deploy/update-from-upload.sh
./deploy/update-from-upload.sh
```

**或者手动更新**：

```bash
# 查找前端volume名称
sudo docker volume ls | grep frontend

# 假设volume名称为 newsapp_app_frontend
VOLUME_NAME="newsapp_app_frontend"

# 获取volume路径
VOLUME_PATH=$(sudo docker volume inspect $VOLUME_NAME | grep -i "Mountpoint" | awk '{print $2}' | tr -d '",')

# 清空旧文件
sudo rm -rf $VOLUME_PATH/*

# 复制新文件
sudo cp -r client/dist/* $VOLUME_PATH/

# 设置权限
sudo chown -R root:root $VOLUME_PATH
sudo chmod -R 755 $VOLUME_PATH
```

### 步骤4：重启应用容器（后端代码自动生效）

由于后端代码是直接挂载的，只需重启容器即可：

```bash
# 重启应用容器
sudo docker compose restart app

# 等待服务启动
sleep 5

# 查看容器状态
sudo docker compose ps
```

### 步骤5：重启Nginx容器（前端更新需要）

```bash
# 重启Nginx容器
sudo docker compose restart nginx

# 查看Nginx状态
sudo docker compose ps nginx
```

### 步骤6：验证更新

```bash
# 1. 检查容器状态（所有容器应该显示为 "Up"）
sudo docker compose ps

# 2. 查看应用日志（确认没有错误）
sudo docker compose logs app --tail 50

# 3. 测试健康检查
curl http://localhost:3001/api/health

# 4. 检查前端文件是否已更新
sudo docker compose exec nginx ls -la /usr/share/nginx/html/ | head -10
```

## ⚠️ 重要注意事项

### 1. 前端必须重新构建

**重要**：前端代码修改后，**必须执行 `npm run build`**，否则修改不会生效！

### 2. 浏览器缓存

更新后，用户需要：
- 清除浏览器缓存（`Ctrl+Shift+Delete`）
- 或强制刷新页面（`Ctrl+F5` 或 `Ctrl+Shift+R`）

### 3. 检查构建是否成功

```bash
# 检查dist目录是否生成
ls -la client/dist/

# 检查index.html是否存在且大小正常（应该大于几KB）
ls -lh client/dist/index.html
```

### 4. 如果构建失败

```bash
# 查看构建错误
cd client
npm run build 2>&1 | tee build.log

# 检查Node版本
node --version  # 应该是 v18.x 或更高

# 检查内存（如果内存不足，可能需要增加swap）
free -h
```

## 🐛 常见问题排查

### 问题1：前端更新后看不到变化

**解决方法**：
```bash
# 1. 确认前端已重新构建
ls -lh client/dist/index.html

# 2. 确认文件已复制到volume
sudo docker volume inspect newsapp_app_frontend | grep Mountpoint

# 3. 重启nginx
sudo docker compose restart nginx

# 4. 清除浏览器缓存
```

### 问题2：应用容器重启失败

**解决方法**：
```bash
# 查看详细错误日志
sudo docker compose logs app --tail 100

# 检查代码语法
sudo docker compose exec app node -c /app/server/routes/news.js

# 检查环境变量
sudo docker compose exec app env | grep -i db
```

### 问题3：构建前端时内存不足

**解决方法**：
```bash
# 增加Node内存限制
cd client
NODE_OPTIONS="--max-old-space-size=4096" npm run build

# 或使用构建脚本
chmod +x deploy/build-frontend-safe.sh
./deploy/build-frontend-safe.sh
```
