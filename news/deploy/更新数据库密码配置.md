# 更新数据库密码配置

## 📋 问题

数据库密码已更改，但应用程序的 `.env` 文件中的密码配置未更新，导致应用无法连接数据库。

## ✅ 解决方法

### 方法1：使用脚本自动更新（推荐）

```bash
# 1. 上传脚本到服务器（如果还没有）
# 脚本路径: deploy/更新数据库密码配置.sh

# 2. 在服务器上执行
cd /opt/newsapp/news
bash /path/to/更新数据库密码配置.sh

# 或者直接执行
bash deploy/更新数据库密码配置.sh
```

### 方法2：手动更新

```bash
# 1. 进入应用目录
cd /opt/newsapp/news

# 2. 备份原文件
cp .env .env.backup.$(date +%Y%m%d_%H%M%S)

# 3. 更新密码
sed -i 's|^DB_PASSWORD=.*|DB_PASSWORD=Mqdqxygyqy!!!klklsys24678|' .env

# 4. 验证更新
cat .env | grep DB_PASSWORD
```

### 方法3：使用 nano 编辑器

```bash
# 1. 进入应用目录
cd /opt/newsapp/news

# 2. 编辑 .env 文件
nano .env

# 3. 找到 DB_PASSWORD 行，修改为：
# DB_PASSWORD=Mqdqxygyqy!!!klklsys24678

# 4. 保存（Ctrl+O, Enter, Ctrl+X）
```

## 🔄 更新后重启应用

```bash
# 重启 PM2 应用
pm2 restart newsapp

# 查看启动日志
pm2 logs newsapp --lines 20

# 检查应用状态
pm2 status
```

## 📋 完整的 .env 配置示例

确保 `.env` 文件包含以下配置：

```env
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=Mqdqxygyqy!!!klklsys24678
DB_NAME=investment_tools
```

## ✅ 验证步骤

```bash
# 1. 检查配置文件
cd /opt/newsapp/news
cat .env | grep DB_

# 2. 重启应用
pm2 restart newsapp

# 3. 查看日志（确认没有数据库连接错误）
pm2 logs newsapp --lines 20 --nostream

# 4. 检查应用状态
pm2 status
```

## 🚀 一键更新和重启

```bash
# 更新密码并重启
cd /opt/newsapp/news && \
sed -i 's|^DB_PASSWORD=.*|DB_PASSWORD=Mqdqxygyqy!!!klklsys24678|' .env && \
echo "✓ 密码已更新" && \
cat .env | grep DB_PASSWORD && \
pm2 restart newsapp && \
sleep 3 && \
pm2 logs newsapp --lines 10 --nostream
```

## 🔍 如果仍然失败

### 检查密码是否正确

```bash
# 测试数据库连接
mysql -u root -p'Mqdqxygyqy!!!klklsys24678' -e "SELECT 1;"
```

### 检查 .env 文件权限

```bash
# 确保应用用户有读取权限
ls -la /opt/newsapp/news/.env
chmod 644 /opt/newsapp/news/.env
```

### 检查应用日志

```bash
# 查看详细错误
pm2 logs newsapp --err --lines 50
tail -50 /var/log/newsapp/error.log
```
