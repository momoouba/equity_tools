# 手动更新前端步骤（FinalShell）

## 当前情况

你已经上传了 `dist.zip` 并解压到了 `/opt/newsapp/news/client/dist` 目录。

## 方法一：使用简化版脚本（推荐，无需Docker镜像）

在 FinalShell 终端中执行：

```bash
cd /opt/newsapp/news

# 给脚本添加执行权限
chmod +x deploy/update-frontend-simple.sh

# 执行更新
./deploy/update-frontend-simple.sh
```

这个脚本会：
- 自动找到运行中的nginx容器
- 直接复制文件到nginx容器
- **不需要拉取Docker镜像**

## 方法二：手动执行命令（如果脚本有问题）

### 步骤1: 查找nginx容器

```bash
# 查看运行中的nginx容器
sudo docker ps | grep nginx
```

应该会看到类似 `newsapp-nginx` 或包含 `nginx` 的容器名称。

### 步骤2: 清空旧文件并复制新文件

```bash
# 替换 CONTAINER_NAME 为实际的nginx容器名称
CONTAINER_NAME="newsapp-nginx"  # 根据实际情况修改

# 清空旧文件
sudo docker exec "$CONTAINER_NAME" sh -c "rm -rf /usr/share/nginx/html/*"

# 复制新文件
sudo docker cp client/dist/. "$CONTAINER_NAME:/usr/share/nginx/html/"
```

### 步骤3: 验证更新

```bash
# 检查文件是否已复制
sudo docker exec "$CONTAINER_NAME" ls -la /usr/share/nginx/html/

# 查看index.html
sudo docker exec "$CONTAINER_NAME" cat /usr/share/nginx/html/index.html | head -20
```

### 步骤4: 重启nginx（可选，通常不需要）

如果文件已更新但页面没变化，可以重启：

```bash
sudo docker restart "$CONTAINER_NAME"
```

## 方法三：通过volume路径直接更新

### 步骤1: 查找volume路径

```bash
# 查看所有volumes
sudo docker volume ls | grep frontend

# 查看volume详细信息
sudo docker volume inspect newsapp_app_frontend
```

从输出中找到 `Mountpoint` 字段，例如：`/var/lib/docker/volumes/newsapp_app_frontend/_data`

### 步骤2: 直接操作volume路径

```bash
# 替换VOLUME_PATH为实际路径
VOLUME_PATH="/var/lib/docker/volumes/newsapp_app_frontend/_data"

# 清空旧文件
sudo rm -rf "$VOLUME_PATH"/*

# 复制新文件
sudo cp -r client/dist/* "$VOLUME_PATH/"

# 设置权限
sudo chown -R root:root "$VOLUME_PATH"
sudo chmod -R 755 "$VOLUME_PATH"
```

### 步骤3: 重启nginx

```bash
sudo docker restart newsapp-nginx
```

## 快速操作（FinalShell终端）

在 FinalShell 终端中，当前在 `/opt/newsapp/news` 目录，直接执行：

```bash
# 查找nginx容器名称
NGINX_CONTAINER=$(sudo docker ps --format "{{.Names}}" | grep nginx | head -1)
echo "Nginx容器: $NGINX_CONTAINER"

# 清空并复制
sudo docker exec "$NGINX_CONTAINER" sh -c "rm -rf /usr/share/nginx/html/*"
sudo docker cp client/dist/. "$NGINX_CONTAINER:/usr/share/nginx/html/"

# 验证
sudo docker exec "$NGINX_CONTAINER" ls -la /usr/share/nginx/html/ | head -10
```

## 验证更新是否成功

### 1. 检查文件

```bash
# 进入nginx容器查看
sudo docker exec -it newsapp-nginx sh

# 在容器内
ls -la /usr/share/nginx/html/
cat /usr/share/nginx/html/index.html | head -20

# 退出容器
exit
```

### 2. 查看nginx日志

```bash
sudo docker logs newsapp-nginx --tail 50
```

### 3. 浏览器验证

1. 清除浏览器缓存（`Ctrl+Shift+Delete`）
2. 强制刷新页面（`Ctrl+F5`）
3. 打开开发者工具（F12），检查控制台是否有错误

## 常见问题

### Q: 找不到nginx容器

```bash
# 查看所有容器（包括停止的）
sudo docker ps -a | grep nginx

# 如果容器停止了，启动它
sudo docker start newsapp-nginx

# 或使用docker compose
sudo docker compose up -d nginx
```

### Q: 权限被拒绝

```bash
# 使用sudo执行所有命令
sudo docker exec ...
sudo docker cp ...
```

### Q: 文件复制后页面没有更新

```bash
# 1. 确认文件已复制
sudo docker exec newsapp-nginx ls -la /usr/share/nginx/html/

# 2. 重启nginx容器
sudo docker restart newsapp-nginx

# 3. 清除浏览器缓存
# 在浏览器中按 Ctrl+Shift+Delete

# 4. 强制刷新
# 在浏览器中按 Ctrl+F5
```

### Q: 找不到volume路径

```bash
# 查看docker-compose.yml中的volume配置
grep -A 3 "app_frontend" docker-compose.yml

# 查看所有volumes
sudo docker volume ls

# 尝试不同的volume名称
sudo docker volume inspect newsapp_app_frontend
sudo docker volume inspect "$(basename $(pwd))_app_frontend"
```

## 推荐方案

**最简单的方法：**

```bash
cd /opt/newsapp/news
NGINX_CONTAINER=$(sudo docker ps --format "{{.Names}}" | grep nginx | head -1)
sudo docker exec "$NGINX_CONTAINER" sh -c "rm -rf /usr/share/nginx/html/*"
sudo docker cp client/dist/. "$NGINX_CONTAINER:/usr/share/nginx/html/"
```

这三行命令就可以完成更新，**不需要任何Docker镜像**！
