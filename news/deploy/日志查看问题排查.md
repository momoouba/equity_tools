# 日志查看问题排查

## ❌ 常见错误：`No such file or directory`

### 错误原因

用户当前不在应用目录中，脚本路径不正确。

### 解决方法

#### 方法1：切换到应用目录（推荐）

```bash
# 1. 切换到应用目录
cd /opt/newsapp/news

# 2. 运行日志查看脚本
./deploy/view-logs.sh --pm2
```

#### 方法2：使用完整路径

```bash
# 直接使用完整路径
/opt/newsapp/news/deploy/view-logs.sh --pm2
```

#### 方法3：检查当前目录

```bash
# 查看当前目录
pwd

# 查看应用目录是否存在
ls -la /opt/newsapp/news/deploy/view-logs.sh

# 如果文件存在，切换到应用目录
cd /opt/newsapp/news
```

## 🔍 完整排查步骤

### 步骤1：检查应用目录

```bash
# 检查应用目录是否存在
ls -la /opt/newsapp/news

# 检查deploy目录
ls -la /opt/newsapp/news/deploy/
```

```bash
# 检查脚本权限

# 如果没有执行权限，添加权限
chmod +x /opt/newsapp/news/deploy/view-logs.sh
```

### 步骤3：切换到正确目录

```bash
# 切换到应用目录
cd /opt/newsapp/news

# 确认当前目录
pwd
# 应该显示: /opt/newsapp/news

# 运行脚本
./deploy/view-logs.sh --pm2
```

## ✅ 正确的使用方式

### 方式1：使用PM2直接查看（最简单）

```bash
# 直接使用PM2命令（不需要脚本）
pm2 logs newsapp

# 只查看错误日志
pm2 logs newsapp --err

# 查看最后100行
pm2 logs newsapp --lines 100
```

### 方式2：使用日志查看脚本

```bash
# 1. 切换到应用目录
cd /opt/newsapp/news

# 2. 运行脚本
./deploy/view-logs.sh --pm2

# 或使用完整路径
/opt/newsapp/news/deploy/view-logs.sh --pm2
```

### 方式3：直接查看日志文件

```bash
# 实时查看合并日志
tail -f /var/log/newsapp/combined.log

# 查看错误日志
tail -f /var/log/newsapp/error.log

# 查看最后100行
tail -n 100 /var/log/newsapp/combined.log
```

## 🛠️ 如果脚本不存在

### 检查文件是否同步到服务器

```bash
# 检查deploy目录内容
ls -la /opt/newsapp/news/deploy/

# 如果view-logs.sh不存在，需要同步文件
```

### 手动创建快捷方式（可选）

```bash
# 创建全局命令（可选）
sudo ln -s /opt/newsapp/news/deploy/view-logs.sh /usr/local/bin/view-logs

# 然后可以在任何目录使用
view-logs --pm2
```

## 💡 建议

1. **使用PM2命令**：最简单直接，不需要脚本
2. **创建别名**：可以在 `~/.bashrc` 中添加别名
3. **使用完整路径**：避免目录问题

### 创建别名（可选）

编辑 `~/.bashrc`：

```bash
nano ~/.bashrc
```

添加：

```bash
# 日志查看别名
alias viewlogs='cd /opt/newsapp/news && ./deploy/view-logs.sh'
alias pm2logs='pm2 logs newsapp'
```

重新加载配置：

```bash
source ~/.bashrc
```

然后可以使用：

```bash
viewlogs --pm2
pm2logs
```
