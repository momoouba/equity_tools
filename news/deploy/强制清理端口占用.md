# 强制清理端口 3306 占用

## ❌ 问题确认

从日志可以看到：
```
[ERROR] Can't start server: Bind on TCP/IP port. Got error: 98: Address already in use
[ERROR] Do you already have another server running on port: 3306 ?
```

端口 3306 仍然被占用，需要找到并强制停止占用进程。

## 🔍 查找占用进程

### 步骤1：查找占用端口的进程

```bash
# 方法1：使用 lsof（最准确）
sudo lsof -i :3306

# 方法2：使用 netstat
sudo netstat -tulpn | grep 3306

# 方法3：使用 ss
sudo ss -tulpn | grep 3306

# 方法4：使用 fuser
sudo fuser 3306/tcp
```

### 步骤2：查看所有 MySQL/MariaDB 相关进程

```bash
# 查看所有相关进程
ps aux | grep mysql
ps aux | grep mariadb
ps aux | grep mysqld
```

## ✅ 强制停止占用进程

### 方法1：根据进程ID停止

```bash
# 从 lsof 输出中找到进程ID（PID），然后停止
# 例如：如果看到 PID 是 12345
sudo kill -9 12345

# 或使用 fuser 直接停止
sudo fuser -k 3306/tcp
```

### 方法2：停止所有 MySQL/MariaDB 进程

```bash
# 停止所有相关进程
sudo pkill -9 mysql
sudo pkill -9 mysqld
sudo pkill -9 mariadb
sudo pkill -9 mariadbd

# 停止可能存在的服务
sudo systemctl stop mysql 2>/dev/null
sudo systemctl stop mysqld 2>/dev/null
sudo systemctl stop mariadb 2>/dev/null
```

### 方法3：使用 fuser 强制释放端口

```bash
# 强制停止占用 3306 端口的所有进程
sudo fuser -k 3306/tcp

# 等待进程完全停止
sleep 3
```

## 🚀 完整清理和启动流程

### 一键清理命令

```bash
# 1. 查找占用端口的进程
echo "=== 查找占用 3306 端口的进程 ==="
sudo lsof -i :3306

# 2. 强制停止所有相关进程
echo -e "\n=== 强制停止所有 MySQL/MariaDB 进程 ==="
sudo fuser -k 3306/tcp 2>/dev/null
sudo pkill -9 mysql 2>/dev/null
sudo pkill -9 mysqld 2>/dev/null
sudo pkill -9 mariadb 2>/dev/null
sudo pkill -9 mariadbd 2>/dev/null
sudo systemctl stop mysql mysqld mariadb 2>/dev/null

# 3. 等待进程完全停止
echo "等待进程停止..."
sleep 5

# 4. 确认端口已释放
echo -e "\n=== 确认端口已释放 ==="
sudo netstat -tulpn | grep 3306 || echo "✓ 端口 3306 已释放"

# 5. 再次检查进程
echo -e "\n=== 检查是否还有相关进程 ==="
ps aux | grep -E "mysql|mariadb" | grep -v grep || echo "✓ 没有相关进程"

# 6. 启动 MariaDB
echo -e "\n=== 启动 MariaDB ==="
sudo systemctl start mariadb

# 7. 检查状态
echo -e "\n=== 检查 MariaDB 状态 ==="
sudo systemctl status mariadb --no-pager -l | head -20
```

## 🔧 如果仍然被占用

### 方法1：查找隐藏进程

```bash
# 查看所有网络连接
sudo netstat -tulpn | grep 3306

# 查看所有进程（包括僵尸进程）
ps auxf | grep mysql

# 查看 systemd 管理的服务
sudo systemctl list-units | grep -i mysql
sudo systemctl list-units | grep -i mariadb
```

### 方法2：检查 Docker 容器

```bash
# 检查是否有 Docker 容器占用端口
sudo docker ps | grep 3306
sudo docker ps -a | grep mysql

# 如果有，停止容器
sudo docker stop <容器ID>
```

### 方法3：检查其他数据库服务

```bash
# 检查是否有其他数据库服务
sudo systemctl list-units --type=service | grep -E "mysql|mariadb|percona|mariadb"
```

## 🚀 最强制的方法

如果以上方法都不行，使用最强制的方法：

```bash
# 1. 查找所有占用 3306 的进程（包括内核）
sudo lsof -i :3306 -t | xargs -r sudo kill -9

# 2. 使用 fuser 强制释放
sudo fuser -k 3306/tcp

# 3. 停止所有可能的服务
sudo systemctl stop mysql mysqld mariadb percona-server 2>/dev/null

# 4. 强制停止所有相关进程
sudo pkill -9 -f mysql
sudo pkill -9 -f mariadb
sudo pkill -9 -f mysqld

# 5. 等待
sleep 5

# 6. 确认
sudo netstat -tulpn | grep 3306 || echo "端口已释放"

# 7. 启动 MariaDB
sudo systemctl start mariadb
```

## 📋 验证步骤

清理后，确认：

```bash
sudo netstat -tulpn | grep 3306
# 应该没有输出

# 2. 没有相关进程
ps aux | grep -E "mysql|mariadb" | grep -v grep
# 应该没有输出（除了 grep 本身）

# 3. 启动 MariaDB
sudo systemctl start mariadb

# 4. 检查状态
sudo systemctl status mariadb

# 5. 测试连接
sudo mysql -u root -e "SELECT 1;"
```
