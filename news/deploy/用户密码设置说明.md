# 用户密码设置说明

## 🔐 密码设置方式

### 方式1：创建用户时设置密码（推荐）

使用 `adduser` 命令创建用户时，会自动提示设置密码：

```bash
sudo adduser guofang
```

执行后会提示：
- 输入新密码
- 确认密码
- 输入用户信息（全名、房间号、工作电话、家庭电话、其他）- 可以直接回车跳过

### 方式2：使用脚本创建（已更新）

使用 `create-user.sh` 脚本创建用户时，也会提示设置密码：

```bash
sudo ./deploy/create-user.sh guofang
```

脚本已更新，现在会：
- ✅ 创建用户时提示设置密码
- ✅ 如果用户已存在，可以选择修改密码

### 方式3：创建后设置密码

如果用户已创建但没有密码，可以手动设置：

```bash
# root用户设置其他用户密码
sudo passwd guofang

# 用户自己修改密码（需要先登录）
passwd
```

## 📋 使用 create-user.sh 脚本

### 创建新用户（会提示设置密码）

```bash
chmod +x deploy/create-user.sh
sudo ./deploy/create-user.sh guofang
```

**执行流程**：
1. 创建用户 `guofang`（会提示输入密码）
2. 添加到 sudo 组
3. 可选：配置 sudo 免密码
4. 设置应用目录权限
5. 可选：配置 SSH 密钥
6. 可选：安装 Node.js 和 PM2
7. 可选：配置 PM2 开机自启

### 如果用户已存在

脚本会检测到用户已存在，并询问：
- 是否继续配置此用户？
- 是否修改密码？

## 🔒 密码安全建议

### 1. 使用强密码

密码应该包含：
- 至少8个字符
- 大小写字母
- 数字
- 特殊字符

### 2. 定期更换密码

```bash
# 用户自己修改密码
passwd

# root修改其他用户密码
sudo passwd guofang
```

### 3. 配置SSH密钥（更安全）

设置SSH密钥后，可以禁用密码登录：

```bash
# 编辑SSH配置
sudo nano /etc/ssh/sshd_config

# 设置
PasswordAuthentication no
PubkeyAuthentication yes

# 重启SSH服务
sudo systemctl restart sshd
```

## 🛠️ 常见问题

### 问题1：忘记密码

**解决方法**：

```bash
# root用户重置密码
sudo passwd guofang

# 输入新密码两次
```

### 问题2：无法使用密码登录

**可能原因**：
1. SSH配置禁用了密码登录
2. 用户账户被锁定
3. 密码错误

**解决方法**：

```bash
# 检查SSH配置
sudo grep PasswordAuthentication /etc/ssh/sshd_config

# 如果显示 no，需要启用或使用SSH密钥
# 编辑配置
sudo nano /etc/ssh/sshd_config
# 设置 PasswordAuthentication yes
sudo systemctl restart sshd

# 检查用户账户状态
sudo passwd -S guofang
```

### 问题3：sudo需要密码

**解决方法**：

```bash
# 检查用户是否在sudo组
groups guofang

# 如果不在，添加到sudo组
sudo usermod -aG sudo guofang

# 重新登录用户
```
