# 修复数据库连接错误指南

## ❌ 错误信息

```
× 数据库连接失败: getaddrinfo EAI AGAIN mysql
数据库初始化过程中出错: getaddrinfo EAI_AGAIN mysql
```

## 🔍 问题原因

`getaddrinfo EAI_AGAIN mysql` 表示系统无法解析主机名 "mysql"。这通常是因为：

1. **.env 文件中 DB_HOST 配置为 "mysql"**，但系统无法解析这个主机名
2. **应该使用 "localhost" 或 "127.0.0.1"** 来连接本地 MySQL
3. 或者需要配置 `/etc/hosts` 文件（如果使用 Docker 等）

## ✅ 解决方法

### 方法1：修改 .env 文件（推荐）

```bash
# 1. 进入应用目录
cd /opt/newsapp/news

# 2. 检查 .env 文件
cat .env

# 3. 编辑 .env 文件
nano .env
# 或
vi .env
```

**修改数据库配置**：

找到这一行：
```
DB_HOST=mysql
```

修改为：
```
DB_HOST=localhost
```

或者：
```
DB_HOST=127.0.0.1
```

**完整的数据库配置示例**：
```
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的数据库密码
DB_NAME=investment_tools
```

### 方法2：如果使用 Docker 或需要配置 hosts

如果确实需要使用 "mysql" 作为主机名（比如在 Docker Compose 环境中）：

```bash
# 编辑 hosts 文件
sudo nano /etc/hosts

# 添加这一行（如果 MySQL 在本地）
127.0.0.1 mysql

# 或如果 MySQL 在其他服务器
MySQL服务器IP mysql
```

### 方法3：检查 MySQL 服务状态

```bash
# 检查 MySQL 是否运行
sudo systemctl status mysql
# 或
sudo systemctl status mysqld

# 如果未运行，启动 MySQL
sudo systemctl start mysql

# 设置开机自启
sudo systemctl enable mysql
```

## 🔧 完整修复流程

### 步骤1：检查 .env 文件

```bash
cd /opt/newsapp/news

# 检查 .env 文件是否存在
ls -la .env

# 查看当前配置
cat .env | grep DB_
```

### 步骤2：修改数据库配置

```bash
# 编辑 .env 文件
nano .env

# 确保以下配置正确：
# DB_HOST=localhost  （不要使用 mysql）
# DB_PORT=3306
# DB_USER=root       （或你的数据库用户名）
# DB_PASSWORD=你的密码
# DB_NAME=investment_tools
```

### 步骤3：测试数据库连接

```bash
# 测试 MySQL 连接
mysql -h localhost -u root -p

# 或使用配置中的用户名
mysql -h localhost -u <DB_USER> -p

# 如果连接成功，说明配置正确
```

### 步骤4：重启应用

```bash
# 重启 PM2 应用
pm2 restart newsapp

# 查看日志确认
pm2 logs newsapp --lines 50
```

## 🚀 快速修复命令

### 一键修复（如果 .env 中 DB_HOST=mysql）

```bash
cd /opt/newsapp/news && \
sed -i 's/DB_HOST=mysql/DB_HOST=localhost/g' .env && \
pm2 restart newsapp && \
pm2 logs newsapp --lines 50
```

### 如果 .env 文件不存在

```bash
cd /opt/newsapp/news && \
cp deploy/env.production.template .env && \
nano .env
# 然后编辑数据库配置
```

## 🔍 验证修复

### 检查应用状态

```bash
# 查看 PM2 状态
pm2 status

# 应该显示所有实例为 online（不是 launching）

# 查看日志（应该没有数据库连接错误）
pm2 logs newsapp --lines 100

# 检查错误日志
pm2 logs newsapp --err --lines 20
```

### 检查数据库连接

```bash
# 查看日志中是否有成功连接的信息
pm2 logs newsapp | grep "数据库"

# 应该看到类似：
# ✓ 数据库连接已就绪
# ✓ 服务器运行在 http://localhost:3001
```

### 测试 API

```bash
# 测试健康检查接口
curl http://localhost:3001/api/health

# 应该返回：
# {"status":"ok","message":"服务器运行正常","database":"connected"}
```

## ⚠️ 常见问题

### 问题1：.env 文件不存在

**解决方法**：

```bash
cd /opt/newsapp/news

# 从模板创建
cp deploy/env.production.template .env

# 编辑配置
nano .env
```

### 问题2：MySQL 服务未启动

**解决方法**：

```bash
# 检查 MySQL 状态
sudo systemctl status mysql

# 启动 MySQL
sudo systemctl start mysql

# 检查端口
netstat -tulpn | grep 3306
```

### 问题3：数据库用户或密码错误

**解决方法**：

```bash
# 测试连接
mysql -h localhost -u root -p

# 如果连接失败，检查：
# 1. 用户名是否正确
# 2. 密码是否正确
# 3. 用户是否有权限访问数据库
```

### 问题4：数据库不存在

**解决方法**：

```bash
# 登录 MySQL
mysql -h localhost -u root -p

# 创建数据库（如果不存在）
CREATE DATABASE IF NOT EXISTS investment_tools DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 退出
exit;
```

## 📋 完整的 .env 配置示例

```env
# 数据库配置
DB_HOST=localhost

DB_USER=root

DB_NAME=investment_tools

# 应用配置
NODE_ENV=production
PORT=3001

# JWT密钥（需要修改）
JWT_SECRET=your-secret-key-here

# 其他配置...
```

## ✅ 修复后验证清单

修复完成后，确认：

- [ ] `.env` 文件中 `DB_HOST=localhost`（不是 mysql）
- [ ] MySQL 服务正在运行：`sudo systemctl status mysql`
- [ ] 可以手动连接数据库：`mysql -h localhost -u root -p`
- [ ] `pm2 status` 显示所有实例为 `online`
- [ ] `pm2 logs newsapp` 没有数据库连接错误
- [ ] `curl http://localhost:3001/api/health` 返回正常

## 📞 如果仍然有问题

如果修复后仍然报错：

1. **查看详细错误**：
   ```bash
   pm2 logs newsapp --err --lines 100
   ```

2. **检查 MySQL 日志**：
   ```bash
   sudo tail -f /var/log/mysql/error.log
   ```

3. **测试数据库连接**：
   ```bash
   mysql -h localhost -u root -p investment_tools
   ```

4. **检查防火墙**：
   ```bash
   sudo ufw status
   # 确保 MySQL 端口 3306 没有被阻止
   ```
