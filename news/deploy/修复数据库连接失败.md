# 修复数据库连接失败

## ❌ 错误信息

```
数据库连接失败: Access denied for user 'newsapp'@'localhost' (using password: YES)
```

## ✅ 问题分析

应用尝试使用 `newsapp@'localhost'` 用户连接数据库，但：
1. 该用户可能不存在
2. 或者密码不正确
3. 或者 `.env` 文件中配置的用户名是 `newsapp`，但应该使用 `root`

## 🔧 解决步骤

### 步骤1：检查 .env 文件配置

```bash
# 查看当前的数据库配置
cd /opt/newsapp/news
cat .env | grep DB_
```

### 步骤2：方案A - 使用 root 用户（推荐）

如果 `.env` 中配置的是 `newsapp`，改为使用 `root`：

```bash
# 更新 .env 文件，使用 root 用户
cd /opt/newsapp/news
sed -i 's|^DB_USER=.*|DB_USER=root|' .env
sed -i 's|^DB_PASSWORD=.*|DB_PASSWORD=Mqdqxygyqy!!!klklsys24678|' .env

# 验证配置
cat .env | grep DB_
```

### 步骤3：方案B - 创建 newsapp@localhost 用户

如果想继续使用 `newsapp` 用户，需要创建本地用户：

```bash
# 使用 root 登录 MySQL
sudo mysql -u root -p'Mqdqxygyqy!!!klklsys24678' <<EOF
CREATE USER IF NOT EXISTS 'newsapp'@'localhost' IDENTIFIED BY '98K6^7s8!9Z8*76p8';
GRANT ALL PRIVILEGES ON investment_tools.* TO 'newsapp'@'localhost';
FLUSH PRIVILEGES;
SELECT user, host FROM mysql.user WHERE user='newsapp';
EOF

# 然后更新 .env 文件
cd /opt/newsapp/news
sed -i 's|^DB_USER=.*|DB_USER=newsapp|' .env
sed -i 's|^DB_PASSWORD=.*|DB_PASSWORD=98K6^7s8!9Z8*76p8|' .env
```

### 步骤4：重启应用

```bash
# 重启应用
pm2 restart newsapp

# 查看启动日志
pm2 logs newsapp --lines 20
```

## 🚀 一键修复（使用 root 用户）

```bash
# 更新配置为使用 root 用户
cd /opt/newsapp/news && \
sed -i 's|^DB_USER=.*|DB_USER=root|' .env && \
sed -i 's|^DB_PASSWORD=.*|DB_PASSWORD=Mqdqxygyqy!!!klklsys24678|' .env && \
echo "✓ 配置已更新" && \
echo "=== 当前配置 ===" && \
cat .env | grep DB_ && \
echo -e "\n=== 测试数据库连接 ===" && \
mysql -u root -p'Mqdqxygyqy!!!klklsys24678' -e "USE investment_tools; SELECT 1;" && \
echo "✓ 数据库连接正常" && \
echo -e "\n=== 重启应用 ===" && \
pm2 restart newsapp && \
sleep 5 && \
echo -e "\n=== 查看最新日志 ===" && \
pm2 logs newsapp --err --lines 10 --nostream
```

```bash
tail -50 /var/log/newsapp/error.log | grep -i "error\|denied\|fail"
```
