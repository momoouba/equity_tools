# 日志查看快速参考

## 🚀 最快的方式

### 如果使用PM2（推荐）

```bash
# 最简单：实时查看所有日志
pm2 logs newsapp

# 只查看错误日志
pm2 logs newsapp --err

# 查看最后100行（不跟踪）
pm2 logs newsapp --lines 100 --nostream
```

### 如果直接运行node（开发环境）

日志会直接输出到运行node的终端窗口。

查看运行中的进程：
```bash
ps aux | grep 'node.*index.js'
```

## 📋 使用日志查看脚本

### 基本用法

```bash
# 给脚本执行权限（只需一次）
chmod +x deploy/view-logs.sh

# 实时查看日志
./deploy/view-logs.sh

# 使用PM2查看
./deploy/view-logs.sh --pm2

# 查看错误日志
./deploy/view-logs.sh --error
```

### 常见问题解决

#### 问题1：日志文件不存在

**原因**：应用未使用PM2运行，或应用未启动

**解决方法**：

1. **使用PM2启动应用**（推荐）：
```bash
# 方式1：使用配置文件
pm2 start deploy/ecosystem.config.js

# 方式2：直接启动
pm2 start server/index.js --name newsapp

# 然后查看日志
pm2 logs newsapp
```

2. **如果直接运行node**：
   - 日志会输出到运行node的终端
   - 建议使用PM2管理以获得更好的日志管理

#### 问题2：PM2未安装

**解决方法**：
```bash
npm install -g pm2
```

#### 问题3：应用未运行

**检查应用状态**：
```bash
# 检查PM2进程
pm2 list

# 检查node进程
ps aux | grep 'node.*index.js'

# 检查端口占用
netstat -tulpn | grep 3001
```

**启动应用**：
```bash
# 使用PM2
pm2 start deploy/ecosystem.config.js

# 或直接运行
node server/index.js
```

## 📁 日志文件位置

### PM2日志文件

- **配置文件指定**：`/var/log/newsapp/`
  - `combined.log` - 合并日志
  - `out.log` - 标准输出
  - `error.log` - 错误日志

- **PM2默认位置**：`~/.pm2/logs/`
  - `newsapp-out.log` - 标准输出
  - `newsapp-error.log` - 错误日志

### 直接运行node

日志输出到运行node的终端，不会写入文件。

## 🔍 常用命令

### 查看日志文件

```bash
# 实时查看
tail -f /var/log/newsapp/combined.log

# 查看最后100行
tail -n 100 /var/log/newsapp/combined.log

# 搜索包含ERROR的日志
grep ERROR /var/log/newsapp/combined.log

# 实时搜索
tail -f /var/log/newsapp/combined.log | grep ERROR
```

### PM2日志命令

```bash
# 实时查看
pm2 logs newsapp

# 只查看错误
pm2 logs newsapp --err

# 只查看输出
pm2 logs newsapp --out

# 查看最后N行
pm2 logs newsapp --lines 100

# 清空日志
pm2 flush newsapp

# 查看所有PM2进程日志
pm2 logs
```

## 🆘 快速故障排查

### 应用无法启动？

```bash
# 1. 查看PM2日志
pm2 logs newsapp --err

# 2. 检查应用状态
pm2 list

# 3. 查看系统日志
journalctl -u newsapp -n 50
```

### 看不到日志？

```bash
# 1. 检查应用是否运行
pm2 list
ps aux | grep node

# 2. 检查日志文件是否存在
ls -lh /var/log/newsapp/

# 3. 检查权限
ls -la /var/log/newsapp/
```

### 日志文件太大？

```bash
# 清空PM2日志
pm2 flush newsapp

# 或手动清理
> /var/log/newsapp/combined.log
```
