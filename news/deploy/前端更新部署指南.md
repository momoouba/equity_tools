# 前端更新部署指南（生产环境）

本文档说明如何在Ubuntu Docker生产环境中只更新前端文件，完全替换旧文件（不保留）。

## 部署方案概述

本方案使用以下步骤：
1. 在本地或服务器上构建前端
2. 找到Docker volume的挂载位置
3. **完全清空旧的前端文件**
4. 复制新的前端文件
5. 重启nginx服务

## 方法一：使用部署脚本（推荐）

### 步骤 1: 上传代码到服务器

```bash
# 在本地，确保代码已提交并推送到Git
git add .
git commit -m "更新前端：优化Arco Design UI"
git push

# 在服务器上，拉取最新代码
cd /path/to/your/project
git pull origin main  # 或你的主分支名称
```

### 步骤 2: 执行部署脚本

```bash
# 进入项目根目录
cd /path/to/your/project/news

# 给脚本添加执行权限（如果还没有）
chmod +x deploy/update-frontend-production.sh

# 执行部署脚本
./deploy/update-frontend-production.sh
```

脚本会自动执行：
- ✅ 检查项目结构
- ✅ 安装依赖（如果需要）
- ✅ 构建前端（`npm run build`）
- ✅ 找到Docker volume
- ✅ **完全清空旧的前端文件**
- ✅ 复制新的前端文件
- ✅ 重启nginx服务
- ✅ 验证更新

### 步骤 3: 验证更新

```bash
# 查看nginx日志
sudo docker logs newsapp-nginx

# 查看应用日志
sudo docker logs newsapp

# 检查容器状态
sudo docker ps | grep newsapp
```

### 步骤 4: 清除浏览器缓存

在浏览器中：
1. 按 `Ctrl+Shift+Delete`（Windows/Linux）或 `Cmd+Shift+Delete`（Mac）
2. 选择"缓存"或"Cached images and files"
3. 清除缓存
4. 刷新页面（`Ctrl+F5` 强制刷新）

## 方法二：手动更新（如果脚本不可用）

### 步骤 1: 构建前端

```bash
cd /path/to/your/project/news/client
npm install  # 如果需要安装依赖
npm run build
```

### 步骤 2: 查找Docker Volume

```bash
# 查看所有volumes
sudo docker volume ls | grep frontend

# 查看volume详细信息
sudo docker volume inspect newsapp_app_frontend
```

### 步骤 3: 清空并复制文件

```bash
# 方式1: 直接操作volume路径（如果可以访问）
VOLUME_PATH=$(sudo docker volume inspect newsapp_app_frontend | grep -i "Mountpoint" | awk '{print $2}' | tr -d '",')
sudo rm -rf "$VOLUME_PATH"/*
sudo cp -r /path/to/your/project/news/client/dist/* "$VOLUME_PATH/"

# 方式2: 使用临时容器（如果无法直接访问volume路径）
sudo docker run --rm -v newsapp_app_frontend:/target -v /path/to/your/project/news/client/dist:/source alpine sh -c "rm -rf /target/* && cp -r /source/* /target/"
```

### 步骤 4: 重启服务

```bash
sudo docker compose restart nginx
# 或
sudo docker restart newsapp-nginx
```

## 方法三：完全重建容器（如果以上方法失败）

如果volume更新有问题，可以完全重建容器：

```bash
cd /path/to/your/project/news

# 停止容器
sudo docker compose down

# 重新构建并启动（只构建前端部分）
sudo docker compose build app
sudo docker compose up -d

# 查看日志
sudo docker compose logs -f nginx
```

## 验证更新是否成功

### 1. 检查文件是否正确更新

```bash
# 进入nginx容器
sudo docker exec -it newsapp-nginx sh

# 查看文件
ls -la /usr/share/nginx/html/
cat /usr/share/nginx/html/index.html | head -20

# 退出容器
exit
```

### 2. 检查浏览器控制台

打开浏览器开发者工具（F12），检查：
- 没有404错误
- 没有导入错误
- 资源加载正常

### 3. 检查功能

测试关键功能：
- ✅ 登录页面
- ✅ 仪表板
- ✅ 主要功能页面
- ✅ 按钮交互（hover、click）
- ✅ 表格显示
- ✅ 表单验证

## 常见问题

### Q1: 更新后页面仍是旧版本

**解决方案：**
1. 清除浏览器缓存（强制刷新 `Ctrl+F5`）
2. 检查nginx是否已重启：`sudo docker ps | grep nginx`
3. 检查volume是否正确更新：`sudo docker exec newsapp-nginx ls -la /usr/share/nginx/html/`

### Q2: 脚本执行失败

**解决方案：**
1. 检查脚本权限：`chmod +x deploy/update-frontend-production.sh`
2. 检查是否在项目根目录执行
3. 查看错误信息，手动执行对应步骤

### Q3: 找不到volume

**解决方案：**
1. 检查docker-compose.yml中的volume名称
2. 使用 `sudo docker volume ls` 查看所有volumes
3. 使用 `sudo docker compose ps` 查看运行的容器

### Q4: 权限问题

**解决方案：**
```bash
# 使用sudo执行脚本
sudo ./deploy/update-frontend-production.sh

# 或添加当前用户到docker组
sudo usermod -aG docker $USER
# 重新登录后生效
```

## 回滚方案

如果需要回滚到旧版本：

```bash
# 1. 切换到旧版本代码
cd /path/to/your/project/news
git checkout <旧版本commit-hash>

# 2. 重新构建并部署
./deploy/update-frontend-production.sh

# 或使用Git分支
git checkout <旧分支>
./deploy/update-frontend-production.sh
```

## 注意事项

1. **备份重要数据**：虽然只更新前端，但建议在执行前备份
2. **测试环境验证**：如果有测试环境，先在测试环境验证
3. **浏览器缓存**：更新后务必清除浏览器缓存
4. **服务不中断**：脚本只重启nginx，不会停止应用服务
5. **文件权限**：确保文件权限正确（通常755）
