# 检查并重启应用

## ❌ 问题

登录系统显示"服务器错误"，可能是：
1. 数据库连接失败（密码已更改，但应用配置未更新）
2. 应用程序未运行
3. 应用程序需要重启以加载新配置

## ✅ 解决步骤

### 步骤1：检查应用程序状态

```bash
# 检查 PM2 进程
pm2 status

# 或者检查 Node.js 进程
ps aux | grep node
```

### 步骤2：检查应用程序日志

```bash
# 查看 PM2 日志
pm2 logs newsapp --lines 50

# 或者查看应用日志文件
tail -50 /var/log/newsapp/error.log
tail -50 /var/log/newsapp/combined.log
```

### 步骤3：检查数据库配置

```bash
# 查看 .env 文件（在应用目录）
cd /opt/newsapp/news
cat .env | grep DB_
```

### 步骤4：更新数据库配置（如果需要）

如果 `.env` 文件中的密码还是旧的，需要更新：

```bash
# 编辑 .env 文件
nano /opt/newsapp/news/.env

# 确保配置如下：
# DB_HOST=localhost
# DB_PORT=3306
# DB_USER=root
# DB_PASSWORD=Mqdqxygyqy!!!klklsys24678
# DB_NAME=investment_tools
```

### 步骤5：重启应用程序

```bash
# 如果使用 PM2
pm2 restart newsapp

# 或者
pm2 restart all

# 如果 PM2 未运行，使用 ecosystem.config.js 启动
cd /opt/newsapp/news
pm2 start ecosystem.config.js

# 或者直接启动
cd /opt/newsapp/news
pm2 start server/index.js --name newsapp
```

### 步骤6：验证应用状态

```bash
# 检查 PM2 状态
pm2 status

# 查看日志（实时）
pm2 logs newsapp --lines 20

# 检查应用是否正常启动
curl http://localhost:3001/api/health 2>/dev/null || echo "应用未响应"
```

## 🚀 一键检查和重启

```bash
# 1. 检查 PM2 状态
echo "=== PM2 状态 ==="
pm2 status

# 2. 检查数据库配置
echo "=== 数据库配置 ==="
cd /opt/newsapp/news
cat .env | grep DB_ || echo "未找到 .env 文件"

# 3. 查看最新日志
echo "=== 最新错误日志 ==="
pm2 logs newsapp --err --lines 20 --nostream

# 4. 重启应用
echo "=== 重启应用 ==="
pm2 restart newsapp || pm2 start ecosystem.config.js

# 5. 等待几秒后检查状态
sleep 3
pm2 status
pm2 logs newsapp --lines 10 --nostream
```

## 📋 如果应用未使用 PM2

### 检查直接运行的 Node.js 进程

```bash
# 查找 Node.js 进程
ps aux | grep "node.*server/index.js" | grep -v grep

# 如果找到进程，记录 PID 并重启
# 停止旧进程
pkill -f "node.*server/index.js"

# 启动新进程（使用 PM2 或直接运行）
cd /opt/newsapp/news
pm2 start server/index.js --name newsapp
```

## 🔍 常见问题排查

### 问题1：数据库连接失败

**错误信息**：`ER_ACCESS_DENIED_ERROR` 或 `数据库连接失败`

**解决方法**：
```bash
# 更新 .env 文件中的密码
cd /opt/newsapp/news
sed -i 's/DB_PASSWORD=.*/DB_PASSWORD=Mqdqxygyqy!!!klklsys24678/' .env

# 重启应用
pm2 restart newsapp
```

### 问题2：应用未运行

**解决方法**：
```bash
# 使用 PM2 启动
cd /opt/newsapp/news
pm2 start ecosystem.config.js

# 或者
pm2 start server/index.js --name newsapp

# 保存 PM2 配置
pm2 save
```

### 问题3：端口被占用

**解决方法**：
```bash
# 检查端口占用
netstat -tulpn | grep 3001

# 如果被占用，停止占用进程或修改应用端口
```

## ✅ 验证清单

重启后，确认：

- [ ] PM2 显示应用状态为 `online`
- [ ] 日志中没有数据库连接错误
- [ ] 可以访问应用（http://news.gf-dsai.com）
- [ ] 登录功能正常
