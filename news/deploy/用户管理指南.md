# 服务器用户管理指南

## 📋 概述

本指南介绍如何在服务器上创建和管理应用用户，避免使用root账号操作，提高系统安全性。

---

## 🚀 快速创建用户

### 方法1：使用脚本创建（推荐）

```bash
# 1. 给脚本执行权限
chmod +x deploy/create-user.sh

# 2. 使用root权限运行（创建用户 guofang）
sudo ./deploy/create-user.sh guofang

# 或创建其他用户
sudo ./deploy/create-user.sh 用户名
```

脚本会自动完成：
- ✅ 创建用户
- ✅ 配置sudo权限
- ✅ 设置应用目录权限
- ✅ 配置SSH密钥（可选）
- ✅ 安装Node.js和PM2（可选）
- ✅ 配置PM2开机自启（可选）

### 方法2：手动创建

```bash
# 1. 创建用户
sudo adduser guofang

# 2. 添加到sudo组
sudo usermod -aG sudo guofang

# 3. 设置应用目录权限
sudo chown -R guofang:guofang /opt/newsapp
sudo chown -R guofang:guofang /var/log/newsapp
```

## 🔐 SSH密钥配置

### 生成SSH密钥对（在本地Windows）

```powershell
# 1. 生成密钥对（如果还没有）
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

# 2. 查看公钥
cat ~/.ssh/id_rsa.pub
```

### 配置服务器SSH密钥

**方法1：使用ssh-copy-id（推荐）**

```bash
# 在本地Windows PowerShell中
ssh-copy-id guofang@服务器IP

# 或手动复制
type $env:USERPROFILE\.ssh\id_rsa.pub | ssh guofang@服务器IP "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```

**方法2：手动配置**

```bash
# 1. 在服务器上创建.ssh目录
sudo mkdir -p /home/guofang/.ssh
sudo chmod 700 /home/guofang/.ssh

# 2. 复制公钥内容到authorized_keys
sudo nano /home/guofang/.ssh/authorized_keys
# 粘贴公钥内容，保存退出

# 3. 设置权限
sudo chmod 600 /home/guofang/.ssh/authorized_keys
sudo chown -R guofang:guofang /home/guofang/.ssh
```

### 测试SSH密钥登录

```bash
# 在本地Windows PowerShell中
ssh guofang@服务器IP

# 如果配置成功，应该可以直接登录，无需密码
```

## 👤 用户权限配置

### sudo权限

用户已添加到sudo组，可以使用sudo执行需要root权限的操作：

```bash
# 使用sudo执行命令
sudo systemctl restart nginx
sudo systemctl restart mysql
sudo apt update
```

### 应用目录权限

应用目录已设置为用户所有：

```bash
# 查看权限
ls -la /opt/newsapp
ls -la /var/log/newsapp

# 应该显示所有者是 guofang
```

### 文件权限说明

- `/opt/newsapp` - 应用目录，用户guofang所有
- `/var/log/newsapp` - 日志目录，用户guofang所有
- `/home/guofang` - 用户主目录

## 🔧 切换到新用户

### 方法1：使用su命令

```bash
# 切换到用户（保留当前目录）
su guofang

# 切换到用户（切换到用户主目录）
su - guofang

# 切换到用户并执行命令
su - guofang -c "cd /opt/newsapp && pm2 status"
```

### 方法2：使用SSH登录

```bash
# 从本地Windows PowerShell
ssh guofang@服务器IP
```

### 方法3：使用sudo执行命令

```bash
# 以guofang用户身份执行命令
sudo -u guofang pm2 logs newsapp
sudo -u guofang cd /opt/newsapp
```

## 🔒 安全最佳实践

### 1. 禁用root SSH登录（推荐）

编辑SSH配置：

```bash
sudo nano /etc/ssh/sshd_config
```

修改以下配置：

```
PermitRootLogin no
PasswordAuthentication no  # 如果使用SSH密钥
PubkeyAuthentication yes
```

重启SSH服务：

```bash
sudo systemctl restart sshd
```

### 2. 配置防火墙

```bash
# 允许SSH（22端口）
sudo ufw allow 22/tcp

# 允许HTTP/HTTPS
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 启用防火墙
sudo ufw enable

# 查看状态
sudo ufw status
```

### 3. 定期更新系统

```bash
# 更新系统包
sudo apt update
sudo apt upgrade -y

# 更新安全补丁
sudo apt dist-upgrade -y
```

### 4. 配置SSH密钥后禁用密码登录

```bash
# 编辑SSH配置
sudo nano /etc/ssh/sshd_config

# 设置
PasswordAuthentication no
PubkeyAuthentication yes

# 重启SSH
sudo systemctl restart sshd
```

```bash
id guofang
```


```bash
# 删除用户但保留主目录
sudo userdel guofang

# 删除用户及主目录

```

## ⚠️ 注意事项

1. **不要使用root运行应用**：应用应该以普通用户身份运行
2. **文件权限**：确保应用目录和日志目录的所有者是应用用户
3. **PM2用户**：PM2进程应该以应用用户身份运行
4. **日志权限**：确保日志目录有写权限
5. **数据库权限**：确保数据库用户有足够权限

## 🔍 故障排查

### 问题1：无法使用sudo

**解决方法**：
```bash
# 检查用户是否在sudo组
groups guofang

# 如果不在，添加到sudo组
sudo usermod -aG sudo guofang

# 重新登录
```

### 问题2：无法访问应用目录

**解决方法**：
```bash
# 检查权限
ls -la /opt/newsapp

# 修改所有者
sudo chown -R guofang:guofang /opt/newsapp
```

### 问题3：PM2无法启动

**解决方法**：
```bash
# 确保以正确用户身份运行
su - guofang
pm2 start deploy/ecosystem.config.js

# 检查PM2配置
pm2 list
```

### 问题4：SSH密钥登录失败

**解决方法**：
```bash
# 检查权限
ls -la /home/guofang/.ssh

# 应该显示：
# drwx------ .ssh
# -rw------- authorized_keys

# 如果不对，修复权限
sudo chmod 700 /home/guofang/.ssh
sudo chmod 600 /home/guofang/.ssh/authorized_keys
sudo chown -R guofang:guofang /home/guofang/.ssh
```
