# 本地文件更新部署步骤（Docker 环境）

## 📋 更新流程

### 情况1：只更新了后端代码（server 目录）

由于 `docker-compose.yml` 中 `server` 目录是直接挂载的，**只需重启容器即可生效**：

```bash
# 1. 进入项目目录
cd /opt/newsapp/news

# 2. 确认文件已更新（检查修改时间）
stat server/db.js
stat server/routes/aiPromptConfig.js
# 或其他更新的文件

# 3. 重启应用容器
docker compose restart app

# 4. 等待几秒让服务启动
sleep 5

# 5. 查看日志确认启动成功
docker compose logs app --tail 50

# 6. 检查容器状态
docker compose ps
```

### 情况2：更新了前端代码（client 目录）

前端代码需要重新构建并更新到 Docker volume：

```bash
# 1. 进入项目目录
cd /opt/newsapp/news

# 2. 进入前端目录
cd client

# 3. 安装依赖（如果 package.json 有变化）
npm install

# 4. 重新构建前端
npm run build

# 5. 返回项目根目录
cd ..

# 6. 更新前端文件到 Docker volume
# 方法A：使用自动脚本（推荐）
chmod +x deploy/clear-cache-and-update.sh
./deploy/clear-cache-and-update.sh

# 方法B：手动更新（如果脚本不存在）
docker run --rm \
  -v newsapp_app_frontend:/target \
  -v $(pwd)/client/dist:/source \
  alpine sh -c "rm -rf /target/* && cp -r /source/* /target/"

# 7. 重启 nginx 容器（如果需要）
docker compose restart nginx
```

### 情况3：同时更新了前端和后端代码

```bash
# 1. 进入项目目录
cd /opt/newsapp/news

# 2. 更新前端（如果需要）
cd client
npm install
npm run build
cd ..

# 3. 更新前端文件到 Docker volume
chmod +x deploy/clear-cache-and-update.sh
./deploy/clear-cache-and-update.sh

# 4. 重启应用容器（后端代码会自动生效）
docker compose restart app

# 5. 等待服务启动
sleep 5

# 6. 查看日志
docker compose logs app --tail 50
```

## ✅ 验证更新是否生效

### 1. 检查容器状态

```bash
docker compose ps
# 所有容器应该显示为 "Up" 状态
```

### 2. 查看应用日志

```bash
# 查看最近日志
docker compose logs app --tail 50

# 持续查看日志
docker compose logs -f app

# 查看错误日志
docker compose logs app --tail 100 | grep -i error
```

### 3. 测试健康检查

```bash
# 测试 API 健康检查
curl http://localhost:3001/api/health

# 应该返回：
# {"status":"ok","message":"服务器运行正常","database":"connected"}
```

### 4. 检查文件修改时间

```bash
# 检查容器内的文件是否已更新
docker compose exec app ls -la /app/server/db.js
docker compose exec app stat /app/server/db.js
```

### 5. 检查前端文件

```bash
# 检查前端文件是否已更新到 volume
docker run --rm -v newsapp_app_frontend:/target alpine ls -la /target | head -10
```

## ⚠️ 注意事项

### 1. 数据库迁移

如果更新了 `server/db.js`，系统启动时会自动执行数据库迁移：

```bash
# 查看迁移日志
docker compose logs app | grep -i "迁移\|migration\|已添加\|已创建"
```

### 2. 提示词初始化

如果更新了 `server/utils/initPrompts.js`，系统启动时会自动初始化提示词：

```bash
# 查看初始化日志
docker compose logs app | grep -i "提示词\|prompt\|初始化"
```

### 3. 前端缓存

更新前端后，用户浏览器可能需要清除缓存：

- 按 `Ctrl+Shift+Delete` 清除缓存
- 或按 `Ctrl+F5` 强制刷新

### 4. 检查文件权限

确保上传的文件有正确的权限：

```bash
# 检查文件权限
ls -la server/db.js
ls -la deploy/update-from-git.sh

# 如果需要，添加执行权限
chmod +x deploy/*.sh
```

## 🐛 故障排查

### 问题1：容器重启失败

```bash
# 查看详细错误日志
docker compose logs app --tail 100

# 检查代码语法
docker compose exec app node -c /app/server/db.js

# 手动测试启动
docker compose exec app node /app/server/index.js
```

```bash
# 1. 确认前端已重新构建
ls -lht client/dist/ | head -5

# 2. 确认文件已复制到 volume
docker run --rm -v newsapp_app_frontend:/target alpine ls -la /target | head -10

# 3. 重启 nginx 容器
docker compose restart nginx

# 4. 清除浏览器缓存
```

```bash

docker compose exec mysql mysql -u newsapp -p investment_tools -e "SHOW TABLES;"
```
