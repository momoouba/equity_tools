## 🚀 Docker部署更新步骤

### 方法一：完整部署流程（推荐）

#### 步骤1：上传更新的文件到服务器

```bash
# 在本地Windows PowerShell执行
# 替换 your-server-ip 为实际服务器IP，root 为实际用户名

# 上传后端文件
scp E:\USER\SUREAL\Desktop\equity_news\news\server\routes\newsShare.js root@your-server-ip:/opt/newsapp/news/server/routes/

# 上传前端源文件
scp E:\USER\SUREAL\Desktop\equity_news\news\client\src\pages\NewsInfo.jsx root@your-server-ip:/opt/newsapp/news/client/src/pages/
scp E:\USER\SUREAL\Desktop\equity_news\news\client\src\pages\ShareNewsPage.css root@your-server-ip:/opt/newsapp/news/client/src/pages/
```

#### 步骤2：SSH登录服务器并执行部署

```bash
# SSH登录到服务器
ssh root@your-server-ip

# 进入项目目录
cd /opt/newsapp/news

# 步骤1：构建前端（因为前端代码有更改）
cd client
npm run build
cd ..

# 步骤2：更新Docker volume中的前端文件
# 查找volume名称
VOLUME_NAME=$(sudo docker compose config | grep "app_frontend:" | awk '{print $2}' | tr -d ':')

# 使用临时容器更新前端文件
TEMP_CONTAINER="temp-frontend-update-$(date +%s)"
sudo docker run -d --name "$TEMP_CONTAINER" -v "$VOLUME_NAME:/target" alpine sleep 3600

# 清空旧文件
sudo docker exec "$TEMP_CONTAINER" sh -c "rm -rf /target/* /target/.[!.]* /target/..?* 2>/dev/null || true"

# 复制新构建的前端文件
sudo docker cp client/dist/. "$TEMP_CONTAINER:/target/"

# 清理临时容器
sudo docker rm -f "$TEMP_CONTAINER"

# 步骤3：重启应用容器（后端代码会自动同步，因为server目录已挂载）
sudo docker compose restart app

# 步骤4：查看日志确认启动成功
sudo docker compose logs app --tail 50
```

## ⚠️ 重要说明

### 1. 后端代码自动同步

由于 `docker-compose.yml` 中 `server` 目录已挂载为 volume：
```yaml
volumes:
  - ./server:/app/server
```

**后端代码更改会自动同步到容器**，只需重启容器即可，无需重新构建镜像。

### 2. 前端代码需要重新构建

前端代码在 `app_frontend` volume 中，需要：
1. 重新构建前端：`cd client && npm run build`
2. 更新 volume 中的文件（见上面的步骤）

### 3. 如果遇到问题

```bash
# 查看详细错误日志
sudo docker compose logs app --tail 200

# 检查容器内部文件是否正确
sudo docker exec newsapp ls -la /app/server/routes/newsShare.js
sudo docker exec newsapp cat /app/server/routes/newsShare.js | head -50

# 检查前端文件
sudo docker exec newsapp-nginx ls -la /usr/share/nginx/html/ | head -20

# 强制重启所有服务
sudo docker compose restart

# 如果问题严重，可以完全重建（谨慎使用）
sudo docker compose down
sudo docker compose up -d
```

### 4. 清除浏览器缓存

**非常重要**：部署后必须清除浏览器缓存，否则可能看到旧版本的代码。

- **Chrome/Edge**: Ctrl+Shift+Delete（Windows）或 Cmd+Shift+Delete（Mac）
- **Firefox**: Ctrl+Shift+Delete（Windows）或 Cmd+Shift+Delete（Mac）
- 或者使用强制刷新：Ctrl+F5（Windows）或 Cmd+Shift+R（Mac）
