# 快速更新前端命令（FinalShell）

## 问题说明

nginx容器的volume挂载是**只读**（read-only），所以不能直接复制文件到nginx容器。
需要通过**volume的实际路径**或**app容器**（可写挂载）来更新。

## 最简单的方法（推荐）

在 FinalShell 终端中执行以下命令：

### 方法1: 直接操作volume路径（最快）

```bash
cd /opt/newsapp/news

# 1. 查找volume路径
VOLUME_PATH=$(sudo docker volume inspect newsapp_app_frontend 2>/dev/null | grep -i "Mountpoint" | awk '{print $2}' | tr -d '",')

# 如果上面找不到，尝试其他名称
if [ -z "$VOLUME_PATH" ]; then
    VOLUME_PATH=$(sudo docker volume inspect "$(basename $(pwd))_app_frontend" 2>/dev/null | grep -i "Mountpoint" | awk '{print $2}' | tr -d '",')
fi

# 如果还是找不到，列出所有volumes
if [ -z "$VOLUME_PATH" ]; then
    echo "查找所有frontend相关的volumes:"
    sudo docker volume ls | grep frontend
    echo ""
    echo "请手动执行: sudo docker volume inspect <volume名称> | grep Mountpoint"
    exit 1
fi

echo "找到volume路径: $VOLUME_PATH"

# 2. 清空旧文件并复制新文件
sudo rm -rf "$VOLUME_PATH"/*
sudo cp -r client/dist/* "$VOLUME_PATH/"

# 3. 设置权限
sudo chown -R root:root "$VOLUME_PATH"
sudo chmod -R 755 "$VOLUME_PATH"

echo "✓ 更新完成！"
```

### 方法2: 使用app容器（如果找不到volume路径）

```bash
cd /opt/newsapp/news

# 1. 找到app容器
APP_CONTAINER=$(sudo docker ps --format "{{.Names}}" | grep -E "newsapp$|^app$" | head -1)
echo "找到app容器: $APP_CONTAINER"

# 2. 清空并复制文件（app容器的挂载是可写的）
sudo docker exec "$APP_CONTAINER" sh -c "rm -rf /app/client/dist/*"
sudo docker cp client/dist/. "$APP_CONTAINER:/app/client/dist/"

echo "✓ 更新完成！"
```

## 一键执行命令（复制粘贴即可）

```bash
cd /opt/newsapp/news && \
VOLUME_PATH=$(sudo docker volume inspect newsapp_app_frontend 2>/dev/null | grep -i "Mountpoint" | awk '{print $2}' | tr -d '",') && \
if [ -z "$VOLUME_PATH" ]; then VOLUME_PATH=$(sudo docker volume inspect "$(basename $(pwd))_app_frontend" 2>/dev/null | grep -i "Mountpoint" | awk '{print $2}' | tr -d '",'); fi && \
if [ -n "$VOLUME_PATH" ] && [ -d "$VOLUME_PATH" ]; then \
    echo "找到volume路径: $VOLUME_PATH" && \
    sudo rm -rf "$VOLUME_PATH"/* && \
    sudo cp -r client/dist/* "$VOLUME_PATH/" && \
    sudo chown -R root:root "$VOLUME_PATH" && \
    sudo chmod -R 755 "$VOLUME_PATH" && \
    echo "✓ 更新完成！"; \
else \
    echo "使用app容器更新..." && \
    APP_CONTAINER=$(sudo docker ps --format "{{.Names}}" | grep -E "newsapp$|^app$" | head -1) && \
    sudo docker exec "$APP_CONTAINER" sh -c "rm -rf /app/client/dist/*" && \
    sudo docker cp client/dist/. "$APP_CONTAINER:/app/client/dist/" && \
    echo "✓ 更新完成！"; \
fi
```

## 分步骤手动执行（如果上面命令有问题）

```bash
# 步骤1: 查找volume名称
sudo docker volume ls | grep frontend

# 步骤2: 查看volume详细信息（替换VOLUME_NAME为实际名称）
sudo docker volume inspect newsapp_app_frontend | grep Mountpoint

# 步骤3: 记录下路径，例如: /var/lib/docker/volumes/newsapp_app_frontend/_data

# 步骤4: 清空并复制（替换VOLUME_PATH为实际路径）
VOLUME_PATH="/var/lib/docker/volumes/newsapp_app_frontend/_data"
sudo rm -rf "$VOLUME_PATH"/*
sudo cp -r client/dist/* "$VOLUME_PATH/"

# 步骤5: 验证
sudo ls -la "$VOLUME_PATH" | head -10
```

## 验证更新

```bash
# 检查文件
VOLUME_PATH=$(sudo docker volume inspect newsapp_app_frontend | grep -i "Mountpoint" | awk '{print $2}' | tr -d '",')
sudo ls -la "$VOLUME_PATH/index.html"

# 或通过nginx容器检查（只读，只能查看）
sudo docker exec newsapp-nginx ls -la /usr/share/nginx/html/ | head -10
```

## 后续操作

1. **清除浏览器缓存**（`Ctrl+Shift+Delete`）
2. **强制刷新页面**（`Ctrl+F5`）
3. **检查浏览器控制台**（F12）是否有错误

通常**不需要重启nginx容器**，因为volume是共享的，文件更新后nginx会自动读取。

如果需要重启nginx（通常不需要）：
```bash
sudo docker restart newsapp-nginx
```
