# 503错误快速排查指南

## 🔍 问题现象

访问网站时出现 **HTTP 503 Service Unavailable** 错误，表示服务器无法处理请求。

## ⚡ 快速修复步骤

### 方法1：使用自动修复脚本（推荐）

在服务器上执行：

```bash
# 进入项目目录
cd /opt/newsapp/news

# 给脚本执行权限
chmod +x deploy/fix-503-error.sh

# 运行修复脚本
./deploy/fix-503-error.sh
```

脚本会自动：
1. 检测部署方式（Docker或PM2）
2. 诊断问题
3. 自动修复常见问题

---

### 方法2：手动排查和修复

#### 步骤1：确定部署方式

**检查是否是Docker部署：**
```bash
cd /opt/newsapp/news
sudo docker compose ps
```

**检查是否是PM2部署：**
```bash
pm2 status
```

---

#### 步骤2：根据部署方式修复

### 🔹 如果是 Docker 部署

**1. 检查容器状态**
```bash
cd /opt/newsapp/news
sudo docker compose ps
```

**2. 查看容器日志**
```bash
# 查看应用容器日志
sudo docker compose logs app --tail 50

# 查看Nginx容器日志
sudo docker compose logs nginx --tail 50

# 查看MySQL容器日志
sudo docker compose logs mysql --tail 50
```

**3. 重启服务**
```bash
# 重启所有容器
sudo docker compose restart

# 或者分别重启
sudo docker compose restart app
sudo docker compose restart nginx
```

**4. 如果容器未启动，重新创建**
```bash
# 重新创建并启动容器
sudo docker compose up -d

# 查看状态
sudo docker compose ps
```

**5. 检查端口监听**
```bash
# 检查端口80和3001
sudo netstat -tulpn | grep -E ":(80|3001) "
# 或
sudo ss -tulpn | grep -E ":(80|3001) "
```

---

### 🔹 如果是 PM2 部署

**1. 检查PM2进程状态**
```bash
pm2 status
```

**2. 查看应用日志**
```bash
# 查看最近50行日志
pm2 logs newsapp --lines 50

# 查看错误日志
pm2 logs newsapp --err --lines 50
```

**3. 重启应用**
```bash
# 重启PM2应用
pm2 restart newsapp

# 如果应用未运行，启动它
pm2 start deploy/ecosystem.config.js
# 或
pm2 start server/index.js --name newsapp

# 保存PM2配置
pm2 save
```

**4. 检查端口监听**
```bash
# 检查端口3001
sudo netstat -tulpn | grep 3001
# 或
sudo ss -tulpn | grep 3001
```

**5. 检查Nginx状态**
```bash
# 检查Nginx服务状态
sudo systemctl status nginx

# 重启Nginx
sudo systemctl restart nginx

# 测试Nginx配置
sudo nginx -t
```

## 🔍 常见原因和解决方法

### 1. 应用未启动

**症状：** 容器或进程不存在或状态为停止

**解决方法：**
- Docker: `sudo docker compose up -d`
- PM2: `pm2 start deploy/ecosystem.config.js`

---

### 2. 应用崩溃

**症状：** 日志中有错误信息，应用反复重启

**解决方法：**
```bash
# 查看错误日志
# Docker:
sudo docker compose logs app --tail 100

# PM2:
pm2 logs newsapp --err --lines 100
```

**常见错误：**
- **数据库连接失败**：检查 `.env` 文件中的数据库配置
- **端口被占用**：检查端口占用情况
- **依赖缺失**：运行 `npm install`

---

### 3. Nginx配置错误

**症状：** Nginx无法启动或无法代理到后端

**解决方法：**
```bash
# 测试Nginx配置
sudo nginx -t

# 查看Nginx错误日志
sudo tail -f /var/log/nginx/error.log

# 检查Nginx配置
cat /etc/nginx/sites-available/newsapp
# 或
cat /opt/newsapp/news/deploy/nginx-site.conf
```

---

### 4. 数据库连接失败

**症状：** 日志中显示数据库连接错误

**解决方法：**
```bash
# 检查数据库配置
cd /opt/newsapp/news
cat .env | grep DB_

# 测试数据库连接
mysql -h localhost -u newsapp -p investment_tools

# Docker部署时，检查数据库容器
sudo docker compose logs mysql --tail 50
```

---

### 5. 端口冲突

**症状：** 端口被其他进程占用

**解决方法：**
```bash
# 查找占用端口的进程
sudo lsof -i :3001
# 或
sudo netstat -tulpn | grep 3001

# 停止占用端口的进程
sudo kill <进程ID>
```

## 📋 完整排查清单

按顺序执行以下检查：

```bash
# 1. 检查部署方式
cd /opt/newsapp/news
sudo docker compose ps 2>/dev/null || pm2 status

# 2. 检查服务状态
# Docker:
sudo docker compose ps
# PM2:
pm2 status

# 3. 检查端口监听
sudo netstat -tulpn | grep -E ":(80|3001) "

# 4. 查看日志
# Docker:
sudo docker compose logs app --tail 50
# PM2:
pm2 logs newsapp --lines 50

# 5. 测试应用健康检查
curl http://localhost:3001/api/health

# 6. 检查Nginx状态
sudo systemctl status nginx

# 7. 检查数据库连接
# Docker:
sudo docker compose exec mysql mysql -u newsapp -p investment_tools -e "SELECT 1;"
# PM2:
mysql -h localhost -u newsapp -p investment_tools -e "SELECT 1;"
```
