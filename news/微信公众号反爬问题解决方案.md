# 微信公众号反爬问题解决方案

## 问题确认

从日志和HTML内容分析确认：**抓取到的是微信公众号的反爬验证页面，而不是实际的文章内容**。

### 问题表现

1. **HTML内容**：包含"环境异常"、"完成验证"等反爬关键词
2. **缺少文章标识**：没有找到 `js_content`、`rich_media_content` 等文章内容标识
3. **缺少图片URL**：HTML中不包含 `mmbiz.qpic.cn` 图片URL
4. **URL重定向**：可能被重定向到验证页面

## 解决方案

### 方案1：使用Playwright无头浏览器（已实现）

**优点**：
- 可以执行JavaScript，更接近真实浏览器
- 能够绕过部分反爬检测
- 可以等待页面完全加载

**缺点**：
- 需要安装Playwright和浏览器（占用空间较大）
- 执行速度较慢
- 资源消耗较大

**实现状态**：✅ 已添加代码支持

**使用方法**：
1. 安装Playwright：
```bash
# 在Docker容器中
pip install playwright
playwright install chromium
```

2. 系统会自动检测反爬页面，并尝试使用Playwright

### 方案2：降级处理（已实现）

当检测到反爬页面时：
1. 返回明确的错误信息
2. 标记为需要人工处理
3. 在日志中记录详细信息

## 部署步骤

### 步骤1：安装Playwright（推荐）

```bash
# 进入Docker容器
sudo docker compose exec app sh

# 安装Playwright
pip install playwright --break-system-packages

# 安装Chromium浏览器
playwright install chromium

# 退出容器
exit
```

### 步骤2：更新Dockerfile（可选，永久安装）

如果需要永久安装Playwright，可以修改 `Dockerfile`：

```dockerfile
# 在安装Python依赖的部分添加
RUN pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple \
    && pip config set global.timeout 300 \
    && pip config set global.retries 5 \
    && pip install --break-system-packages --no-cache-dir -r ./server/utils/requirements.txt \
    && playwright install chromium
```

### 步骤3：重启容器

```bash
cd /opt/newsapp/news
sudo docker compose restart app
```

## 工作流程

1. **首先尝试HTTP请求**：使用requests库直接请求
2. **检测反爬页面**：检查是否包含反爬关键词
3. **自动切换到Playwright**：如果检测到反爬页面，自动使用Playwright无头浏览器
4. **验证内容有效性**：检查是否包含文章内容标识
5. **返回结果**：返回有效的HTML内容或错误信息

## 注意事项

### 1. Playwright安装

- Playwright需要下载Chromium浏览器（约300MB）
- 首次安装需要较长时间
- 确保Docker容器有足够的磁盘空间

### 2. 性能影响

- Playwright比HTTP请求慢（需要启动浏览器）
- 建议只在检测到反爬页面时使用
- 可以设置超时时间避免长时间等待

### 3. 资源消耗

- Playwright会消耗更多内存和CPU
- 建议限制并发数量
- 考虑使用浏览器池复用

## 故障排查

### 问题1：Playwright未安装

**错误信息**：
```
[Playwright] ⚠️ Playwright未安装，无法使用无头浏览器
```

**解决方案**：
```bash
sudo docker compose exec app pip install playwright --break-system-packages
sudo docker compose exec app playwright install chromium
```

### 问题2：Playwright安装失败

**可能原因**：
- 网络问题
- 磁盘空间不足
- 权限问题

**解决方案**：
```bash
# 检查磁盘空间
df -h

# 检查网络连接
ping pypi.org

# 使用国内镜像源
pip install playwright -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 问题3：Playwright仍然无法获取内容

**可能原因**：
- 微信公众号的反爬机制升级
- 需要更长的等待时间
- 需要更真实的浏览器环境

**解决方案**：
- 增加等待时间
- 使用更完整的浏览器配置
- 考虑使用代理服务
