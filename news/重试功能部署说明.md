# 重试功能部署说明

## 📋 概述

本次更新为"新闻接口同步"定时任务添加了重试功能。当接口调用后未返回任何数据时，系统会根据配置的"重新抓取次数"和"重新抓取间隔"自动进行重试。

## 🔧 数据库迁移

系统会在服务器启动时自动执行数据库迁移，添加以下两个字段到 `news_interface_config` 表：

- `retry_count`：重新抓取次数（INT，默认 0）
- `retry_interval`：重新抓取间隔（INT，单位：分钟，默认 0）

## 🚀 服务器端执行步骤

### 方式一：使用 PM2 部署（推荐）

如果您的服务器使用 PM2 管理 Node.js 进程：

```bash
# 1. 进入项目目录
cd /opt/newsapp/news  # 或您的实际项目路径

# 2. 重启 PM2 进程（会自动执行数据库迁移）
pm2 restart newsapp

# 或者使用 reload（零停机重启）
pm2 reload newsapp

# 3. 查看日志确认迁移成功
pm2 logs newsapp --lines 50
```

**预期日志输出：**
```
✓ 数据库连接已就绪
正在初始化数据库...
✓ 服务器运行在 http://localhost:3001
```

### 方式二：使用 Docker 部署

如果您的服务器使用 Docker 部署：

```bash
# 1. 进入项目目录
cd /path/to/equity_news/news

# 2. 重启应用容器（会自动执行数据库迁移）
docker-compose restart app

# 或者重新构建并启动（如果代码有更新）
docker-compose up -d --build app

# 3. 查看容器日志确认迁移成功
docker-compose logs app --tail 50
```

**预期日志输出：**
```
✓ 数据库连接已就绪
正在初始化数据库...
✓ 服务器运行在 http://localhost:3001
```

### 方式三：手动执行 SQL（可选）

如果自动迁移失败，可以手动执行以下 SQL：

```sql
-- 检查字段是否已存在
SELECT COLUMN_NAME 
FROM INFORMATION_SCHEMA.COLUMNS 
WHERE TABLE_SCHEMA = DATABASE() 
AND TABLE_NAME = 'news_interface_config' 
AND COLUMN_NAME IN ('retry_count', 'retry_interval');

-- 如果字段不存在，执行以下 SQL 添加字段
ALTER TABLE news_interface_config 
ADD COLUMN retry_count INT DEFAULT 0 COMMENT '未获取数据时的重新抓取次数，0表示不重试';

ALTER TABLE news_interface_config 
ADD COLUMN retry_interval INT DEFAULT 0 COMMENT '重新抓取间隔（单位：分钟）';
```

## ✅ 验证部署

### 1. 检查数据库字段

连接到 MySQL 数据库，执行：

```sql
DESCRIBE news_interface_config;
```

应该能看到 `retry_count` 和 `retry_interval` 两个字段。

### 2. 检查前端功能

1. 登录系统，进入"定时任务管理"
2. 切换到"新闻接口同步"标签页
3. 点击"编辑"或"新增"按钮
4. 确认表单中出现了"重新抓取次数"和"重新抓取间隔（分钟）"两个输入框

### 3. 测试重试功能

1. 编辑一个"新闻接口同步"任务
2. 设置"重新抓取次数"为 1，"重新抓取间隔"为 5（分钟）
3. 保存配置
4. 等待定时任务执行（或手动点击"立即执行"）
5. 如果未获取到数据，查看服务器日志，应该能看到重试相关的日志：

```
[新闻同步] 配置 xxx 未获取到数据，将在 5 分钟后进行第 1 次重试（共 1 次）
[新闻同步] 配置 xxx 开始第 1 次重试
```

## 📝 注意事项

1. **数据库迁移是自动的**：服务器启动时会自动检查并添加字段，无需手动干预
2. **字段默认值**：新字段的默认值为 0，表示不启用重试功能
3. **重试仅适用于定时任务**：手动触发的同步不会进行重试
4. **重试使用相同时间范围**：重试时会使用与原始调用相同的时间范围
5. **配置更改会取消重试**：如果在重试进行中修改了配置（删除、禁用或更改重试参数），已安排的重试会被取消

## 🐛 故障排除

### 问题：重启后字段未添加

**解决方案：**
1. 检查服务器日志，查看是否有数据库错误
2. 确认数据库连接正常
3. 手动执行 SQL 添加字段（见方式三）

### 问题：前端看不到新字段

**解决方案：**
1. 清除浏览器缓存
2. 重新构建前端：`npm run build`（在 client 目录下）
3. 如果使用 Docker，重新构建前端镜像

### 问题：重试功能不工作

**解决方案：**
1. 确认 `retry_count > 0` 且 `retry_interval > 0`
2. 确认任务状态为"启用"
3. 查看服务器日志，确认是否有错误信息
4. 确认是定时任务触发（手动触发不会重试）
