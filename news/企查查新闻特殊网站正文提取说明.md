# 企查查新闻特殊网站正文提取说明

## 更新时间
2026-01-06

## 概述

企查查接口返回的新闻数据中，部分网站的 `Content` 字段可能为空。系统会自动从新闻的 `source_url` 抓取网页内容并提取正文。针对不同网站的结构特点，系统实现了针对性的提取逻辑。

## 支持的特殊网站

### 1. 新浪新闻（sina.com.cn）

**提取位置**：`news/server/utils/newsAnalysis.js:136-229`

**提取规则**：
- 查找 `<div id="artibody">` 标签
- 使用智能匹配算法处理嵌套的 div 标签
- 跳过 script 标签内的 div，避免误匹配

**示例HTML结构**：
```html
<div id="artibody">
  <p>正文内容...</p>
  <p>更多正文内容...</p>
</div>
```

**内容长度要求**：提取的文本长度必须大于 100 字符

### 2. 每经网（nbd.com.cn）

**提取位置**：`news/server/utils/newsAnalysis.js:231-323`

**提取规则**：
- 查找 `<div class="g-article">` 标签
- 使用智能匹配算法处理嵌套的 div 标签
- 跳过 script 标签内的 div，避免误匹配

**示例HTML结构**：
```html
<div class="g-article">
  <div class="g-article-top">
    <h1>新闻标题</h1>
    <p class="u-time">
      <span class="source">每日经济新闻</span>
      <span class="time">2026-01-05 16:05:04</span>
    </p>
  </div>
  <div class="g-article-abstract">
    <p>文章摘要...</p>
  </div>
  <div class="g-articl-text">
    <p>正文内容...</p>
    <p>更多正文内容...</p>
  </div>
</div>
```

**内容长度要求**：提取的文本长度必须大于 100 字符

**示例URL**：
- https://www.nbd.com.cn/articles/2026-01-05/4207081.html

### 3. 东方财富网（eastmoney.com）

**提取位置**：`news/server/utils/newsAnalysis.js:325-450`

**提取规则**：
- 优先查找 `<div id="ContentBody">` 标签
- 如果未找到，则查找 `<div class="txtinfos">` 标签
- 使用智能匹配算法处理嵌套的 div 标签
- 跳过 script 标签内的 div，避免误匹配

**内容清理**：
- 移除 script、style、noscript 等标签和注释
- 移除 `display:none` 的 p 标签
- 移除 `em_module_extend` div
- 移除 center 标签
- 移除广告链接（如包含"妙想"、"体验"等关键词的链接）

**示例HTML结构**：
```html
<div class="txtinfos" id="ContentBody" style="margin-top: 0px; position: relative;">
  <p>正文内容...</p>
  <p>更多正文内容...</p>
  <p class="em_media">（文章来源：财联社）</p>
</div>
```

**内容长度要求**：提取的文本长度必须大于 50 字符

**示例URL**：
- https://finance.eastmoney.com/a/202601053608506079.html

## 智能匹配算法

所有特殊网站的提取都使用相同的智能匹配算法：

1. **查找开始标签**：使用正则表达式匹配特定的 div 开始标签
2. **处理嵌套**：使用深度计数算法处理嵌套的 div 标签
   - 遇到 `<div>` 时，深度 +1
   - 遇到 `</div>` 时，深度 -1
   - 当深度为 0 时，找到对应的结束标签
3. **跳过 script 标签**：检查 div 标签是否在 script 标签内，如果是则跳过
4. **提取内容**：提取开始标签和结束标签之间的所有内容

## 工作流程

当企查查接口返回的新闻 `Content` 字段为空时：

1. **第一步**：使用 `fetchContentFromUrl` 方法抓取网页 HTML 内容
2. **第二步**：调用 `extractArticleContent` 方法提取正文
   - 首先检查 URL 是否匹配特殊网站（新浪、每经网、东方财富网）
   - 如果匹配，使用对应的特殊提取逻辑
   - 如果未匹配或提取失败，继续使用通用提取逻辑
3. **第三步**：如果 `extractArticleContent` 提取失败或内容太短，使用 AI 提取作为备用方案

## 提取优先级

1. **最高优先级**：特殊网站的特殊提取逻辑（新浪、每经网、东方财富网）
2. **次高优先级**：企查查、格隆汇等网站的 `article class="main-news article-with-html"` 标签
3. **第三优先级**：通用的 `article` 标签匹配
4. **最低优先级**：其他常见的正文容器匹配规则

## 验证方法

### 1. 测试特定网站的新闻提取

```sql
-- 查询最近同步的企查查新闻，检查是否有正文内容
SELECT id, title, 
       LENGTH(content) as content_length,
       source_url,
       APItype,
       created_at
FROM news_detail
WHERE APItype = '企查查'
  AND source_url LIKE '%nbd.com.cn%'  -- 每经网
  AND created_at >= DATE('2026-01-06')
  AND content IS NOT NULL 
  AND content != ''
  AND LENGTH(content) > 50
  AND delete_mark = 0
ORDER BY created_at DESC
LIMIT 10;
```

### 2. 查看提取日志

在应用日志中查找以下信息：
- `[extractArticleContent] 检测到XXXURL，使用特殊提取逻辑`
- `[extractArticleContent] ✓ 成功提取XXX正文，长度: XXX字符`
- `[企查查同步] ✓ 使用extractArticleContent方法成功提取正文，长度: XXX 字符`

### 3. 手动测试提取

可以创建一个测试脚本，手动测试特定URL的正文提取：

```javascript
const newsAnalysis = require('./server/utils/newsAnalysis');
const url = 'https://www.nbd.com.cn/articles/2026-01-05/4207081.html'; // 每经网新闻链接

newsAnalysis.fetchContentFromUrl(url).then(content => {
  console.log('提取的正文长度:', content.length);
  console.log('正文预览:', content.substring(0, 200));
});
```

## 注意事项

1. **匹配优先级**：特殊网站的提取逻辑被设置为最高优先级，确保优先匹配这些网站的特殊结构
2. **内容长度检查**：提取的内容必须满足最小长度要求才会被使用，避免提取到导航栏等非正文内容
3. **备用方案**：如果特殊提取逻辑失败，系统会自动使用通用提取逻辑，如果仍然失败，会使用 AI 提取作为备用方案
4. **兼容性**：新的提取规则不会影响其他网站的正文提取，因为它们是按优先级顺序匹配的
5. **内容清理**：东方财富网的提取会特别清理广告链接和隐藏元素，确保提取到纯净的正文内容

## 添加新网站支持

如果需要添加新的特殊网站支持，可以按照以下步骤：

1. 在 `extractArticleContent` 方法中添加新的 URL 检测逻辑
2. 创建对应的智能匹配函数（参考 `findNbdArticleDiv` 或 `findEastmoneyArticleDiv`）
3. 设置合适的内容长度要求
4. 添加必要的内容清理逻辑
5. 更新本文档
