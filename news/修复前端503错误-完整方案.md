# 修复前端 503 错误 - 完整方案

## 🔍 问题诊断

访问 `/login` 页面返回 503，说明 Nginx 无法提供前端静态文件。原因是 `app_frontend` volume 可能为空。

## 🚀 立即执行以下命令

### 方案1：快速修复（推荐）

```bash
cd /opt/newsapp/news

# 1. 检查 volume 中是否有文件
VOLUME_PATH=$(sudo docker volume inspect news_app_frontend | grep -i "Mountpoint" | awk '{print $2}' | tr -d '",')
echo "Volume 路径: $VOLUME_PATH"
sudo ls -la "$VOLUME_PATH" | head -10

# 2. 如果 volume 为空，从容器复制文件
if [ -z "$(sudo ls -A $VOLUME_PATH 2>/dev/null)" ]; then
    echo "Volume 为空，正在复制文件..."
    sudo docker cp newsapp:/app/client/dist/. "$VOLUME_PATH/"
    echo "✓ 文件已复制"
else
    echo "Volume 中已有文件"
fi

# 3. 重启 Nginx
sudo docker compose restart nginx

# 4. 等待 5 秒
sleep 5

# 5. 测试
curl -v http://localhost/ | head -20
```

### 方案2：重新构建并初始化（最彻底）

```bash
cd /opt/newsapp/news

# 1. 停止服务
sudo docker compose down

# 2. 删除旧的 volume（可选，确保使用新文件）
sudo docker volume rm news_app_frontend 2>/dev/null || true

# 3. 重新构建前端（如果需要）
cd client
npm run build
cd ..

# 4. 重新构建镜像
sudo docker compose build app

# 5. 启动服务
sudo docker compose up -d

# 6. 等待容器启动（60秒）
sleep 60

# 7. 初始化 volume（如果为空）
VOLUME_PATH=$(sudo docker volume inspect news_app_frontend | grep -i "Mountpoint" | awk '{print $2}' | tr -d '",')
if [ -z "$(sudo ls -A $VOLUME_PATH 2>/dev/null)" ]; then
    echo "初始化 volume..."
    sudo docker cp newsapp:/app/client/dist/. "$VOLUME_PATH/"
    sudo docker compose restart nginx
fi

# 8. 测试
curl -v http://localhost/ | head -20
```

## ⚠️ 重要提示

- 如果 volume 为空，Nginx 无法提供文件，会返回 503
- 确保容器启动后，前端文件已复制到 volume
- 清除浏览器缓存很重要，避免缓存旧的错误响应
