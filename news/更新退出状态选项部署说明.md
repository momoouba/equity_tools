# 更新退出状态选项（不再观察）部署说明

## 更新内容

本次更新为退出状态下拉菜单添加"不再观察"选项，涉及以下文件：

1. **前端文件**：
   - `client/src/pages/EnterpriseForm.jsx` - 企业表单组件
   - `client/src/pages/BatchImportModal.jsx` - 批量导入模态框

2. **后端文件**：
   - `server/routes/enterprises.js` - 企业路由（模板说明）

3. **文档文件**（可选）：
   - `被投企业管理功能说明.md`
   - `外部数据库连接及数据同步功能说明.md`

## 部署步骤

### 方法一：使用自动部署脚本（推荐）

1. **上传更新的文件到服务器**

   将以下文件上传到服务器的项目目录（通常是 `/opt/newsapp/news`）：
   - `client/src/pages/EnterpriseForm.jsx`
   - `client/src/pages/BatchImportModal.jsx`
   - `server/routes/enterprises.js`
   - `deploy/update-exit-status-option.sh`（部署脚本）

2. **进入项目目录**

   ```bash
   cd /opt/newsapp/news
   ```

3. **给脚本添加执行权限**

   ```bash
   chmod +x deploy/update-exit-status-option.sh
   ```

4. **运行更新脚本**

   ```bash
   ./deploy/update-exit-status-option.sh
   ```

   脚本会自动：
   - 检查文件是否存在
   - 备份当前代码
   - 重新构建前端
   - 复制前端文件到Docker volume
   - 重启应用容器
   - 检查应用健康状态

5. **清除浏览器缓存并刷新页面**

   - 按 `Ctrl+F5` 强制刷新页面
   - 或使用无痕模式访问
   - 测试退出状态下拉菜单，确认"不再观察"选项显示正常

### 方法二：手动操作

如果自动脚本无法使用，可以手动执行以下步骤：

1. **上传文件到服务器**

   ```bash
   # 确保文件上传到正确位置
   # 项目目录通常是: /opt/newsapp/news
   ```

2. **进入项目目录**

   ```bash
   cd /opt/newsapp/news
   ```

3. **备份当前代码（可选但推荐）**

   ```bash
   mkdir -p backups/$(date +%Y%m%d_%H%M%S)
   cp client/src/pages/EnterpriseForm.jsx backups/$(date +%Y%m%d_%H%M%S)/
   cp client/src/pages/BatchImportModal.jsx backups/$(date +%Y%m%d_%H%M%S)/
   cp server/routes/enterprises.js backups/$(date +%Y%m%d_%H%M%S)/
   ```

4. **重新构建前端**

   ```bash
   cd client
   npm install  # 如果node_modules不存在
   npm run build
   cd ..
   ```

5. **复制前端文件到Docker volume**

   ```bash
   # 查找volume名称
   VOLUME_NAME=$(sudo docker compose config | grep -A 2 "app_frontend:" | grep "driver: local" -B 2 | head -1 | awk '{print $1}' | tr -d ':')
   
   # 使用临时容器复制文件
   TEMP_CONTAINER=$(sudo docker run -d --name temp-frontend-update -v ${VOLUME_NAME}:/target alpine sleep 3600)
   sudo docker cp client/dist/. ${TEMP_CONTAINER}:/target/
   sudo docker rm -f ${TEMP_CONTAINER}
   ```

6. **重启应用容器**

   ```bash
   sudo docker compose restart app
   ```

7. **等待应用启动（约30-60秒）**

   ```bash
   # 查看容器状态
   sudo docker compose ps app
   
   # 查看应用日志
   sudo docker compose logs app --tail 50 -f
   ```

8. **验证应用健康状态**

   ```bash
   curl http://localhost:3001/api/health
   # 应该返回: {"status":"ok"}
   ```

## 常见问题排查

### 1. 前端未更新

**问题**：浏览器中看不到"不再观察"选项

**解决方案**：
1. 清除浏览器缓存（Ctrl+Shift+Delete）
2. 强制刷新页面（Ctrl+F5）
3. 使用无痕模式访问
4. 检查前端文件是否正确复制到volume：
   ```bash
   # 查找volume路径
   VOLUME_PATH=$(sudo docker volume inspect news_app_frontend | grep Mountpoint | awk '{print $2}' | tr -d '",')
   # 检查文件
   sudo ls -la $VOLUME_PATH/
   grep -r "不再观察" $VOLUME_PATH/
   ```

### 2. 前端构建失败

**问题**：`npm run build` 失败

**解决方案**：
```bash
cd client
# 清理并重新安装依赖
rm -rf node_modules package-lock.json
npm install
npm run build
```

### 3. 容器无法启动

**问题**：应用容器启动失败

**解决方案**：
```bash
# 查看详细错误日志
sudo docker compose logs app --tail 100

# 检查文件权限
ls -la client/src/pages/EnterpriseForm.jsx
ls -la server/routes/enterprises.js

# 检查文件语法（如果有Node.js环境）
node -c server/routes/enterprises.js
```

### 4. 应用健康检查失败

**问题**：健康检查返回非200状态

**解决方案**：
```bash
# 查看应用日志
sudo docker compose logs app --tail 200

# 手动测试健康检查端点
curl -v http://localhost:3001/api/health

# 检查容器资源使用
sudo docker stats newsapp
```

### 5. 回滚到之前的版本

如果更新后出现问题，可以回滚：

```bash

ls -lt backups/

# 恢复文件
cp backups/YYYYMMDD_HHMMSS/client/src/pages/EnterpriseForm.jsx client/src/pages/EnterpriseForm.jsx
cp backups/YYYYMMDD_HHMMSS/client/src/pages/BatchImportModal.jsx client/src/pages/BatchImportModal.jsx
cp backups/YYYYMMDD_HHMMSS/server/routes/enterprises.js server/routes/enterprises.js

# 重新构建前端
cd client
npm run build
cd ..

# 复制前端文件到volume（参考步骤5）

# 重启应用
sudo docker compose restart app
```
