# 代码更新部署指南

## 概述

本文档说明如何在Linux服务器的Docker环境中更新代码并使其生效。

## 重要更新：Python支持

**本次更新需要安装Python 3.9+环境**，用于微信公众号文章内容提取功能。

### 新增Python相关文件
- `server/utils/wechatArticleExtractor.py` - Python脚本（微信公众号文章提取）
- `server/utils/requirements.txt` - Python依赖列表
- `Dockerfile` - 已更新，添加Python支持

### 修改的文件

本次更新涉及以下文件：
- `server/db.js` - 添加usage_type字段的数据库迁移
- `server/routes/aiConfig.js` - 支持usage_type字段的API
- `server/routes/news.js` - 新榜接口立即AI分析和微信公众号提取
- `server/utils/newsAnalysis.js` - 微信公众号文章提取和AI分析逻辑
- `Dockerfile` - 添加Python 3和依赖安装

## 部署步骤

### ⚠️ 重要：本次更新需要重新构建Docker镜像

由于添加了Python支持，**必须重新构建Docker镜像**，不能只重启容器。

### 方法一：使用Python部署脚本（推荐）

1. **上传所有更新的文件到服务器**
   ```bash
   # 使用scp或其他方式上传文件到服务器
   # 确保文件路径正确：
   # - /opt/newsapp/news/Dockerfile
   # - /opt/newsapp/news/server/db.js
   # - /opt/newsapp/news/server/routes/aiConfig.js
   # - /opt/newsapp/news/server/routes/news.js
   # - /opt/newsapp/news/server/utils/newsAnalysis.js
   # - /opt/newsapp/news/server/utils/wechatArticleExtractor.py
   # - /opt/newsapp/news/server/utils/requirements.txt
   # - /opt/newsapp/news/deploy/update-with-python.sh
   ```

2. **进入项目目录**
   ```bash
   cd /opt/newsapp/news
   ```

3. **给脚本添加执行权限**
   ```bash
   chmod +x deploy/update-with-python.sh
   ```

4. **运行Python部署脚本**
   ```bash
   ./deploy/update-with-python.sh
   ```

   脚本会自动：
   - 备份当前容器
   - 停止应用容器
   - 重新构建Docker镜像（包含Python支持）
   - 启动应用容器
   - 验证Python环境
   - 检查应用健康状态

### 方法二：手动操作

1. **上传所有更新的文件到服务器**
   ```bash
   # 确保文件上传到正确位置
   # 项目目录通常是: /opt/newsapp/news
   ```

2. **进入项目目录**
   ```bash
   cd /opt/newsapp/news
   ```

3. **备份当前容器（推荐）**
   ```bash
   docker commit newsapp newsapp-backup:$(date +%Y%m%d_%H%M%S)
   ```

4. **验证文件已更新**
   ```bash
   # 检查关键文件是否存在
   ls -lh Dockerfile
   ls -lh server/utils/wechatArticleExtractor.py
   ls -lh server/utils/requirements.txt
   
   # 检查Dockerfile是否包含Python
   grep -n "python3" Dockerfile
   ```

5. **停止应用容器**
   ```bash
   sudo docker compose stop app
   ```

6. **重新构建Docker镜像（重要！）**
   ```bash
   # 重新构建镜像（包含Python支持）
   sudo docker compose build --no-cache app
   
   # 或者使用docker build命令
   sudo docker build -t newsapp:latest .
   ```

7. **启动应用容器**
   ```bash
   sudo docker compose up -d app
   ```

8. **等待应用启动（约30-60秒）**
   ```bash
   # 查看容器状态
   sudo docker compose ps app
   
   # 查看应用日志
   sudo docker compose logs app --tail 50 -f
   ```

9. **验证Python环境**
   ```bash
   # 检查Python版本
   sudo docker exec newsapp python --version
   
   # 检查Python依赖
   sudo docker exec newsapp pip list | grep -E "requests|beautifulsoup4|lxml|Pillow"
   ```

10. **验证应用健康状态**
    ```bash
    # 检查健康检查
    curl http://localhost:3001/api/health
    
    # 应该返回: {"status":"ok"}
    ```

11. **验证数据库迁移**
    ```bash
    # 检查usage_type字段是否存在
    sudo docker exec -it newsapp-mysql mysql -u newsapp -p -e "DESCRIBE investment_tools.ai_model_config;" | grep usage_type
    ```

## 验证更新是否生效

### 1. 检查应用日志

```bash
# 查看应用日志，确认没有错误
sudo docker compose logs app --tail 100 | grep -i "error\|fail"

# 查看企查查同步相关日志
sudo docker compose logs app --tail 200 | grep "企查查同步"
```

### 2. 测试功能

#### 测试企查查接口同步
1. 登录管理后台
2. 进入"新闻接口配置"
3. 手动触发企查查接口同步
4. 观察日志，确认：
   - 正文为空时，会从URL提取正文并存入数据库
   - 摘要字段为NULL，等待后续AI分析

#### 测试AI分析
1. 进入"新闻分析统计"
2. 手动触发AI分析
3. 观察日志，确认：
   - 企查查接口数据会先确保有正文（如果为空则从URL抓取）
   - 基于正文内容生成摘要（150字左右，最多170字）
   - 新榜接口正文为空时，摘要为"正文无文字，无法生成摘要"

### 3. 检查数据库

```bash
# 连接到MySQL容器
sudo docker exec -it newsapp-mysql mysql -u newsapp -p

# 查看最近同步的企查查新闻
SELECT id, title, APItype, 
       LENGTH(content) as content_length,
       news_abstract IS NULL as abstract_is_null,
       created_at 
FROM news_detail 
WHERE APItype = '企查查' 
ORDER BY created_at DESC 
LIMIT 10;

# 查看最近分析的新闻
SELECT id, title, APItype,
       LENGTH(news_abstract) as abstract_length,
       news_sentiment,
       keywords,
       created_at
FROM news_detail 
WHERE news_abstract IS NOT NULL
ORDER BY updated_at DESC 
LIMIT 10;
```

## 常见问题排查

### 1. 容器无法启动

```bash
# 查看详细错误日志
sudo docker compose logs app --tail 100

# 检查文件权限
ls -la server/routes/news.js server/utils/newsAnalysis.js

# 检查文件语法（如果有Node.js环境）
node -c server/routes/news.js
node -c server/utils/newsAnalysis.js
```

### 2. 应用健康检查失败

```bash
# 查看应用日志
sudo docker compose logs app --tail 200

# 手动测试健康检查端点
curl -v http://localhost:3001/api/health

# 检查容器资源使用
sudo docker stats newsapp
```

### 3. 代码更新未生效

```bash
# 确认文件已更新
ls -lh server/routes/news.js server/utils/newsAnalysis.js

# 确认容器已重启
sudo docker inspect newsapp | grep StartedAt

# 检查volume挂载
sudo docker inspect newsapp | grep -A 10 Mounts
```

### 4. 回滚到之前的版本

```bash
# 找到备份目录
ls -lt backups/

# 恢复文件
cp backups/YYYYMMDD_HHMMSS/server/routes/news.js server/routes/news.js
cp backups/YYYYMMDD_HHMMSS/server/utils/newsAnalysis.js server/utils/newsAnalysis.js

# 重启应用
sudo docker compose restart app
```

## 更新内容说明

### 主要修改

1. **企查查接口同步**：
   - 移除同步时的摘要提取
   - 只提取正文内容并存入数据库
   - 摘要、关键词、情感分析在后续AI分析中生成

2. **AI分析逻辑**：
   - 新榜接口正文为空时，摘要改为"正文无文字，无法生成摘要"
   - 摘要长度限制调整为最多170字（从150字）
   - 企查查接口正文为空时，会从URL抓取正文后再进行分析

3. **执行顺序**：
   - 确保先提取正文入库
   - 然后基于正文内容进行摘要、关键词、情感分析
   - 最后进行企业关联性分析
