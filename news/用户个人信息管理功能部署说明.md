# 用户个人信息管理功能部署说明

## 📋 功能更新内容

本次更新包含以下功能：
1. 用户个人信息编辑功能（点击用户名可编辑）
2. 修改密码功能
3. 忘记密码提示功能
4. 管理员重置密码功能

## 🚀 部署步骤

### 方式一：开发环境（本地开发）

如果是在本地开发环境，执行以下命令：

```bash
# 1. 进入项目根目录
cd E:\USER\SUREAL\Desktop\equity_news\news

# 2. 重新构建前端（因为前端代码有更改）
cd client
npm run build
cd ..

# 3. 重启开发服务器
# 如果使用 npm run dev，需要停止后重新启动
# Ctrl+C 停止当前进程，然后执行：
npm run dev
```

### 方式二：生产环境（PM2管理）

如果服务器使用 PM2 管理应用，执行以下命令：

```bash
# 1. 登录服务器并切换到应用用户
su - guofang
# 或
ssh guofang@服务器IP

# 2. 进入应用目录
cd /opt/newsapp/news

# 3. 重新构建前端（前端代码有更改）
cd client
npm run build
cd ..

# 4. 重启PM2应用
pm2 restart newsapp

# 5. 查看应用状态
pm2 status

# 6. 查看日志确认启动成功
pm2 logs newsapp --lines 50
```

**一键执行命令**：
```bash
cd /opt/newsapp/news && cd client && npm run build && cd .. && pm2 restart newsapp && pm2 logs newsapp --lines 50
```

### 方式三：Docker环境

如果使用 Docker 部署，执行以下命令：

```bash
# 1. 进入项目目录
cd /opt/newsapp/news

# 2. 重新构建前端
cd client
npm run build
cd ..

# 3. 重新构建Docker镜像（包含新的前端文件）
sudo docker compose build app

# 4. 重启应用容器
sudo docker compose restart app

# 或完全重启
sudo docker compose down
sudo docker compose up -d

# 5. 查看容器状态
sudo docker compose ps

# 6. 查看应用日志
sudo docker compose logs -f app
```

**使用部署脚本**（如果有）：
```bash
cd /opt/newsapp/news
./deploy/update-deployment.sh
```

## ✅ 验证更新是否生效

### 1. 检查后端API

```bash
# 测试获取用户信息接口
curl http://localhost:3001/api/auth/profile \
  -H "x-user-id: 你的用户ID"

# 测试重置密码接口（需要管理员权限）
curl -X PUT http://localhost:3001/api/auth/users/用户ID/reset-password \
  -H "x-user-role: admin"
```

### 2. 检查前端功能

1. **用户个人信息功能**：
   - 登录系统
   - 点击右上角"欢迎，{用户名}"中的用户名
   - 应该弹出个人信息编辑弹窗

2. **修改密码功能**：
   - 在个人信息弹窗中点击"修改密码"
   - 应该弹出修改密码弹窗

3. **管理员重置密码功能**：
   - 使用管理员账号登录
   - 进入"用户管理"页面
   - 在操作列中应该看到"重置密码"按钮

### 3. 检查应用日志

```bash
# PM2环境
pm2 logs newsapp --lines 100

# Docker环境
sudo docker compose logs app --tail 100

# 查看是否有错误
pm2 logs newsapp --err
```

## 🔍 常见问题

### 问题1：前端功能不生效

**原因**：前端代码需要重新构建

**解决方法**：
```bash
cd client
npm run build
cd ..
# 然后重启应用
```

### 问题2：后端API报错

**原因**：后端代码未重启或代码有语法错误

**解决方法**：
```bash
# 检查代码语法
node -c server/routes/auth.js

# 查看错误日志
pm2 logs newsapp --err

# 重启应用
pm2 restart newsapp
```

### 问题3：修改的文件未生效

**检查清单**：
- [ ] 确认文件已保存
- [ ] 确认文件已上传到服务器（如果是生产环境）
- [ ] 确认前端已重新构建（前端代码更改）
- [ ] 确认应用已重启（后端代码更改）
- [ ] 清除浏览器缓存（前端功能）

## ⚠️ 重要提示

1. **前端代码必须重新构建**：由于新增了组件和修改了页面，前端代码必须执行 `npm run build` 重新构建

2. **后端代码需要重启**：修改了后端路由文件，需要重启Node.js应用才能生效

3. **浏览器缓存**：如果前端功能不显示，尝试清除浏览器缓存或使用无痕模式

4. **权限检查**：确保执行命令的用户有足够的权限（PM2需要应用用户权限，Docker需要sudo权限）
