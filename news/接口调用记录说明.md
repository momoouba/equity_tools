# 系统接口调用记录说明

## 已存储的接口调用记录

### 1. 新闻同步接口执行日志

**表名：** `news_sync_execution_log`

**存储内容：**
- ✅ 执行类型（手动触发/定时任务）
- ✅ 开始执行时间
- ✅ 结束执行时间
- ✅ 执行耗时（秒）
- ✅ 执行状态（成功/失败/执行中）
- ✅ 同步的新闻数量
- ✅ 处理的企业/公众号总数
- ✅ 实际处理数量
- ✅ 错误数量和错误信息
- ✅ 执行详情（时间范围、配置信息等）
- ✅ 操作人ID（手动触发时）

**查看方式：**
1. **前端界面**：定时任务管理 → 新闻接口同步 → 点击"日志"按钮
2. **数据库查询**：
   ```sql
   SELECT * FROM news_sync_execution_log 
   ORDER BY start_time DESC 
   LIMIT 50;
   ```

**记录时机：**
- ✅ 手动触发新闻同步时（`/api/news/sync`）
- ✅ 定时任务执行新闻同步时
- ✅ 企查查接口同步时
- ✅ 新榜接口同步时

## 查看接口调用记录的方法

### 方法1：通过前端界面查看

**新闻同步记录：**
1. 访问：定时任务管理 → 新闻接口同步
2. 点击对应配置的"日志"按钮
3. 查看执行记录详情

**邮件发送记录：**
1. 访问：定时任务管理 → 邮件发送舆情
2. 点击对应配置的"日志"按钮
3. 查看发送记录详情

### 方法2：直接查询数据库

**查看新闻同步记录：**
```sql
-- 查看所有记录
SELECT 
    id,
    config_id,
    execution_type,
    start_time,
    end_time,
    duration_seconds,
    status,
    synced_count,
    total_enterprises,
    processed_enterprises,
    error_count,
    error_message
FROM news_sync_execution_log
ORDER BY start_time DESC
LIMIT 50;

-- 查看特定配置的记录
SELECT * FROM news_sync_execution_log 
WHERE config_id = '配置ID'
ORDER BY start_time DESC;

-- 查看失败的记录
SELECT * FROM news_sync_execution_log 
WHERE status = 'failed'
ORDER BY start_time DESC;
```

**查看邮件发送记录：**
```sql
-- 查看所有邮件发送记录
SELECT 
    id,
    operation_type,
    from_email,
    to_email,
    subject,
    status,
    error_message,
    created_at
FROM email_logs
WHERE operation_type = 'send'
ORDER BY created_at DESC
LIMIT 50;
```

### 方法3：查看应用日志

```bash
# Docker环境
sudo docker compose logs app | grep -E "同步|接口|触发"

# 查看最近的日志
sudo docker compose logs app --tail 100
```

## 记录的数据字段说明

### news_sync_execution_log 表字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | VARCHAR(19) | 日志ID |
| config_id | VARCHAR(19) | 新闻接口配置ID |
| execution_type | ENUM | 执行类型：manual（手动）/ scheduled（定时） |
| start_time | TIMESTAMP | 开始执行时间 |
| end_time | TIMESTAMP | 结束执行时间 |
| duration_seconds | INT | 执行耗时（秒） |
| status | ENUM | 状态：success（成功）/ failed（失败）/ running（执行中） |
| synced_count | INT | 同步的新闻数量 |
| total_enterprises | INT | 企业/公众号总数 |
| processed_enterprises | INT | 实际处理数量 |
| error_count | INT | 错误数量 |
| error_message | TEXT | 错误信息 |
| execution_details | JSON | 执行详情（时间范围、接口类型等） |
| created_by | VARCHAR(19) | 操作人ID |
| created_at | TIMESTAMP | 创建时间 |

## 建议

如果需要记录更多接口调用，可以考虑：

1. **创建通用API日志表**：记录所有API接口的调用情况
2. **添加中间件**：在Express中添加日志中间件，自动记录所有请求
3. **使用日志系统**：集成专业的日志系统（如ELK、Winston等）
