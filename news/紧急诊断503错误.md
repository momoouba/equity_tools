# 紧急诊断 503 错误 - 完整排查步骤

## 🔍 立即执行以下诊断命令

请在服务器上依次执行以下命令，并提供输出结果：

### 1. 检查容器状态

```bash
cd /opt/newsapp/news
sudo docker compose ps
```

### 2. 查看最近的错误日志

```bash
sudo docker compose logs app --tail 100 | grep -i "error\|fail\|exception\|ReferenceError"
```

### 3. 检查服务器是否就绪

```bash
sudo docker compose logs app --tail 200 | grep -i "服务器核心功能已就绪\|serverReady\|503"
```

### 4. 测试健康检查

```bash
curl -v http://localhost:3001/api/health
```

### 5. 查看完整的启动日志

```bash
sudo docker compose logs app --tail 300
```

## 🚀 如果服务器未启动，执行完全重启

```bash
cd /opt/newsapp/news

# 1. 停止所有容器
sudo docker compose down

# 2. 等待 5 秒
sleep 5

# 3. 重新启动
sudo docker compose up -d

# 4. 等待 60 秒让服务器完全启动
sleep 60

# 5. 检查状态
sudo docker compose ps

# 6. 查看日志
sudo docker compose logs app --tail 100
```

## 📋 需要提供的信息

请执行上述命令后，提供以下信息：

1. **容器状态** (`sudo docker compose ps` 的输出)
2. **最近的错误日志** (`sudo docker compose logs app --tail 100 | grep -i error` 的输出)
3. **服务器就绪状态** (`sudo docker compose logs app | grep -i "服务器核心功能已就绪"` 的输出)
4. **健康检查结果** (`curl http://localhost:3001/api/health` 的输出)
