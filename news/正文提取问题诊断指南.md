# 正文提取问题诊断指南

## 问题描述
格隆汇网站（https://www.gelonghui.com/news/5136685）的新闻在AI重新分析后，没有正确提取正文内容和摘要内容。

## 诊断工具

我已经创建了一个诊断API端点，可以帮助我们收集详细信息来定位问题。


## 可能的问题和解决方案

### 问题1: 正则表达式未匹配到内容

**症状**：
- `htmlInfo.hasMainNews = true`
- `extraction.success = false`
- 日志显示"规则 X（main-news匹配）未匹配到内容"

**可能原因**：
1. HTML结构变化，class属性格式不同
2. 有多个article标签，匹配到了错误的标签
3. article标签有嵌套，正则表达式提前结束

**解决方案**：
根据诊断工具返回的 `mainNewsArticleSample` 调整正则表达式。

### 问题2: 提取的内容太短

**症状**：
- `extraction.success = false`
- `extraction.contentLength < 50`
- 日志显示"提取的内容太短"

**可能原因**：
1. 匹配到了错误的标签（如侧边栏、导航等）
2. 内容被截断
3. HTML中有嵌套结构，只匹配到了部分内容

**解决方案**：
检查 `directExtraction` 的结果，看看直接提取是否能得到更多内容。

### 问题3: HTML结构变化

**症状**：
- `htmlInfo.hasMainNews = false`
- `htmlInfo.hasArticleWithHtml = false`

**可能原因**：
格隆汇网站更新了HTML结构，不再使用 `main-news article-with-html` 类。

**解决方案**：
需要查看实际的HTML结构，找到新的正文容器标识。

### 问题4: 摘要未生成

**症状**：
- 正文提取成功，但 `news_abstract` 为空

**可能原因**：
1. AI分析时未正确处理提取的正文
2. 正文内容被判定为无效（太短、乱码等）
3. AI分析失败

**解决方案**：
检查AI分析的日志，查看是否有错误信息。

## 临时解决方案

如果诊断工具显示提取失败，可以：

1. **手动提取并更新数据库**：
```sql
-- 手动更新content字段（需要先提取好内容）
UPDATE news_detail 
SET content = '<提取的正文内容>'
WHERE source_url = 'https://www.gelonghui.com/news/5136685'
  AND delete_mark = 0;
```

2. **重新触发AI分析**：
使用管理后台的"重新分析"功能，或调用API：
```bash
POST /api/news-analysis/reanalyze
{
  "newsIds": [<新闻ID>]
}
```
