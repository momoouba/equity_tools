# 邮件发送新闻功能说明

## 📋 功能概述

邮件发送新闻功能允许系统自动或手动将舆情信息以邮件形式发送给指定的收件人。系统支持定时自动发送和手动立即发送两种方式，邮件内容包含HTML格式的舆情信息摘要和Excel附件，方便收件人快速了解昨日舆情动态。

## 🎯 主要功能

### 1. 收件人管理

#### 功能特点
- 用户可以配置自己的收件人邮箱列表
- 支持多个收件人邮箱（用逗号、分号或换行分隔）
- 可以设置邮件主题
- 可以配置发送频率和发送时间
- 支持启用/禁用收件配置
- 管理员可以查看所有用户的收件配置

#### 配置项说明
- **收件人邮箱**：支持多个邮箱，用逗号、分号或换行分隔
- **邮件主题**：自定义邮件主题，如果不填写则使用默认主题
- **发送频率**：
  - `daily`：每天
  - `weekly`：每周（每周一）
  - `monthly`：每月（每月1号）
- **发送时间**：格式为 `HH:mm:ss`，例如 `09:00:00`
- **启用状态**：启用/禁用该收件配置
- **企查查接口消息类型**：选择需要推送的企查查新闻类别（可选）
  - 点击"选择消息类型"按钮打开类别选择弹窗
  - 支持全选、清空、使用默认类别
  - 支持多选，可以选择任意数量的类别
  - 如果未选择任何类别，系统使用默认类别（80000系列、40000系列、14004）
  - 详细类别列表请参考《企查查邮件推送新闻类型说明.md》

### 2. 邮件内容生成

#### HTML邮件内容
系统会自动生成美观的HTML格式邮件，包含：

- **邮件标题**：舆情信息日报
- **日期信息**：显示昨日日期
- **按企业分组**：新闻按被投企业分组显示
- **新闻详情**：每条新闻包含：
  - 新闻标题
  - 新闻标签（关键词）
  - 新闻情绪（正面/中性/负面）
  - 新闻摘要
  - 发布时间
  - 公众号名称
  - 原文链接

#### Excel附件
邮件会自动附带Excel文件，包含完整的舆情信息：

- **文件命名**：`舆情信息日报_YYYY-MM-DD.xlsx`
- **数据列**：
  - 序号
  - 被投企业全称
  - 新闻标题
  - 新闻标签
  - 新闻情绪
  - 新闻摘要
  - 发布时间
  - 公众号名称
  - 原文链接

### 3. 定时自动发送

#### 功能特点
- 系统根据收件配置的发送频率和时间自动发送邮件
- 使用 `node-cron` 进行定时任务调度
- 支持每天、每周、每月三种频率
- 自动获取昨日舆情信息
- 发送失败会记录错误日志

#### 执行流程
1. 定时任务触发（根据配置的频率和时间）
2. 获取收件配置（只处理启用状态的配置）
3. 查询昨日舆情信息（根据用户权限过滤）
4. 生成邮件内容（HTML + Excel附件）
5. 发送邮件
6. 记录发送日志（成功/失败）

### 4. 手动立即发送

#### 功能特点
- 用户可以手动触发邮件发送
- 管理员可以在定时任务管理中手动执行
- 立即发送昨日舆情信息
- 不依赖定时任务配置

#### 使用场景
- 测试邮件配置是否正确
- 补充发送遗漏的邮件
- 紧急发送舆情信息

### 5. 邮件发送记录

#### 功能特点
- 记录所有邮件发送历史
- 显示发送状态（成功/失败）
- 记录错误信息（如果发送失败）
- 支持分页查看
- 权限控制（普通用户只能查看自己的邮件记录）

#### 记录信息
- 操作类型（发送/接收）
- 发件人邮箱
- 收件人邮箱
- 抄送邮箱（如果有）
- 密送邮箱（如果有）
- 邮件主题
- 发送状态
- 错误信息（如果失败）
- 发送时间

## 🔧 技术实现

### 数据库设计

#### recipient_management表（收件管理）
```sql
CREATE TABLE recipient_management (
  id VARCHAR(19) PRIMARY KEY COMMENT '数据ID',
  user_id VARCHAR(19) NOT NULL COMMENT '用户ID',
  recipient_email TEXT NOT NULL COMMENT '收件人邮箱（多个用逗号分隔）',
  email_subject VARCHAR(255) COMMENT '邮件主题',
  send_frequency ENUM('daily', 'weekly', 'monthly') DEFAULT 'daily' COMMENT '发送频率',
  send_time TIME DEFAULT '09:00:00' COMMENT '发送时间',
  is_active TINYINT DEFAULT 1 COMMENT '是否启用',
  qichacha_category_codes JSON COMMENT '企查查新闻类别编码列表（JSON数组），为空时使用默认类别',
  is_deleted TINYINT DEFAULT 0 COMMENT '删除标志',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  deleted_at DATETIME NULL,
  deleted_by VARCHAR(19) NULL
);
```

**新增字段说明：**
- **qichacha_category_codes**：企查查新闻类别编码列表
  - 类型：JSON
  - 格式：JSON数组，例如：`["80000", "80001", "14004"]`
  - 默认值：NULL（表示使用默认类别）
  - 说明：如果为NULL或空数组，系统使用默认类别；如果包含类别编码，系统只推送这些类别对应的企查查新闻

#### email_logs表（邮件日志）
```sql
CREATE TABLE email_logs (
  id VARCHAR(19) PRIMARY KEY,
  email_config_id VARCHAR(19) NOT NULL COMMENT '邮件配置ID',
  operation_type ENUM('send', 'receive') DEFAULT 'send' COMMENT '操作类型',
  from_email VARCHAR(255) COMMENT '发件人邮箱',
  to_email TEXT COMMENT '收件人邮箱',
  cc_email TEXT COMMENT '抄送邮箱',
  bcc_email TEXT COMMENT '密送邮箱',
  subject VARCHAR(500) COMMENT '邮件主题',
  content TEXT COMMENT '邮件内容',
  status ENUM('success', 'failed') DEFAULT 'success' COMMENT '状态',
  error_message TEXT COMMENT '错误信息',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  created_by VARCHAR(19) COMMENT '创建用户ID'
);
```

### 定时任务调度

系统使用 `node-cron` 进行定时任务调度：

```javascript
// server/utils/scheduledEmailTasks.js
function generateCronExpression(sendFrequency, sendTime) {
  const [hours, minutes] = sendTime.split(':');
  
  switch (sendFrequency) {
    case 'daily':
      return `${minutes} ${hours} * * *`;  // 每天执行
    case 'weekly':
      return `${minutes} ${hours} * * 1`;  // 每周一执行
    case 'monthly':
      return `${minutes} ${hours} 1 * *`;   // 每月1号执行
  }
}
```

## 📊 权限控制

### 用户权限
- **普通用户**：
  - 只能管理自己的收件人配置
  - 只能查看发送给自己配置的收件人的邮件记录
  - 只能看到自己可见的舆情信息（自己创建的被投企业相关新闻）

- **管理员**：
  - 可以查看所有用户的收件人配置
  - 可以查看所有邮件发送记录
  - 可以看到所有舆情信息
  - 可以在定时任务管理中管理所有邮件发送任务

### 数据过滤
- 邮件发送时，系统会根据用户权限过滤舆情信息
- 管理员可以看到所有新闻
- 普通用户只能看到自己创建的被投企业相关的新闻

## 🚀 使用方法

### 1. 配置收件人

1. 进入"邮件管理" → "收件人管理"页面
2. 点击"新增"按钮
3. 填写收件人信息：
   - 收件人邮箱（多个用逗号分隔）
   - 邮件主题（可选）
   - 发送频率（每天/每周/每月）
   - 发送时间（例如：09:00:00）
   - 启用状态
   - **企查查接口消息类型**（可选）：
     - 点击"选择消息类型"按钮
     - 在弹窗中选择需要的企查查类别
     - 支持全选、清空、使用默认类别
     - 如果未选择，系统使用默认类别（80000系列、40000系列、14004）
4. 点击"保存"完成配置

### 2. 手动发送邮件

**方法一：在收件人管理页面**
1. 找到要发送的收件配置
2. 点击"发送"按钮
3. 确认发送操作
4. 系统会立即发送昨日舆情信息

**方法二：在定时任务管理页面**
1. 进入"定时任务管理" → "邮件发送舆情"Tab
2. 找到要执行的任务
3. 点击"立即执行"按钮
4. 确认执行操作

**方法三：在邮件收发管理页面**
1. 进入"邮件管理" → "邮件收发管理"页面
2. 选择邮件配置
3. 点击"发送邮件"按钮

### 3. 查看发送记录

1. 进入"邮件管理" → "邮件收发管理"页面
2. 选择邮件配置
3. 切换到"收发记录"或"邮件日志"Tab

5. 使用分页浏览更多记录

### 4. 管理定时任务

1. 进入"定时任务管理" → "邮件发送舆情"Tab
2. 查看所有邮件发送定时任务

## 📝 注意事项

### 1. 邮件配置要求
- 必须先配置邮件服务器（在系统配置中）
- 邮件配置必须关联"新闻舆情"应用
- 邮件配置必须处于启用状态

### 2. 收件人邮箱格式
- 支持多个邮箱，用逗号、分号或换行分隔
- 邮箱格式必须正确（包含@符号）

### 3. 企查查类别选择
- 企查查类别选择功能仅影响企查查数据源的新闻过滤
- 新榜等其他数据源的新闻不受类别限制，全部推送
- 如果未选择任何类别，系统自动使用默认类别（80000系列、40000系列、14004）
- 类别选择后，邮件发送时会根据选择的类别过滤企查查新闻
- 详细类别列表和说明请参考《企查查邮件推送新闻类型说明.md》
- 系统会自动过滤无效邮箱

### 3. 舆情信息过滤
- 系统只发送有企业全称的新闻
- 普通用户只能看到自己创建的被投企业相关新闻
- 管理员可以看到所有新闻
- 如果昨日没有可见的新闻，邮件可能不发送或发送空内容

### 4. 定时任务执行
- 定时任务使用服务器本地时区（Asia/Shanghai）
- 确保服务器时间准确
- 定时任务在服务器启动时自动初始化
- 修改收件配置后，定时任务会自动更新

### 5. 邮件发送限制
- 受邮件服务器发送频率限制
- 大量收件人时可能需要较长时间
- 发送失败会记录错误日志，不会影响其他收件人

### 6. Excel附件
- Excel文件包含完整的舆情信息
- 文件大小取决于新闻数量
- 建议控制单次发送的新闻数量

### 7. 错误处理
- 发送失败会记录详细错误信息
- 可以在邮件日志中查看错误原因
- 常见错误：邮件服务器配置错误、网络问题、收件人邮箱无效等

## 🔍 常见问题

### Q: 为什么没有收到邮件？
**A:** 请检查：
1. 收件配置是否处于启用状态
2. 邮件服务器配置是否正确
3. 昨日是否有可见的舆情信息
4. 查看邮件日志了解发送状态和错误信息
5. 检查收件人邮箱是否正确

### Q: 如何修改邮件发送时间？
**A:** 
1. 进入"收件人管理"页面
2. 找到要修改的收件配置
3. 点击"编辑"按钮
4. 修改"发送时间"字段
5. 保存后定时任务会自动更新

### Q: 可以同时发送给多个收件人吗？
**A:** 可以。在"收件人邮箱"字段中，用逗号、分号或换行分隔多个邮箱地址即可。

### Q: 邮件内容可以自定义吗？
**A:** 目前邮件内容由系统自动生成，包含HTML格式的舆情摘要和Excel附件。邮件主题可以自定义。

### Q: 定时任务没有执行怎么办？
**A:** 请检查：
1. 收件配置是否处于启用状态
2. 服务器是否正常运行
3. 查看服务器日志了解任务执行情况
4. 可以手动执行任务测试

### Q: 如何查看邮件发送历史？
**A:** 
1. 进入"邮件管理" → "邮件收发管理"页面
2. 选择邮件配置
3. 切换到"收发记录"或"邮件日志"Tab
4. 查看发送历史记录

### Q: 普通用户和管理员看到的邮件内容一样吗？
**A:** 不一样。普通用户只能看到自己创建的被投企业相关的新闻，管理员可以看到所有新闻。
