# 额外公众号数据同步功能说明

## 功能概述

在原有的新闻数据同步功能基础上，新增了对`additional_wechat_accounts`表中额外公众号的数据同步支持。现在系统会同时从两个数据源抓取公众号文章数据：

1. **被投企业公众号**：来自`invested_enterprises`表，退出状态不为"完全退出"和"已上市"的企业公众号
2. **额外公众号**：来自`additional_wechat_accounts`表，状态为"active"的额外公众号

## 技术实现

### 1. 数据源合并

修改了`syncNewsData`函数中的公众号查询逻辑：

```javascript
// 查询invested_enterprises表中退出状态不为"完全退出"和"已上市"的数据
const enterprises = await db.query(
  `SELECT DISTINCT wechat_official_account_id 
   FROM invested_enterprises 
   WHERE exit_status NOT IN ('完全退出', '已上市')
   AND wechat_official_account_id IS NOT NULL 
   AND wechat_official_account_id != ''
   AND delete_mark = 0`
);

// 查询additional_wechat_accounts表中状态为"active"的数据
const additionalAccounts = await db.query(
  `SELECT DISTINCT wechat_account_id 
   FROM additional_wechat_accounts 
   WHERE status = 'active' 
   AND wechat_account_id IS NOT NULL 
   AND wechat_account_id != ''
   AND delete_mark = 0`
);

// 合并两个数据源的公众号ID
const enterpriseAccountIds = enterprises.map(e => e.wechat_official_account_id);
const additionalAccountIds = additionalAccounts.map(a => a.wechat_account_id);
const allAccountIds = [...enterpriseAccountIds, ...additionalAccountIds];

// 去重公众号ID
const uniqueAccounts = [...new Set(allAccountIds)];
```

### 2. 企业全称匹配逻辑

更新了企业全称的匹配逻辑，支持从两个表中查找：

```javascript
// 根据微信账号匹配被投企业全称
let enterpriseFullName = null;
try {
  // 首先从invested_enterprises表中查找
  const enterpriseResult = await db.query(
    `SELECT enterprise_full_name 
     FROM invested_enterprises 
     WHERE wechat_official_account_id = ? 
     AND delete_mark = 0 
     LIMIT 1`,
    [article.account || account]
  );
  
  if (enterpriseResult.length > 0) {
    enterpriseFullName = enterpriseResult[0].enterprise_full_name;
  } else {
    // 如果在invested_enterprises中没找到，则从additional_wechat_accounts表中查找
    const additionalResult = await db.query(
      `SELECT account_name 
       FROM additional_wechat_accounts 
       WHERE wechat_account_id = ? 
       AND delete_mark = 0 
       LIMIT 1`,
      [article.account || account]
    );
    
    if (additionalResult.length > 0) {
      enterpriseFullName = additionalResult[0].account_name;
    }
  }
} catch (e) {
  console.warn('匹配企业全称时出错:', e.message);
}
```

## 功能特点

### 1. 数据源优先级

- **优先级1**：`invested_enterprises`表中的企业全称
- **优先级2**：`additional_wechat_accounts`表中的公众号名称

### 2. 去重机制

- 对所有公众号ID进行去重处理，避免重复抓取
- 文章去重仍基于`source_url`字段

### 3. 状态筛选

- `invested_enterprises`：只同步退出状态不为"完全退出"和"已上市"的企业
- `additional_wechat_accounts`：只同步状态为"active"的公众号

### 4. 软删除支持

- 两个表都支持软删除机制（`delete_mark = 0`）
- 只同步未被删除的记录

## 使用场景

1. **扩展数据源**：管理员可以添加更多公众号作为数据源补充
2. **灵活管理**：可以独立管理额外公众号的状态，不影响被投企业数据
3. **数据完整性**：确保所有相关公众号的文章都能被同步到系统中

## 注意事项

1. **性能考虑**：增加数据源会增加同步时间，建议合理控制额外公众号数量
2. **API限制**：需要确保新闻接口的调用频率限制能够支持更多公众号的数据抓取
3. **存储空间**：更多的数据源意味着更多的文章数据，需要关注存储空间使用情况

## 监控和日志

系统会在同步过程中输出详细的日志信息，包括：
- 从两个数据源查询到的公众号数量
- 合并后的唯一公众号数量
- 每个公众号的同步结果
- 企业全称匹配的详细信息

通过这些日志可以监控新功能的运行状态和效果。
